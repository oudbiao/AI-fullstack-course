# Role-Based Learning Paths

![Role-based path selection map](/img/course/intro-role-based-paths-map-en.png)

If you are unsure, follow the main path from stages 1-9. Use this page only to decide where to read deeply after the first quick pass.

## Pick in 30 Seconds

| Goal | Read deeply | Skim first | Portfolio project |
|---|---|---|---|
| AI application engineer | 1, 2, 3, 7, 8, 9 | 4-6 details | AI app with API, RAG, logs, deployment notes |
| RAG engineer | 2, 3, 5, 7, 8 | advanced Agent | Knowledge base with retrieval logs and citation evaluation |
| Agent developer | 2, 3, 7, 8, 9 | advanced training | Tool-calling Agent with trace and permission boundary |
| Model engineering | 4, 5, 6, 7, 8 model deployment | frontend details | Model comparison, fine-tuning, or inference report |
| Portfolio job seeker | 1-9, then one capstone direction | niche extras | A connected project story with README, demo, failures |

## Minimum Project Set

| Path | Build these first |
|---|---|
| Application | Python API + Prompt assistant + RAG Q&A + simple deployment |
| RAG | data cleaning + chunking demo + RAG Q&A + evaluation set |
| Agent | function-calling demo + multi-step Agent + safe confirmation example |
| Model | ML baseline + DL training run + model comparison |
| Portfolio | data report + ML baseline + RAG assistant + Agent assistant + capstone |

## Switching Rule

Do not switch roles every week. Finish one small project, review what felt difficult, then decide whether to go deeper into RAG, Agent, models, or product delivery.
