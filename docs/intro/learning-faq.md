# Learning Path FAQ

![FAQ decision tree](/img/course/appendix-faq-decision-tree-en.png)

Use this page to make a decision and keep moving. Most questions are better answered by one small project than by more planning.

## Fast Answers

| Question | Short answer |
|---|---|
| Can a complete beginner learn this? | Yes. Start with tools, Python, and data. |
| I already know Python. Can I skip? | Skim, but still complete the stage project check. |
| I only want RAG or Agent. Do I need math? | You need enough intuition for vectors, similarity, metrics, and evaluation. |
| Do I need frontend? | Not at first. CLI, Notebook, Streamlit, or FastAPI is enough to start. |
| Do I need a GPU? | Not for the first pass. APIs and small local examples cover most stages. |
| How long does it take? | 4 weeks for a sprint, 8 weeks for portfolio shape, 12 weeks for a steady full pass. |
| Theory or project first? | Run the smallest project, then read the theory with a concrete problem in mind. |
| Do I need a project every stage? | Keep at least one small runnable artifact per major stage. |
| When can I start the capstone? | After a minimal RAG or Agent loop works. |
| What if I get stuck? | Narrow it to one command, one file, or one input before changing routes. |

## The Default Route

If you are unsure, follow this order:

1. Quick experience.
2. Environment setup.
3. Stages 1-3 for tools, Python, and data.
4. Stages 4-6 for enough model intuition.
5. Stages 7-9 for LLM, RAG, and Agent.
6. One capstone direction from stages 10-12 or deployment electives.

## Getting Unstuck

Do not restart the whole course. Go back to the current stage task, reproduce the issue with the smallest command, check the troubleshooting index, and record the fix in your project notes.
