# Project Roadmap and Portfolio

![Project portfolio growth ladder](/img/course/intro-project-portfolio-roadmap-en.png)

Learn through projects, but keep them small enough to finish. A project only becomes portfolio material when it can run, be explained, and be evaluated.

## Three Project Levels

| Level | Goal | Do not chase first |
| --- | --- | --- |
| Stage project | Input, output, run command | Many features or polished UI |
| Integrated project | Logs, metrics, failure cases | One success screenshot only |
| Capstone project | README, evaluation, demo script | Every technology in one system |

## Recommended Project Ladder

| Course area | Project output |
| --- | --- |
| Tools + Python | CLI learning assistant |
| Data analysis | EDA report with charts and limits |
| ML / DL | baseline model, metrics, error samples |
| LLM / Prompt | structured plan or review-card assistant |
| RAG | course Q&A with citations and eval questions |
| Agent | tool-using assistant with trace and permissions |
| Multimodal / Capstone | one demoable final project |

## Minimum Portfolio README

````md
# Project Name

## Goal
What problem does it solve?

## Input and Output
Input:
Output:

## How to Run
```bash
python main.py
```

## Evaluation
How do you judge the result?

## Failure Samples
What broke, and what did you learn?

## Next Improvements
What would you improve first?
````

If every project has these sections, your portfolio will grow naturally as the course progresses.
