# Choose a 4, 8, or 12 Week Pace

![Learning pace cards](/img/course/intro-learning-pace-cards-en.png)

Do not plan this course by article count. Plan it by weekly deliverables: one visible output, one short review, and one improvement note.

## 1. Pick the Right Pace

| Your situation | Use this pace | Main goal |
|---|---|---|
| You already code and want a quick AI app | 4 weeks | Build one explainable RAG or Agent prototype |
| You want a job-ready portfolio | 8 weeks | Produce staged projects and connect them into a story |
| You are a beginner or want full foundations | 12 weeks | Build steadily from tools to capstone |

## 2. Pace Cards

| Pace | Weekly focus | Final evidence |
|---|---|---|
| 4 weeks | Review foundations fast, then spend most time on LLM, RAG, Agent, and deployment notes | One runnable AI app, README, evaluation set, failure samples |
| 8 weeks | One project ability every week or two: tools, Python, data, ML, DL, LLM, RAG, Agent | Portfolio README, screenshots, demo script, project retrospectives |
| 12 weeks | Follow the course order, keep one small result per week, and leave more time for debugging | Complete learning repository, staged outputs, capstone draft |

Do not skip evaluation, logs, or failure samples, even on the 4-week path. They are what turn a demo into a project.

## 3. Weekly Operating Loop

| Step | What to do |
|---|---|
| Monday | Choose one chapter goal and one deliverable |
| During the week | Run the smallest code path first, then add features |
| Before finishing | Save one screenshot, one command, one sample output, and one failure |
| Weekend | Update README and decide the next bottleneck |

## 4. Adjustment Rules

| If you are... | Adjust like this |
|---|---|
| Already strong in Python | Compress chapters 1-2 and spend the saved time on RAG and Agent evaluation |
| Strong in ML/DL | Skim theory, but still keep metrics, curves, and error analysis |
| Often blocked by setup | Use the 12-week pace and keep a troubleshooting log |
| Short on time | Reduce features, not evidence. Keep README, run command, examples, and failures |

## 5. Weekly Review Template

```md
## Week N Review

- Deliverable:
- Command that runs:
- Sample input/output:
- One failure:
- What I checked:
- Next bottleneck:
```

Passing the course does not mean reading every page once. It means leaving with runnable work that another person can understand, reproduce, and discuss with you.
