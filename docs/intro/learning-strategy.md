# Learning Strategy

![Continuous learning flywheel](/img/course/appendix-continuous-learning-flywheel-en.png)

Use one learning loop for the whole course:

> See the idea -> run a tiny example -> build a small artifact -> record evidence -> review the next step.

## Three Rules

| Rule | What it prevents | Minimum action |
| --- | --- | --- |
| Learn by project | Watching many tutorials without proof | Leave one runnable artifact per stage |
| Learn theory near practice | Reading math/model theory for weeks without feedback | Pair one intuition with one small experiment |
| Learn with evidence | Forgetting what you actually did | Save command, output, error, or metric |

## Weekly Rhythm

| Time box | Action |
| --- | --- |
| 25 minutes | Read one small section or look at one diagram |
| 25 minutes | Run or modify one example |
| 10 minutes | Write a README note or failure note |
| End of week | Choose one result to keep as portfolio evidence |

## Avoid These Traps

| Trap | Better move |
| --- | --- |
| Tutorial collecting | Run one example before opening another resource |
| Perfectionism | Make the ugly version work first |
| Route switching | Finish the current stage checkpoint before changing routes |
| Theory-only learning | Add a tiny experiment |
| Demo-only learning | Add evaluation or failure samples |

When in doubt, return to the current stage's `0.0 Study Guide and Task Sheet` and finish the smallest task.
