# Skill Evidence Cards

![Stage checkpoint cards](/img/course/intro-stage-checkpoint-cards-en.png)

Badges are only useful if they point to evidence. Instead of collecting many names, keep one evidence card per stage.

## Evidence Card Rule

| Field | What to write |
| --- | --- |
| Artifact | What you built |
| Run command | How to reproduce it |
| Evidence | Screenshot, log, chart, metric, trace, or citation check |
| One failure | A real problem you met |
| Improvement | What you fixed or will fix next |

## First-Pass Cards

| Stage | Card focus | Example evidence |
| --- | --- | --- |
| 1 Tools | Runnable workspace | README, Git commit, terminal output |
| 2 Python | Program saves data | JSON file, CLI output, error handling |
| 3 Data | Data becomes insight | cleaned CSV, chart, conclusion |
| 4 Math | Formula becomes code | small experiment and explanation |
| 5 ML | Baseline and metrics | metric table, error samples |
| 6 DL | Training signal | loss curve, config, failed run |
| 7 LLM/Prompt | Stable structured output | prompt versions, schema checks |
| 8 RAG | Answer with evidence | retrieval logs, citation table |
| 9 Agent | Controllable tool use | trace, permission table, stop rule |
| 10-12 | One extension direction | before/after examples and evaluation |

## Card Template

```md
## Stage Evidence Card

Artifact:
Run command:
Evidence file:
One failure:
Fix or next step:
```

If time is limited, one clear card per stage is better than many empty badges.
