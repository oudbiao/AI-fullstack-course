# AI Full-Stack Glossary

![Concept sorting board](/img/course/intro-concept-sorting-board-en.png)

This glossary is for quick lookup, not memorization. If a term still feels unclear, remember only the problem it solves and return to the chapter where you use it.

## Lookup Rule

| If the term sounds like... | First look in... |
|---|---|
| command, repo, package, API, JSON | Development basics |
| dataset, feature, label, baseline, metric | Data and machine learning |
| tensor, loss, embedding, attention, token | Deep learning and LLM basics |
| Prompt, RAG, chunk, citation, hallucination | LLM application |
| Agent, tool, trace, memory, guardrails | Agent and engineering |

## Core Terms

| Term | One-line meaning |
|---|---|
| Terminal | The place where you run commands |
| Current directory | The folder a command is running from |
| Package manager | Installs dependencies such as Python or Node packages |
| Virtual environment | Keeps one Python project’s dependencies isolated |
| Git commit | A saved version of code changes |
| README | The file that explains what a project does and how to run it |
| API | A contract for calling a service |
| JSON | A structured text format used for APIs, configs, and logs |
| Dataset | Data used for analysis, training, or evaluation |
| Feature | Input information used by a model |
| Label | The target a supervised model learns to predict |
| Baseline | A simple starting model or rule for comparison |
| Data leakage | Information the model should not see during training |
| Overfitting | Performs well on training data but poorly on new data |
| Metric | A number or rule used to judge results |
| Tensor | A multi-dimensional array used in deep learning |
| Loss | The gap between prediction and target |
| Backpropagation | The method for sending error signals back through a neural network |
| Embedding | A vector representation of text, image, or category |
| Attention | A mechanism for choosing which input parts matter more |
| Transformer | The architecture behind many modern language models |
| Token | The text unit processed by a model |
| Context window | The amount of input a model can see at one time |
| Prompt | Instructions and input given to a model |
| Structured output | Asking the model to return fixed fields such as JSON |
| Function calling | Letting a model fill tool-call arguments |
| RAG | Retrieving external material before answering |
| Chunk | A document piece used for retrieval |
| Vector database | Stores and retrieves vectors for similarity search |
| Rerank | Reorders retrieved results before generation |
| Citation | The source an answer claims to use |
| Hallucination | A plausible but unsupported model output |
| Agent | A system that plans steps, uses tools, and keeps state |
| Tool | An external capability the system can call |
| Trace | A record of steps, inputs, outputs, and tool calls |
| Guardrails | Rules and checks around inputs, outputs, tools, and permissions |
| Observability | Logs, metrics, traces, and errors that make a system inspectable |
| Deployment | Making a project accessible outside your local machine |

## How to Use It

Do not stop a learning session because one word is unfamiliar. First classify it, read the one-line meaning, run the related example, and write a better explanation after you use it once.
