# Beginner-Friendly Learning Guide

![30-minute AI quick experience loop](/img/course/intro-quick-experience-loop-en.png)

If the course map feels large, use this page as a pressure reducer. You do not need to understand everything before starting.

The first pass has one rule:

> Run something small, explain one input and one output, save one piece of evidence, then move to the next step.

## First-Week Rhythm

| Day | Do this | Stop when you have |
| --- | --- | --- |
| 1 | Run the 30-minute quick experience | One AI result or screenshot |
| 2 | Prepare Python, VS Code, Git, and a project folder | `python --version` and `git --version` work |
| 3 | Open the capability map | You can name the current layer: tools, data, model, LLM, app, Agent, or engineering |
| 4 | Start Chapter 1 | One project folder and one README |
| 5 | Write a tiny Python script | One command and one output |
| 6 | Record one real error | Error message, cause, and fix note |
| 7 | Review and choose the next lesson | One sentence about what to do next |

Do not use “pages read” as the main score. Use **result + evidence + next step**.

## What to Ignore on the First Pass

| Ignore for now | Why |
| --- | --- |
| Perfect theory notes | Concepts become clearer after you run examples |
| Every optional route | Choose the default route first |
| Large model training | Expensive and slow before the foundations are ready |
| Complex frameworks | They hide the basic input-output loop |
| Perfect portfolio polish | First make the project runnable |

Bookmark optional pages instead of opening all of them. They are useful later, but they are not the starting line.

## The Smallest Daily Loop

![Stage checkpoint cards](/img/course/intro-stage-checkpoint-cards-en.png)

Each study session can be as small as this:

| Part | Example |
| --- | --- |
| One input | command, file, prompt, question |
| One output | printed line, chart, JSON file, model answer |
| One record | README note, screenshot, error note, metric table |

If all three exist, the session counts.

## When a Chapter Feels Unclear

Do not reread the same page five times. Ask these questions instead:

| Question | Why it helps |
| --- | --- |
| What problem does this page solve? | Finds the purpose |
| What is the input? | Finds where the work starts |
| What is the output? | Finds what success looks like |
| Can I run the smallest example? | Turns reading into action |
| Which project task uses it? | Connects the lesson back to the course |

If you can answer these, continue. Details can be filled in when the project needs them.

## Tiny Review Prompt

Use this only after you have real commands, output, or errors:

```text
I am at stage X of the AI full-stack course.
Here is what I ran today:
[commands, output, error]

Please explain what I learned, classify the blocker layer, and give me one next step that takes less than 30 minutes.
```

The useful part is not “AI explains everything.” The useful part is forcing your learning record into a small, reviewable shape.
