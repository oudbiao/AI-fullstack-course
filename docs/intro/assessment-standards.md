# Competency Checkpoints

![Stage checkpoint cards](/img/course/intro-stage-checkpoint-cards-en.png)

This page answers: “Can I move on?” Do not measure progress only by finished articles. Measure it by visible ability.

## 1. Four Levels

| Level | You can... | Evidence |
|---|---|---|
| Understand | Explain what problem the idea solves | Notes, concept map, short explanation |
| Run | Execute the smallest example yourself | Command, screenshot, runnable script |
| Build | Complete a small project with input and output | README, source code, examples |
| Improve | Evaluate, find failures, and iterate | Metrics, logs, failure samples, change notes |

Early chapters can stop at Run or Build. RAG, Agent, and capstone work should move toward Improve.

## 2. Before Moving On

Ask these five questions:

| Question | Passes when... |
|---|---|
| Can I explain it simply? | You can describe the use case without reading the article |
| Did I run code myself? | You have the command and output saved |
| Did I keep one project artifact? | README, script, notebook, chart, or demo exists |
| Did I record one failure? | You know what broke and why it might have broken |
| Can I rerun it next week? | The command and environment notes are clear |

If two or more answers are weak, strengthen the project evidence before moving on.

## 3. Higher Bar for AI Application Stages

| Stage type | Extra checkpoint |
|---|---|
| Prompt | Prompt versions and fixed test inputs exist |
| RAG | Retrieval logs and citation checks are visible |
| Agent | Tool schema, trace, step limit, and permission boundary are recorded |
| Deployment | Env vars, startup command, logs, and rollback note exist |

Real progress is not knowing more tool names. It is connecting problem, data, model, engineering, evaluation, and review into a loop you can explain.
