# Recommended Learning Path

![Recommended Learning Path Selection Diagram](/img/course/intro-learning-path-selection-en.png)

The course is large. Do not flatten every branch on the first pass. Choose one path, build small outputs, and come back for depth when a project needs it.

## Pick One Default Path

| Your situation | First path | First-pass rule |
| --- | --- | --- |
| Not sure where to start | Beginner completion | Chapters 1-9 in order, one small output per stage |
| Already code comfortably | Fast application | Skim 1-6, focus on Prompt, RAG, Agent, and deployment |
| Need portfolio evidence | Portfolio path | Keep README, screenshots, logs, and failure notes from every project |
| Care about models deeply | Model understanding | Spend more time on math, ML, deep learning, and Transformer |

If you hesitate, choose **Beginner completion**. A complete loop beats a perfect plan.

## Time-Based Shortcut

| Time available | What to do |
| --- | --- |
| 3-5 hours/week | Read core pages only and finish the minimum project task |
| 6-10 hours/week | Follow the main path and write short project notes |
| Job or portfolio sprint | Build first, then fill the theory gaps that the project exposes |
| Research/model focus | Slow down in math, ML, deep learning, and evaluation |

## Do Not Switch Too Often

Switch paths only when the current project exposes a real gap:

| Symptom | Return to |
| --- | --- |
| You cannot run or reproduce code | Tools and Python |
| Your data or citations are unreliable | Data, RAG, and evaluation |
| Model results are hard to explain | Math, ML, and deep learning |
| The Agent is hard to control | Tool calling, traces, permissions, and recovery |

The right rhythm is: read briefly, run something, save evidence, then continue.
