# Stage Transition Guide

![AI main relay map](/img/course/appendix-ai-main-relay-map-en.png)

Each stage adds one missing capability. When the next topic feels sudden, use this page to see what gap it fills.

## Transition Map

| From | To | New capability |
|---|---|---|
| Tools | Python | Express a workflow in code |
| Python | Data analysis | Read files, tables, logs, and messy inputs |
| Data analysis | Machine learning | Turn data into features, targets, metrics, and baselines |
| Machine learning | Deep learning | Learn representations with tensors and neural networks |
| Deep learning | LLM | Understand tokens, embeddings, context, Transformer, and model behavior |
| LLM | RAG | Answer with retrieved sources instead of model memory only |
| RAG | Agent | Use tools, state, traces, and permissions to complete tasks |
| Agent | Capstone | Combine the right pieces into a reproducible system |

## Before You Move On

| Next stage | Check one thing |
|---|---|
| Python | Can you run a script and make one Git commit? |
| Data | Can you read a file and explain its fields? |
| ML | Can you define the target and metric? |
| DL | Can you explain train/test split, loss, overfitting, and baseline? |
| LLM | Can you call an API or run a small text model? |
| RAG | Can you inspect retrieved chunks before generation? |
| Agent | Can you replay tool calls and stop unsafe actions? |
| Capstone | Can another person run your README? |

The course order is practical, not decorative: every later AI application depends on earlier evidence, debugging, and project habits.
