# Course Page Guide

![Recommended learning path selection diagram](/img/course/intro-learning-path-selection-en.png)

This page answers: **which page should I open next?**

## 1. First Pass Order

| Step | Open | Do this |
|---|---|---|
| 1 | [30-Minute AI Quick Experience](/intro/quick-experience) | Run one tiny AI example |
| 2 | [Environment Setup](/intro/environment-setup) | Prepare Python, Git, and a project folder |
| 3 | [AI Full-Stack Capability Map](/intro/ai-fullstack-map) | Read the picture, not every detail |
| 4 | [Recommended Learning Path](/intro/learning-path) | Pick one route |
| 5 | [Chapter 1: Developer Tools](/ch01-tools) | Start building your workspace |

## 2. How Course Pages Work

| Page type | Use it for |
|---|---|
| Intro pages | Quick start, setup, map, route, and terms |
| Stage home page | Why the stage matters and what you will build |
| `0.0 Study Guide and Task Sheet` | Learning order, required tasks, and pass criteria |
| Lessons | Concepts, code, output, and common mistakes |
| Workshop or project | A runnable artifact after the main theory |
| Extra help | FAQ, troubleshooting, portfolio, career, or deeper route choices |

## 3. Default Learning Loop

1. Look at the picture first.
2. Run the smallest code example.
3. Compare the expected output.
4. Save one command, result, error, or screenshot.
5. Return to the stage task.

If you feel lost, ask: is the blocker about tools, code, data, model behavior, LLM app logic, Agent actions, or deployment? Then open the matching chapter or troubleshooting page.
