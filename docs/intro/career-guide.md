# AI Career Direction Guide

![Role-based AI learning paths](/img/course/intro-role-based-paths-map-en.png)

Do not choose a job title before you have built anything. Finish the core foundation first, notice which tasks you enjoy, then choose one direction for your portfolio project.

## Direction Cards

| Direction | You may like it if you enjoy | Course focus | Portfolio proof |
| --- | --- | --- | --- |
| LLM Application Engineer | building products, APIs, and useful workflows | Chapters 7-8 | RAG or LLM app with evaluation |
| AI Agent Engineer | multi-step automation, tools, traces, permissions | Chapter 9 | Agent trace, tool schema, safety boundary |
| Data / ML Engineer | data quality, metrics, experiments, baselines | Chapters 3-6 | model report with metrics and failures |
| CV / Multimodal | images, screenshots, visual results | Chapters 10 and 12 | before/after visual examples and review |
| NLP / RAG | text extraction, search, classification, Q&A | Chapters 7-8 and 11 | labeled examples, citations, evaluation set |
| MLOps / Deployment | making systems reliable and reproducible | deployment and engineering references | Docker/runbook/logs/monitoring notes |

## Quick Self-Check

| Question | If yes, lean toward |
| --- | --- |
| Do I want visible product feedback quickly? | LLM application or Agent |
| Do I like measuring and explaining errors? | Data / ML |
| Do I like images, video, or documents? | CV / multimodal |
| Do I like text, search, and structured extraction? | NLP / RAG |
| Do I like reliability, cost, logs, and deployment? | MLOps / AI engineering |

## When to Decide

| Stage | Decision |
| --- | --- |
| Before Chapter 3 | Do not decide yet; build basic tools and Python/data ability |
| After Chapter 6 | Choose whether you prefer model depth or application building |
| After Chapter 9 | Pick one portfolio direction and make it stronger |
| Capstone | Choose based on available materials and evaluation method |

The safest default is still LLM application + RAG + Agent, because it creates visible projects quickly while still requiring real engineering evidence.
