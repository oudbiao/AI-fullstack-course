# 第零阶段：开发者工具基础

| 信息 | 说明 |
|---|---|
| **预估学时** | 8～12h |
| **前置要求** | 无 |

掌握终端、Git 和开发环境，这是后面所有代码学习的起点。

## 阶段导读

很多新人不是输在模型难，而是输在最前面的工具链不稳。

这一阶段的目标很明确：

1. 会在终端里定位、运行、管理文件
2. 会用 Git 保存历史和回滚代码
3. 会搭出一个稳定可复现的开发环境

如果这三件事不稳，后面你会在“环境坏了、代码丢了、命令不会用”上浪费很多时间。

## 本阶段包含什么

| 章节 | 主题 | 主要解决什么问题 |
|---|---|---|
| 第一章 | 终端与命令行 | 学会进入目录、查看文件、运行命令 |
| 第二章 | Git 版本控制 | 学会提交、查看历史、分支与协作基础 |
| 第三章 | 开发环境配置 | 学会 Python 环境、编辑器、依赖安装 |

## 建议学习顺序

1. 先学第一章“终端与命令行”  
   因为后面所有操作几乎都会在终端里完成。

2. 再学第二章“Git 版本控制”  
   因为你需要养成“边写边保存历史”的习惯。

3. 最后学第三章“开发环境配置”  
   把编辑器、Python 环境、依赖管理一次搭顺。

## 学这一阶段最容易忽略什么

- 以为命令行只是“会一点就够了”
- 写代码但不提交 Git
- 在一个混乱的 Python 环境里装所有东西

这三件事在前面看起来小，到了后面会不断放大。

## 学完后的出口能力

- 能独立用终端完成基本开发操作
- 能用 Git 管理学习项目
- 能为后面课程搭建和切换 Python 环境
- 遇到最基础的环境报错时，知道先查什么
