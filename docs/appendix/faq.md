# 常见问题

这一页不讲概念，而是集中回答新人最常碰到的实际问题。你可以把它当成“遇到犹豫时先翻一页”的附录。

## 1. 数学不好能学 AI 吗？

可以，而且大多数人都是边学边补的，不是先把数学学完才开始。

更适合新人的顺序是：

1. 先把 Python、数据处理、画图跑顺。
2. 再回头看线性代数、概率统计在代码里到底用在哪里。
3. 最后再补公式、推导和更抽象的理解。

如果你一上来就被公式压住，通常不是你不适合，而是顺序不对。

一个更稳的办法是：

- 学矩阵时，同时看 `NumPy` 的数组运算。
- 学梯度下降时，同时看一个最小训练循环。
- 学概率时，同时看分类任务里的损失函数和预测概率。

你的目标不是先“会证明”，而是先“知道这个数学工具在模型里负责什么”。

## 2. 需要买 GPU 吗？

大多数新人一开始不需要。

| 学习阶段 | 主要任务 | 是否必须 GPU | 更现实的建议 |
|---|---|---|---|
| 第 0～4 阶段 | Python、数据分析、数学、传统机器学习 | 不需要 | 个人电脑足够 |
| 第 5～7 阶段 | 深度学习、CV、NLP 入门 | 可选 | 先用 Colab 或云 GPU |
| 第 8A 阶段 | LLM 原理、微调理解 | 视实验而定 | 先读懂流程，再决定是否租卡 |
| 第 8B～10 阶段 | RAG、Agent、应用工程 | 通常不必须 | CPU + API 就能做很多项目 |

如果你是第一次系统学习，优先级通常是：

1. 稳定的电脑环境
2. 足够的内存和硬盘
3. 能长期坚持的学习节奏
4. 最后才是买显卡

## 3. 所有阶段都要学吗？

不一定，但主线建议尽量不断。

更实用的理解方式是：

- 第 0～5 阶段：主干课，建议绝大多数人都学
- 第 6～7 阶段：方向课，帮你理解 CV / NLP 的传统主线
- 第 8A：理解大模型从哪里来
- 第 8B～9：做应用、做工程、做 Agent
- 第 10 阶段：多模态和 AIGC 视野拓展

如果你目标是“尽快做 AI 应用项目”，可以先走：

`第 0～5 阶段 -> 第 8B -> 第 9 阶段`

如果你目标是“模型原理更扎实”，可以走：

`第 0～7 阶段 -> 第 8A -> 第 8B -> 第 9 阶段`

## 4. 一周学多少比较合理？

比起“每天学多久”，更重要的是你能不能连续 12 周以上稳定推进。

| 每周时间 | 节奏建议 | 适合人群 |
|---|---|---|
| 4～6 小时 | 慢速但可长期坚持 | 在职、课业繁忙 |
| 7～10 小时 | 比较理想 | 大多数自学者 |
| 12～18 小时 | 进度快，但容易疲劳 | 有明确项目目标的人 |

新人常见误区是第一周猛冲，第三周就断掉。更稳的方式是：

- 工作日每天 1 小时
- 周末做 1 次复盘和 1 次代码练习
- 每 2～3 周做一个小项目或实验总结

## 5. 什么时候开始做项目最合适？

越早越好，但项目难度要跟阶段匹配。

比较合适的节奏是：

- 学完 Python 基础后：做小脚本、小工具
- 学完数据分析后：做数据清洗和可视化项目
- 学完机器学习后：做一个完整分类或回归项目
- 学完深度学习后：做训练闭环项目
- 学到 RAG / Agent 后：做真正可展示的应用项目

不要把“项目”理解成一定要做大系统。对新人最有价值的是：

- 项目目标清楚
- 输入输出明确
- 能跑通最小闭环
- 你自己讲得明白为什么这么做

## 6. 代码总是跑不起来怎么办？

先不要急着怀疑自己，大多数问题都出在环境、路径、依赖版本和输入数据。

你可以先按这个顺序排：

1. 看报错的最后 10 行，不要只看第一行。
2. 确认你是不是在正确的 Python 环境里。
3. 确认依赖有没有装到当前环境。
4. 确认文件路径、数据路径、模型路径是不是写对了。
5. 把问题缩小成一个最小可复现示例。

建议优先掌握这些排查命令：

```bash
python --version
which python
pip --version
pip list
pwd
ls
```

如果你还没形成排障习惯，可以同时看附录里的 [学习卡点救援](./troubleshooting.md)。

## 7. 看不懂论文是不是就学不好 AI？

不是。

对新人来说，论文应该是“补充理解和拓展视野”，不是学习入口。

更推荐的顺序是：

1. 先看教程，把任务、输入输出、评估方式搞清楚。
2. 再看实现或最小代码。
3. 最后带着问题去看论文。

如果一开始就硬啃论文，很容易出现两个问题：

- 只记住术语，没有任务直觉
- 觉得自己什么都不会，从而越来越不敢动手

## 8. 学到什么程度可以开始找工作或接项目？

一个比较现实的标准不是“我是不是把所有知识学完了”，而是你是否具备下面这 4 个能力：

1. 能独立搭环境并跑通一个中等复杂度项目
2. 能解释项目的数据流、模型选择、评估方式和失败案例
3. 能读懂常见 Python / ML / LLM 工程代码
4. 能把需求拆成可执行的小任务并逐步完成

如果你已经有 2～3 个自己能讲清楚的项目，其实就可以开始准备简历和面试了，而不是等“全学完”。

## 9. 我总觉得自己学得慢，怎么办？

这几乎是所有自学者都会经历的阶段。

你真正应该比较的不是“别人一天学了多少”，而是：

- 这个月的我是不是比上个月更能看懂代码
- 我是不是开始能自己改示例，而不是只会复制
- 我是不是越来越能把一个概念解释给别人听

如果这三个问题的答案越来越偏向“是”，你就在进步。

## 10. 遇到卡点时，先补哪三样最有效？

如果你现在已经有点乱，最值得优先补的是：

1. Python 基础和调试能力
2. 数据处理和可视化能力
3. 一个完整项目的最小闭环意识

很多人以为自己缺的是“更高级的模型”，其实真正缺的是：

- 看懂输入输出
- 会查错
- 会拆问题
- 会验证结果

这四件事补起来之后，后面的学习会顺很多。
