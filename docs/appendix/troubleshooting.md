# 学习卡点救援

这页的目标不是解释理论，而是帮你在“卡住的时候尽快恢复推进”。很多学习中断，不是因为太难，而是因为一个小问题拖了太久。

## 1. 先判断你卡在哪一类

| 现象 | 最可能的问题类型 | 第一反应 |
|---|---|---|
| `ModuleNotFoundError` | 环境或依赖没装对 | 先查当前 Python 环境 |
| 代码能跑但结果很怪 | 输入、标签或评估理解错了 | 先打印中间结果 |
| 训练很慢或显存爆了 | batch 太大、模型太重、设备不匹配 | 先缩小实验规模 |
| 一章看完还是不会做题 | 概念没有和代码绑定 | 回到最小例子重跑 |
| 项目不知道怎么开始 | 题目太大，没有拆分 | 先画最小闭环 |
| 学了很多但记不住 | 缺复盘和输出 | 做笔记和小总结 |

## 2. 环境问题怎么查

环境问题最容易把人打懵，但其实很多都是重复问题。

先跑这一组命令：

```bash
python --version
which python
pip --version
pip list
pwd
ls
```

如果你有 GPU，再补一条：

```bash
nvidia-smi
```

### 2.1 最常见的环境问题

#### 问题一：包装了，但导入失败

通常原因：

- 你装到了别的 Python 环境
- 终端和 IDE 用的不是同一个解释器
- `pip` 指向的不是当前环境

更稳的做法是：

```bash
python -m pip install numpy
python -m pip install pandas
```

这样能减少“装错环境”的概率。

#### 问题二：路径找不到

先确认：

- 当前工作目录是不是你以为的那个目录
- 相对路径是不是基于当前目录
- 文件名大小写有没有写错

可以先临时打印：

```python
from pathlib import Path

print(Path.cwd())
print(Path("data").exists())
```

#### 问题三：版本冲突

如果你已经开始反复遇到依赖问题，优先做两件事：

1. 把常用依赖写进统一环境文件
2. 每个大方向尽量用独立虚拟环境

不要把所有实验都塞在一个脏环境里。

## 3. 代码能跑，但你不知道它是不是对的

这类问题比“直接报错”更危险。

### 3.1 先查输入

你至少要确认：

- 数据条数对不对
- 输入 shape 对不对
- 标签范围对不对
- 有没有空值、乱码、重复样本

示例：

```python
print("样本数:", len(texts))
print("前两条文本:", texts[:2])
print("前两条标签:", labels[:2])
print("标签集合:", sorted(set(labels)))
```

### 3.2 再查中间结果

不要只看最终准确率。更重要的是：

- 模型到底输出了什么
- 概率是否合理
- 中间特征是不是全 0 或全一样

### 3.3 最后查评估方式

常见错误：

- 训练集和测试集混了
- 类别极不平衡却只看准确率
- 文本任务只看单条样例，不看整体错误分布

## 4. 训练不收敛怎么办

先别一下子怀疑模型结构，很多时候问题没那么复杂。

可以按这个顺序查：

1. 先拿极小数据集试试能不能过拟合
2. 再看学习率是不是太大或太小
3. 再看损失函数和标签格式是不是匹配
4. 最后才去怀疑模型结构和更复杂技巧

如果一个模型连很小的一批数据都学不会，通常要先查：

- 输入有没有喂错
- 标签有没有错位
- 损失函数是否选错
- 优化器和学习率是否离谱

## 5. 显存不够怎么办

最直接的顺序是：

1. 减小 `batch size`
2. 降低输入尺寸
3. 先关掉不必要的日志和缓存
4. 用更小的模型或只训练头部
5. 需要时再用梯度累积、混合精度

很多新人一上来就想“换更大显卡”，但先缩小实验规模通常更有效。

## 6. 项目不知道怎么落地怎么办

如果你感觉“什么都会一点，但不知道怎么做项目”，通常是因为题目太大。

一个更稳的拆法是：

### 6.1 先写一句话目标

例如：

- 做一个能回答公司制度问题的知识库助手
- 做一个能识别图片类别的小系统
- 做一个能给文章生成摘要的工具

### 6.2 再拆成最小闭环

最小闭环一般只包含：

1. 一个明确输入
2. 一个可运行处理流程
3. 一个可观察输出
4. 一种基本评估方式

### 6.3 最后再逐步补功能

常见补充项：

- 日志
- 错误处理
- 缓存
- 用户界面
- 评估集
- 部署

## 7. 学习很焦虑、总觉得跟不上怎么办

先记住一件事：AI 领域更新快，但基础能力更新没那么快。

真正应该稳住的能力是：

- 读代码
- 查错
- 拆问题
- 做小闭环
- 解释自己的方案

如果你已经在变得更会这五件事，就不算落后。

## 8. 提问时怎样更容易得到有效帮助

不要只发一句“报错了，帮我看看”。更好的提问模板是：

```text
我在做什么：
我期望看到什么：
实际发生了什么：
完整报错最后 20 行：
我已经试过哪些方法：
最小可复现代码：
```

这会让别人更容易帮你，也会逼你自己先把问题说清楚。

## 9. 一份最小复现模板

当你怀疑自己的大项目太乱时，先把问题缩成下面这种体量：

```python
def predict(x):
    return x * 2

data = [1, 2, 3]
preds = [predict(x) for x in data]
print(preds)
```

然后一点点把真实逻辑加回来。这样做的意义是：你可以快速定位，到底是哪一层加进去后问题开始出现。

## 10. 什么时候该暂停，什么时候该硬啃？

适合暂停的情况：

- 你已经连续半小时都在乱试
- 你开始机械复制命令，不知道自己在做什么
- 你已经看不懂报错信息

适合继续硬啃的情况：

- 问题已经缩小到 1～2 个可验证假设
- 你知道下一步要试什么
- 你能解释每次修改的目的

真正高效的学习，不是一直顶着冲，而是知道什么时候该停下来整理。
