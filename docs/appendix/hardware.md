# 硬件与云资源指南

这页不是让你“堆配置”，而是帮你判断：在不同阶段，到底要不要买设备、什么时候租云更划算、什么配置是真的影响学习体验。

## 先看结论

- 第 0～4 阶段基本不用 GPU。
- 第 5～7 阶段可以先用云 GPU，不必急着买显卡。
- 第 8B～9 阶段大量项目其实更依赖工程能力，不一定依赖本地大显卡。
- 新人花钱优先级通常是：内存 > 硬盘 > 稳定环境 > GPU。

## 1. 按阶段看硬件需求

| 阶段 | 常见任务 | 本地最低建议 | 更舒适配置 | 云端替代 |
|---|---|---|---|---|
| 第 0～2 阶段 | Python、数据清洗、画图、Notebook | 8GB 内存 / 256GB SSD | 16GB / 512GB SSD | 不需要 |
| 第 3～4 阶段 | 数学实验、传统机器学习 | 8GB 内存 | 16GB 内存 | 不需要 |
| 第 5 阶段 | PyTorch、CNN、RNN、Transformer 基础 | 16GB 内存 | 16GB+ / 512GB SSD | Colab / 云 GPU |
| 第 6～7 阶段 | CV、NLP 实验 | 16GB 内存 | 16GB+ / 外接显示器更舒服 | Colab / AutoDL / 云服务器 |
| 第 8A 阶段 | LLM 原理、微调概念实验 | 16GB 内存 | 32GB 内存更好 | 云 GPU 更现实 |
| 第 8B～9 阶段 | RAG、Agent、应用开发 | 16GB 内存 | 16GB+ / 稳定网络 | CPU + API 就能做大部分项目 |
| 第 10 阶段 | 多模态试玩、项目拼装 | 16GB 内存 | 32GB 内存更顺 | 图像视频生成更适合云端 |

## 2. 三档设备方案

### 2.1 够用档

适合人群：

- 刚开始系统学习
- 预算有限
- 主要先跑教学示例和中小项目

建议：

- 16GB 内存
- 512GB SSD
- 普通 CPU 笔记本或台式机
- 需要 GPU 时临时用云

### 2.2 舒适档

适合人群：

- 会频繁跑本地实验
- 同时开浏览器、IDE、Notebook、终端
- 希望长时间学习体验稳定

建议：

- 32GB 内存
- 1TB SSD
- 多核 CPU
- 必要时配中端独显，或者继续以云 GPU 为主

### 2.3 重度实验档

适合人群：

- 明确要做训练实验
- 经常跑较大的视觉或深度学习任务
- 需要本地快速迭代

建议：

- 32GB～64GB 内存
- 1TB SSD 以上
- 较新显卡
- 但要提前想清楚：这是不是学习需要，还是“设备焦虑”

## 3. 对新人最值钱的配置，不一定是显卡

很多人一开始最容易高估 GPU 的价值，低估内存和硬盘的重要性。

更影响体验的通常是：

1. 内存够不够  
   浏览器、VS Code、Jupyter、终端一起开时，16GB 和 8GB 的差距非常明显。

2. SSD 够不够  
   数据集、模型缓存、虚拟环境都很占空间。硬盘太小会让你频繁清理，学习节奏容易被打断。

3. 环境稳定不稳定  
   同样一台机器，如果环境混乱，你会把大量时间浪费在排错上。

4. 显示器和输入设备顺不顺手  
   长时间看代码和文档，外接显示器、键盘和鼠标带来的提升很实在。

## 4. 云资源怎么选更合理

### 4.1 免费或低成本试玩

适合：

- 跑课程示例
- 做小实验
- 熟悉 GPU 训练流程

特点：

- 上手快
- 成本低
- 不适合长期稳定训练

### 4.2 按小时租用云 GPU

适合：

- 训练时间较长
- 需要更稳定的显存和算力
- 想控制成本，不想一次性买硬件

建议：

- 先在小数据上跑通流程，再上云
- 开机前准备好代码、数据和实验计划
- 训练结束及时导出结果、关机止损

### 4.3 API 优先的应用路线

适合：

- 做 RAG、Agent、智能助手
- 更关注产品和工程，而不是自己训练底座模型

这种路线通常不需要本地 GPU，重点在于：

- 接口封装
- 日志和监控
- 检索和工具调用
- 用户体验和成本控制

## 5. 什么时候值得考虑买本地 GPU？

只有当下面至少满足两条时，买本地 GPU 才更有意义：

- 你已经确定会长期做训练实验
- 你经常被云端排队和时长限制打断
- 你对本地迭代速度有明确需求
- 你已经知道自己主要跑哪些任务、需要多大显存

如果你还停留在“也许以后会用到”，先别急着买。

## 6. 常见误区

### 6.1 误区一：没有显卡就学不了 AI

这是最常见的误解。前半段很多能力和显卡无关，后半段大量应用工程项目也可以靠 API 和云资源完成。

### 6.2 误区二：先把设备买齐再开始

学习最怕的不是设备普通，而是迟迟不开始。先用现有设备推进，遇到真实瓶颈再升级，通常更合理。

### 6.3 误区三：为了显卡牺牲整体稳定性

如果你买了高配 GPU，但内存、硬盘、散热、系统稳定性都跟不上，实际体验可能并不好。

## 7. 一份现实的购买建议

如果你现在正准备开始，最稳的方案通常是：

- 先用现有电脑完成第 0～4 阶段
- 第 5～7 阶段按需租云 GPU
- 第 8B～9 阶段优先做应用项目
- 等你明确自己会长期做训练，再决定要不要升级本地硬件

这比一开始把预算全部砸在设备上更理性，也更适合长期学习。
