# 求职准备清单

这一页不是让你“学完再找工作”，而是帮你把学习过程逐步转换成能投递、能面试、能讲清楚的成果。

## 1. 先判断你想投什么岗位

不同方向，准备重点不同。

| 岗位方向 | 更看重什么 | 你应该重点准备什么 |
|---|---|---|
| AI / 算法工程 | 模型理解、训练、评估 | 机器学习、深度学习、实验项目 |
| LLM 应用工程 | 产品闭环、RAG、Agent、后端工程 | 项目完整度、接口设计、日志和评估 |
| 数据科学 / 数据分析 | 数据处理、统计、建模、可视化 | SQL、Python、分析报告、业务表达 |
| AI 产品 / 技术产品 | 场景理解、需求拆解、评估与协作 | 项目方案、产品判断、沟通能力 |

如果你是新人，不要试图同时准备所有方向，先选一个主目标更有效。

## 2. 简历里最有分量的东西是什么

对大多数新人来说，最有价值的通常不是“看过多少课”，而是下面三样：

1. 2～3 个你能讲清楚的项目
2. 一套稳定的技术栈和工具链
3. 明确可验证的结果

项目比“学过 XXX、了解 XXX”更有说服力，因为项目能同时展示：

- 动手能力
- 排障能力
- 取舍能力
- 表达能力

## 3. 项目怎么选才对求职有帮助

比起做很大的题目，更重要的是做“边界清楚”的题目。

更合适的项目通常满足这些条件：

- 有明确输入输出
- 有 baseline
- 有评估方式
- 有失败案例
- 有你自己的改进点

例如：

- 图像分类系统
- 文本分类 / 情感分析项目
- 企业知识库问答系统
- 智能助手或 Agent 工作流
- 数据分析 Agent

## 4. 简历项目应该怎么写

不要只写“使用了某某框架完成某项目”。更好的写法是：

### 4.1 先写问题

项目解决什么问题？面向谁？输入输出是什么？

### 4.2 再写你的方案

你做了哪些模块？为什么这样设计？

### 4.3 最后写结果

结果可以是：

- 指标提升
- 处理速度提升
- 召回质量改善
- 用户体验变好
- 部署或维护成本下降

### 4.4 一个简历表述对比

不够好的写法：

```text
使用 Python 和 LangChain 做了一个知识库问答系统。
```

更好的写法：

```text
基于 Python 构建企业知识库问答系统，完成文档切分、向量检索、权限过滤和答案引用链路；
设计评估集对比不同 chunk 策略，减少误召回并提升答案可追溯性。
```

第二种写法更能体现你真的做过。

## 5. GitHub 项目页怎么准备

一个能加分的项目仓库，至少应包含：

- `README`
- 运行方法
- 项目结构说明
- 示例输入输出
- 截图或效果图
- 已知问题和后续计划

如果别人点开你的仓库，3 分钟内应该能知道：

- 这是干什么的
- 怎么跑
- 你做了哪些关键部分
- 效果大概怎样

## 6. 面试前要补哪些基础

### 6.1 通用基础

- Python 基础
- 数据结构与常见复杂度意识
- Git、环境管理、常见命令行操作
- HTTP / API 基础

### 6.2 机器学习 / 深度学习基础

- 常见任务和指标
- 过拟合与欠拟合
- 训练集、验证集、测试集
- 优化器、学习率、损失函数

### 6.3 LLM / Agent 方向基础

- Prompt、RAG、Fine-tuning 的区别
- 向量检索和 chunking 的作用
- 工具调用为什么需要 schema 和校验
- Agent 为什么需要状态、记忆和监控

## 7. 项目面试时最常被问什么

高频问题通常不是“你用了哪个框架”，而是：

- 为什么选这个方案
- 有没有 baseline
- 遇到过什么失败
- 怎么评估效果
- 如果线上出问题，你先查什么

所以准备时要能讲清：

1. 你的项目目标
2. 你的最小闭环
3. 你的关键设计
4. 你的评估方法
5. 你的失败案例和改进方向

## 8. 一份投递前自查清单

- 我的简历是否能在 30 秒内看懂主线
- 我是否至少有 2 个能深聊的项目
- 我的 GitHub 是否有 README 和运行说明
- 我是否能口头解释自己的项目架构
- 我是否能说出一个做错过但后来修正的问题
- 我是否知道目标岗位更看重什么

如果这 6 条里大多数还不稳，优先补项目和表达，而不是继续无止境囤知识。

## 9. 一个四周求职冲刺建议

### 第 1 周

- 定岗位方向
- 梳理已有项目
- 选 2～3 个作为主打项目

### 第 2 周

- 补 README、截图、运行方式
- 重写简历项目描述
- 开始准备项目讲解稿

### 第 3 周

- 刷通用基础题
- 针对岗位补 ML / LLM / 工程基础
- 做模拟面试

### 第 4 周

- 开始投递
- 记录常被问到的问题
- 根据反馈继续修项目和表达

## 10. 最重要的一件事

对新人来说，求职准备不是把知识背完，而是把“学习痕迹”整理成“别人能看懂的成果”。

你最终拿出去竞争的，不是你看过多少资料，而是：

- 你做过什么
- 你理解到什么程度
- 你能不能讲清楚
- 你能不能继续把事情做下去
