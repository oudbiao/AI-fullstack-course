# 项目：医学影像分析【选修】

:::tip 本节定位
医学影像项目和普通视觉项目最大的不同在于：

- 错误代价更高
- 数据更稀缺
- 标注更昂贵

所以这类项目特别适合用来理解：

> **高风险视觉系统为什么必须同时重视模型、数据和评价标准。**
:::

## 学习目标

- 理解医学影像项目为什么要格外重视数据与评估
- 学会定义一个最小医学影像任务边界
- 通过可运行示例建立项目规划直觉
- 理解高风险视觉项目的展示重点

---

## 一、为什么医学影像项目不能照搬普通视觉套路？

### 1. 数据更少、更贵

### 2. 标注更依赖专家

### 3. 错误后果更严重

这意味着：

- 不能只追求炫酷 demo
- 更要关注风险边界和评价指标

---

## 二、一个最小项目规划示例

```python
from dataclasses import dataclass, field


@dataclass
class MedicalProject:
    task: str
    input_type: str
    labels: list
    metrics: list
    risks: list = field(default_factory=list)


project = MedicalProject(
    task="肺部病灶区域分割",
    input_type="CT slice",
    labels=["background", "lesion"],
    metrics=["dice", "iou", "sensitivity"],
    risks=["标注不一致", "类别极度不平衡", "假阴性代价高"],
)

print(project)
```

### 2.1 这个例子最关键的地方是什么？

它提醒你医学影像项目一开始就该把：

- 任务
- 标签
- 指标
- 风险

写清楚。

尤其是：

- 假阴性代价高

这种约束，往往直接影响你怎么看模型。

---

## 三、真实项目里还应该补什么？

### 3.1 数据与标注协议

至少要明确：

- 标注由谁做
- 是否有多专家复核
- 如何处理标注分歧

### 3.2 指标优先级

这类项目通常不会只看一个总准确率。  
更常见会一起看：

- sensitivity / recall
- specificity
- dice / IoU

### 3.3 风险边界说明

项目页最好显式写清：

- 这是辅助系统，不替代临床判断
- 哪类样本不适用
- 是否需要人工复核

### 3.4 结果展示方式

更有说服力的展示通常包括：

- 原图
- 标注 mask / 预测 mask
- 典型失败案例

---

## 四、最容易踩的坑

### 4.1 只看总体准确率

在医学场景里，这常常远远不够。

### 4.2 忽略标注一致性

如果专家之间本身分歧很大，  
模型上限就会受影响。

### 4.3 只做模型，不谈风险边界

高风险项目必须同时交代：

- 适用边界
- 人工复核
- 不能单独替代临床决策

---

## 小结

这节最重要的是建立一个高风险项目判断：

> **医学影像项目不只是视觉建模问题，更是数据质量、指标设计和风险控制共同决定的系统问题。**
