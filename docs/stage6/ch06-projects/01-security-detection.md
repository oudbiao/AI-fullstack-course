# 项目：智能安防系统

:::tip 本节定位
安防项目是很典型的“模型 + 系统”任务。  
它不只是检测一个人，而是要考虑：

- 告警时机
- 误报
- 实时性
- 跟踪一致性

所以它非常适合拿来练视觉系统的完整闭环。
:::

## 学习目标

- 学会定义一个安防检测项目的最小范围
- 学会把检测结果转成告警逻辑
- 通过可运行示例理解规则和模型的协作
- 建立视觉系统项目展示思路

---

## 一、项目目标怎么定才稳？

建议先从非常清楚的问题开始，例如：

- 检测禁区内是否有人进入

而不是一开始就做特别多目标和特别复杂规则。

---

## 二、先跑一个最小告警逻辑示例

```python
detections = [
    {"frame": 1, "label": "person", "box": (40, 40, 80, 120)},
    {"frame": 2, "label": "person", "box": (42, 42, 82, 122)},
    {"frame": 3, "label": "person", "box": (150, 150, 180, 210)},
]

danger_zone = (30, 30, 100, 140)


def is_inside(box, zone):
    bx1, by1, bx2, by2 = box
    zx1, zy1, zx2, zy2 = zone
    return bx1 >= zx1 and by1 >= zy1 and bx2 <= zx2 and by2 <= zy2


alerts = []
for det in detections:
    if det["label"] == "person" and is_inside(det["box"], danger_zone):
        alerts.append({"frame": det["frame"], "alert": "intrusion"})

print(alerts)
```

### 2.1 这个例子为什么有工程味？

因为它把：

- 检测结果
- 区域规则
- 告警输出

串起来了。  
这正是很多安防系统最小闭环。

---

## 三、真实项目里还应该补什么？

### 3.1 数据与标注

至少要说清：

- 监控画面来自哪里
- 标注的是哪几类对象
- 禁区规则怎么定义

### 3.2 评估

建议至少同时看：

- 检测本身的召回 / 精确率
- 告警级误报率
- 告警延迟

### 3.3 去重与跟踪

安防场景一个很真实的问题是：

- 同一个人连续 30 帧都被检测到

如果你没有去重逻辑，系统会疯狂重复报警。

### 3.4 部署展示

这个项目做作品集时，最有说服力的通常不是一张静态截图，而是：

- 一个短视频片段
- 告警轨迹
- 一组误报 / 漏报分析

---

## 四、最容易踩的坑

### 4.1 误报过多

如果误报太多，真实用户很快就不信系统。

### 4.2 只看模型精度，不看告警逻辑

项目真正交付的是告警质量，  
不是裸检测框质量。

### 4.3 实时性不达标

安防场景对时延通常比普通离线分析更敏感。

---

## 小结

这节最重要的是建立一个项目判断：

> **安防系统的关键，不只是检测到目标，而是把检测结果稳定地转成可信告警。**
