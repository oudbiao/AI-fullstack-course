# 第六阶段：计算机视觉

| 信息 | 说明 |
|---|---|
| **预估学时** | 120～180 小时 |
| **前置要求** | 完成第五阶段 |

## 阶段概述

深入掌握计算机视觉核心任务与前沿技术

## 阶段导读

这一阶段建议把视觉任务按“粒度”来学：

1. 先看图像本身和基础处理
2. 再看整图分类
3. 再看框级检测
4. 最后看像素级分割和高级视觉任务

## 第五阶段是怎么自然流到这里的

如果你刚学完第五阶段，最值得先看清的一件事是：

- 第五阶段已经教会你神经网络怎么训练
- 第六阶段开始回答“这些网络怎么真正去看图”

更直白地说：

- 第五阶段更强调“模型怎么学起来”
- 第六阶段开始强调“视觉任务到底在学什么、输入输出长什么样、指标怎么评”

你可以先把这条过渡记成一句话：

> 第五阶段给了你卷积网络和训练闭环，第六阶段开始把它们放进真正的视觉任务里。

## 这一阶段的教学安排是否由浅入深？

整体上是顺的，而且对第一次学 CV 的人来说，这种按任务粒度推进的方式是合理的。

更适合新人的理解主线是：

```mermaid
flowchart LR
    A["视觉基础"] --> B["图像分类<br/>先学整图判断"]
    B --> C["目标检测<br/>再学目标位置"]
    C --> D["图像分割<br/>再学像素级理解"]
    D --> E["高级视觉专题"]
    E --> F["项目实践"]
```

也就是说：

- **前四章是主干**
- **第五章更像扩展视野**
- **第六章项目要在你已经分清分类 / 检测 / 分割之后再做**

## 这一阶段真正新增了什么能力

从第五阶段走到第六阶段，真正新增的不是“又多了一批模型名”，而是：

1. 开始把图像任务按输出粒度拆开理解
2. 开始更认真地区分分类、检测、分割这三条主线
3. 开始更频繁地接触视觉特有的评估和可视化方式

### 建议学习顺序

1. 第一章：视觉基础
2. 第二章：图像分类
3. 第三章：目标检测
4. 第四章：图像分割
5. 第五章：高级视觉
6. 第六章：项目实践

## 更适合新人的学习节奏

如果你是第一次系统学视觉，更稳的节奏通常是：

1. 先学第一章  
   把图像表示、像素、颜色、OpenCV 基础和最基本处理打稳。

2. 再学第二章  
   先用图像分类建立“视觉模型到底在看什么”的第一层感觉。

3. 然后学第三章  
   先把 IoU、边界框、检测任务的输入输出看懂。

4. 再学第四章  
   这时再进入像素级任务，会更容易知道“检测和分割到底差在哪”。

5. 第五章按需回看  
   OCR、视频、人脸、3D 更适合作为方向拓展。

6. 最后做第六章项目  
   把数据、模型、指标和错误分析真正串起来。

## 如果你是第一次系统学视觉，最稳的起步方式

建议先不要急着追：

- 哪个检测器最新
- 哪个分割模型更强
- 哪个视觉大模型最火

更稳的顺序通常是：

1. 先把图像本身的表示看懂  
   像素、通道、颜色空间、尺寸这些东西要先有感觉。

2. 先用分类任务建立第一层视觉直觉  
   因为它最容易看清“模型到底在整张图里学什么”。

3. 再进入检测和分割  
   这时你更容易理解：
   为什么检测要看框，分割要看 mask。

## 本阶段章节地图

| 章节 | 主题 | 主要解决什么问题 |
|---|---|---|
| 第一章 | 视觉基础 | 理解图像表示、像素、颜色、卷积和 OpenCV 基础 |
| 第二章 | 图像分类 | 学整图分类任务、迁移学习和分类训练 |
| 第三章 | 目标检测 | 学边界框、IoU、mAP 和检测模型主线 |
| 第四章 | 图像分割 | 学像素级预测、语义分割和实例分割 |
| 第五章 | 高级视觉 | 学 OCR、视频、人脸与 3D 等拓展方向 |
| 第六章 | 项目实践 | 把视觉任务真正做成可评估的小系统 |

### 学这一阶段时最容易卡住的地方

- 把分类、检测、分割三类任务混在一起
- 只记模型名，不理解评估指标
- 只看样例图，不做错误分析

## 学这一阶段最该养成的 3 个习惯

1. 每学一个任务，先问输入是什么、输出是什么
2. 每学一个任务，先问评估指标在量什么
3. 每看一个失败例子，都先问模型到底错在分类、定位还是边界

## 适合怎么学这阶段

- 如果你是第一次学视觉，先以任务为主，不要以模型名为主
- 每学一个任务，都先搞清输入、输出和评估
- 做项目时，优先完成一个最小闭环，再追求更高指标

## 这一阶段最值得优先补强的能力

- 能从一张图想清楚它是分类、检测还是分割问题
- 能看懂 shape、框、mask 各自在表达什么
- 能根据任务选对评估指标
- 能通过错例可视化去真正理解模型问题

### 学完后的出口能力

- 能区分 CV 三大主线任务及其输入输出
- 能看懂 IoU、mAP、分割 IoU 这类核心指标
- 能完成一个最小视觉项目并做基本评估

## 学完第六阶段后，你应该能自己回答什么

- 一张图的问题到底是分类、检测还是分割
- 为什么视觉任务不能只靠“展平后做全连接”
- 为什么检测比分类多了一层“位置”难度
- 为什么分割比检测又更细一层
- 我现在最该用什么最小项目来练视觉闭环
