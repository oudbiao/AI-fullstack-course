# 学前导读：LLM 概览这一章到底在学什么

这一章解决的是：

> **第一次系统接触大模型时，先把地图建立起来。**

## 零、先建立一张桥接线

如果你刚学完 NLP 核心速成，这一章最值得先看清的一件事是：

- 前面你已经知道文本、token、embedding 和预训练模型这些最小对象
- 这一章开始把它们放进“大模型时代的全景图”里

所以这一章不是在讲某一个具体模型，而是在回答：

> **大模型这条主线到底是怎么长出来的，我们现在站在哪一段。**

## 这一章的主线

```mermaid
flowchart LR
    A["发展历史"] --> B["核心概念"]
    B --> C["产业格局"]
```

## 这一章更适合新人的学习顺序

1. 先看发展历史  
   先知道为什么会从传统 NLP 一路走到大模型。

2. 再看核心概念  
   先把参数、上下文长度、能力、对齐这些词放对位置。

3. 最后看产业格局  
   这时再去理解开源 / 闭源、模型家族和应用生态会更顺。

## 这一章最该先抓住什么

- 这一章的目标是“建地图”，不是“抄定义”
- 先把路线图看顺，比一开始就死磕某个模型细节更重要
- 后面 Transformer 深入和预训练章节，都会建立在这里的地图感之上
