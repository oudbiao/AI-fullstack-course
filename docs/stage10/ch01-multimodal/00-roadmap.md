# 学前导读：多模态基础这一章到底在学什么

这一章解决的是：

> **真实世界不是单模态的，系统怎样把不同形式的信息拼起来理解。**

## 零、先建立一张桥接线

如果你是从前面的 LLM 和 Agent 主线过来的，这一章最值得先看清的一件事是：

- 前面系统多数还在围绕文本工作
- 这一章开始回答：如果输入和输出不止一种形式，系统要怎样统一理解和组织

所以这一章真正重要的不是“多模态这个词”，而是：

> **开始把不同模态放进同一条理解链路里。**

## 这一章的主线

```mermaid

    A["多模态基础"] --> B["视觉语言模型"]
```

## 这一章更适合新人的学习顺序

1. 先把“模态”看清楚  
   先知道文本、图像、音频为什么不能直接混成一团。

2. 再看对齐和融合  
   先把多模态里最核心的两个动作放对位置。

3. 最后再看视觉语言模型和应用  
   这时你更容易理解图文问答、图文检索这些系统到底在干什么。

## 这一章最该先抓住什么

- 多模态不是“图片加文字”这么简单
- 对齐和融合是后面所有多模态系统的基础动作
- 这一章会成为后面图像生成和多模态产品工作流的入口
