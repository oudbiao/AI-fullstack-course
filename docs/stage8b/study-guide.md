# 学习指南：大模型应用与工程怎么学最不容易学乱

:::tip 这页适合什么时候看
如果你来到第八 B 阶段后开始觉得：

- 框架很多
- 工程名词很多
- 不知道该先做知识库、部署，还是应用功能

那这页就是给你的。
:::

## 这一阶段最重要的总原则

第八 B 阶段最容易学乱的原因，不是组件太多，而是容易没有分层意识。

更适合新人的主线其实是：

> **先把知识层、模型层、应用层、工程层分开，再看系统怎么接起来。**

## 最推荐的阅读顺序

1. 先学 RAG  
   先把知识怎么接进来搞清楚。

2. 再学部署  
   先把模型怎么稳定调用搞清楚。

3. 再学应用开发  
   先把会话、函数调用、结构化输出接顺。

4. 再学工程化  
   并发、日志、监控、部署这时才更自然。

5. 最后做项目

## 这一阶段最容易卡住的地方

### 1. 一上来就追框架

先把：

- 文档怎么进来
- 模型怎么调用
- 结果怎么返回

这条链走顺。

### 2. 把 RAG、Agent、应用逻辑混在一起

先分清：

- RAG 是知识增强
- 应用层是功能组织
- Agent 是更动态的系统编排

### 3. 只做 Demo，不补工程

日志、监控、超时、回退，这些不是可选题。

## 第一次做 LLM 应用，最稳的默认顺序

1. 先做最小知识库或 API 调用闭环
2. 再做会话或工具调用
3. 再补工程化能力
4. 最后再考虑更复杂系统

## 如果你想做“知识库驱动的课件生成助手”，最短学习路径是什么？

这个项目最稳的最短路径通常是：

1. 先学 `RAG 基础 + 文档处理`
2. 再学 `函数调用 + 对话系统`
3. 再学新增的：
   - [文档解析与知识抽取](./ch03-app-dev/07-document-parsing.md)
   - [模板化文档生成（Word / PPT）](./ch03-app-dev/08-template-doc-generation.md)
4. 最后做：
   - [项目：知识库驱动的课件生成助手](./ch05-projects/04-courseware-assistant.md)

如果你后面想把它继续做成更动态的系统，再往第九阶段补：

- 工具调用
- 记忆
- Agent 编排

会更顺。

## 如果你准备进入第九阶段，最值得先带走什么

进入 Agent 之前，最值得先带走的是：

1. 层次分明的系统观
2. 稳定的模型调用方式
3. 最小可维护的日志和评估意识

## 一句话版的过关标准

- 能把知识、模型、应用、工程分成四层来看
- 能完成一个最小 LLM 应用闭环
- 不再把“能跑起来”和“能上线”混为一谈

如果这三件事已经开始成立，第八 B 阶段就已经学得很值了。
