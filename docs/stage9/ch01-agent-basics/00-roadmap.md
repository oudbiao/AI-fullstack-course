# 学前导读：Agent 基础这一章到底在学什么

这一章解决的是：

> **先把 Agent 的边界、能力和结构说清楚。**

## 零、先建立一张桥接线

如果你是从第八 B 阶段过来的，这一章最值得先看清的一件事是：

- 前面你已经会做“带模型的应用系统”
- 这一章开始回答“什么时候系统会从应用，跨进 Agent”

所以这一章真正重要的不是定义本身，而是：

> **先把 Agent 和聊天系统、工作流、工具调用系统之间的边界讲清楚。**

## 这一章的主线

```mermaid
flowchart LR
    A["什么是 Agent"] --> B["发展脉络"]
    B --> C["能力层级"]
    C --> D["系统结构"]
```

## 这一章更适合新人的学习顺序

1. 先把“什么是 Agent”看清楚  
   先把边界立住。

2. 再看发展脉络  
   先知道它为什么不是突然凭空出现的。

3. 再看能力层级  
   先把“会回答、会调用工具、会规划、会协作”放进同一条线里。

4. 最后看系统结构  
   这时你更容易理解模块为什么会长成那个样子。

## 这一章最该先抓住什么

- Agent 不是模型名，而是一种系统组织方式
- 这一章是整个第九阶段的边界页
- 如果这里没稳，后面推理、工具、记忆都会容易学散
