# 学前导读：推理与规划这一章到底在学什么

这一章解决的是：

> **Agent 到底怎样把复杂任务拆开、思考并推进。**

## 先建立一张桥接线

如果你刚学完 Agent 基础，这一章最值得先看清的一件事是：

- Agent 之所以不像普通工作流，关键不只是多了工具
- 更关键的是它开始需要自己决定：下一步做什么、按什么顺序做

所以这一章真正重要的不是“多几个推理术语”，而是：

> **让 Agent 从“会调用能力”走到“会组织行动”。**

## 这一章的主线

```mermaid
flowchart LR
    A["推理能力"] --> B["链式推理"]
    B --> C["ReAct"]
    C --> D["Plan-and-Execute"]
    D --> E["更复杂规划"]
    E --> F["推理评估"]
```

## 这一章更适合新人的学习顺序

1. 先看 LLM 推理能力  
   先把“知道答案”和“推出来答案”的差别看清楚。

2. 再看链式推理  
   先把多步中间状态的意义立住。

3. 然后再看 ReAct 和 Plan-and-Execute  
   这时你更容易理解为什么 Agent 会把“想”和“做”交错起来。

4. 最后再看复杂规划和推理评估  
   真正建立“推理好不好，不能只看最后一句话”的判断。

## 这一章最该先抓住什么

- 推理不是“答案更长”，而是中间状态更稳
- Agent 的推理和工具动作是交织的
- 这一章会决定你后面怎么看规划、工具调度和错误恢复

## 这一章更适合新人的学习顺序

1. 先看 LLM 推理能力  
   先把“记住答案”和“推出来答案”的差别立住。

2. 再看链式推理  
   先把多步中间状态这件事看顺。

3. 再看 ReAct 和 Plan-and-Execute  
   这时你更容易理解为什么 Agent 会把推理和动作交错起来。

4. 最后看复杂规划和评估  
   真正建立“推理不是写长点答案，而是能不能稳定推进任务”的直觉。

## 这一章最该先抓住什么

- 推理不是“说得像有道理”，而是能不能把中间状态维持正确
- Agent 推理和工具调用是交织在一起的
- 这一章会决定你后面怎么看 ReAct、规划和工具策略
