# 学习指南：Agent 系统怎么学最不容易学乱

:::tip 这页适合什么时候看
如果你来到第九阶段后开始觉得：

- Agent 名词特别多
- 很想一上来就做多 Agent
- 框架、协议、工具、记忆容易混在一起

那这页就是给你的。
:::

## 这一阶段最重要的总原则

第九阶段最容易学乱的原因，不是系统太复杂，而是太容易在“基础没稳”时过早拉高复杂度。

更适合新人的主线其实是：

> **先把单 Agent 做稳，再慢慢加推理、工具、记忆、多 Agent、评估和部署。**

## 最推荐的阅读顺序

1. 先学 Agent 基础  
   先分清 Agent 和普通工作流的边界。

2. 再学推理与工具  
   这是 Agent 最关键的运转层。

3. 再学记忆  
   让系统开始有连续性。

4. 再看 MCP 和框架  
   理解生态和系统组织方式。

5. 最后再看多 Agent、评估、安全、部署

## 这一阶段最容易卡住的地方

### 1. 太早做多 Agent

单 Agent 如果都没做稳，多 Agent 只会把问题放大。

### 2. 只演示成功路径

真正的 Agent 系统必须把：

- 失败恢复
- 权限边界
- 日志观察

一起设计。

### 3. 框架学得比系统还多

先搞清系统层次，再看框架抽象，会更稳。

## 第一次做 Agent 项目，最稳的默认顺序

1. 先做单 Agent
2. 先把推理和工具闭环做出来
3. 再补记忆
4. 再做评估和日志
5. 最后再考虑多 Agent

## 如果你准备进入第十阶段，最值得先带走什么

进入多模态和生成系统前，最值得先带走的是：

1. 目标 / 状态 / 动作这条 Agent 主线
2. 工具调用和失败恢复意识
3. 评估与可观测性习惯

## 一句话版的过关标准

- 能设计一个最小单 Agent 系统
- 能解释它为什么会成功、为什么会失败
- 不再把“会说话”误当成“能稳定完成任务”

如果这三件事已经开始成立，第九阶段就已经学得很值了。
