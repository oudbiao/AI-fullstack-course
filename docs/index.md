# AI Full-Stack Engineering Course

![Start here four-step map](/img/course/intro-start-here-four-step-map-en.webp)

This course trains you to build AI applications that another engineer can inspect: runnable code, reproducible environments, data notes, model evidence, prompt/RAG/Agent traces, and clear limits.

Start simple: experience one AI example, prepare the smallest environment, look at the map, then enter Chapter 1. The career goal is not to memorize AI buzzwords. It is to collect project evidence that proves you can turn model behavior into a usable product workflow.

The course has one main engineering line. Chapters 1-9 build the core path from tools to Python, data, models, LLMs, RAG, and Agents. Chapters 10-12 extend the same project mindset into vision, NLP, and multimodal/AIGC work. Elective modules deepen deployment, advanced Python, and classic ML when your project needs them.

## What You Are Building Toward

By the end of the main line, your strongest project should explain one AI application end to end:

| Evidence | What it proves |
|---|---|
| Setup and README commands | Another person can run the work |
| Data, prompt, retrieval, and tool traces | The application has observable behavior, not hidden magic |
| Metrics, comparison notes, and failure cases | You can judge quality instead of trusting one demo |
| Safety, privacy, cost, and latency notes | You understand product constraints |
| Screenshots or short demos | A reviewer can understand the user experience quickly |

## One Rule

Read briefly, run something, keep evidence. At the end of each stage, you should have something another person can inspect: a README command, a saved output, a metric table, a trace, a failure note, or a small demo. This is also how you turn learning into a portfolio story.
