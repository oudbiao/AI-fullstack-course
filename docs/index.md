# AI 全栈学习教程

这套课程的目标很直接：把你从“会一点编程或准备开始编程”，带到“能独立做 AI 应用、RAG 系统、Agent 项目和作品集”。你不需要一开始读完所有导览页，也不需要先把所有数学和模型细节都学透。更好的方式是先跑通主线，再按目标补深度。

## 让学习不那么枯燥的玩法

如果你觉得长课程容易坚持不下去，可以把课程当成一条任务线来学：[AI 学习助手剧情任务线](/intro/story-quest-line) 负责给每阶段一个故事目标，[课程 Boss 战挑战地图](/intro/boss-challenge-map) 负责阶段验收，[Debug 侦探任务集](/intro/debug-detective-missions) 负责把报错变成破案练习，[技能徽章与成就系统](/intro/badges-achievements) 负责记录每次可验证进步。

这些页面不是额外负担，而是把原本枯燥的任务换一种更轻松的表达。你仍然会学到终端、Python、数据、模型、Prompt、RAG、Agent 和部署，只是每一步都有更清楚的目标和反馈。

## 最推荐的主线

对大多数学习者来说，推荐先走这一条主线：

```text
开发工具 -> Python -> 数据分析 -> AI 数学最小基础 -> 机器学习 -> 深度学习与 Transformer -> 大模型原理与 Prompt -> LLM 应用与 RAG -> AI Agent -> 毕业项目
```

这条路线的核心逻辑是：先能写程序，再能处理数据；先理解模型如何学习，再理解大模型为什么有效；先能调用模型完成任务，再把模型接入知识库、工具、记忆和工作流；最后再考虑部署、评估、安全和方向拓展。

## 你一路会做出什么

| 学习段 | 重点能力 | 代表作品 |
| --- | --- | --- |
| 打基础 | 环境、Python、数据处理 | 命令行工具、API 小工具、数据分析报告 |
| 理解模型 | 机器学习、深度学习、Transformer | 预测模型、分类实验、训练曲线复盘 |
| 做应用 | Prompt、LLM API、RAG | 课程问答助手、知识库问答系统 |
| 做系统 | Agent、工具调用、记忆、评估 | 学习规划 Agent、自动化助手 |
| 做作品集 | 部署、日志、安全、复盘 | 可演示的 AI 全栈毕业项目 |

学习时一直问自己一个问题：这一章学完后，我的作品能多完成哪一步？这样你就不会只是“看教程”，而是在持续升级一个项目。

## 如果你的目标不同，可以这样走

如果你想尽快做 AI 应用，优先学习开发工具、Python、数据分析、LLM 应用、RAG 和 Agent。数学、深度学习和模型原理先学到能理解关键概念即可，后面再补。

如果你想理解模型原理，就按 Python、数据分析、数学、机器学习、深度学习、Transformer、预训练、微调和对齐的顺序深入。项目仍然要做，但重点放在实验、指标和错误分析。

如果你想做作品集，就不要等学完所有章节才开始项目。每个阶段都保留一个可运行作品，持续更新 README、截图、实验记录、失败样本和下一步计划。

## 学习过程中最重要的原则

不要只看概念，要尽早跑代码。AI 学习真正卡住的地方，往往不是定义本身，而是环境依赖、数据格式、模型输入输出、接口错误、评估方式和项目边界。

不要一开始追求“大而全”。第一次学习最重要的是跑通主线：能处理数据，能训练或调用模型，能做一个可用的 AI 应用，能解释系统为什么这样设计。

不要把大模型应用和传统机器学习割裂。RAG、Agent、工具调用和多模态应用看起来很新，但底层仍然离不开数据、表示、检索、评估、工程化和安全意识。
