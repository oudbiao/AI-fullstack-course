# 阶段过渡指南

![AI 主线接力图](/img/course/appendix-ai-main-relay-map.png)

每个阶段都补一个新能力。如果下一个主题看起来很突然，就用这页判断它补的是哪个缺口。

## 过渡地图

| 从 | 到 | 新增能力 |
|---|---|---|
| 工具 | Python | 用代码表达流程 |
| Python | 数据分析 | 读取文件、表格、日志和混乱输入 |
| 数据分析 | 机器学习 | 把数据变成特征、目标、指标和 baseline |
| 机器学习 | 深度学习 | 用张量和神经网络学习表示 |
| 深度学习 | LLM | 理解 token、embedding、上下文、Transformer 和模型行为 |
| LLM | RAG | 不只靠模型记忆，而是带检索来源回答 |
| RAG | Agent | 用工具、状态、trace 和权限完成任务 |
| Agent | 毕业项目 | 把合适组件组合成可复现系统 |

## 进入下一阶段前

| 下一阶段 | 检查一件事 |
|---|---|
| Python | 能运行脚本并完成一次 Git commit 吗？ |
| 数据 | 能读取文件并解释字段吗？ |
| 机器学习 | 能定义目标和指标吗？ |
| 深度学习 | 能解释训练/测试划分、loss、过拟合和 baseline 吗？ |
| LLM | 能调用 API 或运行小文本模型吗？ |
| RAG | 能在生成前查看检索到的 chunk 吗？ |
| Agent | 能回放工具调用并阻止危险动作吗？ |
| 毕业项目 | 别人能按 README 跑起来吗？ |

课程顺序不是装饰，而是工程顺序：后面的 AI 应用都依赖前面的证据、排障和项目习惯。
