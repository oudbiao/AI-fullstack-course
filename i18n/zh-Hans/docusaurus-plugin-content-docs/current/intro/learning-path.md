# 推荐学习路线

![推荐学习路线选择图](/img/course/intro-learning-path-selection.png)

课程内容很多。第一遍不要试图推平所有分支。先选一条路线，做出小成果；项目需要更深知识时，再回来补。

## 先选一条默认路线

| 你的情况 | 第一条路线 | 第一遍规则 |
| --- | --- | --- |
| 不确定从哪开始 | 新手完整路线 | 第 1-9 章顺序走，每阶段留下一个小成果 |
| 已经会写代码 | 快速应用路线 | 快速扫过 1-6，重点做 Prompt、RAG、Agent 和部署 |
| 需要作品集 | 作品集路线 | 每个项目都保留 README、截图、日志和失败记录 |
| 想深入模型 | 模型理解路线 | 在数学、机器学习、深度学习和 Transformer 多停留 |

如果犹豫，就选 **新手完整路线**。跑通一个闭环，比设计完美路线更重要。

## 按时间做取舍

| 可投入时间 | 做法 |
| --- | --- |
| 每周 3-5 小时 | 只读核心页，完成最小项目任务 |
| 每周 6-10 小时 | 走主线，并写简短项目记录 |
| 求职或作品集冲刺 | 先做项目，再补项目暴露出来的理论缺口 |
| 研究或模型方向 | 在数学、ML、深度学习和评估上放慢 |

## 不要频繁换路线

只有当前项目暴露真实缺口时，才切换补课：

| 症状 | 回到哪里 |
| --- | --- |
| 代码无法运行或复现 | 工具和 Python |
| 数据或引用不可靠 | 数据、RAG 和评估 |
| 模型结果说不清楚 | 数学、ML 和深度学习 |
| Agent 难以控制 | 工具调用、trace、权限和恢复 |

正确节奏是：短读、运行、保存证据，然后继续。
