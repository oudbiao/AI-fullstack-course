# 课程页面使用说明

![推荐学习路线选择图](/img/course/intro-learning-path-selection.png)

这一页回答：**下一步应该打开哪个页面？**

## 1. 第一遍顺序

| 步骤 | 打开 | 做什么 |
|---|---|---|
| 1 | [30 分钟 AI 快速体验](/intro/quick-experience) | 先跑一个极小 AI 示例 |
| 2 | [环境准备](/intro/environment-setup) | 准备 Python、Git 和项目文件夹 |
| 3 | [AI 全栈能力地图](/intro/ai-fullstack-map) | 先看图，不急着读完细节 |
| 4 | [推荐学习路线](/intro/learning-path) | 选一条路线 |
| 5 | [第 1 章：开发者工具](/ch01-tools) | 开始搭学习工作区 |

## 2. 课程页面怎么用

| 页面类型 | 用途 |
|---|---|
| Intro 页面 | 快速开始、环境、地图、路线、术语 |
| 阶段首页 | 说明本阶段为什么重要、要做什么 |
| `0.0 学习指南与任务表` | 学习顺序、必做任务、通过标准 |
| 正课文章 | 概念、代码、输出和常见错误 |
| 工作坊或项目 | 学完主要理论后做可运行产物 |
| 补充帮助 | FAQ、排障、作品集、职业方向或深入路线 |

## 3. 默认学习闭环

1. 先看图。
2. 跑最小代码。
3. 对照预期输出。
4. 保存一个命令、结果、错误或截图。
5. 回到阶段任务。

如果迷路，先问：当前卡点属于工具、代码、数据、模型行为、LLM 应用逻辑、Agent 动作，还是部署？然后再打开对应章节或排障页。
