# 全课程项目矩阵

![项目作品集路线图](/img/course/intro-project-portfolio-roadmap.png)

这一页只回答一个问题：“每个阶段到底做什么项目？”第一遍先做小，只有准备展示的项目才升级成作品集版本。

## 1. 阶段项目地图

| 阶段 | 最小有用项目 | 应保留的证据 |
|---|---|---|
| 1 工具 | 创建仓库并运行 Python | README、命令记录、截图 |
| 2 Python | CLI 任务管理器或简单 API | 运行命令、示例输入输出 |
| 3 数据 | 分析一个 CSV 并画图 | 清洗记录、图表、结论 |
| 4 AI 数学 | 向量、概率或梯度小实验 | 图解、小代码、解释 |
| 5 机器学习 | 分类或回归 baseline | 指标、baseline、错误样本 |
| 6 深度学习 | PyTorch 训练循环 | 配置、曲线、失败样本 |
| 7 LLM 与 Prompt | Prompt 助手或复习卡生成器 | Prompt 版本、输出对比 |
| 8 RAG | 带引用的 Markdown 问答 | 问题集、检索日志、引用检查 |
| 9 Agent | 工具调用规划 Agent | 工具 schema、执行 trace、边界 |
| 10 视觉 | 分类、OCR 或检测实验 | 标注、指标、可视化结果 |
| 11 NLP | 文本分类或信息抽取 | 标签规则、指标、错误案例 |
| 12 多模态 | 图片、音频、视频或多模态流程 | 原始素材、生成日志、复核规则 |

## 2. 升级规则

| 版本 | 目标 | 增加的证据 |
|---|---|---|
| 最小版 | 证明能跑 | README、命令、示例输出 |
| 标准版 | 证明别人能复现 | 依赖文件、配置说明、日志 |
| 作品集版 | 证明能评估、能解释 | 评估集、失败样本、截图、复盘 |

挑战项目是选做。它适合主线已经稳定之后升级作品，而不是在第一个闭环跑通前就开始扩展。

## 3. 推荐仓库结构

```text
ai-fullstack-portfolio/
├── ch01-tools/
├── ch02-python-cli/
├── ch03-data-report/
├── ch05-ml-baseline/
├── ch08-rag-assistant/
├── ch09-agent-planner/
└── final-ai-app/
```

每个文件夹至少放 README、源码、样例输入或数据、输出证据、一个失败样本和一个下一步计划。
