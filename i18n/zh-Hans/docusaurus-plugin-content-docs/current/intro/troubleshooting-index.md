# 常见错误与排障索引

![排障急救图](/img/course/appendix-troubleshooting-rescue-map.png)

很多学习卡点不是“AI 概念没懂”，而是目录、环境、依赖、数据格式、API、检索、工具或部署不一致。排障要按顺序来。

## 1. 按现象先做第一步

| 现象 | 先检查 | 最小证明 |
|---|---|---|
| 命令失败 | 当前目录、命令是否存在 | `pwd`、`which <command>`、项目 scripts |
| Python import 失败 | 解释器和包安装位置 | `python -m pip show <package>` |
| 文件读不到 | 相对路径、文件名大小写、编码 | 打印 `pwd`，再打开一个小文件 |
| JSON 输出不稳定 | Prompt schema 和校验 | 先让模型只返回一个最小 JSON |
| RAG 回答错误 | 模型调用前的检索片段 | 打印 top-k 检索结果 |
| Agent 循环 | 步数限制、停止条件、trace | 限制最多 3 步运行 |
| 部署失败 | 环境变量、启动命令、文件路径 | 在干净环境按 README 重跑 |

## 2. 排障闭环

1. 保存完整错误和准确命令。
2. 确认目录、运行环境、依赖版本和配置。
3. 用最小输入复现。
4. 定位失败层：环境、代码、数据、模型、检索、工具或部署。
5. 修复后，把预防说明写回 README 或脚本。

不要只看错误栈最后一行。真正有用的线索经常在中间，比如实际读取的路径、解释器、模型名、字段名或工具参数。

## 3. 分层检查表

| 层级 | 快速问题 |
|---|---|
| 环境 | 我在正确目录吗？运行时对吗？ |
| 依赖 | 包是不是装到了同一个 Python 或 Node 环境里？ |
| 数据 | 路径、字段、编码、样例行是否正确？ |
| 模型 | 输入形状、标签、模型名、额度、超时是否正确？ |
| RAG | Chunk、metadata、top-k、rerank、引用是否出现在日志里？ |
| Agent | 工具 schema、步数限制、权限、trace 是否记录？ |
| 部署 | 环境变量、端口、启动命令、线上日志是否可见？ |

## 4. 排障记录模板

```md
## 问题标题

### 现象
我运行了什么命令，看到了什么错误。

### 关键错误
粘贴关键行，不要只贴最后一行。

### 复现方式
1. 目录：
2. 命令：
3. 输入或配置：

### 检查过程
我检查了哪些环境、路径、依赖、数据、参数或日志。

### 修复
改了什么，为什么有效。

### 预防
应该补到 README、校验脚本、日志字段或测试用例里的内容。
```

每解决一个真实错误，都应该给下一次运行留下更清楚的文档或更安全的命令。
