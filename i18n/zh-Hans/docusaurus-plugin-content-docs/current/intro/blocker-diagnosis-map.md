# 学习卡点诊断图

![学习卡点诊断流程图](/img/course/intro-blocker-diagnosis-flow.png)

卡住时，先不要继续加资料。先定位卡在哪一层，跑最小测试，再记录修复。

## 快速诊断表

| 现象 | 可能层级 | 优先回到 | 最小测试 | 证据 |
| --- | --- | --- | --- | --- |
| 命令跑不起来 | 工具/环境 | 第 1 章、环境准备 | 新开终端，按 README 重新跑 | 命令日志 |
| Python 读不到文件 | 代码/项目结构 | 第 2 章 | 打印 `Path.cwd()`，做最小读写 | 输入输出文件 |
| 数据结论不可信 | 数据质量 | 第 3 章 | 打印列名、缺失、重复 | 数据字典 |
| 指标或 loss 看不懂 | 数学/模型评估 | 第 4-6 章 | 用 5 个样本手动复现一个指标 | 指标笔记 |
| LLM JSON 不稳定 | Prompt/schema | 第 7 章 | 固定 10 个输入，用解析器检查 | Prompt 版本表 |
| RAG 来源不可靠 | 检索/引用 | 第 8 章 | 先不生成，只打印检索结果 | 检索日志 |
| Agent 循环或越权 | 工具/权限 | 第 9 章 | 限制 3 步并保存 trace | `agent_traces.jsonl` |
| 项目只在本机能跑 | 交付/部署 | 交付标准、部署章节 | 干净 clone 后按 README 跑 | 干净运行日志 |
| 讲不清项目 | 作品集表达 | 作品集清单 | 补问题、输入输出、评估、失败 | README 更新 |

## 搜索前先问四个问题

1. 哪个命令或输入触发了问题？
2. 我原本期待什么？
3. 实际发生了什么？
4. 能不能用更小的例子复现？

## 卡点记录模板

## 卡点

顺利学习不是从不失败，而是下一次遇到类似失败时能更快定位。
