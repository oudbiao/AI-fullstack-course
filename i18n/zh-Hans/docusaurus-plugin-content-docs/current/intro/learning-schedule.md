# 选择 4、8 或 12 周学习节奏

![学习节奏卡](/img/course/intro-learning-pace-cards.png)

不要按“读了多少篇文章”来安排课程。更有效的方式是按每周产出：一个可见结果、一段简短复盘、一个下一步改进点。

## 1. 先选节奏

| 你的情况 | 建议节奏 | 主要目标 |
|---|---|---|
| 已经会写代码，只想快速做 AI 应用 | 4 周 | 做出一个能讲清楚的 RAG 或 Agent 原型 |
| 想准备求职作品集 | 8 周 | 分阶段做项目，再串成一条作品故事 |
| 新手，或想系统补基础 | 12 周 | 从工具到毕业项目稳步搭起来 |

## 2. 节奏卡

| 节奏 | 每周重点 | 最终证据 |
|---|---|---|
| 4 周 | 快速补基础，把主要时间放在 LLM、RAG、Agent 和部署说明上 | 一个可运行 AI 应用、README、评估集、失败样本 |
| 8 周 | 每一到两周增加一个项目能力：工具、Python、数据、机器学习、深度学习、LLM、RAG、Agent | 作品集 README、截图、演示脚本、项目复盘 |
| 12 周 | 按课程顺序走，每周留下一个小成果，给排障留足时间 | 完整学习仓库、阶段产出、毕业项目草稿 |

即使选择 4 周快跑，也不要跳过评估、日志和失败样本。它们才是 Demo 变成项目的证据。

## 3. 每周操作闭环

| 步骤 | 要做什么 |
|---|---|
| 周一 | 选一个章节目标和一个产出 |
| 学习中 | 先跑通最小代码路径，再加功能 |
| 收尾前 | 保存一张截图、一个命令、一个样例输出、一个失败 |
| 周末 | 更新 README，确定下周最需要解决的瓶颈 |

## 4. 调整规则

| 如果你... | 这样调整 |
|---|---|
| Python 已经不错 | 压缩第 1-2 章，把时间留给 RAG 和 Agent 评估 |
| 有机器学习或深度学习经验 | 理论可以快读，但指标、曲线、误差分析不能省 |
| 经常卡在环境 | 选 12 周路线，并保留排障日志 |
| 时间很少 | 减功能，不减证据。保留 README、运行命令、示例和失败样本 |

## 5. 每周复盘模板

```md
## 第 N 周复盘

- 本周产出：
- 运行命令：
- 示例输入输出：
- 一个失败：
- 我检查了什么：
- 下个瓶颈：
```

学完课程不是把每篇文章都看过一遍，而是留下别人能理解、能复现、能与你讨论的可运行作品。
