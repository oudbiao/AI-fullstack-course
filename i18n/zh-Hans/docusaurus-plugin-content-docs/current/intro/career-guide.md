# AI 职业方向指南

![按角色选择 AI 学习路线](/img/course/intro-role-based-paths-map.png)

不要在还没做出东西前就急着选岗位。先完成核心基础，观察自己喜欢哪类任务，再选择一个方向做作品集项目。

## 方向卡

| 方向 | 如果你喜欢这些，可能适合 | 课程重点 | 作品集证明 |
| --- | --- | --- | --- |
| LLM 应用工程师 | 做产品、接 API、搭实用流程 | 第 7-8 章 | 带评估的 RAG 或 LLM 应用 |
| AI Agent 工程师 | 多步骤自动化、工具、trace、权限 | 第 9 章 | Agent trace、工具 schema、安全边界 |
| 数据 / ML 工程师 | 数据质量、指标、实验、baseline | 第 3-6 章 | 带指标和失败样本的模型报告 |
| CV / 多模态 | 图片、截图、视觉结果 | 第 10 章和第 12 章 | 前后对比视觉样例与审核 |
| NLP / RAG | 文本抽取、搜索、分类、问答 | 第 7-8 章和第 11 章 | 标注样例、引用、评估集 |
| MLOps / 部署 | 可靠性、成本、日志、部署 | 部署与工程参考 | Docker、运行手册、日志、监控说明 |

## 快速自测

| 问题 | 如果是，更偏向 |
| --- | --- |
| 我想很快看到产品反馈吗？ | LLM 应用或 Agent |
| 我喜欢度量和解释错误吗？ | 数据 / ML |
| 我喜欢图像、视频或文档吗？ | CV / 多模态 |
| 我喜欢文本、搜索和结构化抽取吗？ | NLP / RAG |
| 我喜欢可靠性、成本、日志和部署吗？ | MLOps / AI 工程 |

## 什么时候决定

| 阶段 | 决策 |
| --- | --- |
| 第 3 章前 | 先不决定，打好工具、Python、数据基础 |
| 第 6 章后 | 判断自己更偏模型深度还是应用构建 |
| 第 9 章后 | 选一个作品集方向做强 |
| 毕业项目 | 根据手头材料和评估方式选择 |

最稳的默认方向仍然是 LLM 应用 + RAG + Agent，因为它能较快做出可见项目，同时也需要真实工程证据。
