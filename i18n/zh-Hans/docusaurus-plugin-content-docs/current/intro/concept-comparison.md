# 概念分类板

![概念分类板](/img/course/intro-concept-sorting-board.png)

看到一个新词时，先不要急着背定义。先判断它属于哪一类，再看短解释，这样更不容易把概念混在一起。

## 1. 先分类，再记忆

| 分类盒子 | 常见术语 | 一句话理解 |
|---|---|---|
| 调用与接入 | API、SDK、库、框架、CLI、Web API | 你的代码怎样接到外部能力 |
| 模型与服务 | 模型、推理、训练、微调、部署 | 你是在使用、改变，还是对外提供模型 |
| 大模型流程 | Prompt、System Prompt、Few-shot、结构化输出、函数调用、工具使用 | 怎样指挥模型，并让它接工具 |
| 数据与向量 | Token、Embedding、Chunk、Metadata、向量数据库、上下文窗口 | 内容怎样切分、表示、检索并放给模型看 |
| 工程交付 | 日志、评估、延迟、成本、安全、监控 | Demo 能不能变成稳定项目 |

## 2. 高频术语一句话

| 术语 | 简单理解 | 不要混成 |
|---|---|---|
| API | 调用服务的约定入口 | SDK，SDK 只是帮你更方便调用 |
| SDK | 开发工具包 | 服务本身 |
| 模型 | 学到参数、负责预测或生成的核心 | 包在外面的 API |
| 推理 | 用模型得到输出 | 训练，训练会改变参数 |
| 微调 | 在已有模型上继续训练 | 改 Prompt |
| Prompt | 本次给模型的任务说明和输入 | 存私有知识的地方 |
| RAG | 先检索外部文档，再带着上下文回答 | 微调模型 |
| Agent | 能规划、多步执行、调用工具的系统 | 只有一次 API 调用的聊天机器人 |
| Embedding | 把文本、图片等变成向量表示 | 存向量的向量数据库 |
| Chunk | 文档切出来的检索片段 | 原始整篇文档 |

## 3. 三个快速判断

| 如果问题是... | 先用... | 原因 |
|---|---|---|
| 输出格式不稳定 | Prompt 和结构化输出 | 任务边界还不清楚 |
| 模型不知道你的资料 | RAG | 缺的是外部知识 |
| 任务需要反复行动 | 工作流或 Agent | 系统需要决策或执行步骤 |

新手通常按这个顺序处理：先改 Prompt；缺知识再加检索；真的需要外部动作时，再加工具调用。

## 4. 小练习

看这三句话，判断它属于哪类问题：

| 现象 | 正确分类 |
|---|---|
| “回答引用了错误段落。” | 数据与向量，然后看 RAG 评估 |
| “JSON 字段有时缺失。” | 大模型流程 |
| “本地能跑，部署后失败。” | 工程交付 |

写 README 时也尽量说清楚：用了哪个模型服务、通过 API 还是 SDK 调用、是否用了 RAG 或工具、结果怎么评估、已知边界在哪里。
