# 学习策略

![持续学习飞轮](/img/course/appendix-continuous-learning-flywheel.png)

整门课都用一个学习闭环：

> 看懂思路 -> 跑最小例子 -> 做一个小成果 -> 记录证据 -> 复盘下一步。

## 三条规则

| 规则 | 防止什么 | 最小行动 |
| --- | --- | --- |
| 用项目学 | 看了很多教程却没有能力证明 | 每阶段留下一个可运行成果 |
| 理论贴近实践 | 连续几周只看数学/模型没有反馈 | 一个直觉配一个小实验 |
| 用证据学 | 学过但忘了自己做过什么 | 保存命令、输出、错误或指标 |

## 每周节奏

| 时间盒 | 行动 |
| --- | --- |
| 25 分钟 | 读一个小节或先看一张图 |
| 25 分钟 | 运行或修改一个例子 |
| 10 分钟 | 写 README 笔记或失败笔记 |
| 周末 | 选一个结果作为作品集证据 |

## 避开这些坑

| 坑 | 更好的动作 |
| --- | --- |
| 收藏教程 | 打开下一个资料前先跑一个例子 |
| 完美主义 | 先让丑版本跑起来 |
| 频繁换路线 | 先完成当前阶段检查卡 |
| 只学理论 | 加一个最小实验 |
| 只做 Demo | 加评估或失败样本 |

拿不准时，回到当前阶段的 `0.0 学习指南与任务表`，完成最小任务。
