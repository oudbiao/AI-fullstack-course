# 教学与自学使用指南

![学习节奏卡](/img/course/intro-learning-pace-cards.png)

同一套课程可以按不同场景使用。内容顺序大体不变，变化的是节奏、作业和复盘方式。

## 1. 先选使用方式

| 方式 | 每周重点 | 证据 |
|---|---|---|
| 自学 | 跑出一个小结果，写一次复盘 | README、命令、截图、失败记录 |
| 讲师授课 | 讲框架，演示闭环，点评作业 | 讲义或板书、演示代码、评分标准 |
| 学习小组 | 同步产出，而不是只同步阅读进度 | 互评过的 README 和示例输出 |

## 2. 推荐节奏

| 步骤 | 自学者 | 讲师 | 学习小组 |
|---|---|---|---|
| 预习 | 看地图和目标 | 选本周最容易误解的点 | 分配角色 |
| 练习 | 先跑最小代码 | 先演示一次，再让学员改 | 一人演示 |
| 项目 | 留下一个小产物 | 设置基础任务和挑战任务 | 互相 review 产物 |
| 复盘 | 记录一个失败 | 讨论共性失败 | 收集卡点 |

## 3. 作业规则

好作业应该能运行、能检查、能复盘。不要只布置“阅读第 X 章”。更好的形式：

| 阶段 | 作业形态 |
|---|---|
| Python | 带样例输入输出的 CLI 脚本 |
| 数据 | 有图表结论的 Notebook 或报告 |
| 机器学习/深度学习 | 带指标和失败样本的 baseline 或训练实验 |
| RAG | 带检索日志和引用的问答 demo |
| Agent | 带 trace 和停止条件的工具调用任务 |

## 4. 验收规则

![作品集验收阶梯](/img/course/intro-portfolio-acceptance-ladder.png)

每次项目 review 都问四个问题：能不能运行，别人能不能复现，结果能不能评估，学习者能不能讲清一个失败和下一步。

维护课程时，优先更新地图、项目、评估标准和常见错误。不要为了追每个新模型名而打乱主线，除非它真的改变了教学路径。
