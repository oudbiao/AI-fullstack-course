# 项目路线与作品集

![项目作品集成长阶梯](/img/course/intro-project-portfolio-roadmap.png)

用项目学习，但项目要小到能完成。只有能运行、能解释、能评估的项目，才会变成作品集材料。

## 三类项目层级

| 层级 | 目标 | 一开始不要追 |
| --- | --- | --- |
| 阶段项目 | 输入、输出、运行命令 | 很多功能或精美 UI |
| 综合项目 | 日志、指标、失败样本 | 只有一张成功截图 |
| 毕业项目 | README、评估、演示脚本 | 把所有技术塞进一个系统 |

## 推荐项目阶梯

| 课程区域 | 项目产出 |
| --- | --- |
| 工具 + Python | CLI 学习助手 |
| 数据分析 | 带图表和限制说明的 EDA 报告 |
| ML / DL | baseline 模型、指标、错误样本 |
| LLM / Prompt | 结构化计划或复盘卡助手 |
| RAG | 带引用和评估问题的课程问答 |
| Agent | 带 trace 和权限的工具型助手 |
| 多模态 / 毕业项目 | 一个可演示最终项目 |

## 最小作品集 README

````md
# 项目名

## 目标
解决什么问题？

## 输入和输出
输入：
输出：

## 如何运行
```bash
python main.py
```

## 评估
怎样判断结果好不好？

## 失败样本
哪里坏过，你学到了什么？

## 下一步改进
如果继续做，先改什么？
````

如果每个项目都有这些部分，作品集会随着课程自然长出来。
