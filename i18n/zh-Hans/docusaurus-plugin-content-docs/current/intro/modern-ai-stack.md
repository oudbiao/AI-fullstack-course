# 现代 AI 应用技术栈

![现代 AI 应用技术栈全景图](/img/course/intro-modern-ai-stack-map.png)

课程后半段不是只学习“怎么调用大模型”，而是学习怎样做出能用知识、能执行动作、能处理真实输入、能评估、能长期运行的系统。

## 1. 先记住五块

| 模块 | 解决的问题 | 主要阶段 |
|---|---|---|
| RAG | 模型需要私有或最新知识 | 8 |
| Agent | 任务需要规划、工具和多步执行 | 9 |
| 多模态 AI | 输入或输出包含图片、PDF、音频、视频、截图 | 10-12 |
| 模型工程 | 系统要平衡质量、延迟、成本和隐私 | 6-8 |
| Ops | Prompt、评估、日志、trace、部署和回滚要可管理 | 7-9 和毕业项目 |

## 2. 最小系统视角

| 层级 | 要能检查什么 |
|---|---|
| 输入 | 用户任务、文件、图片、权限 |
| 上下文 | 文档、chunk、metadata、对话状态 |
| 模型或工具 | 模型调用、工具调用、参数、成本 |
| 输出 | 回答、引用、动作、生成素材 |
| 评估 | 固定测试、人工复核、指标 |
| 可观测性 | 日志、trace、错误、延迟 |
| 改进 | 改了 Prompt、检索、工具 schema、模型或流程 |

如果项目只有输入和输出，它还只是 Demo。能说清用了什么数据、调了什么工具、为什么失败、怎么改进，才开始像工程项目。

## 3. 什么情况用什么

| 需求 | 先从哪里开始 |
|---|---|
| 根据文档回答 | 带检索日志的 RAG |
| 安全执行动作 | 先用工作流，步骤不确定再用 Agent |
| 理解截图或 PDF | 多模态模型加来源追踪 |
| 降低成本 | 小模型、路由、缓存或更短上下文 |
| 稳定质量 | 评估集、日志、trace、Prompt 版本 |

第一遍学习建议：先记住这张图，再用最小可运行项目理解每一块。
