# AI 学习助手版本路线图

![AI 学习助手版本路线图](/img/course/ai-learning-assistant-roadmap.png)

每个版本只增加一个能力，并留下四类证据：

| 必要证据 | 为什么重要 |
| --- | --- |
| 运行命令 | 证明功能可复现 |
| 示例输入输出 | 说明功能长什么样 |
| 失败样本 | 说明系统边界 |
| 评估或检查 | 避免只是碰巧成功的 Demo |

## 版本卡

| 版本 | 对应阶段 | 能力 | 最小证据 |
| --- | --- | --- | --- |
| v0.1 | 第 1 章 | 项目骨架 | README、依赖说明、第一次 commit |
| v0.2 | 第 2 章 | CLI 学习任务助手 | `tasks.json`、新增/查看/完成命令、坏文件处理 |
| v0.3 | 第 3 章 | 学习数据分析 | 图表、清洗说明、一个结论 |
| v0.4 | 第 4-5 章 | 问题分类或风险 baseline | 规则或 ML baseline、指标、错误样本 |
| v0.5 | 第 6 章 | 训练诊断实验 | 配置、loss 曲线、失败记录 |
| v0.7 | 第 7 章 | Prompt 学习助手 | Prompt 版本、schema 检查、失败输出 |
| v0.8 | 第 8 章 | RAG 课程问答 | 文档清单、检索日志、引用检查 |
| v0.9 | 第 9 章 | 学习规划 Agent | 工具 trace、权限规则、停止条件 |
| v1.0 | 第 10-12 章 / 毕业项目 | 可演示产品或方向拓展 | Demo、README、评估报告 |

版本号可以调整，但证据不能省。

## 版本记录模板

```md
## v0.x 版本名

目标：
运行命令：
示例输入：
示例输出：
评估/检查：
失败样本：
下一版本：
```

展示项目时，不要只给最终截图。更有说服力的说法是：“我从 CLI 开始，逐步增加数据、评估、RAG、trace 和部署证据。”
