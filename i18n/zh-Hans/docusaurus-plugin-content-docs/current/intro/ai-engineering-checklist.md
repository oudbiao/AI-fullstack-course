# AI 工程上线检查表

![AI 项目速查总览图](/img/course/appendix-project-quick-reference-map.png)

Demo 不够。称一个项目“准备好”之前，先检查它是否可复现、可评估、可排障、有边界。

## 从 Demo 到可用系统

| 层级 | 检查 |
| --- | --- |
| 问题边界 | 用户、输入、输出、不做什么、拒答条件 |
| 评估 | 固定问题/任务、预期结果、失败样本 |
| 日志/trace | 请求、检索、Prompt 版本、工具调用、错误 |
| 成本/延迟 | token、调用次数、重试、慢步骤 |
| 安全 | API key、敏感数据、高风险工具、人工确认 |
| 部署 | `.env.example`、启动命令、干净运行记录 |
| 作品集 | README、截图、评估、限制、下一步 |

## 按项目类型检查

| 项目类型 | 必须展示 |
| --- | --- |
| RAG | 文档来源、切分方式、检索日志、引用检查、无答案处理 |
| Agent | 工具 schema、最大步数、停止条件、trace、权限边界 |
| 多模态 | 输入素材、生成版本、人工审核、导出限制 |
| 部署 | 环境变量、健康/错误日志、回滚或重置命令 |

## 上线前最后一问

如果系统明天答错、检索错来源、调用错工具，或突然变贵，你能判断是哪一层出问题吗？

如果不能，先加日志、评估和边界，再加功能。
