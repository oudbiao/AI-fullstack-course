# 能力检查点

![阶段检查卡](/img/course/intro-stage-checkpoint-cards.png)

这一页只回答：“我能不能进入下一阶段？”不要只按读完多少文章来判断进度，要看可见能力。

## 1. 四个层级

| 层级 | 你能做到 | 证据 |
|---|---|---|
| 理解 | 说明这个概念解决什么问题 | 笔记、概念图、简短解释 |
| 跑通 | 自己执行最小示例 | 命令、截图、可运行脚本 |
| 构建 | 完成有输入和输出的小项目 | README、源码、示例 |
| 改进 | 评估、发现失败并迭代 | 指标、日志、失败样本、修改记录 |

早期章节做到“跑通”或“构建”即可。RAG、Agent 和毕业项目要尽量接近“改进”。

## 2. 进入下一阶段前

问自己五个问题：

| 问题 | 达标标准 |
|---|---|
| 我能简单解释吗？ | 不看文章也能说出用途 |
| 我亲自跑过代码吗？ | 保存了命令和输出 |
| 我留下项目证据了吗？ | 有 README、脚本、Notebook、图表或 Demo |
| 我记录过失败吗？ | 知道哪里坏过，以及可能原因 |
| 下周还能复现吗？ | 命令和环境说明足够清楚 |

如果有两个以上答案很弱，先补项目证据，再进入下一阶段。

## 3. AI 应用阶段的更高要求

| 阶段类型 | 额外检查点 |
|---|---|
| Prompt | 有 Prompt 版本和固定测试输入 |
| RAG | 能看到检索日志和引用检查 |
| Agent | 记录了工具 schema、trace、步数限制和权限边界 |
| 部署 | 有环境变量、启动命令、日志和回滚说明 |

真正的进步不是记住更多工具名，而是能把问题、数据、模型、工程、评估和复盘连成一个自己讲得清的闭环。
