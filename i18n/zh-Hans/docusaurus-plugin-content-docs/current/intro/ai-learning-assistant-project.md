# 贯穿项目：AI 学习助手

![AI 学习助手版本路线图](/img/course/ai-learning-assistant-roadmap.png)

AI 学习助手是本课程最推荐的贯穿项目。它能避免每个阶段都从零开始：每学完一个阶段，就给同一个产品增加一个小能力。

这页只做总览。实现细节看 [版本路线图](/intro/ai-learning-assistant-version-roadmap) 和 [项目仓库模板](/intro/ai-learning-assistant-template)。

## 一个产品，每阶段一个能力

| 阶段 | 新增能力 | 证据 |
| --- | --- | --- |
| 1 工具 | 项目工作台存在 | README、Git commit、运行截图 |
| 2 Python | 助手记录学习任务 | CLI 输出、`tasks.json`、错误处理 |
| 3 数据 | 助手汇总学习记录 | 清洗后数据、图表、结论 |
| 4-6 模型 | 助手分类或预测学习风险 | baseline、指标、错误样本 |
| 7 LLM/Prompt | 助手生成计划或复盘卡 | Prompt 版本、schema 检查 |
| 8 RAG | 助手基于课程资料回答 | 检索日志、引用、评估问题 |
| 9 Agent | 助手规划步骤并调用工具 | trace、权限表、停止规则 |
| 10-12 拓展 | 助手处理一个专项媒介 | 前后对比样例和审核记录 |
| 毕业项目 | 助手成为可演示产品 | Demo、README、评估报告 |

## 产品规则

不要一开始就做巨大产品。每阶段只增加：

1. 一个用户动作。
2. 一个运行命令。
3. 一个示例输入和输出。
4. 一个失败样本。
5. 一个下一步记录。

这样不用每章另开项目，也能自然形成作品集故事。

## 最终标准

最终助手不必功能很多，但要有清晰闭环：

```text
学习目标或材料 -> 读取上下文 -> 检索或工具调用 -> 回答或计划 -> 证据、日志、评估
```

如果别人能运行它、查看示例、看到失败并理解你的下一步改进，这个项目就可以进入作品集。
