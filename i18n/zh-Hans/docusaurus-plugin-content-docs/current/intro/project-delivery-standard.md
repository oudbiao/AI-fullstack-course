# 项目交付标准

![项目速查图](/img/course/appendix-project-quick-reference-map.png)

这一页只回答：“项目做完后到底交什么？”每完成一个阶段项目，就按这里检查。

## 1. 交付闭环

| 步骤 | 证据 |
|---|---|
| 目标 | 谁使用、解决什么问题、输出什么 |
| 运行 | 安装说明、环境变量、一个最小运行命令 |
| 示例 | 真实输入和真实输出 |
| 评估 | 指标、清单、固定问题集或人工复核规则 |
| 失败 | 一个可复现坏案例和可能原因 |
| 下一步 | 下一版要改的一个具体点 |

时间紧时，先保留运行命令、示例输出和失败样本。

## 2. 最小 README

```md
# 项目名

## 目标

## 如何运行

## 示例输入输出

## 评估

## 已知失败

## 下一步
```

## 3. AI 项目额外证据

| 项目 | 额外补充 |
|---|---|
| Prompt | Prompt 版本、固定测试样本、结构化校验 |
| RAG | 切分策略、检索日志、引用检查 |
| Agent | 工具 schema、trace、最大步数、权限边界 |
| 多模态 | 原始素材、生成设置、人工复核清单 |

交付可以小，但必须具体。别人能按 README 跑起来，比写很长计划更重要。
