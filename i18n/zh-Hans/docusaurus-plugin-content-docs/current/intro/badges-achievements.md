# 能力证据卡

![阶段检查卡](/img/course/intro-stage-checkpoint-cards.png)

徽章只有指向证据时才有用。与其收集很多称号，不如每个阶段保留一张证据卡。

## 证据卡规则

| 字段 | 写什么 |
| --- | --- |
| 成果物 | 你做出了什么 |
| 运行命令 | 别人如何复现 |
| 证据 | 截图、日志、图表、指标、trace 或引用检查 |
| 一个失败 | 你真实遇到的问题 |
| 改进 | 你修了什么，或下一步要修什么 |

## 第一遍阶段卡

| 阶段 | 卡片重点 | 示例证据 |
| --- | --- | --- |
| 1 工具 | 可运行工作台 | README、Git commit、终端输出 |
| 2 Python | 程序能保存数据 | JSON 文件、CLI 输出、错误处理 |
| 3 数据 | 数据变成结论 | 清洗后 CSV、图表、结论 |
| 4 数学 | 公式变成代码 | 小实验和解释 |
| 5 ML | baseline 与指标 | 指标表、错误样本 |
| 6 DL | 训练信号 | loss 曲线、配置、失败记录 |
| 7 LLM/Prompt | 稳定结构化输出 | Prompt 版本、schema 检查 |
| 8 RAG | 带证据的回答 | 检索日志、引用表 |
| 9 Agent | 可控工具调用 | trace、权限表、停止规则 |
| 10-12 | 一个拓展方向 | 前后对比样例与评估 |

## 卡片模板

```md
## 阶段证据卡

成果物：
运行命令：
证据文件：
一个失败：
修复或下一步：
```

时间有限时，每阶段一张清楚的证据卡，比很多空徽章更有用。
