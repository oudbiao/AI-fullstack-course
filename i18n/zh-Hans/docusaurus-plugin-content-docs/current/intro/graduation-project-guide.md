# 毕业项目设计指南

![毕业项目闭环设计图](/img/course/intro-graduation-project-loop.png)

毕业项目要证明你能把真实问题做成可运行、可评估的 AI 系统。它不需要功能很多，但必须闭环完整。

## 毕业项目闭环

| 层级 | 最小要求 |
| --- | --- |
| 问题 | 用户是谁，解决什么痛点？ |
| 材料/数据 | 输入来自哪里，如何清洗？ |
| AI 能力 | Prompt、RAG、Agent、模型或多模态流程 |
| 交互入口 | CLI、Notebook、API 或简单网页 |
| 评估/日志 | 固定测试、失败样本、引用、trace 或指标 |
| Demo/复盘 | README、截图、演示脚本、下一步计划 |

## 选择题目

| 你手上有什么 | 适合题目 | 评估重点 |
| --- | --- | --- |
| 课程文档或知识库 | RAG / AI 学习助手 | 检索命中、引用支撑、无答案处理 |
| CSV、Excel 或业务数据 | 数据分析 Agent | 结论正确性、图表解释、代码安全 |
| 图片、截图、PDF | 多模态助手 | 解析质量、不确定性、人工审核 |
| 领域文本 | NLP / 垂直助手 | 抽取准确率、标签边界、事实一致性 |
| 没有明确想法 | AI 学习助手 | 可复现、日志、评估、Demo |

## 最小版本

以 AI 学习助手为例，最小毕业项目可以是：

```text
Markdown 文档 -> 切分/索引 -> 用户问题 -> 检索来源 -> 带引用回答 -> Q&A 日志 -> 10 个评估问题
```

第一版不需要复杂 UI、多 Agent 系统或长期记忆。

## README 必须包含

| 部分 | 必须回答 |
| --- | --- |
| 目标 | 解决什么问题，给谁用？ |
| 运行 | 命令和环境变量是什么？ |
| 示例 | 输入和输出是什么？ |
| 架构 | LLM、RAG、Agent、数据、日志在哪里？ |
| 评估 | 固定测试或指标是什么？ |
| 失败样本 | 哪里会坏？ |
| 下一步 | 为什么要这样改进？ |

## 演示脚本

| 时间 | 展示内容 |
| --- | --- |
| 1 分钟 | 问题和用户 |
| 2 分钟 | 架构和数据流 |
| 3 分钟 | 一个成功样例 |
| 2 分钟 | 一个失败样例和诊断 |
| 2 分钟 | 评估结果和下一步 |

能清楚展示一个失败，往往比把失败藏起来更强。它证明你有工程判断力。
