# AI 全栈术语表

![概念分类板](/img/course/intro-concept-sorting-board.png)

这不是背诵表，而是查阅表。如果一个词暂时还不懂，先记住它解决什么问题，等在章节里真正用到时再补细节。

## 查阅规则

| 如果这个词像是在说... | 先看哪一类 |
|---|---|
| 命令、仓库、包、API、JSON | 开发基础 |
| 数据集、特征、标签、baseline、指标 | 数据与机器学习 |
| tensor、loss、embedding、attention、token | 深度学习与大模型基础 |
| Prompt、RAG、chunk、citation、hallucination | 大模型应用 |
| Agent、tool、trace、memory、guardrails | Agent 与工程 |

## 核心术语

| 术语 | 一句话理解 |
|---|---|
| Terminal | 输入命令的地方 |
| 当前目录 | 命令正在执行的文件夹 |
| 包管理器 | 安装 Python 或 Node 依赖的工具 |
| 虚拟环境 | 隔离一个 Python 项目的依赖 |
| Git commit | 保存一次可追踪的代码变化 |
| README | 说明项目做什么、怎么运行的文件 |
| API | 调用服务的约定入口 |
| JSON | 常用于 API、配置和日志的结构化文本 |
| Dataset | 用于分析、训练或评估的数据 |
| Feature | 模型使用的输入信息 |
| Label | 监督学习中模型要预测的目标 |
| Baseline | 用来对比的简单起点模型或规则 |
| 数据泄漏 | 训练时看到了本不该看的信息 |
| 过拟合 | 训练集表现好，新数据表现差 |
| 指标 | 判断结果好坏的数字或规则 |
| Tensor | 深度学习里使用的多维数组 |
| Loss | 预测和目标之间的差距 |
| 反向传播 | 把错误信号传回神经网络各层的方法 |
| Embedding | 文本、图片或类别的向量表示 |
| Attention | 让模型判断哪些输入更重要的机制 |
| Transformer | 许多现代语言模型的基础架构 |
| Token | 模型处理文本的基本片段 |
| 上下文窗口 | 模型一次能看到的输入范围 |
| Prompt | 给模型的指令和输入 |
| 结构化输出 | 要求模型返回 JSON 等固定字段 |
| Function calling | 让模型填写工具调用参数 |
| RAG | 回答前先检索外部资料 |
| Chunk | 用于检索的文档片段 |
| 向量数据库 | 存储和检索向量的系统 |
| Rerank | 在生成前重新排序检索结果 |
| Citation | 回答声称引用的来源 |
| 幻觉 | 看起来合理但没有依据的模型输出 |
| Agent | 能规划步骤、使用工具、保存状态的系统 |
| Tool | 系统可以调用的外部能力 |
| Trace | 记录步骤、输入、输出和工具调用的轨迹 |
| Guardrails | 围绕输入、输出、工具和权限的规则与检查 |
| 可观测性 | 让系统可检查的日志、指标、trace 和错误 |
| 部署 | 让项目离开本地电脑也能访问 |

## 怎么用

不要因为一个词不熟就停下整段学习。先分类，看一句话解释，跑相关例子，用过一次后再回来写自己的解释。
