# 每部分要读多深？

![学习路线选择图](/img/course/intro-learning-path-selection.png)

不要用同样力气读每一页。用三种阅读深度即可。

| 深度 | 什么时候用 | 通关线 |
| --- | --- | --- |
| 必读 | 当前项目离不开它 | 能跑最小例子，并解释输入和输出 |
| 快读 | 现在只需要知道它放在哪里 | 能说出它解决什么问题、以后去哪补 |
| 选修 | 只服务某个方向或毕业项目 | 项目需要时再补实验 |

## 默认阅读深度

| 课程区域 | 第一遍深度 | 原因 |
| --- | --- | --- |
| 工具、Python、数据 | 必读 | 后面所有内容都依赖运行代码和处理数据 |
| 数学、机器学习、深度学习 | 先快读；模型方向学习者再精读 | 需要理解指标、训练、相似度和失败分析 |
| LLM、Prompt、RAG、Agent | 做应用时必读 | 这是大模型应用主线 |
| CV、NLP、多模态 | 第一遍选修 | 毕业项目需要哪个方向再选哪个 |
| 部署、评估、安全 | 做作品集时必读 | 别人不能运行或评估，Demo 就不够硬 |

## 可以继续往前走的规则

满足三件事就可以先前进：

1. 有一个能运行的最小例子。
2. 能用一句话解释输入和输出。
3. 保存一份证据：截图、日志、指标、README 笔记或失败样本。

如果这三件事还没有，少读一点，先做出最小成果。
