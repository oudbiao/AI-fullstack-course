# 新手轻松学习指南

![30 分钟 AI 快速体验闭环](/img/course/intro-quick-experience-loop.png)

如果你觉得课程地图很大，就把这页当成减压入口。你不需要先理解所有内容再开始。

第一遍只记住一个规则：

> 跑一个小结果，解释一个输入和一个输出，保存一份证据，然后进入下一步。

## 第一周节奏

| 天数 | 做什么 | 有什么就停 |
| --- | --- | --- |
| 第 1 天 | 跑 30 分钟快速体验 | 一个 AI 结果或截图 |
| 第 2 天 | 准备 Python、VS Code、Git 和项目文件夹 | `python --version` 与 `git --version` 能运行 |
| 第 3 天 | 打开能力地图 | 能说出当前在哪一层：工具、数据、模型、LLM、应用、Agent 或工程 |
| 第 4 天 | 开始第 1 章 | 一个项目文件夹和一份 README |
| 第 5 天 | 写一个最小 Python 脚本 | 一个命令和一个输出 |
| 第 6 天 | 记录一个真实错误 | 错误信息、原因和修复笔记 |
| 第 7 天 | 复盘并选下一节 | 一句话写清下一步做什么 |

不要把“读了多少页”当成主要成绩。先看 **结果 + 证据 + 下一步**。

## 第一遍先忽略什么

| 暂时忽略 | 原因 |
| --- | --- |
| 完美理论笔记 | 跑过例子后概念更容易懂 |
| 所有选修路线 | 先走默认路线 |
| 大模型训练 | 基础没稳时成本高、反馈慢 |
| 复杂框架 | 容易遮住最基本的输入输出闭环 |
| 作品集精装修 | 先让项目能运行 |

可选页面先收藏，不必一开始全打开。它们以后有用，但不是起跑线。

## 每天最小闭环

![阶段检查卡](/img/course/intro-stage-checkpoint-cards.png)

一次学习可以很小：

| 部分 | 例子 |
| --- | --- |
| 一个输入 | 命令、文件、Prompt、问题 |
| 一个输出 | 一行结果、一张图、一个 JSON 文件、一个模型回答 |
| 一个记录 | README 笔记、截图、错误记录、指标表 |

三样都有，就算有效学习。

## 一章看不懂怎么办

不要把同一页重读五遍。改问下面几个问题：

| 问题 | 为什么有用 |
| --- | --- |
| 这一页解决什么问题？ | 找到目的 |
| 输入是什么？ | 找到工作从哪里开始 |
| 输出是什么？ | 找到怎样算成功 |
| 我能跑最小例子吗？ | 把阅读变成行动 |
| 哪个项目任务会用到它？ | 把课程拉回主线 |

能回答这些，就可以继续。细节等项目需要时再补。

## 简短复盘提示词

只有在你真的有命令、输出或错误时再用：

```text
我正在学习 AI 全栈课程第 X 阶段。
这是我今天运行的内容：
[命令、输出、错误]

请解释我学到了什么，判断卡点属于哪一层，并给我一个 30 分钟内能完成的下一步。
```

它有用的地方不是“让 AI 解释所有东西”，而是逼自己把学习记录整理成小而可复盘的形状。
