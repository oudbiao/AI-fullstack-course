# 学习指南与任务单：机器学习怎么学最不容易学乱

![机器学习学习循环图](/img/course/ml-study-loop.png)

如果你来到 `第 5 章 机器学习入门到实战` 后觉得模型名很多，先不要急着逐个背算法。机器学习第一遍最重要的是完整建模流程。

## 本阶段总原则

机器学习要抓住一条项目主线：把问题翻译成任务，准备数据，建立 baseline，用指标评估，再通过特征和模型改进效果。

![机器学习学习指南项目循环](/img/course/ch05-study-guide-project-loop.png)

## 本阶段必须完成的任务

![机器学习通关任务清单图](/img/course/ml-task-checklist.png)

用这张清单把本阶段落到实处。机器学习项目不只是“跑出一个分数”，而是要能复跑、能解释失败、能继续改进。

| 任务 | 产出物 | 通过标准 |
|---|---|---|
| 建立 ML 问题意识 | 一份问题定义说明 | 能区分分类、回归、聚类和异常检测 |
| 跑通 sklearn baseline | 一个最小训练脚本 | 能完成 train/test split、fit、predict、score |
| 完成引导式证据包工作坊 | 生成的 `ml_workshop_run/` 证据包 | 能复跑脚本，并解释 `model_comparison.csv`、`threshold_review.csv`、`error_samples.csv` 和 `leakage_check.md` |
| 完成特征工程 | 一份特征处理记录 | 能说明缺失值、类别变量、标准化和泄漏风险 |
| 完成模型评估 | 一份指标对比表 | 能解释准确率、召回率、F1、AUC 或 RMSE 的适用场景 |
| 完成一个阶段项目 | 一个可复现实验项目 | 有 README、数据说明、指标、失败样本和改进计划 |

## 推荐学习顺序

第一轮先学机器学习历史突破、基础概念和 Scikit-learn 基本用法。你要先知道 Bayes、线性模型、决策树、SVM、随机森林、Boosting、XGBoost 为什么依次出现，再理解训练集、测试集、特征、标签、模型、指标和 pipeline 是什么。

第二轮学监督学习。线性回归、逻辑回归、决策树和集成学习足够支撑很多入门项目。

第三轮学无监督学习。聚类、降维和异常检测帮助你理解没有标签时如何发现结构。

第四轮学模型评估与选择。指标、交叉验证、偏差方差和调参决定你是否能判断模型真的好不好。

第五轮学特征工程和项目。很多表格数据项目里，特征处理比换模型更重要。

如果你想在进入大项目之前先做一次具体彩排，可以完成 [实操工作坊：构建可复现的 ML 证据包](./ch06-projects/05-hands-on-ml-workshop.md)。它会把抽象建模闭环变成一个可运行脚本和一整套证据文件夹。

## 新人建模术语翻译表

当机器学习术语像背单词一样让人发懵时，先把它翻译成项目动作：

| 术语 | 第一直觉 | 在代码或报告里应该做什么 |
|---|---|---|
| 特征 | 模型能看到的输入列 | 判断它是数值、类别、时间还是文本 |
| 标签 / 目标 | 模型要学习预测的答案 | 和特征分开，避免泄漏 |
| Baseline | 最简单、用来比较的模型或规则 | 先训练它，再追求复杂模型 |
| 指标 | 判断模型好坏的尺子 | 根据任务选择 accuracy、F1、AUC、MAE、RMSE 或轮廓系数 |
| 训练 / 验证 / 测试集 | 学习、选方案、最终验收的三类数据 | 不要让测试集信息提前进入预处理 |
| 过拟合 | 模型把训练数据记得太死 | 对比训练分数和测试分数，简化或正则化 |
| 欠拟合 | 模型太弱，学不到规律 | 增加有用特征或换表达能力更强的模型 |
| 交叉验证 | 换不同切分反复评估 | 当单次切分不稳定时使用 |
| Pipeline | 把预处理和模型打包在一起 | 防止泄漏，让实验可复现 |
| 超参数 | 训练前由人设定的模型配置 | 用验证集或交叉验证调，不要用测试集调 |

如果你能把这些术语解释成项目动作，就不只是学 API，而是在学如何做一次建模调查。

## 建议学习节奏

| 内容类型 | 建议时间 | 学习目标 |
|---|---|---|
| 基础概念页 | 2～3 小时 | 能解释任务、数据和指标 |
| 算法页 | 2～4 小时 | 能知道算法适用场景和直觉 |
| 评估页 | 2～4 小时 | 能判断模型效果是否可信 |
| 项目页 | 8～16 小时 | 跑通完整建模闭环 |

## 阶段项目路线

在选择更大的项目主题前，建议先跑一遍 [实操证据包工作坊](./ch06-projects/05-hands-on-ml-workshop.md)。把它当作热身：你会在同一次练习中生成数据、划分数据、训练 baseline、比较模型、检查错误，并写出项目证据。

第一个项目建议做房价预测，练习回归、特征处理和误差分析。

第二个项目建议做客户流失预测，练习分类、混淆矩阵、AUC、召回率和业务解释。

第三个项目建议做用户分群，练习无监督学习、聚类结果解释和可视化。

如果时间有限，至少完整做一个项目，不要只学算法页。

## 常见卡点

最常见的卡点是“不知道该用哪个模型”。解决方式是先做 baseline。很多时候，先用简单模型确认数据和指标没问题，比一开始尝试复杂模型更重要。

第二个卡点是指标混乱。分类、回归、聚类指标不同，要先问任务是什么，再选指标。

第三个卡点是数据泄漏。标准化、编码、特征选择都要注意训练集和测试集边界。

## 阶段作品集交付物

如果想把本阶段成果沉淀到作品集，建议至少保留下面这些文件或等价材料。

| 交付物 | 说明 |
|---|---|
| `train.py` 或 Notebook | 可复现的 baseline 训练流程，包含数据切分、训练和评估 |
| `feature_report.md` | 特征含义、缺失值处理、编码、标准化和泄漏风险检查 |
| `metrics.md` | 记录 accuracy、F1、AUC、RMSE 等指标及选择理由 |
| `error_analysis.md` | 保存预测错误样本，分析数据、特征、模型或指标问题 |
| `README.md` | 问题定义、运行命令、模型结果、限制和下一步计划 |

## 阶段通关问题

学完本阶段后，你应该能独立完成一个机器学习项目：定义任务、处理数据、训练 baseline、选择指标、改进模型、解释结果。

进入第 6 章前，确认自己能回答这些问题：

- 为什么要划分训练集和测试集？
- 什么是数据泄漏，`Pipeline` 如何降低风险？
- 什么时候准确率不可靠？
- 为什么简单 baseline 是机器学习项目的起点？
- 模型哪里做得好、哪里做得不好、下一步应该怎么改？

如果你能把一个表格数据项目整理成报告，并回答这些问题，就可以进入深度学习阶段。
