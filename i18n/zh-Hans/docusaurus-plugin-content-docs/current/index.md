# AI 全栈工程课程

![先从这里开始四步图](/img/course/intro-start-here-four-step-map.webp)

这门课训练你做出别人能检查的 AI 应用：可运行代码、可复现环境、数据说明、模型证据、Prompt/RAG/Agent trace，以及清楚的边界。

先简单开始：体验一个 AI 示例，准备最小环境，看懂能力地图，然后进入第 1 章。就业/转型目标不是背 AI 热词，而是持续积累项目证据，证明你能把模型行为变成可用的产品工作流。

课程只有一条工程主线。第 1-9 章从工具、Python、数据、模型、LLM、RAG 到 Agent。第 10-12 章把同一种项目思维扩展到视觉、NLP、多模态/AIGC。选修模块会在项目需要时加深部署、进阶 Python 和经典机器学习。

## 你最终要做出的东西

学完主线后，你最强的项目应该能完整说明一个 AI 应用：

| 证据 | 证明什么 |
|---|---|
| 环境设置和 README 命令 | 别人可以运行你的工作 |
| 数据、Prompt、检索和工具 trace | 应用行为可观察，不是黑箱演示 |
| 指标、对比笔记和失败样本 | 你会判断质量，而不是相信一次 demo |
| 安全、隐私、成本和延迟说明 | 你理解产品约束 |
| 截图或短演示 | 审阅者能快速理解用户体验 |

## 一个规则

短读，跑起来，留证据。每个阶段结束时，都应该留下别人能检查的东西：README 命令、保存的输出、指标表、trace、失败笔记，或者一个小演示。这也是把学习转成作品集故事的方法。
