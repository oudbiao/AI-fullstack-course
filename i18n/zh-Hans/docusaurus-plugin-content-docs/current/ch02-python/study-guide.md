# 2.0 学习指南与任务单：Python 编程基础

把这一页当成第 2 章的控制台。目标不是背完每条语法规则，而是能写出一个会运行、会保存数据、能处理错误、还能解释清楚的小程序。

## 2.0.1 推荐学习顺序

![Python 学习指南最小闭环](/img/course/ch02-study-guide-program-loop.png)

先记住一条主线：**输入 -> 数据结构 -> 函数 -> 文件或输出**。

| 顺序 | 小节 | 重点 | 需要留下的证据 |
|---|---|---|---|
| 1 | `2.1 Python 语言基础` | 变量、类型、运算符、输入输出、控制流 | 5 个自己敲过并改过的小脚本 |
| 2 | `2.1 数据结构与函数` | list、dict、函数、模块 | 一个拆成小函数的脚本 |
| 3 | `2.2 Python 进阶` | 类、异常、文件、迭代器、类型提示 | 一个文件读取或 JSON 保存示例 |
| 4 | `2.3 阶段项目` | CLI、爬虫、Web API、AI API 体验 | 一个可运行项目文件夹 |
| 5 | `2.5 跟做工作坊` | CLI 命令、JSON 持久化、统计、报告导出 | `ch02_output/` 和 README 记录 |

不要一开始追高级语法。先让程序跑起来，保存一点东西，再读回来，并能说明每个函数为什么存在。

## 2.0.2 开始前需要认识的术语

| 术语 | 英文全称 | 本章先怎么理解 |
|---|---|---|
| `CLI` | Command-Line Interface | 用文字命令操作的程序。 |
| `I/O` | Input/Output | 数据进入程序，以及结果离开程序。 |
| `JSON` | JavaScript Object Notation | 用来保存嵌套数据的轻量文本格式。 |
| `API` | Application Programming Interface | 让一个程序调用另一个程序的接口。 |
| `SDK` | Software Development Kit | 把 API 包成更容易调用的库。 |

把缩写理解成能力：界面、存储、请求、响应或工作流。

## 2.0.3 本阶段必须完成的任务

| 任务 | 产出物 | 通过标准 |
|---|---|---|
| 掌握最小编程闭环 | 5 个小脚本 | 能用变量、条件、循环和函数解决简单任务 |
| 组织数据 | list/dict 练习文件 | 能选择 list、dict、set 或 tuple，并说出原因 |
| 拆分函数 | 一个包含多个函数的小程序 | 每个函数都有清楚的输入、输出和职责 |
| 读写文件 | 文本、JSON 或 JSONL 示例 | 能保存结果并读回来，不靠猜路径 |
| 处理错误 | 一份排错记录 | 能定位行号、检查变量值，并修复至少 2 个常见错误 |
| 完成跟做工作坊 | `ch02_output/` | 能运行命令、保存 JSON、完成任务、查看统计并导出报告 |

## 2.0.4 小项目模板

本阶段每个 Python 项目都按同一套结构做：

1. 写清楚运行命令。
2. 准备一个很小的输入。
3. 打印或保存一个清楚的输出。
4. 加一个错误场景。
5. 在 README 里记录你改了什么。

这套模板足够覆盖任务管理器、爬虫、Web API 和 AI API 示例。

## 2.0.5 阶段作品集交付物

| 交付物 | 证明什么 |
|---|---|
| `todo_cli.py` 或项目文件夹 | 你能做一个小型命令行工具。 |
| `data/tasks.json` | 你能保存并重新读取结构化数据。 |
| `api_demo.py` | 你能调用外部服务并检查响应。 |
| `README.md` | 别人能复现你的项目。 |
| `debug_notes.md` | 你能从错误中学习，而不是把错误藏起来。 |
| `ch02_output/` | 你完成了跟做工作坊的证据包。 |

## 2.0.6 阶段通关问题

进入第 3 章前，确认自己能回答这些问题：

- 什么时候应该把重复代码改成函数？
- 什么时候字典比列表更合适？
- 文件路径是相对于哪个文件夹计算的？
- `print` 和 `return` 的区别是什么？
- 一个脚本怎样逐步变成带模块、数据和 README 的小项目？

只要你能从零做出命令行任务管理器，并增加搜索、排序、分类或导出之类的一个小功能，就可以继续往后学。
