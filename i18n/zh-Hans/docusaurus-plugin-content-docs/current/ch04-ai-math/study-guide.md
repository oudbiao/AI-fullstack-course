# 4.0 学习指南与任务单：AI 数学基础

![AI 数学学习循环图](/img/course/math-study-loop.png)

把这一页当成第 4 章的控制台。本章目标不是学完所有证明，而是建立足够的数学直觉：看懂模型输入、不确定性、损失和参数更新。

## 4.0.1 推荐学习顺序

![AI 数学学习指南最小闭环](/img/course/ch04-study-guide-math-minimum-loop.png)

先学理论，再做完整工作坊。这样工作坊不会变成一堆没解释的命令。

| 顺序 | 小节 | 重点 | 需要留下的证据 |
|---|---|---|---|
| 1 | `4.1 线性代数实战` | 向量、矩阵、点积、范数、余弦相似度 | 一个很小的向量相似度对比 |
| 2 | `4.2 概率与统计实战` | 概率、分布、均值、方差、熵 | 一份概率或不确定性小笔记 |
| 3 | `4.3 微积分与优化实战` | 导数、梯度、学习率、梯度下降 | 一张参数更新表或短图 |
| 4 | `4.4 数学实操工作坊` | 用一个可运行脚本串起三类概念 | 自动生成的证据文件夹 |

如果你只有一天时间，就先读每个理论小节的 roadmap，跑一个最小代码例子，再完成 `4.4`。

## 4.0.2 开始前需要认识的术语

| 术语 | 英文全称 | 本章先怎么理解 |
|---|---|---|
| `ML` | Machine Learning | 机器学习：用数据训练模型，让模型预测或决策。 |
| `RAG` | Retrieval-Augmented Generation | 检索增强生成：先找资料，再让模型基于证据回答。 |
| `LLM` | Large Language Model | 大语言模型：会预测和生成 token 的大型文本模型。 |
| `Embedding` | Vector representation | 向量表示：把文本、图片、用户或物品变成可比较的向量。 |
| `Notebook` | Jupyter Notebook | 把代码、图表、笔记和输出放在一起的学习实验文件。 |

现在不需要吃透这些工程方向。它们只是帮你知道：这些数学概念以后会在哪里用到。

| 任务 | 产出物 | 通过标准 |

## 4.0.4 公式到直觉翻译表

公式看起来吓人时，先翻译成模型语言，再考虑推导。

| 术语 | 第一直觉 | 后面会在哪里见到 |
|---|---|---|
| 向量 | 一行特征，或者“含义”的坐标 | Embedding、相似度搜索、模型输入 |
| 矩阵 | 很多行数据，或者变换向量的机器 | 神经网络层、Attention 权重 |
| 点积 | 两个方向有多一致 | 余弦相似度、Attention 分数 |
| 范数 | 长度或强度 | 距离、正则化、梯度裁剪 |
| PCA | 旋转坐标轴，只保留信息最多的方向 | 降维、可视化 |
| MLE | 选择最能解释观测数据的参数 | 损失函数、逻辑回归 |
| 熵 | 不确定性或惊讶程度 | 分类损失、语言模型 |
| GD / SGD | 在损失地形上往下坡走 | 模型训练、优化器 |

如果你能把右边说出来，就已经掌握了第一遍最有用的版本。

## 4.0.6 阶段通关问题

进入第 5 章前，确认自己能做到这五件事：

- 解释为什么一个样本可以表示成向量，整个数据集可以表示成矩阵。
- 解释为什么模型输出可以理解成概率或置信度。
- 解释熵、交叉熵和不确定性、损失之间的关系。
- 用代码或表格演示一个最小梯度下降过程。
- 跑完 [4.4 实操：第 4 章完整数学工作坊](./hands-on-math-workshop.md)，并保留生成的证据文件夹。

只要你能把每个公式连接到一个模型动作：表示数据、衡量不确定性、衡量损失或更新参数，就可以继续往后学。
