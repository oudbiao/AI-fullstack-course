# 段階学習タスクシート：ディープラーニングと Transformer の基礎

この段階の目標は、ニューラルネットワークがどのように学習するのか、PyTorch でモデルをどう組み立てるのか、そして Transformer がなぜ現代の大規模モデルの基盤になったのかを理解することです。いきなり大規模モデルの学習を目指すのではなく、まずは Tensor、モデル、損失、オプティマイザ、学習ループ、評価の流れをしっかり動かせるようにしましょう。

## この段階で必ず完了するタスク

| タスク | 成果物 | 合格基準 |
| --- | --- | --- |
| ニューラルネットワーク学習の全体フローを理解する | 手書きの学習フローチャート | 順伝播、損失、逆伝播、パラメータ更新を説明できる |
| PyTorch の基礎を一通り動かす | 最小構成の学習スクリプト | Dataset、DataLoader、nn.Module、optimizer を使える |
| 画像またはテキストの小タスクを完成させる | 実行可能な学習プロジェクト | 学習曲線、検証指標、エラーサンプルを記録できる |
| Attention と Transformer を理解する | 構造説明ノート | Query、Key、Value、Self-Attention、位置エンコーディングを説明できる |
| 段階プロジェクトを完成させる | ディープラーニング実践プロジェクト | 学習ログ、指標、再現可能なコマンド、振り返りがある |

## おすすめの学習順序

まずニューラルネットワークの学習プロセスを理解し、その後に PyTorch の基礎を学び、最後に CNN/RNN/Attention/Transformer を学びます。Transformer を単独の数式として捉えないようにしましょう。本質的には、系列情報、文脈関係、並列計算効率を扱うための仕組みです。

PyTorch のコードを書くときは、まずデータの形状に注目してください。初心者の多くのつまずきは、Tensor の shape、batch 次元、loss の入力形式、device の不一致に関係しています。モジュールを 1 つ書くたびに、入力と出力の形状を 1 回出力するのがおすすめです。

## AI 学習アシスタントプロジェクトとの関係

この段階は、AI 学習アシスタントの v0.5「表現学習理解版」に対応します。学習アシスタントのために必ず大規模モデルを学習する必要はありませんが、embedding、系列モデリング、Transformer の基礎は理解しておくべきです。これは、その後の RAG、Prompt、微調整、Agent の理解に直接影響します。

簡単な実験として、シンプルなテキスト分類や類似度タスクで、異なるテキスト表現方法の効果を観察してみましょう。大事なのは高得点を取ることではなく、「テキストがどうベクトルになるのか」「なぜベクトルの類似度が検索に使えるのか」を理解することです。

## よくあるつまずき

よくある問題には、Tensor の次元不一致、学習 loss が下がらない、検証セットの性能がとても悪い、学習率が大きすぎる・小さすぎる、過学習、GPU/CPU の device 不一致、学習指標を汎化性能と勘違いする、などがあります。学習で問題が起きたら、まず小さなデータセットで過学習テストを行い、コードがちゃんと学習できることを確認してから、データを増やしましょう。


## やさしい版 / 標準版 / 挑戦版タスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| やさしい版 | 最小構成の学習ループを動かす | 初回学習の人、時間が少ない人、入門したばかりの人 |
| 標準版 | 学習曲線と検証指標を保存する | この段階を作品集に入れたい人 |
| 挑戦版 | shape mismatch か loss が下がらない問題を意図的に作って修正する | 基礎があり、より強いプロジェクトの証拠を作りたい人 |

## この段階のバッジと Boss 戦

| 種類 | 内容 |
|---|---|
| Boss 戦 | Shape 巨獣 |
| 解放できるバッジ | Loss 監視員、Shape 追跡者 |
| 最低合格の合言葉 | まず動かす、次に説明する、最後に失敗を記録する |
| 証拠保存のおすすめ | スクリーンショット、ログ、失敗サンプル、評価表を `reports/`、`evals/`、または `logs/` に保存する |

やさしい版を終えれば先へ進めます。標準版を終えたら、作品集に載せるのがおすすめです。挑戦版は、余力があるときだけ取り組みましょう。

## 段階作品集の提出物

この段階の成果を作品集にまとめたいなら、少なくとも以下のファイル、または同等の資料を残すのがおすすめです。

| 提出物 | 説明 |
| --- | --- |
| `train.py` | Dataset、DataLoader、モデル、loss、optimizer を含む PyTorch 学習スクリプト |
| `config.yaml` | 学習率、batch size、epoch、モデル構造などの実験設定 |
| `training_log.csv` | 各エポックの loss、指標、所要時間、検証結果 |
| `curves/` | 学習曲線、検証曲線、混同行列、予測の可視化 |
| `failure_cases.md` | 誤ったサンプル、過学習/未学習の現象、改善アクション |
| `README.md` | データ説明、実行コマンド、モデル結果、制限、振り返り |

これらの資料があると、ディープラーニングのプロジェクトは「学習が動いた」状態から、「学習過程を診断できる」「実験を再現できる」「モデルの失敗を説明できる」状態へと進化します。

## 段階クリアの確認問題

学習後には、次の問いに答えられるようになっているはずです。なぜ逆伝播が必要なのか、optimizer は何を更新しているのか、Dataset と DataLoader はそれぞれ何を解決するのか、Attention はなぜ文脈をモデル化できるのか、Transformer とその後の大規模モデルにはどんな関係があるのか。
