# 6.1.8 任意の背景：深層学習の歴史的ブレークスルー

## 一、まずは深層学習の歴史にある三つの「空気の変化」をつかむ

深層学習は、最初から順調に発展してきたわけではありません。むしろ、「期待が高まる、壁にぶつかる、条件が整う、また爆発する」というサイクルを何度か繰り返してきました。

| 段階 | 当時の期待 | 主なボトルネック | 後のブレークスルー |
|---|---|---|---|
| 第1波のニューラルネットワーク | パーセプトロンで機械がデータからルールを学べる | 単層モデルの表現力が限られ、XOR ができない | 多層ネットワークと逆伝播 |
| 第2波のニューラルネットワーク | 逆伝播で多層ネットワークを訓練できる | 勾配消失、データ不足、計算資源不足 | LSTM、初期化、事前学習の考え方 |
| 深層学習の復興 | データ、GPU、ネットワーク構造がそろう | 深いネットワークの学習の難しさ、系列モデリングのボトルネック | AlexNet、ResNet、Attention、Transformer |

この歴史の流れは、第6章の学習順序とかなり対応しています。

| 歴史上の課題 | 本章で対応する学習 |
|---|---|
| 単一ニューロンで何ができるか | 1.4 ニューロンから MLP へ |
| 多層ネットワークをどう学習するか | 1.5 順伝播と逆伝播 |
| なぜ学習が不安定なのか | 1.6 最適化手法、1.7 正則化、1.8 初期化 |
| CNN はなぜ画像に向いているのか | 3.2 畳み込み操作の原理、3.4 代表的な CNN |
| 画像はなぜ CNN に向いているのか | 第3章 CNN |
| 系列に記憶機構が必要なのはなぜか | 第4章 RNN / LSTM |
| 長期依存と並列学習をどうするか | 第5章 Attention / Transformer |

## 二、1943～1958：人工ニューロンからパーセプトロンへ

1943年、McCulloch と Pitts は人工ニューロンの初期的な抽象化を提案しました。ニューロンは入力を受け取り、簡単な計算をして出力を返す、という考え方です。かなり粗いモデルですが、「脳のような計算」を計算可能なモデルとして表現した最初の一歩でした。

1958年、Rosenblatt はパーセプトロンを提案しました。パーセプトロンが本当に人をわくわくさせたのは、次の点です。

> **機械は人が書いたルールを実行するだけではなく、サンプルからパラメータを調整できる。**

初心者の方は、パーセプトロンを最小のニューラルネットモデルだと考えるとよいです。

```text
入力特徴 -> 重み付き和 -> 活性化判定 -> 出力クラス
```

第5章の線形モデルとよく似ていますが、ニューラルネットワークという道を開いた点が重要です。もし1つのニューロンが少しでも規則を学べるなら、多くのニューロンや多層構造なら、もっと複雑な規則を学べるのではないか、という発想につながりました。

対応して学ぶとよい箇所：

| 対応位置 | 理解したいこと |
|---|---|
| 1.4 ニューロンから MLP へ | ニューロン、重み、バイアス、活性化関数 |
| 第5章 ロジスティック回帰 | 線形スコアと分類確率がどうつながるか |

## 三、1969：XOR 問題が第1波のニューラルネット熱を冷やした

パーセプトロンの限界はすぐに明らかになりました。Minsky と Papert は、単層パーセプトロンでは XOR のような非線形分離問題を解けないことを指摘しました。

XOR の重要な点は、その難しさそのものではなく、次の事実を示したことです。

> **直線では分けられないデータは、単層モデルでは学べない。**

この出来事の歴史的影響は大きく、初期のニューラルネットワークの表現力が非常に不十分であることを示しました。その結果、第1波のニューラルネット熱は明らかに冷え込みました。

しかし今の学習の観点から見ると、XOR はとても優れた教材でもあります。

| 問題 | なぜ重要か |
|---|---|
| 単層モデルでは XOR を分けられない | 線形境界には限界があることを示す |
| 隠れ層を追加すると解ける | 多層ネットワークは非線形を組み立てられる |
| 非線形活性化が必要 | 活性化関数は飾りではなく、表現力の源である |

対応して学ぶとよい箇所：

| 対応位置 | 理解したいこと |
|---|---|
| 1.4 ニューロンから MLP へ | 多層構造と活性化関数がなぜ重要か |
| 第5章 タスクの種類と決定境界 | 線形モデルが万能ではない理由 |

## 四、1980：ネオコグニトロンが CNN の重要な考え方を先取りした

AlexNet よりずっと前の1980年、Fukushima は新認知機（Neocognitron）を提案しました。ここには、後の CNN にとって非常に重要な考え方がすでに含まれていました。

- 局所受容野：すべての画素を全位置につなぐのではなく、まず局所領域を見る
- 階層的特徴：まず単純なエッジを見て、徐々により複雑な形へ組み立てる
- 空間不変性の直感：同じ特徴が別の位置に現れても認識できるべき

初心者の方には、次のように考えるとわかりやすいです。

> **画像はバラバラな画素の集まりではなく、局所的なテクスチャ、エッジ、形が層ごとに組み合わさってできている。**

ネオコグニトロンは今日の主流の実装フレームワークにそのままつながったわけではありませんが、CNN の核となる直感をかなり早い段階で示しました。その後、LeNet、AlexNet、ResNet がこの流れを、学習可能で、拡張可能で、実用的なものへと押し進めていきました。

対応して学ぶとよい箇所：

| 対応位置 | 理解したいこと |
|---|---|
| 3.2 畳み込み操作の原理 | 局所受容野、畳み込み核、特徴マップ |
| 3.4 代表的な CNN アーキテクチャ | LeNet、AlexNet、ResNet が CNN の考え方をどう受け継ぎ、拡張したか |

## 五、1986：逆伝播が多層ネットワークの学習をついに実用的にした

ネットワークが1層だけなら、パラメータの更新は比較的直感的です。しかし多層ネットワークでは、前の層のパラメータが最終的な損失にどう影響するかが複雑になります。

逆伝播が解決した核心はここです。

> **最終的な誤差を計算グラフに沿って一層ずつ戻し、各パラメータをどの方向に動かせばよいかを教える。**

これは第4章の連鎖律に依存しており、第6章の学習ループを動かすエンジンでもあります。

逆伝播は、プロジェクトの振り返りにたとえるとわかりやすいです。

| 学習の動作 | たとえ |
|---|---|
| 順伝播 | まず一度予測してみる |
| loss を計算 | 結果が目標とどれだけ違うかを見る |
| 逆伝播 | どの工程がどれだけ誤差に効いたかをたどる |
| オプティマイザの更新 | 責任分担に基づいてパラメータを調整する |

対応して学ぶとよい箇所：

| 対応位置 | 理解したいこと |
|---|---|
| 1.5 順伝播と逆伝播 | 計算グラフ、loss、gradient |
| PyTorch の自動微分 | `loss.backward()` の意味 |
| 第4章 連鎖律 | 数学的に逆伝播ができる理由 |

## 六、1989～1997：表現力、勾配消失、そして LSTM

1989年、Cybenko の一般近似定理は、非線形を持つフィードフォワードネットワークが非常に強い関数近似能力を持つことを理論的に示しました。これは、ニューラルネットワークの路線にとって重要なメッセージでした。構造と学習が適切であれば、ネットワークは本当に複雑な関数を表現できる、ということです。

ただし、理論的に表現できることと、実際にうまく学習できることは別です。1994年、Bengio らは長い系列の学習における勾配消失問題を体系的に指摘しました。普通の RNN は、長いテキストや長時間系列を扱うと、かなり簡単に昔の情報を「忘れて」しまい、勾配も昔の時刻へ安定して伝わりにくくなります。

1997年、LSTM はゲート機構でこの問題を緩和しました。LSTM は、RNN により信頼できる記憶ノートを付けたようなものだと考えられます。

| モデル | 解決する問題 |
|---|---|
| 普通の RNN | 順序は扱えるが、遠い情報を忘れやすい |
| LSTM | ゲートで「何を記憶するか」「何を忘れるか」「何を出力するか」を制御する |
| GRU | より簡潔なゲート構造で似た能力を実現する |

対応して学ぶとよい箇所：

| 対応位置 | 理解したいこと |
|---|---|
| 1.8 重みの初期化 | なぜ信号と勾配の安定性が重要か |
| 4.2 RNN の基礎 | 系列モデリングと隠れ状態 |
| 4.3 LSTM と GRU | ゲートが長期依存をどう緩和するか |

## 七、2006：RBM / DBN が深いネットワークを再び注目させた

2006年ごろ、Hinton らの Deep Belief Net と RBM の事前学習研究によって、深いネットワークが再び注目されました。当時は深層ネットワークを直接学習するのが難しく、事前学習は「層ごとに表現を学んでから、タスク向けに微調整する」という考え方を提供しました。

今のプロジェクトで RBM を自分で実装する必要はないかもしれませんが、その歴史的意義は次の点にあります。

> **多層の表現学習は、本当に実現可能かもしれない、という期待を再び取り戻した。**

これも「深層学習の復興」に向かう重要な前触れの1つでした。その後、データ規模、GPU、初期化、正則化、最適化手法、ネットワーク構造がそろい、深層学習は本当に大きく広がっていきました。

対応して学ぶとよい箇所：

| 対応位置 | 理解したいこと |
|---|---|
| 学習テクニック | なぜ深いネットワークに初期化、正則化、診断が必要か |
| 生成モデルの補足 | RBM、VAE、GAN はデータ分布を学ぶ異なる考え方 |
| 第7章 事前学習 | 「まず汎用表現を学び、その後タスクへ移す」という考え方の継承 |

## 八、2012～2015：AlexNet、ImageNet、ResNet が深層学習で画像認識を大きく押し切った

2012年、AlexNet は ImageNet の画像分類コンペティションで画期的な成果を出しました。このブレークスルーはモデル構造だけの話ではなく、複数の条件が同時に成熟した結果でもあります。

- 大規模なラベル付きデータセット ImageNet
- GPU による高速化
- より深い CNN
- ReLU、Dropout、データ拡張などの学習テクニック

AlexNet によって、多くの人が「深層学習は画像タスクで本当に従来手法を大きく上回れる」と実感しました。

2015年、ResNet は残差接続で深いネットワークの学習困難を解決しました。残差接続の直感は、各層に「入力からどれだけ変えるか」だけを学ばせればよい、というものです。完全な変換をゼロから学ばせる必要はありません。

対応して学ぶとよい箇所：

| 対応位置 | 理解したいこと |
|---|---|
| 3.2 畳み込み操作の原理 | CNN がなぜ画像に向いているか |
| 3.4 代表的な CNN アーキテクチャ | LeNet、AlexNet、VGG、ResNet の進化 |
| 3.5 転移学習 | なぜ事前学習済みの視覚モデルを新しいタスクへ移せるのか |
| 第10章 コンピュータビジョン | 画像分類、検出、セグメンテーションがどう発展してきたか |

## 九、2017：Attention と Transformer が系列モデリングの主流を書き換えた

RNN と LSTM は系列を順番に処理しますが、そこには自然な問題があります。並列化しにくく、長距離情報への経路も長くなることです。Transformer のブレークスルーは次の点にあります。

> **self-attention によって、系列中の任意の位置同士が直接つながれるようにした。**

`Attention Is All You Need` は単なる新しいモデル提案ではなく、その後の NLP、LLM、多モーダルシステムの土台を変えました。

Transformer はいくつかの重要な問題を解決しました。

| 旧い問題 | Transformer での変化 |
|---|---|
| RNN は順番に計算するので並列化しにくい | self-attention で token を並列に処理できる |
| 長距離依存への経路が長すぎる | attention により遠い token 同士が直接注目し合える |
| さまざまなタスク構造がばらばら | Encoder、Decoder、事前学習の枠組みで多くのタスクを統一しやすい |

対応して学ぶとよい箇所：

| 対応位置 | 理解したいこと |
|---|---|
| 5.2 Attention 機構 | Q/K/V と self-attention |
| 5.3 Transformer アーキテクチャ | Block、残差、LayerNorm、FFN |
| 第7章 大規模モデルの原理 | Transformer がどう LLM の土台になるか |
| 第8～9章 RAG / Agent | 大規模モデルが知識やツールとどうつながるか |

## 十、深層学習のブレークスルーを第6章の学習ルートに割り当てる

| 歴史的ブレークスルー | 解決した問題 | 本講座で対応する章 |
|---|---|---|
| McCulloch-Pitts / Perceptron | ニューロンが計算可能で、パラメータを学習できる | 1.4 ニューロンから MLP へ |
| XOR の限界 | 単層線形モデルの表現力不足 | 1.4 MLP、活性化関数 |
| Neocognitron | 局所受容野と階層的な視覚特徴 | 3.2 畳み込み操作、3.4 代表的な CNN |
| Backpropagation | 多層ネットワークで誤差を分配し、パラメータを更新する方法 | 1.5 順伝播と逆伝播、PyTorch autograd |
| Cybenko の一般近似 | 多層非線形ネットワークの強い表現力 | 1.4 MLP の背景 |
| 勾配消失 | 深いネットワークや長い系列の学習不安定 | 1.8 初期化、4.3 LSTM |
| LSTM / GRU | 長い系列の記憶とゲート制御 | 第4章 RNN と系列モデル |
| RBM / DBN | 深いネットワークが再び学習可能になる歴史的前段階 | 生成モデル、事前学習の背景 |
| AlexNet / ImageNet | データ + GPU + CNN で画像認識を大きく前進 | 第3章 CNN、第10章 画像認識 |
| ResNet | 深い CNN の学習困難 | 3.4 代表的な CNN アーキテクチャ |
| Attention / Transformer | 長期依存、並列学習、統一的な系列モデリング | 第5章 Transformer、第7章 LLM |

## 十一、この節を学び終えたときに持っておきたい直感

深層学習の歴史は、たくさんのモデル名の一覧ではなく、連続した問題の連鎖です。

| 旧い問題 | 新しいブレークスルー | 今身につけたい力 |
|---|---|---|
| ルールを書ききれない | パーセプトロンとニューロン | パラメータ学習を理解する |
| 単層モデルが弱い | 多層ネットワークと活性化関数 | 表現力を理解する |
| 多層ネットワークの学習が難しい | 逆伝播 | 学習ループを理解する |
| 長い系列を覚えられない | LSTM / GRU | ゲート付き記憶を理解する |
| 画像タスクが難しい | CNN / AlexNet / ResNet | 局所特徴と深い構造を理解する |
| 系列の並列化が難しい、長期依存が難しい | Attention / Transformer | LLM の土台を理解する |

それぞれのモデル名を「前の世代の何を解決したのか」で答えられるようになれば、第6章はただのアーキテクチャ一覧ではなく、はっきりした技術進化の流れとして見えてきます。
