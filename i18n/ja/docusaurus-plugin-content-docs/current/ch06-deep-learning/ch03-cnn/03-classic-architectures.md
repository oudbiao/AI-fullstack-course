# 古典的な CNN アーキテクチャ

![古典的な CNN アーキテクチャの進化図](/img/course/imagenet-cnn-evolution.png)

:::tip 本節の位置づけ
古典的なアーキテクチャを学ぶ目的は、モデル名を暗記することではありません。とても重要な進化の流れを見抜くためです。

> **画像タスクがどんどん複雑になると、CNN はどうやって少しずつ強くなってきたのか？**

この流れが理解できると、後でより現代的なビジョンモデルに出会っても、単なる用語の羅列には見えなくなります。
:::

## 学習目標

- LeNet、AlexNet、VGG、ResNet がそれぞれ何を解決したのかを理解する
- 形だけでなく、古典的な CNN の進化のロジックを理解する
- 「小さい畳み込みカーネルの積み重ね」と「残差接続」がなぜ重要かを理解する
- 最小の残差ブロックを書いて、ResNet の核心アイデアを本当に理解する
- 工学的な観点から、異なるアーキテクチャの長所と短所を判断できるようになる

---

## 一、なぜ「古典的なアーキテクチャ」を学ぶのか？

### 1.1 古典モデルは古い知識ではなく、ビジョンモデリングの進化史

CNN アーキテクチャを学ぶとき、初心者は次のようになりがちです。

- LeNet：名前を1つ覚える
- AlexNet：また1つ覚える
- VGG：さらに1つ覚える
- ResNet：なんだか重要らしい

この学び方だと、知識がバラバラになりやすいです。

より良い方法は、これらを1本の進化の鎖として見ることです。

1. LeNet：畳み込みネットワークで画像認識ができることを示した
2. AlexNet：深い CNN を本格的に大きくして、大規模データで成果を出した
3. VGG：「小さい畳み込みカーネルを重ねる」という考え方を標準的な手法にした
4. ResNet：深いネットワークの学習が難しい問題を解決した

ですから、古典的なアーキテクチャを学ぶ本当の目的は「名前を知ること」ではなく、次を理解することです。

> **それぞれの世代が、どの重要な弱点を補ったのか。**

### 1.1.1 これらの名前の背後には、実は「深層ビジョン進化史」がある

もしモデル名としてだけ見ると、すぐに退屈に感じるかもしれません。  
でも進化の歴史として見ると、ぐっとイメージしやすくなります。

- `LeNet` は「畳み込みの道は通用する」と示した存在
- `AlexNet` は「この道は有効なだけでなく、一気に強い」と世界に伝えた存在
- `VGG` は、実務上の経験をより安定したやり方に整理した存在
- `ResNet` は、「さらに深くすると学習が難しい」という本当の壁を解決した存在

この節では、単にモデルを説明しているのではなく、  
次の問いを考えています。

> **ビジョン分野は、なぜ世代ごとに今の形へ進化してきたのか。**

### 1.2 覚えやすい比喩

古典的な CNN アーキテクチャは、「建物を建てること」にたとえられます。

- LeNet：まず建物が本当に建てられることを示した
- AlexNet：建物が高くなり、実際に商用レベルになった
- VGG：より統一された、再現しやすい施工ルールが生まれた
- ResNet：高い建物をさらに高くすると壊れやすい問題を解決した

これが直感的な違いです。

## 二、LeNet：畳み込みネットワークの初期原型

### 2.1 LeNet の歴史的位置づけ

LeNet はもともと手書き数字認識に使われました。  
重要なのは、今どれほど強いかではなく、かなり早い段階で次の主流を形にしたことです。

> **畳み込み層で特徴を取り、プーリング層で圧縮し、最後に全結合層で分類する。**

### 2.2 LeNet の典型的な構造

ざっくり言うと、LeNet は次のような形です。

```text
入力 -> Conv -> Pool -> Conv -> Pool -> FC -> 出力
```

すでに CNN の基本骨格がそろっています。

### 2.3 LeNet は私たちに何を教えたのか？

LeNet が後の研究者に教えたのは、次の点です。

- 画像を最初から平坦化すべきではない
- 局所特徴の抽出は有効である
- 層を重ねて特徴を学ぶことは意味がある

今見ると自然ですが、当時としては大きな突破でした。

### 2.4 なぜ LeNet は古いのに、今でも学ぶ価値があるのか？

LeNet は、本当に「原型機」のような存在だからです。

- すべての問題を解決したわけではない
- しかし、後の CNN の主流となる骨格を初めて示した

だから古典的なアーキテクチャの授業では、  
LeNet の価値は「今どれほど強いか」ではなく、  
次の点にあります。

> **後の人たちに、「この建物はこうやって建てればよい」と示した。**

---

## 三、AlexNet：深い CNN が本当に爆発した始まり

### 3.1 AlexNet がなぜ里程標なのか？

AlexNet が有名なのは、ImageNet で深い CNN の威力を本当に証明したからです。

その重要性は、ざっくり次のように覚えるとよいです。

- モデルがより深くなった
- データがより大きくなった
- GPU が本格的に活躍した
- ReLU、Dropout などの技術が広く使われ始めた

### 3.1.1 なぜ AlexNet は「ディープラーニング復活の号砲」と見なされるのか？

それ以前は、多くの人が本当に信じていたわけではありませんでした。

- もっと深いネットワーク
- もっと大きなデータ
- もっと強い計算資源

この3つがそろうと、これほど大きな性能向上が起きるとは、あまり思われていなかったのです。

AlexNet が衝撃的だったのは、「少しずつ良くなる」感じではなく、  
むしろ次のような変化だったからです。

- 研究の選択肢の1つだったものが、一気に主流になった

そのため歴史的な意味は、ImageNet で勝ったことだけではありません。  
多くの人が初めて本気で次を信じるきっかけになりました。

> **ディープラーニングは単なる流行語ではなく、本当に決定的な結果を出せる。**

### 3.2 AlexNet は何を解決したのか？

初期の小さなモデルと比べると、AlexNet は次を示しました。

> **データ、計算資源、学習の工夫がそろえば、深い畳み込みネットワークは画像認識性能を大きく高められる。**

### 3.3 AlexNet から得られるヒント

AlexNet は単に「深くなった」だけではありません。  
次のことを強く印象づけました。

- 深いネットワークは作る価値がある
- GPU 学習は未来の方向性である
- 活性化関数や正則化の工夫が重要である

つまり、深層ビジョン時代の本当のスタート信号だったのです。

## 四、VGG：なぜ「小さい畳み込みカーネルの積み重ね」がそんなに重要なのか？

### 4.1 VGG の核心思想

VGG で最も覚えやすいポイントは次です。

> **大きな畳み込みカーネルをいきなり使うのではなく、たくさんの `3x3` 小さい畳み込み層を重ねる。**

### 4.2 なぜ大きいカーネルではなく、小さいカーネルを何層も重ねるのか？

連続して小さな畳み込み層を重ねる利点は、主に次の3つです。

1. 感受野を大きくできる
2. パラメータ数をより抑えやすい
3. 中間により多くの非線形性を入れられる

直感的には、次のように考えられます。

- 1つの `7x7` 畳み込み：1回で広い範囲を見る
- 3つ連続の `3x3` 畳み込み：最終的には広い範囲を見つつ、各段階で非線形性を追加できる

そのため、こちらのほうが柔軟なことが多いです。

### 4.3 パラメータ量の直感的な比較

入力チャネル数と出力チャネル数が同じだと仮定して、ざっくり比べると次のようになります。

- `7x7` の畳み込み：パラメータ量は `49` に比例
- `3x3` の畳み込み：パラメータ量は `9` に比例

もちろん、何層も重ねれば必ずしも総パラメータが少なくなるとは限りません。  
ただし、「小さい核 + 多層」は、よりきめ細かい表現を作りやすいことが多いです。

### 4.4 実行できる例：VGG 風の小さなブロック

```python
import torch
from torch import nn

vgg_block = nn.Sequential(
    nn.Conv2d(3, 16, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.Conv2d(16, 16, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2)
)

x = torch.randn(2, 3, 32, 32)
y = vgg_block(x)

print("input shape :", x.shape)
print("output shape:", y.shape)
```

このブロックは、典型的な VGG の考え方にとても近いです。

- まず畳み込みを2回連続で行う
- その後にプーリングする

## 五、ResNet：なぜネットワークが深くなると、学習が難しくなるのか？

### 5.1 直感に反する問題

理屈の上では、ネットワークが深いほど表現力は高くなり、性能も上がりそうです。  
しかし実際には、次のことが分かりました。

> 深くなりすぎると、学習がむしろ難しくなり、性能も必ずしも良くならない。

これは「モデルが強すぎて過学習する」という単純な話ではありません。  
主な原因は次のようなものです。

- 勾配が伝わりにくくなる
- 最適化が難しくなる
- とても深いネットワークは、「少なくとも悪化しない写像」すら学びにくい

### 5.2 ResNet の核心思想

ResNet が提案した鍵は、残差接続です。

> 直接 `H(x)` を学ぶのではなく、`F(x) = H(x) - x` を学ぶ。

すると出力は次のように書けます。

> `y = F(x) + x`

### 5.2.1 ResNet が本当にすごいのはどこか？

ResNet の価値は、「さらに深い」という点そのものではありません。  
本当にすごいのは、次の現実的な問題に答えたことです。

- 深いネットワークには可能性があるのに、なぜ深くすると学習が難しいのか？

ResNet はこの問題を避けずに、  
かなり工学的な答えを示しました。

- 深い層が、元の情報を保ちやすいバイパスを持てるようにする

だから多くの人は ResNet を学んだとき、初めて次を強く実感します。

> **ニューラルネットワークの構造設計は、単に層を積むことではなく、最適化の難しさをきちんと扱うことでもある。**

### 5.3 なぜこれが役立つのか？

これにより、モデルは次のように学びやすくなります。

- その層が役立つなら、新しい特徴を少し学ぶ
- その層で大きく変える必要がないなら、入力情報をできるだけ保つ

言い換えると、

> 残差接続は、深いネットワークに「元の情報を完全に失わない」ためのバイパスを与えるのです。

## 六、実際に手で書いてみる価値のある最小残差ブロック

### 6.1 まずコードを見る

```python
import torch
from torch import nn

class ResidualBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, kernel_size=3, padding=1)
        self.relu = nn.ReLU()
        self.conv2 = nn.Conv2d(channels, channels, kernel_size=3, padding=1)

    def forward(self, x):
        identity = x
        out = self.conv1(x)
        out = self.relu(out)
        out = self.conv2(out)
        out = out + identity
        out = self.relu(out)
        return out

block = ResidualBlock(8)
x = torch.randn(2, 8, 16, 16)
y = block(x)

print("input shape :", x.shape)
print("output shape:", y.shape)
```

### 6.2 このコードで本当に重要なのはどの行か？

最も重要なのは次の行です。

```python
out = out + identity
```

これが残差接続そのものです。

新しい特徴を学びながら、元の入力情報を完全には失わないようにします。

### 6.3 なぜ shape をそろえる必要があるのか？

要素ごとの加算をするためです。  
入力と出力の shape が一致していなければ、直接残差加算はできません。

そのため、実際の ResNet ではときどき次のものが使われます。

- `1x1 conv`

これは次元をそろえるためです。

## 七、古典的なアーキテクチャから見える、はっきりした進化の線

### 7.1 進化のロジックまとめ

| アーキテクチャ | 核心的な貢献 | 解決した問題 |
|---|---|---|
| LeNet | CNN の骨格の原型 | 畳み込みが画像認識に使えることを証明した |
| AlexNet | より深く、より大きく、GPU 学習 | 深い CNN を本格的に爆発させた |
| VGG | 小さい畳み込みカーネルの多層積み重ね | 表現力を高め、構造をより統一した |
| ResNet | 残差接続 | 深いネットワークの学習困難を解決した |

### 7.2 本当に覚えるべきこと

覚えるべきなのは、「何層目に何個の畳み込みがあるか」ではありません。  
次の点です。

> **古典的な各世代のアーキテクチャは、より深く、より安定し、より強いビジョン表現学習への道を切り開いた。**

## 八、今でもこれらの古典的なアーキテクチャを学ぶべきか？

### 8.1 学ぶべきだが、元のまま再現するためではない

今の実プロジェクトでは、LeNet や AlexNet から始めないことも多いです。  
それでも、これらが重要である理由は、次のことを教えてくれるからです。

- CNN がなぜこの形になったのか
- 深さがなぜ重要なのか
- 小さい畳み込みカーネルがなぜ広まったのか
- 残差がなぜほぼ標準になったのか

### 8.2 多くの現代モデルにも、これらの考え方は残っている

より新しいモデルが登場していても、その核心には今でも古典的な CNN の思想が見えます。

- 階層的な特徴抽出
- チャネルの拡張
- 深い積み重ね
- 残差パス

## 九、初心者がよくハマる落とし穴

### 9.1 古典的なアーキテクチャを「モデル名の暗記」として学ぶ

これが一番忘れやすく、応用もしにくいです。

### 9.2 構造図だけを見て、「なぜそう設計されたのか」を考えない

設計の動機を理解していないと、新しいアーキテクチャに出会ったとき、その変更に意味があるかどうか判断しにくくなります。

### 9.3 ResNet の重点は「より深いこと」だと思い込む

ResNet の本当の鍵は、深さそのものではなく、次の点です。

> **より深くしても学習できるようにしたこと。**

## まとめ

この節で最も大事なのは、LeNet、AlexNet、VGG、ResNet の名前を覚えることではありません。  
次の主線をつかむことです。

> **古典的な CNN アーキテクチャの進化は、本質的には「画像ネットワークをどうやってより深く、より安定に、より効果的に学習させるか」という問いへの答えである。**

この流れが理解できれば、後でより現代的なビジョンモデルを見ても、それがどの考え方を引き継ぎ、どこを改良したのかが分かるようになります。
