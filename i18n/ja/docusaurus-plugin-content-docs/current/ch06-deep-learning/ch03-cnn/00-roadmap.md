# 学習ガイド：この章のCNNで何を学ぶのか

この章で解決するのは、次のことです。

> **なぜ画像は、普通の表形式データのようにそのまま学習できないのか。なぜ畳み込みネットワークが必要なのか。**

## まずは橋渡しの線を引こう

前の MLP から来た人にとって、この章でまず押さえたいのは次の点です。

- MLP が間違っているわけではない
- ただし、画像のような「空間構造を持つデータ」にはあまり自然ではない

より安定した理解のしかたは、次の図のようなイメージです。

![CNNの章の関係図](/img/course/ch06-cnn-chapter-flow-ja.png)

つまり、この章は全結合ネットワークを否定するのではなく、次の問いに答えています。

> **データが画像のとき、なぜネットワーク構造もそれに合わせて変える必要があるのか。**

## この章の主な流れ

## この章は、初心者にこういう順番で読むのがおすすめ

1. まず、畳み込みが何をしているのかを理解する  
   いきなりアーキテクチャ名を覚えずに、「局所接続」「パラメータ共有」「受容野」の3つをまず押さえましょう。

2. 次に、CNN の基本構造を見る  
   まずは畳み込みブロック、プーリング、チャネル数、分類ヘッドをつなげて理解します。

3. それから、定番アーキテクチャの進化を見る  
   この段階で LeNet / AlexNet / VGG / ResNet を見ると、単なるモデル名の一覧ではなく、設計の進化として見えてきます。

4. その次に、転移学習を見る  
   これで「なぜ画像分野ではゼロから学習しないことが多いのか」を初めて実感できます。

5. 最後に、画像分類プロジェクトをやってみる  
   学習、評価、誤り分析までを本当に一通りつなげます。

## この章で最初に押さえるべきこと

- 画像は普通の表データではない
- 畳み込みのいちばん重要な価値は、空間構造を保てること
- CNN の多くの設計は、表現力、パラメータ数、学習の安定性のバランスを取るためのもの
- 後ろに出てくる分類、検出、セグメンテーションは、実はこの章の直感の上に成り立っている

## 初心者がつまずきやすいポイント

- 「畳み込み核が動く」ということだけ覚えて、なぜそうするのか分からない
- shape の変化が多くて混乱する
- モデル名は覚えたのに、なぜ構造が進化してきたのか説明できない
- いきなり大きなモデルを作ろうとして、最小の画像分類の流れを先に作らない

## この章を終えたら、自分で答えられるようになりたいこと

- なぜ画像タスクは、平坦化してそのまま学習するより畳み込みに向いているのか
- 1つの畳み込み層は、実際に何を抽出しているのか
- CNN の中で、チャネル、プーリング、受容野はそれぞれ何をしているのか
- なぜ転移学習は、画像タスクでこんなに一般的なのか

## 初心者と上級者では、どう読めばいいか

初心者がこの章を初めて学ぶときは、まず主線と最小の実行例を押さえましょう。すべての細部を一度で理解する必要はありません。この章が何を解決するのか、入力と出力は何か、最小プロジェクトをどう動かすかが説明できれば、先へ進んで大丈夫です。

経験のある学習者は、この章を知識の抜けを埋める場、そして実践的な練習の場として使えます。境界条件、失敗例、評価方法、コードの再現性、前後の学習段階とのつながりに注目しましょう。読み終わったら、この章の内容を自分の作品の README や実験記録にまとめておくとよいです。
