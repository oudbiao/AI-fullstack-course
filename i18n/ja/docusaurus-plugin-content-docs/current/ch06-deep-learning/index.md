# 6 ディープラーニングと Transformer 基礎

![ディープラーニングと Transformer のメインビジュアル](/img/course/ch06-deep-learning-ja.png)

この段階で解決するのは、「モデルの内部ではいったいどうやって学習しているのか」ということです。機械学習の段階では既成のモデル API を使うことが多いですが、ディープラーニングの段階では、パラメータ、勾配、学習ループ、ネットワーク構造、表現学習を本当に見ることになります。

## 物語で導入：モデルのエンジンルームに入ってみよう

機械学習の段階が車を運転することだとしたら、ディープラーニングの段階はボンネットを開けて、どのように動力が生まれるのかをはっきり見ることです。テンソルは燃料、ネットワーク層は部品、損失関数はメーターパネルのようなもの、勾配と最適化器は調整を続ける役割を担います。あなたは初めて、モデルがどのように間違いから学ぶのかを本当に見ることになります。

## 学習クリアマップ

![ディープラーニング学習クリアマップ](/img/course/ch06-learning-quest-map-ja.png)

## 体験練習：学習ループの4つの数字に注目する

小さなモデルを学習するときは、最終的な正解率だけを見ないでください。毎回の実験で、学習損失、検証損失、学習正解率、検証正解率を観察しましょう。学習データはどんどん良くなるのに検証データが変わらないなら、過学習の可能性があります。両方とも悪いなら、モデル、データ、学習率に問題があるかもしれません。こうした曲線を読み取れるようになることは、むやみにモデルを変えるよりずっと大切です。

## プロジェクトのおまけ要素

この段階のおまけ作品は、「小さなモデル実験室」にできます。同じデータセットで、異なるネットワーク構造、学習率、batch size、学習回数の結果をそれぞれ記録し、比較曲線を描いてみましょう。この実験室は、微調整、大規模モデル学習、マルチモーダルモデルを理解するための土台テンプレートになります。

## 初心者向けの最小クリアルート

初心者はまず、テンソル、自動微分、データ読み込み、モデル定義、損失計算、逆伝播、最適化器の更新という学習の一連の流れを通しましょう。PyTorch で小さな分類モデルを学習できて、学習損失と検証指標を読み取れるようになれば、最小クリアです。

## 応用者向けの発展ルート

経験のある学習者は、CNN、RNN、Attention、Transformer、正則化、初期化、学習診断を深く掘り下げるとよいでしょう。さらに、異なるネットワーク構造やハイパーパラメータの実験結果を記録して、自分だけの小さなモデル実験室を作ってみましょう。

## ディープラーニングが重要な理由

ディープラーニングによって、モデルはデータから複雑な表現を自動で学習できるようになります。画像の輪郭、質感、物体、テキストの語義や文脈は、複数層のネットワークを通じて少しずつ表現として形成されます。Transformer はさらに、大規模言語モデルやマルチモーダルモデルの中核アーキテクチャになりました。

![ディープラーニング学習ループの主線図](/img/course/ch06-training-loop-backbone-ja.png)

歴史の流れに沿って技術のブレークスルーを理解したいなら、まず [1.2 ディープラーニングの歴史的ブレークスルー主線](./ch01-nn-basics/06-history-breakthroughs.md) を読んでみてください。感知器、XOR の挫折、逆伝播、勾配消失、LSTM、RBM/DBN、AlexNet、ResNet、Attention、Transformer を本章の各小節に対応づけて、モデルがなぜ代を重ねて進化してきたのかを理解する助けになります。

## 初心者は何を先にやる？上級者は何をする？

初めてこの段階を学ぶ人は、まずニューラルネットワーク学習の最小ループを理解しましょう。データを準備し、モデルを定義し、損失を計算し、逆伝播し、パラメータを更新し、曲線を観察します。最初から複雑な構造を追い求める必要はありません。

経験のある学習者は、学習診断に重点を置けます。過学習をどう見つけるか、学習率が曲線にどう影響するか、データ拡張や正則化はいつ有効か、Transformer がなぜ系列モデリングを変えたのか。あなたの目標は、1回の学習がなぜ成功したのか、あるいは失敗したのかを説明できるようになることです。

## この段階の学習パス

第1章では、ニューラルネットワークの基礎と歴史的ブレークスルーの主線を学びます。感知器、XOR の限界、逆伝播、ニューロン、活性化関数、順伝播、逆伝播、最適化器、正則化、パラメータ初期化が理解できるようになります。

第2章では PyTorch を学びます。テンソル、自動微分、`nn.Module`、データ読み込み、学習ループから始めて、実際に学習可能なモデルを組み立てます。

第3章では CNN を学びます。画像タスクは最も直感的なので、深いネットワーク構造を最初に理解する入口として適しています。

第4章では RNN と系列モデルを学びます。系列データが普通の表形式データとどう違うのかが見えてきて、LSTM、GRU の歴史的意義も理解できます。

第5章では Attention と Transformer を学びます。これは、後続の大規模モデルの主線につながる重要な橋です。

第6章と第7章は拡張として、生成モデルと学習チューニングの理解を助けます。

## 学習後にできるようになること

- ニューラルネットワークの順伝播、損失計算、パラメータ更新を説明できる
- PyTorch で最小の学習ループを書ける
- 簡単な CNN やテキスト分類モデルを学習できる
- RNN、Attention、Transformer の基本的な違いを理解できる
- 後続の LLM の原理、微調整、マルチモーダル学習の基礎を作れる

## よくある落とし穴

学習コードをただコピーするだけで、それぞれのステップが何をしているのか分からないのはよくありません。少なくとも、データがどうモデルに入るのか、出力からどう損失を計算するのか、勾配がどう逆伝播するのか、最適化器がどうパラメータを更新するのかを説明できる必要があります。

最初から大規模モデルの学習を目指しすぎないことも大切です。ディープラーニングの第1関門で最も重要なのは、小さなモデルと小さなデータセットを最後まで動かし、学習ループをきちんと理解することです。

## 学習エラー劇場：モデル名より曲線が大事

loss が下がらないなら、まず学習率、ラベル形式、入力の正規化、損失関数の対応を確認しましょう。学習データは良いのに検証データが悪いなら、まず過学習やデータ分割を疑います。GPU メモリが足りないなら、まず batch size、画像サイズ、モデル規模を小さくしてみましょう。

## 最小実行可能実験：1本の完全な PyTorch 学習ループ

この段階の最小実験は、大規模モデルを学習することではなく、小さなデータ、小さなモデル、小さな epoch で学習ループを通すことです。各ステップを説明できるようにしましょう。データがモデルに入り、出力から損失を計算し、損失を逆伝播し、最適化器がパラメータを更新します。

```python
for x, y in dataloader:
    optimizer.zero_grad()
    pred = model(x)
    loss = loss_fn(pred, y)
    loss.backward()
    optimizer.step()
```

学習 loss と検証 loss を描けて、それらがなぜ変化するのか説明できれば、すでにディープラーニング実装の主線をつかめています。

## ディープラーニング失敗事例集：まず shape、loss、曲線を見る

| 現象 | よくある原因 | 確認方法 | 修復の方向 |
|---|---|---|---|
| shape mismatch | 入力次元、batch 次元、またはクラス数が合っていない | 各層の入力出力 shape を出力する | reshape、モデルヘッド、データ形式を調整する |
| loss が下がらない | 学習率、ラベル形式、正規化、または loss の不一致 | まず小さな batch で過学習テストをする | 学習率を調整し、ラベルと入力スケールを確認する |
| 学習は良いのに検証が悪い | 過学習、データ分割が不適切 | 学習/検証曲線を比較する | データ拡張、正則化、早期停止 |
| メモリ不足 | batch、画像サイズ、またはモデルが大きすぎる | メモリ使用量を確認する | batch を小さくする、解像度を下げる、軽量モデルに変える |

## 段階プロジェクト

基礎版は、簡単な画像分類またはテキスト感情分類モデルを学習し、データ読み込み、学習、評価を完了することです。標準版では、検証セット、指標曲線、過学習分析、モデル保存・読み込みを追加する必要があります。チャレンジ版では、CNN、RNN、Transformer、または転移学習の案を比較し、なぜモデルが良くなったのか、あるいは悪くなったのかを説明する実験レポートを書きます。

より細かい学習リズムを見たい場合は、[学習ガイド：ディープラーニング基礎はどう学ぶと一番混乱しにくいか](./study-guide.md) を読んでください。

## AI 学習アシスタントとの貫通プロジェクトとの関係

この段階は AI 学習アシスタント v0.6 に対応できます。テキストまたは画像分類の小さな実験を行い、学習曲線、指標、失敗サンプルを記録しましょう。貫通プロジェクトの流れで学んでいるなら、この段階の終わりに少なくとも1回、バージョン記録を提出することをおすすめします。この段階で何ができるようになったか、どう実行するか、入力と出力の例は何か、どんな問題があったか、次にどう改善するかを残しておきましょう。
