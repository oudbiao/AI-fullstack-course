# 段階学習タスクリスト：データ分析と可視化

この段階の目標は、元データを理解できる結論に変えられるようになることです。今後の機械学習、RAG の評価、Agent ログ分析でもデータ処理能力が必要になるため、ここではデータの読み込み、クリーニング、統計、可視化、説明を重点的に練習します。

## この段階で必ず完了するタスク

| タスク | 成果物 | 合格基準 |
| --- | --- | --- |
| NumPy を使って配列を扱う | 配列計算の練習ファイル | shape、ブロードキャスト、スライス、行列演算を説明できる |
| Pandas を使ってデータを読み込む | データ読み込みスクリプトまたは Notebook | CSV/Excel/JSON を読み込み、基本情報を確認できる |
| データクリーニングを完了する | クリーニング前後の比較記録 | 欠損値、重複値、異常値、型変換を処理できる |
| 探索的分析を完了する | EDA Notebook | 統計とグラフで主な発見を説明できる |
| 段階プロジェクトを完了する | データ分析レポート | 問題定義、グラフ、結論、限界が含まれている |

## おすすめの学習順序

まず NumPy で配列の感覚をつかみ、その後で Pandas の表データ処理に進み、最後に可視化を学びましょう。最初から複雑なグラフを目指さず、まずはそれぞれの図が明確な問いに答えられることを確認してください。

データ分析で大切なのは「どれだけ多くの図を描いたか」ではなく、各ステップで「なぜそのように処理したのか」を説明できることです。たとえば、なぜ一部の行を削除したのか、なぜ欠損値を補完したのか、なぜ折れ線グラフではなく棒グラフを選んだのか、を考えましょう。

## AI 学習アシスタントプロジェクトとの関係

この段階は、AI 学習アシスタントの v0.3 学習記録分析版に対応します。前半で記録した学習質問ログを読み込み、ユーザーが最もよく質問する章、最も多い質問タイプ、週ごとの学習頻度を集計して、簡単な分析レポートを作成できます。

おすすめの出力内容は、質問数の推移、質問の分類統計、高頻度キーワード、つまずきやすい段階、次に学ぶべき内容です。これにより、プロジェクトは「記録ツール」から「学習を支援するツール」へと進化します。

## よくあるつまずき

よくある問題には、DataFrame の行と列を逆にしてしまう、インデックスと列名を混同する、チェーン代入の警告、日付フィールドを変換していない、グラフで日本語が表示できない、相関関係を因果関係と勘違いする、などがあります。分析結果に出会ったら、次のことを自分に問いましょう。この結論はどの列から出たのか、どの処理が影響している可能性があるのか、反例はないか。

## ゆるめ版 / 標準版 / チャレンジ版タスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| ゆるめ版 | 欠損値、重複値、異常値の確認を完了する | 1回目の学習、時間が少ない、または入門したばかりの学習者 |
| 標準版 | クリーニング前後の比較と 2 つのグラフを出力する | この段階を作品集に入れたい学習者 |
| チャレンジ版 | 1 つの誤った結論が、どのように汚れたデータから生まれたかを説明する | 基礎があり、より強いプロジェクトの証拠を作りたい学習者 |

## 段階の作品集提出物

この段階の成果を作品集に残したいなら、少なくとも次のファイル、または同等の材料を保存するのがおすすめです。

| 提出物 | 説明 |
| --- | --- |
| `analysis.ipynb` | 読み込み、クリーニング、統計、グラフ、結論を含む完全な EDA Notebook |
| `data_dictionary.md` | データ項目の意味、型、単位、欠損状況の説明 |
| `cleaning_log.md` | 欠損値、異常値、重複値、型変換の処理記録 |
| `figures/` | 重要なグラフの出力ファイル。各グラフは 1 つの分析問題に対応する |
| `report.md` | 分析目標、主な発見、限界、次のモデリング提案 |

これらの材料があると、データ分析の段階は「Pandas が使える」から「データで判断を支え、その過程をわかりやすく説明できる」へとレベルアップします。

## 段階の合格チェック問題

学習後は、次の問いに答えられるようになっているはずです。NumPy の shape がなぜ重要なのか、Pandas の Series と DataFrame の違いは何か、欠損値処理が結論にどのような影響を与えるか、1 枚の図はどんな問題に役立つべきか、なぜデータ分析レポートに限界を書く必要があるのか。

## 完了状態 Checklist

- [ ] CSV、Excel、JSON データを読み込み、行・列、型、欠損値を確認できる。
- [ ] 基本的なデータクリーニングを実施し、それぞれの処理理由を説明できる。
- [ ] 統計表とグラフを使って、明確な問いに答えられる。
- [ ] 小規模な EDA レポート、または学習記録分析を 1 つ完了している。
- [ ] データ分析の結論の限界を説明でき、グラフだけで終わらせない。
