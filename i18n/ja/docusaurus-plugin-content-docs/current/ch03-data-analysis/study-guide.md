# 学習ガイド：データ分析と可視化はどう学ぶと一番混乱しにくいか

`02 データ分析と可視化` に入って、ライブラリが多い、API が細かいと感じたら、まずはメソッド名を暗記しようとしないでください。この段階で本当に身につけるべきなのは、データの流れを意識することです。

## この段階の基本原則

データ分析の最初の1周は、1本の主線だけを追います。データを読み込んだら、まず項目を理解し、次に整えて整理し、そのあと統計的に分析し、最後に図表で結論を表現します。

![データ分析学習ガイドの最小ループ](/img/course/ch03-study-guide-data-loop-ja.png)

## おすすめの学習順序

1回目は、まず純 Python でのデータ処理を軽く体験して、なぜ専用ツールが必要なのかを感じます。

2回目は NumPy を学びます。重点は配列、shape、インデックス、ブロードキャスト、ベクトル化、行列演算、基本統計です。最初からすべての関数を覚えようとしなくて大丈夫です。

3回目は Pandas を学びます。重点は DataFrame、ファイルの読み書き、選択と絞り込み、欠損値、グループ集計、結合、時系列です。

4回目は可視化を学びます。まず「どのグラフを選ぶか」を学び、そのあとで見た目を整えます。グラフは、きれいに見せるためではなく、問題を説明するために描くものです。

5回目は EDA プロジェクトをやります。読み込み、クリーニング、分析、可視化、結論の文章化までをつなげて、1つの完全なレポートにまとめます。

## おすすめの学習ペース

| 内容の種類 | 目安時間 | 学習目標 |
|---|---|---|
| NumPy 基礎 | 1～2 時間 | 配列、shape、ベクトル化を理解する |
| Pandas 処理ページ | 2～4 時間 | 絞り込み、クリーニング、集計、結合ができる |
| 可視化ページ | 1～3 時間 | 適切なグラフを選び、図表を説明できる |
| プロジェクトページ | 6～12 時間 | 読めるデータ分析レポートを1本完成させる |

## 段階的なプロジェクトの進め方

最初のプロジェクトは、単一データセットの EDA から始めるのがおすすめです。たとえば Titanic、住宅価格、映画評価、EC 注文、または公開されている運用データなどです。項目の理解、欠損値処理、統計要約、重要な図表、結論までを完成させましょう。

2つ目のプロジェクトでは、複数ソースのデータ分析に挑戦できます。複数の CSV、Web データ、またはデータベースのテーブルを結合し、実務でよくあるデータ整理の流れを練習します。

## つまずきやすいポイント

一番よくあるつまずきは、「API が多すぎて覚えられない」というものです。これは普通のことです。まず覚えるべきなのは、よく使う動きだけです。読む、見る、選ぶ、絞る、変更する、グループ化する、結合する、グラフを描く。その他のメソッドは、必要になったときに調べれば大丈夫です。

2つ目のつまずきは、「このグラフが何を示しているのか分からない」ことです。グラフを1枚描く前に、まずそれが答えるべき質問を書いておきましょう。たとえば、「どのカテゴリが一番多いか」「価格と面積に関係はあるか」「外れ値はどこか」などです。

3つ目のつまずきは、データをクリーニングするときに記録を残さないことです。Notebook には、各手順の理由を残すようにしましょう。そうしないと、後で振り返るのがとても難しくなります。

## 合格基準

この段階を学び終えたら、CSV ファイルを1つ渡されても、読み込み、クリーニング、探索、可視化、結論の整理までを自力でできるようになっているはずです。

少なくとも3枚の重要な図表、1つのデータクリーニングの流れ、3つの明確な結論を含むデータ分析レポートを書けるなら、AI 数学と機械学習の段階に進んで大丈夫です。
