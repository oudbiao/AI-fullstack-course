# 3.0 学習ガイドとタスクシート：データ分析と可視化

このページを第 3 章のコントロールパネルとして使ってください。目標は Pandas のメソッドを全部覚えることではありません。生データを、他の人が信頼できる結論に変えることです。

## 3.0.1 この順番で学ぶ

![データ分析学習ガイドの最小ループ](/img/course/ch03-study-guide-data-loop-vertical-ja.png)

まずこのワークフローを覚えます：**読む -> 確認する -> きれいにする -> 集計する -> 可視化する -> 説明する**。

| 順番 | セクション | 重点 | 残す証拠 |
|---|---|---|---|
| 1 | `3.1 Python からデータ分析へ` | 表データを純 Python で扱うつらさ | 小さな before/after メモ |
| 2 | `3.2 NumPy 科学計算` | 配列、shape、ブロードキャスト、ベクトル化 | 配列練習ファイル 1 つ |
| 3 | `3.3 Pandas データ処理` | DataFrame、選択、クリーニング、groupby、merge | クリーンな表とクリーニングログ |
| 4 | `3.4 データ可視化` | グラフ選択、ラベル、結論、限界 | 3 つの質問に対応する 3 つのグラフ |
| 5 | `3.5 データベース基礎` | SQL とリレーショナルデータ。選択だが有用 | クエリまたは join の例 |
| 6 | `3.6 ステージプロジェクト` | EDA レポートと再現可能なパイプライン | レポート、図表ファイル、ワークショップ出力 |

最初にグラフを描かないでください。まずフィールド、単位、欠損値、サンプルの出所を理解します。

## 3.0.2 始める前に知っておく用語

| 用語 | この章での最初の意味 |
|---|---|
| `CSV` | プレーンテキストの表。1 行が 1 レコード。 |
| `JSON` | API がよく返す入れ子構造のデータ。 |
| `DataFrame` | 行、列、列名、インデックスを持つ Pandas の表。 |
| `Series` | DataFrame の 1 列。 |
| `dtype` | 列や配列のデータ型。 |
| `EDA` | 探索的データ分析。モデリング前の最初の確認。 |
| `groupby` | カテゴリで分け、統計量を計算し、再びまとめる処理。 |
| `merge` / `join` | 共通キーで表を結合する処理。 |
| `vectorization` | NumPy/Pandas に多くの値をまとめて処理させること。 |

API を覚えにくいときは、ワークフローに戻してください。読む、確認する、きれいにする、変換する、可視化する、説明する、のどれかです。

## 3.0.3 最小タスクシート

| タスク | 成果物 | クリア基準 |
|---|---|---|
| NumPy 配列を使う | 配列練習ファイル | shape、スライス、ブロードキャスト、ベクトル化演算を説明できる |
| Pandas でデータを読み込む | スクリプトまたは Notebook | CSV/Excel/JSON を読み、行列、型、欠損値を確認できる |
| データをクリーニングする | before/after クリーニング記録 | 欠損値、重複、外れ値、型変換を扱える |
| データを探索する | EDA Notebook | 統計とグラフで明確な質問に答えられる |
| ガイド付きパイプラインを完了する | `ch03_output/` | 生データ、クリーンデータ、SQLite クエリ、グラフ、レポートを再現できる |
| ステージプロジェクトを完了する | データ分析レポート | 問い、手順、グラフ、結論、限界が含まれる |

## 3.0.4 グラフ選択ルール

グラフを描く前に、先に質問を書きます。

| 質問 | 最初に試すグラフ |
|---|---|
| どのカテゴリが最も大きいか？ | 棒グラフ |
| 値は時間とともにどう変化するか？ | 折れ線グラフ |
| 2 つの数値は関係しているか？ | 散布図 |
| 分布はどんな形か？ | ヒストグラムまたは箱ひげ図 |
| グループ間に差はあるか？ | グループ化棒グラフまたは箱ひげ図 |

質問のないグラフは装飾です。質問のあるグラフは証拠になります。

## 3.0.6 ステージチェックポイント

第 4 章に進む前に、次の質問に答えられるか確認してください。

- 配列の `shape` はなぜ重要か？
- Series と DataFrame は何が違うか？
- 欠損値は結論をどう変えるか？
- それぞれのグラフは何の質問に答えているか？
- データ分析レポートに限界を書くべき理由は何か？

1 つの CSV を読み込み、クリーニング、分析、可視化、短い結論まで進められれば、次へ進んで大丈夫です。
