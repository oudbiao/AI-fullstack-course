# 学習ガイド：データ分析と可視化はどう学ぶと一番混乱しにくいか

`02 データ分析と可視化` に入って、ライブラリが多い、API が細かいと感じたら、まずはメソッド名を暗記しようとしないでください。この段階で本当に身につけるべきなのは、データの流れを意識することです。

## この段階の基本原則

データ分析の最初の1周は、1本の主線だけを追います。データを読み込んだら、まず項目を理解し、次に整えて整理し、そのあと統計的に分析し、最後に図表で結論を表現します。

![データ分析学習ガイドの最小ループ](/img/course/ch03-study-guide-data-loop-vertical-ja.png)

## 初心者向けの考え方：データ探偵として調べる

データセットは、1つの事件ファイルだと考えてください。目標は「Pandas を使うこと」や「グラフを描くこと」ではありません。散らかった証拠を、他の人が信頼できる結論に変えることです。

| ステップ | 問うべきこと | よく使う道具 |
|---|---|---|
| 読み込む | データはどこから来たのか？ | `read_csv()`、`read_excel()`、SQL |
| 観察する | 行と列は何を意味するのか？ | `head()`、`info()`、`shape`、`dtypes` |
| きれいにする | 欠損、重複、表記ゆれ、極端な値はあるか？ | `isna()`、`drop_duplicates()`、`fillna()`、絞り込み |
| 変換する | どんな新しい列や集計が必要か？ | 列計算、`assign()`、`groupby()` |
| 可視化する | どのグラフが問いに一番まっすぐ答えるか？ | 棒グラフ、折れ線、散布図、ヒストグラム、箱ひげ図 |
| 説明する | 何を変え、何が分かり、限界はどこか？ | Notebook のメモ、レポート本文、グラフ注釈 |

迷ったら、この一文に戻ってください。**表を読み、列を理解し、明らかな問題を直し、重要なグループを集計し、本当に答えるべき問いに合うグラフだけを描く。**

## API を暗記する前に理解したい基本用語

| 用語 | やさしい説明 |
|---|---|
| `CSV` (Comma-Separated Values) | テキスト形式の表ファイル。1行が1レコードで、カンマで列を区切る |
| `JSON` (JavaScript Object Notation) | API や Web サービスでよく使われる入れ子構造のデータ形式 |
| `DataFrame` | Pandas の表。行、列、列名、インデックスを持つ |
| `Series` | DataFrame の1列 |
| `Index` | DataFrame の行ラベル。意味がある場合も、ただの行番号の場合もある |
| `shape` | データの大きさ。通常は `(行数, 列数)` |
| `dtype` (data type) | 整数、浮動小数点、文字列、日時などのデータ型 |
| `missing value` | 欠損値。多くの場合 `NaN` や `None` で表される |
| `outlier` | 多くの値から大きく離れた外れ値 |
| `EDA` (Exploratory Data Analysis) | 探索的データ分析。モデリング前の最初の調査 |
| `groupby` | カテゴリごとに分け、各グループへ統計処理を行い、結果をまとめる |
| `merge` / `join` | ユーザー ID や商品 ID など共通キーで表を結合する |
| `axis` | 処理の方向。表では `axis=0` は行方向、`axis=1` は列方向を指すことが多い |
| `vectorization` | Python のループではなく、NumPy/Pandas にまとめて計算させること |
| `broadcasting` | 形が合うとき、NumPy が小さい配列を大きい配列に自動でそろえて計算する仕組み |

## おすすめの学習順序

1回目は、まず純 Python でのデータ処理を軽く体験して、なぜ専用ツールが必要なのかを感じます。

2回目は NumPy を学びます。重点は配列、shape、インデックス、ブロードキャスト、ベクトル化、行列演算、基本統計です。最初からすべての関数を覚えようとしなくて大丈夫です。

3回目は Pandas を学びます。重点は DataFrame、ファイルの読み書き、選択と絞り込み、欠損値、グループ集計、結合、時系列です。

4回目は可視化を学びます。まず「どのグラフを選ぶか」を学び、そのあとで見た目を整えます。グラフは、きれいに見せるためではなく、問題を説明するために描くものです。

5回目は EDA プロジェクトをやります。読み込み、クリーニング、分析、可視化、結論の文章化までをつなげて、1つの完全なレポートにまとめます。

自分でデータセットを選ぶ前にガイド付きで練習したい場合は、まず [6.3 ハンズオンワークショップ：再現可能なデータ分析パイプラインを作る](./ch06-projects/03-hands-on-data-workshop.md) を完了してください。固定の入力、期待される出力、生成ファイル、トラブルシューティング表があるので、次に何をするか迷わずに一連の流れを練習できます。

## おすすめの学習ペース

| 内容の種類 | 目安時間 | 学習目標 |
|---|---|---|
| NumPy 基礎 | 1～2 時間 | 配列、shape、ベクトル化を理解する |
| Pandas 処理ページ | 2～4 時間 | 絞り込み、クリーニング、集計、結合ができる |
| 可視化ページ | 1～3 時間 | 適切なグラフを選び、図表を説明できる |
| 6.3 ハンズオンワークショップ | 1～2 時間 | CSV クリーニング、SQLite、グラフ、レポートの一連のパイプラインを実行できる |
| プロジェクトページ | 6～12 時間 | 読めるデータ分析レポートを1本完成させる |

## 段階的なプロジェクトの進め方

最初のプロジェクトは、単一データセットの EDA から始めるのがおすすめです。たとえば Titanic、住宅価格、映画評価、EC 注文、または公開されている運用データなどです。項目の理解、欠損値処理、統計要約、重要な図表、結論までを完成させましょう。

2つ目のプロジェクトでは、複数ソースのデータ分析に挑戦できます。複数の CSV、Web データ、またはデータベースのテーブルを結合し、実務でよくあるデータ整理の流れを練習します。

## つまずきやすいポイント

一番よくあるつまずきは、「API が多すぎて覚えられない」というものです。これは普通のことです。まず覚えるべきなのは、よく使う動きだけです。読む、見る、選ぶ、絞る、変更する、グループ化する、結合する、グラフを描く。その他のメソッドは、必要になったときに調べれば大丈夫です。

2つ目のつまずきは、「このグラフが何を示しているのか分からない」ことです。グラフを1枚描く前に、まずそれが答えるべき質問を書いておきましょう。たとえば、「どのカテゴリが一番多いか」「価格と面積に関係はあるか」「外れ値はどこか」などです。

3つ目のつまずきは、データをクリーニングするときに記録を残さないことです。Notebook には、各手順の理由を残すようにしましょう。そうしないと、後で振り返るのがとても難しくなります。

## 合格基準

この段階を学び終えたら、CSV ファイルを1つ渡されても、読み込み、クリーニング、探索、可視化、結論の整理までを自力でできるようになっているはずです。

少なくとも3枚の重要な図表、1つのデータクリーニングの流れ、3つの明確な結論を含むデータ分析レポートを書けるなら、AI 数学と機械学習の段階に進んで大丈夫です。
