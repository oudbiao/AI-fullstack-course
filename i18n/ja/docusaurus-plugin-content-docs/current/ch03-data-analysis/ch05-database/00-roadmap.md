## まずは全体地図を作ろう

このデータベースの章を学ぶとき、新人にいちばん合っている順番は「SQL の文法を先に丸暗記する」ことではありません。まずは次のことをはっきりさせるのが大切です。

![データベース選択学習ロードマップ](/img/course/ch03-database-roadmap-ja.png)

この章が本当に解決したいのは、次のようなことです。

- データはなぜいつも CSV の中に静かに置いておけないのか
- データが長期保存され、複数人で使われ、検索や権限管理が必要になるとき、なぜシステムには自然にデータベースが必要になるのか

## この章でよく出てくる用語

| 用語 | 英語の正式名称 | まず覚えること |
|---|---|---|
| `SQL` | Structured Query Language | データベースに質問するための言語 |
| `SQLite` | SQLite database engine | 初学者に向いた軽量データベース。通常は 1 つのローカルファイルに保存される |
| `DBA` | Database Administrator | データベース管理者。運用、セキュリティ、バックアップ、性能を担当する専門職 |
| `RAG` | Retrieval-Augmented Generation | 検索拡張生成。AI が回答前に外部資料を検索するアプリケーションパターン |
| `AI Agent` | Artificial Intelligence Agent | 手順を計画し、ツールを使い、タスク状態や記憶を持てる AI システム |

この章で `DBA` になる必要はありません。目標はもっと小さく、実用的です。データベースが現実のデータ分析や AI プロジェクトのどこに出てくるのかを理解し、神秘的なブラックボックスとして怖がらないようにすることです。

## なぜ 3 データ分析と可視化のあとにデータベースを補うのか？

新人にとって、より安定しやすい順番はたいてい次のようになります。

1. まず 1 枚の表を扱えるようになる
2. 次に複数の表を扱えるようになる
3. 最後に、それらの表がなぜデータベースに長期保存されるのかを理解する

つまり、データベースの章は `Pandas` の代わりではありません。  
むしろ、次のことを教えてくれます。

- 現実の表データは、よくデータベースから来る

## 新人向けの、もっとわかりやすい全体のたとえ

これまで学んだ内容は、次のように考えるとわかりやすいです。

- データを自分の机の上に持ってきて分析する

一方で、データベースはもっとこういうイメージです。

- 会社で共通して使うデータ保管室

机の上の表はもちろん便利です。  
でも、データが次のようになると、散らばったファイルだけでは足りません。

- 量が増える
- 複数人で使う
- 長く保存する必要がある
- すばやく検索したい

## はじめて学ぶときの、いちばん安定した順番

おすすめの順番は次のとおりです。

1. まず [リレーショナルデータベース基礎](./01-relational-db.md) を学ぶ  
   先にデータベースと表の概念を固めます。
2. 次に [SQL 基礎](./02-sql-basics.md) を学ぶ  
   まずは検索、絞り込み、グループ化ができるようにします。
3. その次に [Python データベース操作](./03-python-db.md) を学ぶ  
   コードとデータベースをつなぎます。
4. 最後に [データベース設計](./04-db-design.md) を見る  
   なぜ表をそのように整理するのかを理解します。

## はじめてデータベースを学ぶとき、最初に覚えるべきことは？

最初に覚えるべきことは、実はこの 4 つだけでも十分です。

1. データベースは、データを長期保存する場所
2. リレーショナルデータベースの中心は、「表と表を鍵で関連づけられること」
3. SQL はデータベースと会話するための言語
4. `Pandas` と SQL は競争相手ではなく、よく一緒に使う

## 新人がよくつまずくポイント

- いきなり SQL の全文法を暗記しようとして、表の構造を理解しない
- データベースを「保存するためのもの」だと思い、業務の問題に沿って検索しない
- データベースを学ぶと `Pandas` と関係なくなると思ってしまう
- `JOIN` を見てこわがるが、実は本質は `merge` にかなり近い

## この章を学んだあと、少なくともできるようになりたいこと

- データベースと CSV が、それぞれどんな場面に向いているかを知る
- シンプルな表の主キー・外部キーの関係を読める
- 基本の `SELECT / WHERE / JOIN / GROUP BY` が書ける
- Python で SQLite に接続して、簡単な読み書きができる

## この章でいちばん持ち帰るべきこと

- データベースは「まったく別の話題」ではなく、現実のデータシステムが自然に発展したもの
- まず表、鍵、関係を理解してから SQL を学ぶと、かなり安定する
- この章の目的は DBA になることではなく、今後データベースを見ても不安にならないこと
