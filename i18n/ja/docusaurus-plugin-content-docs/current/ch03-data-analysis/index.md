# 3 データ分析と可視化

![データ分析と可視化のメインビジュアル](/img/course/ch03-data-visualization-ja.png)

この段階で解決するのは、「データを理解できるか、整理できるか、規則性を見つけて結論として伝えられるか」です。今後、機械学習、RAG、Agent、またはプロダクトのデータ分析を行う場合でも、データの力は土台になります。

## ストーリーで導入：データ探偵のように事件を解く

この段階に入ったら、自分をデータ探偵だと思ってみましょう。CSV、Excel、ログ、データベースの表は、たくさんの手がかりのようなものです。手がかりの中には欠けているものもあれば、重複しているものもあり、見た目は普通でも実は異常値というものもあります。あなたの役割は、すぐにモデルを当てはめることではありません。まず手がかりをきれいに整理し、次に統計やグラフで規則性を見つけ、最後に信頼できる結論を伝えることです。

## 体験型演習：どのデータセットも3回質問する

1回目の質問は「これは何か」を見ることです。何行あるか、何列あるか、各列は何を表すか。2回目の質問は「きれいか」を見ることです。欠損値はどこにあるか、異常値はどこにあるか、型は正しいか。3回目の質問は「何を示しているか」を見ることです。どの変数が最も高いか、どの傾向が最もはっきりしているか、どのグループ差がさらに調べる価値があるか。

## プロジェクトの特典

この段階の特典作品は、実際に見せられるデータ分析レポートです。Notebook にしてもよいですし、図と文章のレポートにまとめてもよいです。最初に問題を提示し、中間でクレンジングと分析の過程を示し、最後に結論と次のモデリング提案を出します。後で機械学習を学ぶとき、このレポートはそのまま「データ分析から予測モデルへ」の完全なプロジェクトに発展させられます。

## 初心者の最小クリアルート

初心者はまず、データの読み込み、フィールドの確認、欠損値の処理、基本的な絞り込みとグループ集計を学び、次に Matplotlib または Seaborn で問題を説明できるグラフを描きます。データの読み込みから結論のまとめまでを含む Notebook レポートを1つ完成できれば、最小クリア達成です。

## 応用・発展ルート

経験のある学習者は、複数ソースのデータ統合、時系列、データベース読み込み、可視化ストーリーテリング、分析レポートの構成まで深く学べます。さらに、データクレンジングの過程を再利用可能な関数としてまとめ、後続の機械学習プロジェクト向けにきれいなデータセットを出力してみましょう。

## 初学者は何を先にやるか、上級者は何をするか

この段階を初めて学ぶ人は、まずデータを「整理が必要な表」として扱います。データを読み込めること、列の意味を理解できること、欠損値を処理できること、重要なグラフを描けること、3つの結論を書けること。ここまでできれば、主な流れはつかめています。

経験のある学習者は、分析の信頼性に重点を置きます。データソースは信頼できるか、クレンジングルールは説明できるか、グラフは誤解を招かないか、結論に限界はないか。目標は、たくさんのグラフを描くことではなく、後続のモデリングや業務判断を支えられるデータ分析を作ることです。

## なぜ AI にデータが欠かせないのか

モデルは空気から学ぶわけではありません。データの品質、構造、分布、ラベル付けの方法は、モデルの結果に直接影響します。たとえ将来、主に大規模言語モデルのアプリケーションを扱うことになっても、文書クレンジング、ログ分析、評価データセットの作成、ユーザーフィードバックの集計、検索品質の分析に必ず出会います。

3.5 節のデータベースは選択科目ですが、実際のアプリケーションや企業データのプロジェクトをやりたいなら、関係データベースと SQL の基礎は少なくとも理解しておくのがおすすめです。

## よくある誤解

データ分析を「API を暗記すること」だと思わないでください。Pandas のメソッドはたくさんありますが、1回目で全部覚える必要はありません。むしろ、データ処理の流れを理解することが大切です。まずデータの見た目を確認し、次に欠損と異常を調べ、続いて変換と集計を行い、最後にグラフで判断を確かめます。

また、きれいなグラフを描くだけで終わらせないでください。グラフの目的は飾ることではなく、質問に答えることです。どのグラフも、「どの特徴量が目的変数と最も関係があるか」「明らかな異常はあるか」「異なるカテゴリの差は何か」といった明確な問いに対応しているべきです。

## データ失敗劇場：なぜグラフや結論は人をだますのか

グラフがきれいなのに結論がおかしいときは、まずデータに欠損、重複、異常値、単位の混乱がないか確認します。グループ結果が直感に合わないときは、分類フィールドに空白、大小文字の違い、別名がないか確認します。相関がとても高いときは、時間、規模、サンプリング方法による共通の影響ではないかを先に疑いましょう。

## 最小実行実験：汚れたデータから信頼できる1枚のグラフへ

この段階の最小実験は、欠損値、重複行、異常値を含む小さな CSV を用意し、読み込み、確認、クレンジング、集計、可視化まで行うことです。あなたが示すべきなのは、Pandas API を呼べることではなく、各ステップをなぜそのように処理するのかを説明できることです。

```python
import pandas as pd

df = pd.read_csv("learning_log.csv")
print(df.info())
print(df.isna().sum())

clean_df = df.drop_duplicates()
summary = clean_df.groupby("topic")["minutes"].sum().sort_values(ascending=False)
print(summary)
```

本当の合格基準はコードが動くことではありません。どのデータをどう変えたのか、なぜそう変えたのか、それが結論にどんな影響を与えるのかを説明できることです。

## データ失敗事例集：まずフィールド、単位、サンプルの出所を確認する

| 現象 | よくある原因 | 切り分け方法 | 修復の方向 |
|---|---|---|---|
| グラフはきれいだが結論が信頼できない | 欠損、重複、異常値が未処理 | `info()`、欠損統計、分布を確認する | クレンジングルールと影響範囲を記録する |
| グループ集計結果が異常 | カテゴリフィールドに空白、大小文字、別名がある | ユニーク値と頻度を表示する | フィールド名とカテゴリ対応を統一する |
| 相関が高く見える | 時間、規模、サンプリング方法による共通影響 | 層別比較、散布図を描く | 業務上の説明と限界を追加する |
| Notebook を再現できない | データパス、依存関係、実行順が混乱している | 最初から再起動して実行する | データパス、依存関係、実行順を固定する |


## 段階プロジェクト

基本版は、公開されている CSV データセットを1つ選び、フィールドの理解、欠損値の確認、基本統計、3～5 枚の重要なグラフを完成させます。標準版では、分析過程を構成の整った Notebook またはレポートにまとめ、問題、クレンジング、分析、結論、次の提案を含めます。チャレンジ版では、複数のデータソースを統合し、データベースや Web データも取り入れて、より実際の業務に近いデータ分析フローを作ります。
