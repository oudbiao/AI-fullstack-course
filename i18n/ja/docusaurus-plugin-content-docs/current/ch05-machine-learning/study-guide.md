# 学習ガイド：機械学習をどう学ぶと一番混乱しにくいか

![機械学習の学習サイクル図](/img/course/ml-study-loop.png)

`第 5 章 機械学習入門から実践まで` に来て、モデル名がたくさんあって戸惑ったら、まずはアルゴリズムを1つずつ暗記しようとしなくて大丈夫です。機械学習で最初にいちばん大事なのは、全体のモデリング手順を理解することです。

## この段階の基本原則

機械学習では、1本のプロジェクトの流れをしっかりつかむことが大切です。つまり、問題をタスクに言い換え、データを準備し、baseline を作り、指標で評価し、その後に特徴量やモデルを改善していきます。

![機械学習学習ガイドのプロジェクトサイクル](/img/course/ch05-study-guide-project-loop.png)

## おすすめの学習順序

1周目は、まず機械学習の歴史的なブレークスルー、基本概念、Scikit-learn の基本的な使い方を学びます。Bayes、線形モデル、決定木、SVM、ランダムフォレスト、Boosting、XGBoost がなぜ順番に登場したのかを先に知り、そのうえで訓練データ、テストデータ、特徴量、ラベル、モデル、指標、pipeline が何かを理解しましょう。

2周目は教師あり学習を学びます。線形回帰、ロジスティック回帰、決定木、アンサンブル学習で、多くの入門プロジェクトに十分対応できます。

3周目は教師なし学習を学びます。クラスタリング、次元削減、異常検知を通して、ラベルがないときにどうやって構造を見つけるかを理解します。

4周目はモデル評価と選択を学びます。指標、交差検証、バイアス・バリアンス、ハイパーパラメータ調整によって、モデルが本当に良いかどうかを判断できるようになります。

5周目は特徴量エンジニアリングとプロジェクトに進みます。多くの表形式データのプロジェクトでは、モデルを変えることよりも、特徴量の処理のほうが重要です。

## おすすめの学習ペース

| 内容タイプ | おすすめ時間 | 学習目標 |
|---|---|---|
| 基礎概念ページ | 2～3 時間 | タスク、データ、指標を説明できる |
| アルゴリズムページ | 2～4 時間 | アルゴリズムの適用場面と直感がわかる |
| 評価ページ | 2～4 時間 | モデルの結果が信頼できるか判断できる |
| プロジェクトページ | 8～16 時間 | モデリングの一連の流れを通して実行できる |

## 段階別プロジェクトの進め方

最初のプロジェクトは住宅価格予測がおすすめです。回帰、特徴量処理、誤差分析を練習できます。

2つ目のプロジェクトは顧客離脱予測がおすすめです。分類、混同行列、AUC、再現率、ビジネス上の説明を練習できます。

3つ目のプロジェクトはユーザーのセグメンテーションがおすすめです。教師なし学習、クラスタリング結果の解釈、可視化を練習できます。

時間が限られているなら、少なくとも1つはプロジェクトを最初から最後までやりきってください。アルゴリズムページだけを学ぶのは避けましょう。

## よくあるつまずきポイント

いちばんよくあるつまずきは「どのモデルを使えばいいかわからない」ことです。解決方法は、まず baseline を作ることです。多くの場合、最初にシンプルなモデルでデータと指標に問題がないか確認するほうが、いきなり複雑なモデルを試すより大切です。

2つ目のつまずきは、指標がごちゃごちゃになることです。分類、回帰、クラスタリングでは指標が違うので、まずタスクが何かを確認してから指標を選びましょう。

3つ目のつまずきは、データリークです。標準化、エンコーディング、特徴量選択では、訓練データとテストデータの境界に注意が必要です。

## 合格基準

この段階を学び終えたら、1つの機械学習プロジェクトを自力で完了できるようになっているはずです。つまり、タスク定義、データ処理、baseline の学習、指標の選択、モデル改善、結果の説明まで行えます。

表形式データのプロジェクトをレポートとして整理し、モデルの良い点・悪い点・次にどう改善するかを説明できれば、深層学習の段階に進んで大丈夫です。
