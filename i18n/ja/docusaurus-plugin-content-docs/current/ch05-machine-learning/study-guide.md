# 5.0 学習ガイドとタスクシート：機械学習をどう学ぶと一番混乱しにくいか

![機械学習の学習サイクル図](/img/course/ml-study-loop-ja.png)

`第 5 章 機械学習入門から実践まで` に来て、モデル名がたくさんあって戸惑ったら、まずはアルゴリズムを1つずつ暗記しようとしなくて大丈夫です。機械学習で最初にいちばん大事なのは、全体のモデリング手順を理解することです。

## この段階の基本原則

機械学習では、1本のプロジェクトの流れをしっかりつかむことが大切です。つまり、問題をタスクに言い換え、データを準備し、baseline を作り、指標で評価し、その後に特徴量やモデルを改善していきます。

![機械学習学習ガイドのプロジェクトサイクル](/img/course/ch05-study-guide-project-loop-ja.png)

## この段階で必ず完了するタスク

![機械学習クリア条件チェックリスト図](/img/course/ml-task-checklist-ja.png)

このチェックリストで、学習を実作業に落とし込みます。機械学習プロジェクトは、スコアを出すだけでは終わりません。再実行でき、失敗を説明でき、次の改善を言える状態まで進めます。

| タスク | 成果物 | 合格基準 |
|---|---|---|
| ML の問題意識を身につける | 問題定義の説明書 1 つ | 分類、回帰、クラスタリング、異常検知を区別できる |
| sklearn の baseline を動かす | 最小限の学習スクリプト 1 つ | train/test split、fit、predict、score を実行できる |
| ガイド付き証拠パックワークショップを完了する | 生成された `ml_workshop_run/` 証拠パック | スクリプトを再実行し、`model_comparison.csv`、`threshold_review.csv`、`error_samples.csv`、`leakage_check.md` を説明できる |
| 特徴量エンジニアリングを完成させる | 特徴量処理の記録 1 つ | 欠損値、カテゴリ変数、標準化、リークリスクを説明できる |
| モデル評価を完成させる | 指標比較表 1 つ | accuracy、recall、F1、AUC、RMSE の適用場面を説明できる |
| 段階プロジェクトを 1 つ完成させる | 再現可能な実験プロジェクト 1 つ | README、データ説明、指標、失敗サンプル、改善計画がある |

## おすすめの学習順序

1周目は、まず機械学習の歴史的なブレークスルー、基本概念、Scikit-learn の基本的な使い方を学びます。Bayes、線形モデル、決定木、SVM、ランダムフォレスト、Boosting、XGBoost がなぜ順番に登場したのかを先に知り、そのうえで訓練データ、テストデータ、特徴量、ラベル、モデル、指標、pipeline が何かを理解しましょう。

2周目は教師あり学習を学びます。線形回帰、ロジスティック回帰、決定木、アンサンブル学習で、多くの入門プロジェクトに十分対応できます。

3周目は教師なし学習を学びます。クラスタリング、次元削減、異常検知を通して、ラベルがないときにどうやって構造を見つけるかを理解します。

4周目はモデル評価と選択を学びます。指標、交差検証、バイアス・バリアンス、ハイパーパラメータ調整によって、モデルが本当に良いかどうかを判断できるようになります。

5周目は特徴量エンジニアリングとプロジェクトに進みます。多くの表形式データのプロジェクトでは、モデルを変えることよりも、特徴量の処理のほうが重要です。

大きなプロジェクトに入る前に具体的な練習をしたい場合は、[5.6.6 実践ワークショップ：再現可能な ML 証拠パックを作る](./ch06-projects/05-hands-on-ml-workshop.md) を完了してください。抽象的なモデリングの流れを、1つの実行可能なスクリプトと証拠フォルダに変えます。

## 初学者向けモデリング用語の翻訳表

機械学習の用語が単語暗記のように感じたら、まずプロジェクト上の行動に翻訳してみましょう。

| 用語 | 最初の直感 | コードやレポートで何をするか |
|---|---|---|
| 特徴量 | モデルが見る入力列 | 数値、カテゴリ、時間、テキストのどれかを判断する |
| ラベル / 目的変数 | モデルが予測したい答え | 特徴量と分け、リークを避ける |
| Baseline | まず超えるべき最も単純なモデルやルール | 複雑なモデルより先に学習する |
| 指標 | モデルを評価するものさし | タスクに応じて accuracy、F1、AUC、MAE、RMSE、シルエット係数を選ぶ |
| 訓練 / 検証 / テスト | 学習、選択、最終確認のための分割 | テストデータの情報を前処理に漏らさない |
| 過学習 | 訓練データを覚えすぎること | 訓練スコアとテストスコアを比べ、単純化や正則化を行う |
| 未学習 / 欠拟合 | モデルが弱すぎて規則を学べないこと | 有用な特徴量を足すか、より表現力のあるモデルを試す |
| 交差検証 | 分割を変えて何度も評価すること | 1回の分割だけでは不安定なときに使う |
| Pipeline | 前処理とモデルをまとめた流れ | リークを防ぎ、実験を再現しやすくする |
| ハイパーパラメータ | 学習前に人が決める設定 | テストセットではなく、検証や交差検証で調整する |

これらをプロジェクト上の行動として説明できれば、単に API を覚える段階から、モデリング調査を進める段階へ進めています。

## おすすめの学習ペース

| 内容タイプ | おすすめ時間 | 学習目標 |
|---|---|---|
| 基礎概念ページ | 2～3 時間 | タスク、データ、指標を説明できる |
| アルゴリズムページ | 2～4 時間 | アルゴリズムの適用場面と直感がわかる |
| 評価ページ | 2～4 時間 | モデルの結果が信頼できるか判断できる |
| プロジェクトページ | 8～16 時間 | モデリングの一連の流れを通して実行できる |

## 段階別プロジェクトの進め方

大きなテーマを選ぶ前に、まず [5.6.6 実践証拠パックワークショップ](./ch06-projects/05-hands-on-ml-workshop.md) を一度実行しましょう。ウォームアップとして、データ生成、分割、baseline 学習、モデル比較、エラー確認、プロジェクト証拠の作成を同じ流れで練習できます。

最初のプロジェクトは住宅価格予測がおすすめです。回帰、特徴量処理、誤差分析を練習できます。

2つ目のプロジェクトは顧客離脱予測がおすすめです。分類、混同行列、AUC、再現率、ビジネス上の説明を練習できます。

3つ目のプロジェクトはユーザーのセグメンテーションがおすすめです。教師なし学習、クラスタリング結果の解釈、可視化を練習できます。

時間が限られているなら、少なくとも1つはプロジェクトを最初から最後までやりきってください。アルゴリズムページだけを学ぶのは避けましょう。

## よくあるつまずきポイント

いちばんよくあるつまずきは「どのモデルを使えばいいかわからない」ことです。解決方法は、まず baseline を作ることです。多くの場合、最初にシンプルなモデルでデータと指標に問題がないか確認するほうが、いきなり複雑なモデルを試すより大切です。

2つ目のつまずきは、指標がごちゃごちゃになることです。分類、回帰、クラスタリングでは指標が違うので、まずタスクが何かを確認してから指標を選びましょう。

3つ目のつまずきは、データリークです。標準化、エンコーディング、特徴量選択では、訓練データとテストデータの境界に注意が必要です。

| 成果物 | 説明 |

## 段階通過質問

この段階を学び終えたら、1つの機械学習プロジェクトを自力で完了できるようになっているはずです。つまり、タスク定義、データ処理、baseline の学習、指標の選択、モデル改善、結果の説明まで行えます。

第 6 章へ進む前に、次の質問に答えられるか確認してください。

- なぜ訓練データとテストデータを分けるのか。
- データリークとは何か、`Pipeline` はどうリスクを下げるのか。
- accuracy が信頼できないのはどんなときか。
- なぜシンプルな baseline が機械学習プロジェクトの出発点なのか。
- モデルは何が得意で、何が苦手で、次に何を改善するべきか。

表形式データのプロジェクトをレポートとして整理し、これらに答えられれば、深層学習の段階に進んで大丈夫です。
