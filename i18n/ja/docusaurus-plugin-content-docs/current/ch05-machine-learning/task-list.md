# 段階学習タスクシート：機械学習の入門から実践まで

![機械学習クリア条件チェックリスト図](/img/course/ml-task-checklist.png)

この段階の目標は、機械学習プロジェクトの全体の流れを理解することです。つまり、問題を定義する、データを準備する、特徴量を作る、モデルを学習する、効果を評価する、間違いを分析する、という流れです。アルゴリズムの数式を暗記することに重点を置くのではなく、各アルゴリズムを「どんな問題に向いているのか」「どうやって良し悪しを判断するのか」という場面に戻して考えましょう。

## この段階で必ず完成させるタスク

| タスク | 成果物 | 合格基準 |
| --- | --- | --- |
| ML の問題意識を身につける | 問題定義の説明書 1 つ | 分類、回帰、クラスタリング、異常検知を区別できる |
| sklearn の baseline を動かす | 最小限の学習スクリプト 1 つ | train/test split、fit、predict、score を実行できる |
| 特徴量エンジニアリングを完成させる | 特徴量処理の記録 1 つ | 欠損値、カテゴリ変数、標準化、リークリスクを説明できる |
| モデル評価を完成させる | 指標比較表 1 つ | accuracy、recall、F1、AUC、RMSE の適用場面を説明できる |
| 段階プロジェクトを完成させる | 再現可能な実験プロジェクト 1 つ | README、データ説明、指標、失敗サンプル、改善計画がある |

## おすすめの学習順序

まず、機械学習の基本概念と sklearn のワークフローを学び、そのあとに教師あり学習、教師なし学習、評価方法、特徴量エンジニアリングを学びます。特徴量エンジニアリングは最後まで後回しにしないでください。実際のプロジェクトでは、モデルの効果はまずデータと特徴量で決まることが多いからです。

アルゴリズムを 1 つ学ぶたびに、必ず次の 3 つを同時に考えましょう。入力は何か、出力は何か、どんな場合に失敗しやすいのか。この考え方のほうが、アルゴリズム名だけを覚えるよりずっと役立ちます。

## AI 学習アシスタントプロジェクトとの関係

この段階は、AI 学習アシスタントの v0.4 学習提案 baseline に対応します。過去の学習質問と段階ラベルを使って、簡単な分類器を学習させ、新しい質問がどの段階、またはどのトピックに属するかを予測できます。このモデルはそれほど強力でなくてもかまいませんが、「ルールからモデルへ」の変化を理解する助けになります。

おすすめは、まず簡単な版を実装することです。手作業で数十件の学習質問サンプルを用意し、TF-IDF か簡単なテキスト特徴量を使って分類モデルを学習し、推奨章を出力します。そのあと、ルール一致方式と比較して、それぞれの長所と短所を記録しましょう。

## よくあるつまずきポイント

よくある問題には、学習データとテストデータを混ぜてしまうこと、テストデータでパラメータ調整をしてしまうこと、accuracy だけを見ること、クラス不均衡を無視すること、分割前に特徴量処理を終えてしまいデータリークが起きること、モデル結果を再現できないことなどがあります。機械学習プロジェクトでは、モデルが高度かどうかよりも、評価プロセスが正しいかどうかのほうが重要なことが多いです。


## やさしい版 / 標準版 / チャレンジ版タスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| やさしい版 | Dummy baseline を学習して指標を出力する | 初めて学ぶ人、時間が少ない人、入門者 |
| 標準版 | 本物のモデルを学習し、baseline と比較する | この段階を作品集に入れたい人 |
| チャレンジ版 | 1 回、データリーク、クラス不均衡、または誤ったサンプルを確認する | 基礎があり、より強いプロジェクト証拠を作りたい人 |

## この段階のバッジと Boss 戦

| 種類 | 内容 |
|---|---|
| Boss 戦 | Baseline 守門員 |
| 解放できるバッジ | Baseline 守門員、リーク探偵 |
| 最低クリアの合言葉 | まず動かす、次に説明する、最後に失敗を記録する |
| 証拠の保存おすすめ | スクリーンショット、ログ、失敗サンプル、評価表を `reports/`、`evals/`、または `logs/` に保存する |

やさしい版を終えれば先へ進めます。標準版を終えたら、作品集に書くのがおすすめです。チャレンジ版は余裕があるときに取り組めば十分です。

## 段階作品集の提出物

この段階の成果を作品集に残したいなら、少なくとも次のファイル、または同等の資料を保存することをおすすめします。

| 提出物 | 説明 |
| --- | --- |
| `train.py` または Notebook | データ分割、学習、評価を含む、再現可能な baseline 学習フロー |
| `feature_report.md` | 特徴量の意味、欠損値処理、エンコード、標準化、リークリスクの確認 |
| `metrics.md` | accuracy、F1、AUC、RMSE などの指標と、その選択理由を記録 |
| `error_analysis.md` | 予測ミスのサンプルを保存し、データ、特徴量、モデル、指標の問題を分析 |
| `README.md` | 問題定義、実行コマンド、モデル結果、制限、次の計画 |

これらの資料があると、機械学習プロジェクトは単に「スコアを出す」だけでなく、再現可能で、説明可能で、改善できる建模の流れになります。

## 段階クリア問題

学習後、次の質問に答えられるようにしてください。なぜ学習データとテストデータを分けるのか、データリークとは何か、いつ accuracy が信頼できないのか、Pipeline は何を解決するのか、なぜシンプルな baseline がすべての ML プロジェクトの出発点なのか。
