# 学習前ガイド：この無监督学習の章で何を学ぶのか

![無监督学習のロードマップ](/img/course/unsupervised-learning-roadmap-ja.png)

無监督学習と监督学習の最大の違いは、**ラベルがない**ことです。

つまり、モデルに直接「正解した？」と聞くことはできません。まずは次のことを考える必要があります。

- データの中に自然なグループ分けがあるか
- データをもっと少ない次元に圧縮できるか
- データの中に目立つ異常点があるか

## まず、とても大事な学習の心構え

この章で初心者がいちばん戸惑いやすいのは、アルゴリズムそのものよりも、次の点です。

- ラベルがない
- 正解がない
- 「なんとなく何でも説明できそう」に見えるのに、何が妥当なのか分からない

最初に持っておくとよい理解は、次のとおりです。

> **無监督学習は、直接正解・不正解を判定するのではなく、データの中にありそうな構造を見つける手助けをするものです。**

そのため、この章は前の监督学習のような「どう判定するかをそのまま学ぶ」章というより、「探索して仮説を立てる」章に近いです。

## この章の3つの節はどうつながっているか

![無监督学習の章の関係図](/img/course/ch05-unsupervised-chapter-flow-ja.png)

- クラスタリング：ラベルがないときに、まずデータを自動でグループ分けできるかを見る
- 次元削減：次に、高次元データをもっと見やすく、計算しやすく圧縮できるかを見る
- 異常検知：最後に、少数の「普通ではない」点をどう見つけるかを見る

## 初めて無监督学習を学ぶなら、いちばん安定した順番

初心者には、次の順番がおすすめです。

1. まず [3.2 クラスタリングアルゴリズム](./clustering) を読む  
   「ラベルがなくても、データには構造があるかもしれない」という感覚をつかみます。

2. 次に [3.3 次元削減アルゴリズム](./dimensionality-reduction) を読む  
   「モデリングのための前処理」と「可視化のための探索」を分けて理解します。

3. 最後に [3.4 異常検知](./anomaly-detection) を読む  
   ここで初めて、「すべてのタスクがグループ分けではない。大多数に属さない点を探すタスクもある」と受け入れやすくなります。

この順番で学ぶメリットは、次のとおりです。

- まず、いちばん分かりやすい「グループ分け」から入れる
- 次に、「表現を圧縮する」考え方を理解できる
- 最後に、しきい値や業務判断に強く依存する「少数の異常を探す」タスクに進める

## この章で混乱しやすいポイント

- 無监督学習の結果を「唯一の真実」だと思ってしまう
- 図が見やすいかどうかばかり気にして、業務上意味があるかを考えない
- クラスタリング、次元削減、異常検知を全部学んだのに、それぞれが何を解決するのか分からない

なので、この章でまず持ち帰るべきなのは、たくさんのモデル名ではなく、次の3つの問いです。

1. 今やっているのは、グループを探しているのか、表現を圧縮しているのか、異常を探しているのか？
2. 手元の結果は、業務で説明できるか？
3. ラベルがないとき、何を根拠にこの結果に価値があると判断するのか？

## 初心者がこの章で持ち帰るべきこと

- ラベルがないときに、問題をどう言い換えるかが分かる
- K-Means、PCA、異常検知がそれぞれ何を解決するのかが分かる
- 無监督学習の結果は、数値のスコアだけでなく、説明や業務理解がより重要だと分かる

## 初心者と上級者でどう読むか

初心者がこの章を初めて学ぶときは、まず主線と最小実行例をつかんでください。細部を一度に全部理解する必要はありません。この章が何を解決するのか、入力と出力は何か、最小プロジェクトをどう動かすのかを説明できれば、先へ進んで大丈夫です。

経験のある学習者は、この章を「抜け漏れの確認」と「実務化の練習」として使えます。境界条件、失敗例、評価方法、コードの再現性、そして前後の学習内容とのつながりに注目しましょう。読み終わったら、この章の内容を自分の作品の README や実験記録にまとめておくとよいです。

## 学習時間と難易度の目安

| 学習方法 | 目安の時間 | 目標 |
|---|---|---|
| ざっと読む | 20～30 分 | この章が何を解決するかを理解し、後でどこで使うかを知る |
| 最小到達 | 1～2 時間 | 最小例を動かし、本章のミニ課題を完了する |
| 深く練習する | 半日～1 日 | エラー分析、比較実験、または README 記録を追加する |

## この章の自己チェック問題

| 自己チェック問題 | 合格基準 |
|---|---|
| この章は何を解決する？ | このコース全体の中での位置を一文で説明できる |
| 最小の入力と出力は何？ | 例に何が必要で、どんな結果が出るかを説明できる |
| よくある失敗点はどこ？ | エラー、精度の悪さ、理解のずれの原因を少なくとも1つ挙げられる |
| 学習後に何を残せる？ | 本章の成果を README、実験記録、または作品集に書ける |
## この章のミニプロジェクトの出口

この章を学び終えたら、最小の練習として、本章でいちばん重要な概念またはツールを1つ選び、実行できて、スクリーンショットを残せて、README に書ける小さな成果を作るのがおすすめです。複雑である必要はありませんが、入力、処理、出力が何かを説明できることが大切です。

## 合格ライン

この章の終わりには、本章が何を解決するのか、前後の学習ステップとどうつながるのかを自分の言葉で説明でき、さらに本章のミニプロジェクトの最小版を完成できるようになっているはずです。

もし、よくあるミスを1回、デバッグの過程を1回、または結果改善を1回記録できたなら、もう「見ただけ」ではなく、この章を自分のプロジェクト経験に変えられていると言えます。
