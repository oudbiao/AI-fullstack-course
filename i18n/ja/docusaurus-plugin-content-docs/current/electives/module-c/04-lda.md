# 線形判別分析

:::tip この節の位置づけ
LDA は、ほかの略語と混同しやすく、  
「また別の線形分類器なのでは？」と誤解されやすいです。

より正確には、次のように理解します。

> **LDA が関心を持つのは、どの投影方向を見つければ、同じクラスのサンプルはよりまとまり、異なるクラスはより分かれやすくなるか、ということです。**

そのため、分類にも使えますし、教師あり次元削減法として見ることもできます。
:::

![LDA 教師あり投影の直感図](/img/course/elective-lda-projection-map-ja.png)

## 学習目標

- LDA の中心的な目標である「クラス内のまとまり」と「クラス間の分離」を理解する
- 普通の線形分類器との違いを理解する
- 実行できる例を通して、LDA の次元削減と分類の効果を理解する
- どんなときに LDA を試すべきか、基本的な判断ができるようにする

---

## 一、LDA は何を解決するのか？

### 1.1 単に「クラスを分ける」だけではない

LDA の目標は、もっと具体的です。

- 同じクラスの中のサンプルは、できるだけ近くに集める
- 異なるクラス同士は、できるだけ引き離す

### 1.2 なぜこれは普通の線形分割よりおもしろいのか？

なぜなら、境界線を 1 本引くだけではなく、  
「より判別しやすい表現空間」を探しているからです。

つまり、分類だけでなく、次のことにも使えます。

- 教師あり次元削減

### 1.3 たとえ話

PCA がどちらかというと、

- 全体の変化を最もよく説明する方向を見つける

のに対して、LDA は、

- クラスを見分けるのに最も役立つ方向を見つける

というイメージです。

---

## 二、LDA がよく「ラベル付きの次元削減」と見なされるのはなぜか？

### 2.1 クラスラベルを使うから

PCA はクラスを気にせず、全体の分散だけを見ます。  
LDA はラベルを明確に使って、次のように考えます。

- どの方向が分類に最も役立つか？

### 2.2 では、どんな場面に向いているのか？

次のような場面に向いています。

- すでに教師ラベルがある
- より判別力の高い低次元表現を作りたい
- あるいは軽量な線形分類器として使いたい

---

## 三、まずは最小限の実行例を動かしてみよう

この例では、2 つのことを同時に行います。

1. LDA で分類する
2. データをより低次元の空間に投影する

```python
import numpy as np
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis

X = np.array([
    [1.0, 2.0],
    [1.5, 1.8],
    [2.0, 1.5],
    [4.0, 5.0],
    [4.5, 4.8],
    [5.0, 4.5],
])
y = np.array([0, 0, 0, 1, 1, 1])

lda = LinearDiscriminantAnalysis(n_components=1)
lda.fit(X, y)

pred = lda.predict([[1.4, 1.9], [4.8, 4.6]])
projection = lda.transform(X)

print("predictions:", pred.tolist())
print("projection shape:", projection.shape)
print("projected values:", projection.ravel().round(3).tolist())
```

### 3.1 なぜこのコードは、単なる `predict` よりも価値があるのか？

それは、次の 2 つを同時に見られるからです。

- 分類結果
- 投影後の低次元表現

これによって、LDA の 2 つの役割がよく分かります。

- 分類できる
- 教師あり次元削減もできる

### 3.2 なぜ `n_components=1` なのか？

今回はクラスが 2 つだけだからです。  
この場合、LDA が投影できる判別方向は最大で次の 1 つです。

- 1 個の判別方向

これも、クラス数と関係する LDA の特徴の 1 つです。

---

## 四、LDA は SVM / Logistic Regression と何が違うのか？

### 4.1 SVM との違い

SVM がより強く重視するのは、

- マージンの最大化

です。

一方、LDA が重視するのは、

- クラス内分散が小さいこと
- クラス間の平均の差が大きいこと

です。

### 4.2 Logistic Regression との違い

Logistic Regression は、どちらかというと次のようなものを学習します。

- 条件付き確率の境界

LDA は、まずデータ分布を仮定してから、より判別しやすい方向を探すイメージです。

### 4.3 なぜこれは学ぶ価値があるのか？

それは、次のことが分かるからです。

- 典型的なモデルでも、「線形分類」の考え方は 1 つではない

---

## 五、LDA はどんなときに試すとよいか？

### 5.1 データ量は大きくないが、クラス構造が比較的はっきりしているとき

このような場面では、LDA が役立つことがあります。

### 5.2 より解釈しやすい低次元表現が欲しいとき

たとえば、

- まず投影してから可視化する
- まず投影してから簡単な分類器につなぐ

といった使い方です。

### 5.3 あまり向いていない場面

クラス境界が非常に複雑で、明らかに非線形な場合は、  
LDA では苦戦しやすいです。

---

## 六、よくある誤解

### 6.1 誤解 1：LDA は単なる別の分類器である

それだけではありません。  
「判別的な表現を作る」という価値もとても重要です。

### 6.2 誤解 2：ラベルがあるなら、必ず PCA より良い

必ずしもそうではありません。  
タスクの目的やデータ分布によります。

### 6.3 誤解 3：LDA とトピックモデルの LDA は同じである

違います。  
ここでの LDA は次の意味です。

- Linear Discriminant Analysis

トピックモデルの LDA は次の意味です。

- Latent Dirichlet Allocation

---

## まとめ

この節で最も大事なのは、次の判断を持つことです。

> **LDA の核心は、ラベルを利用してより判別力の高い投影方向を見つけることにある。そのため、軽量な分類にも、教師あり次元削減にも使える。**

この理解ができると、LDA はもう混同しやすい略語ではなくなります。
