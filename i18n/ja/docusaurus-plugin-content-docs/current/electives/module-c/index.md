# 選修モジュール：古典的 ML 補足アルゴリズム

:::tip モジュールの位置づけ
これらのアルゴリズムは、多くの中小規模データのタスクで今でもとても役立ちます。学ぶ目的は、判断のための道具箱を補うことです。
:::

![古典的 ML 補足アルゴリズムモジュールマップ](/img/course/elective-classic-ml-module-map.png)

## 学習目標

- 古典的機械学習補足モジュールが学習全体の流れの中でどこにあるのかを理解する
- このモジュールの各レッスンが何を解決するのかを知る
- どれを先に学び、どれを後に学ぶべきかをはっきりさせる
- 最小限の例で、すばやく感覚をつかむ

---

## 一、このモジュールは何を解決するのか？

### 1.1 モジュールの位置づけ

古典的機械学習の補足は、「内容を少し多く学ぶ」ためではなく、実務で上限を左右しやすい力を補うためのものです。

まずは、次のようなテーマ別ツール箱だと考えるとよいでしょう。

- 関連するプロジェクトに出会ったときに見返す
- 一度に全部学ぶ必要はない
- ただし、対応する場面に入ると非常に価値がある

### 1.2 おすすめの学習順序

比較的安定した学び方は、通常次のようになります。

1. まず全体像を見て、各レッスンがだいたい何をするのかを知る
2. いちばん基礎的で、すぐ役に立つテーマから始める
3. その後で、より実務寄り・プロジェクト寄りの内容に進む

---

## 二、このモジュールにはどんなテーマがあるのか？

### 2.1 章の一覧

| 章 | テーマ |
|---|---|
| 第1講 | サポートベクターマシン |
| 第2講 | K 近傍法 |
| 第3講 | ナイーブベイズ |
| 第4講 | 線形判別分析 |

### 2.2 このモジュールをどう使うべきか？

とても実用的なやり方は次の通りです。

- まずは主線のコースで全体をひと通り通す
- 専門的なニーズが出てきたら、そのときに選修モジュールへ戻って深める

こうすると、テーマが多すぎて主線のリズムを失うことを防げます。

## 三、最小の実行可能な例

:::info 実行のヒント
```bash
pip install numpy scikit-learn
```
:::

```python
import numpy as np
from sklearn.svm import SVC

X = np.array([[0, 0], [1, 1], [1, 0], [0, 1]])
y = np.array([0, 1, 1, 0])

clf = SVC(kernel="linear")
clf.fit(X, y)
print(clf.predict([[0.8, 0.9]]))
```

### 3.2 この例でつかんでほしいこと

この短いコードは、モジュール全体をカバーするためのものではありません。  
「このモジュールは何をしているのか」をすばやくつかむためのものです。

読むときは、次の3点を優先して見てください。

- 入力は何か
- 中間で何が起きているのか
- 出力は実際のプロジェクトでどう対応するのか

## 四、学習のすすめ方

### 4.1 時間が限られているなら、何を先に学ぶべきか？

後のプロジェクトで高い頻度で出てきて、すぐにコスト削減や効率向上に役立つテーマを優先しましょう。

### 4.2 よくある誤解

- 選修だからまったく学ばない
- いきなり選修を全部やり切ろうとする
- 概念だけ見て、最小例を実行しない

---

## 五、いつこのモジュールに戻って学ぶのが最適か？

次のようなサインが出てきたら、この内容を補い直すのにとても良いタイミングです。

- 手元のタスクが中小規模データで、深層学習が必ずしも最適ではない
- 強い baseline が必要だが、どこから始めればよいかわからない
- 「なぜこのモデルを選ぶのか」という判断力を高めたい
- 木モデルや線形モデルだけでなく、古典的 ML の道具箱を補いたい

## 六、このモジュールを学び終えると何ができるか？

- 中小規模データのタスクで、より柔軟にモデルを選べる
- SVM、KNN、ナイーブベイズ、LDA の適用範囲を理解できる
- 主線プロジェクトに、よりしっかりした古典的 baseline を足せる
