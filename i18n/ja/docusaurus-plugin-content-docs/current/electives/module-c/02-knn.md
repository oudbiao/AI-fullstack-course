# E.C.2 K近傍

![KNN 近傍投票図](/img/course/elective-knn-neighbor-voting-ja.png)

:::tip この節の位置づけ
KNN は、直感をつかむのにとても向いているアルゴリズムの一つです。  
ほとんど複雑な学習過程がなく、考え方の中心はとてもシンプルです。

> **新しいサンプルの周りにいる、いちばん似ている数人の近所の人を見て、その人たちの判断を参考にする。**

それだけでなく、KNN は次のような理解にも役立ちます。

- 距離計量
- 特徴量スケーリング
- 局所的な判断

これらは、機械学習の定番でとても重要な概念です。
:::

## 学習目標

- KNN の基本的な仕組みを理解する
- K の値と距離計量が結果に大きく影響する理由を理解する
- 特徴量スケーリングが KNN に特に重要な理由を理解する
- 実行できるサンプルを通して、KNN の最小限の使い方を身につける

---

## 一、KNN はいったい何をしているの？

### ほとんど「学習」しない

KNN はよく次のように呼ばれます。

- lazy learner

なぜなら、多くのモデルのように学習段階で明確なパラメータを学ぶわけではないからです。  
どちらかというと、次のようなイメージです。

- サンプルをそのまま覚えておく
- 予測するときに最近傍を探す

### なぜ理解しやすいの？

なぜなら、考え方が人間の直感にとても近いからです。

- この新しいお客さんは誰にいちばん似ている？
- この新しい画像は、どの古い画像にいちばん似ている？

もし近くにいる数個のサンプルの多くがあるクラスに属していれば、  
新しいサンプルもそのクラスに属する可能性が高いと考えます。

### たとえば

KNN は次のような質問に似ています。

- 「あなたの近所の人たちは、どんな職業の人が多いですか？」

周りの 5 人のうち 4 人がエンジニアなら、  
その人もエンジニアっぽいと判断しやすいですよね。

---

## 二、K の値がなぜこんなに重要なの？

### K が小さすぎると

K=1 だと、モデルはとても敏感になります。

- ノイズ点の影響を受けやすい

### K が大きすぎると

K が大きいと、  
モデルは平均化されすぎます。

- 局所的な違いがならされてしまう

### では K は本質的に何を調整しているの？

ざっくり言うと、次のように理解できます。

- K が小さい: 局所の細部を見る
- K が大きい: 全体の平均を見る

だからこそ、KNN はシンプルでも、チューニングがかなり重要です。

---

## 三、まずは「近所投票」を実感できる例を動かしてみよう

```python
import numpy as np
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier

X = np.array([
    [1, 1],
    [2, 2],
    [2, 1],
    [8, 8],
    [9, 9],
    [8, 9],
])
y = np.array([0, 0, 0, 1, 1, 1])

clf = make_pipeline(
    StandardScaler(),
    KNeighborsClassifier(n_neighbors=3),
)

clf.fit(X, y)
pred = clf.predict([[3, 3], [8.5, 8.2]])
print(pred.tolist())
```

### このコードで一番つかんでほしいことは？

KNN の最も基本的な 2 つのポイントです。

1. `n_neighbors=3`  
   これは「最近傍を 3 個見る」という意味です
2. `StandardScaler()`  
   これは、距離を計算する前に特徴量をスケーリングするという意味です

### なぜ KNN でもスケーリングが必要なの？

KNN の中心は距離だからです。  
ある次元の値の範囲が、別の次元よりずっと大きいと、その次元が距離の計算を支配してしまいます。

たとえば、

- 年齢は数十
- 収入は数十万

という場合、スケーリングしないと、収入の次元が年齢の次元をほとんど「飲み込んで」しまいます。

## 四、KNN の良い点とコストは何？

### 良い点

- 考え方が直感的
- 学習段階が軽い
- 小さなデータのベースラインとしてよく使える

### コスト

- 予測時に遅くなることがある
- データ量が大きいと検索コストが高い
- 特徴量スケーリングと距離の定義に強く左右される

### エンジニアリング上の判断

KNN が向いているのは、たとえば次のような場面です。

- サンプル数が少ない
- 特徴量が複雑すぎない
- まずは素早く、説明しやすいベースラインを作りたい

あまり向いていないのは、たとえば次のような場面です。

- 超大規模データ
- 低レイテンシ推論が必要

---

## 五、距離計量に注意する理由

### ユークリッド距離は、よく使われる一つにすぎない

多くの場合、デフォルトで使われるのは次の距離です。

- Euclidean distance

でも、タスクによっては次のような距離も検討されます。

- Manhattan distance
- Cosine distance

### なぜ距離の定義でモデルの挙動が変わるの？

KNN の判断はすべて、

- 「誰がより近いか」

に基づいているからです。

「近い」の定義が変われば、近所の人の集まりも変わります。

---

## 六、KNN で最もよくある落とし穴

### 罠その一：データが大きくなっても、そのまま KNN を使う

予測時の近傍検索は、どんどん重くなります。

### 罠その二：スケーリングを忘れる

これは SVM と同じで、よくある落とし穴です。

### 罠その三：K だけを調整して、特徴量の質を見ない

特徴量そのものに識別力がないと、  
K をどれだけうまく調整しても、なかなか改善できません。

---

## まとめ

この節で一番大事なのは、KNN を「局所投票」のモデルとして捉えることです。

> **KNN は複雑なパラメータ学習に頼るのではなく、距離計量と近傍投票で判断する。だから、距離、特徴量スケーリング、局所的な判断の直感を身につけるのにとても役立つ。**

この理解ができれば、KNN は単なる入門アルゴリズムではなく、ベースライン作成でとても便利な小さな道具になります。
