# サポートベクターマシン

![SVM 最大マージンとサポートベクターの図](/img/course/elective-svm-margin-support-vectors-ja.png)

![SVM のパラメータ C と kernel 選択図](/img/course/elective-svm-c-kernel-decision-map-ja.png)

:::tip 図の読み方
SVM のポイントは公式を暗記することではなく、`C` が「許容誤差 vs マージン」を、`kernel` が「線形境界 vs 非線形境界」をどう決めるかを理解することです。図を見るときは、特徴量のスケーリングをほぼ必ず最初に行うことも一緒に覚えましょう。
:::

:::tip この節の位置づけ
SVM は「古いアルゴリズム」ではありません。  
多くの中小規模データのタスクでは、今でもとても強いベースラインになります。特に次のような場合です。

- 特徴量の次元が高い
- データ量がそれほど多くない
- クラス境界が比較的はっきりしている

この講義の重点は、数式を詰め込むことではなく、まず工程上の直感をつかむことです。

> **SVM の核心は分類境界を見つけること、そしてその境界を両側のサンプルからできるだけ遠くに置くことです。**
:::

## 学習目標

- 最大マージンとサポートベクターの直感を理解する
- 線形 kernel と kernel 技巧がそれぞれどんな場面に向くかを理解する
- なぜ特徴量のスケーリングが SVM で特に重要なのかを理解する
- 実行できる例を通して、SVM の実戦的な第一判断を身につける

---

## 一、SVM は何をしているのか？

### 1.1 ただ分ける線を探すのではなく、「いちばん安定した」線を探す

2 クラスのサンプルを分けたいとします。  
分ける線はいくつも考えられますが、SVM は適当に選ぶのではなく、次のような線を選びやすいです。

- 両側で最も近いサンプルから、できるだけ遠い線

これが：

- 最大マージン（max margin）

### 1.2 なぜ「マージンが大きい」と良いのか？

マージンが大きいほど、境界付近の小さな揺らぎの影響を受けにくくなることが多いからです。  
つまり、こう理解できます。

- ただ分けるだけではない
- より安定して分ける

### 1.3 サポートベクターとは？

すべてのサンプルが同じように重要なわけではありません。  
実際に境界線の位置を決めるのは、境界に最も近い少数のサンプルであることが多いです。

このサンプルたちを：

- サポートベクター

と呼びます。

つまり、  
SVM の境界は「最も重要な少数のサンプル」に支えられています。

---

## 二、どんなときに SVM を試す価値が高いのか？

### 2.1 中小規模データセット

データ量が特に大きくないなら、  
SVM はベースラインとして試す価値が高いです。

### 2.2 高次元特徴量

たとえば：

- TF-IDF のテキスト特徴
- 手作業の特徴量設計をした表形式特徴

SVM はこうしたタスクでよく良い結果を出します。

### 2.3 クラス境界が比較的はっきりしている

異なるクラスが特徴空間で比較的きれいに分かれているなら、  
SVM は力を発揮しやすいです。

---

## 三、まずは本当に意味のある最小例を動かしてみよう

この例では 2 つのことを行います。

1. 線形 SVM で分類する
2. サポートベクターの数を表示する

これで、SVM の重要な出力を 2 層で確認できます。

- 分類器の結果
- 境界がどの点で決まっているか

```python
import numpy as np
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC

X = np.array([
    [1.0, 1.2],
    [1.3, 0.9],
    [1.1, 1.0],
    [4.0, 4.2],
    [4.3, 3.8],
    [3.9, 4.1],
])
y = np.array([0, 0, 0, 1, 1, 1])

clf = make_pipeline(
    StandardScaler(),
    SVC(kernel="linear", C=1.0),
)

clf.fit(X, y)
pred = clf.predict([[1.2, 1.1], [4.2, 4.0]])

svc = clf.named_steps["svc"]

print("predictions:", pred.tolist())
print("n_support_ :", svc.n_support_.tolist())
print("support_vectors shape:", svc.support_vectors_.shape)
```

### 3.1 なぜこのコードは `fit/predict` だけより役に立つのか？

それは、次の 2 つを同時に見られるからです。

- 予測結果

だけでなく、

- クラスごとのサポートベクター数

も確認できます。

これにより、SVM の「最大マージン」という直感とモデルの挙動を結びつけやすくなります。

### 3.2 なぜここで `StandardScaler()` を入れるのか？

SVM は特徴量のスケールにとても敏感だからです。  
ある列の値の範囲だけが極端に大きいと、距離計算でその列が過度に効いてしまいます。

これも SVM でよくある工程上の落とし穴です。

- スケーリングをしないと、なぜか性能が大きく下がる

---

## 四、kernel 技巧は何を解決しているのか？

### 4.1 線形 SVM は、境界がだいたい線形なときに向いている

2 クラスのサンプルがもともと直線で分けられるなら、  
線形 kernel で十分なことが多いです。

### 4.2 kernel 技巧の直感

元の空間で分けにくいとき、kernel によって暗黙的により高次元の空間へ写像し、  
新しい空間では線形分離しやすくする、という考え方です。

最もよく使われるのは：

- RBF kernel

### 4.3 いつ kernel 方法に注意すべきか？

kernel SVM は柔軟ですが、その分、

- パラメータ調整に敏感
- 学習と予測が重くなりやすい

という面もあります。

そのため、実務では一般的に次の順番がおすすめです。

1. まず線形を試す
2. 線形では明らかに足りないなら、より複雑な kernel を考える

---

## 五、SVM でよく出る 2 つのパラメータ

### 5.1 `C`

これは「誤りの許容」と「境界の厳しさ」のバランスを決めます。

ざっくり覚えるなら：

- `C` が大きい：訓練データをより正確に分けようとするが、過学習しやすいことがある
- `C` が小さい：どれくらいの誤差を許すが、境界はゆるやかになる

### 5.2 `kernel`

これはモデルが使う境界を決めます。

- 線形境界
- それとも、より柔軟な非線形境界

---

## 六、SVM でよくある落とし穴

### 6.1 落とし穴 1：特徴量のスケーリングをしない

これはほぼ最もよくある問題です。

### 6.2 落とし穴 2：データ量が大きいのに、デフォルトで kernel SVM を使う

データ量が増えると、kernel 方法は重くなりやすいです。

### 6.3 落とし穴 3：SVM を「自動で最適なモデル」と思い込む

SVM は強力なベースラインであることが多いですが、  
すべてのタスクでデフォルト最適解ではありません。

---

## まとめ

この節で本当に大事なのは、サポートベクターマシンを数式の塊として覚えることではなく、次の判断を持つことです。

> **SVM の強みは、中小規模データ、高次元特徴量、境界が比較的はっきりした問題にある。最大マージンでより安定した分類境界を見つけ、サポートベクターがその境界の本当の鍵になる。**

この理解ができていれば、今後は「どんな場面で使うべきか」「どんな場面では無理に使わないべきか」が分かるようになります。
