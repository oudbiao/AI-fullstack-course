# E.B.2 反復子とジェネレータの応用

:::tip この節の位置づけ
反復子とジェネレータは、もっとも「文法テクニック」だと誤解されやすい部分です。  
でも実際の開発では、いちばん大事な価値は次の点にあります。

> **データを一度に全部メモリへ入れるのではなく、生成しながら消費できること。**

これはデータ処理、ログのストリーム、バッチ処理、サーバーサイドのコードでとてもよく使われます。
:::

![ジェネレータのストリーミングパイプライン図](/img/course/elective-generator-stream-pipeline-ja.png)

## 学習目標

- 反復子とジェネレータの、実務での中心的な価値を理解する
- 惰性評価がなぜメモリ負荷を大きく下げるのかを理解する
- 簡単なジェネレータパイプラインを作れるようになる
- 実行可能な例を通して `yield` と `yield from` の使いどころを身につける

---

## 一、なぜ実務コードはジェネレータを好むのか？

### 多くのデータは「塊」ではなく「流れ」だから

たとえば次のようなものです。

- ログストリーム
- ファイルを1行ずつ読む処理
- ネットワークのレスポンス結果
- 大量サンプルの処理

毎回すべてを先にリストへ読み込むと、  
すぐに次の問題が起きやすくなります。

- メモリの無駄
- 遅延の増加

### ジェネレータの本質的な価値

ジェネレータを使うと、次のことができます。

- 必要になったときに、次の値を生成する

これが惰性評価です。

### たとえ話

リストは、最初からたくさんの料理を一度に並べるイメージです。  
ジェネレータは、注文番号ごとに1品ずつ料理を出すイメージです。

お客さんが多く、料理の数も多いなら、後者のほうがたいてい省資源です。

## 二、まずはスライディングウィンドウのジェネレータを見てみよう

```python
def sliding_window(nums, size):
    for i in range(len(nums) - size + 1):
        yield nums[i : i + size]


for window in sliding_window([1, 2, 3, 4, 5], 3):
    print(window)
```

### このコードにどんな価値があるのか？

ジェネレータの本質がすでに表れています。

- すべてのウィンドウを一度に返すのではない
- 1つずつ順番に出力する

### こういう書き方はどこでよく使う？

たとえば次のような場面です。

- 時系列のウィンドウ処理
- NLP の分割処理
- バッチ処理のスライス

## 三、ジェネレータパイプライン：複数の手順をつなぐ

実務では、1つのジェネレータだけよりも、  
複数のジェネレータをつないだパイプラインのほうがよく使われます。

```python
def read_lines():
    lines = [
        "INFO request ok",
        "ERROR db timeout",
        "INFO cache hit",
        "ERROR auth failed",
    ]
    for line in lines:
        yield line


def filter_errors(lines):
    for line in lines:
        if "ERROR" in line:
            yield line


def normalize(lines):
    for line in lines:
        yield line.lower()


pipeline = normalize(filter_errors(read_lines()))

for item in pipeline:
    print(item)
```

### この例でいちばん伝えたいことは？

実務のデータ処理は、たいてい次のように分けられます。

- 読み込み
- フィルタリング
- 変換

もし各ステップで完全なリストを作ると、  
処理のつながりが重くなります。  
ジェネレータパイプラインにすると、より自然です。

### なぜこれは AI 工学でも役立つのか？

よく次のようなものを扱うからです。

- サンプルの流れ
- ログの流れ
- 検索結果の流れ

こうした場面は、ジェネレータパイプラインととても相性が良いです。

## 四、`yield from` はなぜ学ぶ価値があるのか？

### 何を解決するのか？

あるジェネレータが、別のイテラブルの内容をそのまま外へ渡したいだけなら、  
`yield from` を使うとコードがすっきりします。

```python
def chunk_batches():
    yield [1, 2]
    yield [3, 4]


def flatten():
    for batch in chunk_batches():
        yield from batch


print(list(flatten()))
```

### なぜ二重ループより学ぶ価値があるのか？

意図がよりはっきり伝わるからです。

- 「子の反復子の内容を、そのまま外へ出力する」

---

## 五、いちばんよくある落とし穴

### 誤解その1：ジェネレータは必ず速い

たしかにメモリは節約しやすいですが、  
すべての場面で絶対に速いとは限りません。

### 誤解その2：ジェネレータは1回しか使えない

多くの場合、これはバグではなく設計上の特徴です。  
何度も使いたいなら、もう一度作り直す必要があります。

### 誤解その3：ジェネレータを使うために無理に使う

データ量が少なく、ロジックも単純なら、  
そのままリストのほうが読みやすいこともあります。

## まとめ

この節でいちばん大事なのは、実務的な感覚を持つことです。

> **ジェネレータと反復子は、「少しずつ生成して、少しずつ消費する」データストリームの処理に最適です。主な価値は、メモリ節約、中間コピーの削減、そしてパイプラインの整理にあります。**

この理解ができれば、  
今後ログ処理、サンプルパイプライン、ストリーミングサービスを作るときに、自然とこれらを思い出せるようになります。
