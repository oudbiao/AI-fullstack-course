# 4.0 学習ガイドとタスクシート：AI 数学基礎

![AI 数学学習の循環図](/img/course/math-study-loop-ja.png)

このページを第 4 章のコントロールパネルとして使ってください。目標はすべての証明を学ぶことではありません。モデル入力、不確実性、損失、パラメータ更新を理解するための数学的直感を作ることです。

## 4.0.2 始める前に知っておく用語

| 用語 | 英語の正式名称 | この章での最初の意味 |
|---|---|---|
| `ML` | Machine Learning | 機械学習。データでモデルを訓練し、予測や判断を行うこと。 |
| `RAG` | Retrieval-Augmented Generation | 検索拡張生成。先に資料を検索し、その証拠を使って回答させる方法。 |
| `LLM` | Large Language Model | 大規模言語モデル。token を予測・生成する大きなテキストモデル。 |
| `Embedding` | Vector representation | テキスト、画像、ユーザー、商品などを比較できるベクトルに変換すること。 |
| `Notebook` | Jupyter Notebook | コード、グラフ、メモ、出力をまとめる学習実験ファイル。 |

これらの実務テーマを今すぐ完全に理解する必要はありません。数学の行き先を知るための目印として使ってください。

## 4.0.3 最小タスクシート

| タスク | 成果物 | クリア基準 |
|---|---|---|

## 4.0.4 数式を直感に翻訳する表

数式が難しく見えたら、導出の前にモデルの言葉へ翻訳してください。

| 用語 | 最初の直感 | あとで出てくる場所 |
|---|---|---|
| ベクトル | 1 行の特徴量、または意味の座標 | Embedding、類似度検索、モデル入力 |
| 行列 | 多くの行のデータ、またはベクトルを変換する機械 | ニューラルネットワーク層、Attention の重み |
| 内積 | 2 つの方向がどれくらい合っているか | コサイン類似度、Attention スコア |
| ノルム | 長さや強さ | 距離、正則化、勾配クリッピング |
| PCA | 座標軸を回し直し、情報量の多い軸だけ残す | 次元削減、可視化 |
| MLE | 観測データを最もよく説明するパラメータを選ぶ | 損失関数、ロジスティック回帰 |
| エントロピー | 不確実性や驚きの大きさ | 分類損失、言語モデル |
| GD / SGD | 損失の地形を下り坂へ歩く | モデル学習、最適化器 |

右側を自分の言葉で説明できれば、最初に必要な実用版はもうつかめています。

## 4.0.5 ステージチェックポイント

第 5 章へ進む前に、次の 5 つを確認してください。

- 1 つのサンプルをベクトル、データセットを行列として表せる理由を説明できる。
- モデル出力を確率や信頼度として読める理由を説明できる。
- エントロピー、クロスエントロピー、不確実性、損失の関係を説明できる。
- コードまたは表で最小限の勾配降下法を示せる。
- [4.4 実践：第 4 章フル数学ワークショップ](./hands-on-math-workshop.md) を実行し、生成された証拠フォルダを残している。

各数式を「データを表す」「不確実性を測る」「損失を測る」「パラメータを更新する」のどれかに結びつけられれば、先へ進んで大丈夫です。
