# 学習ガイド：AI 数学の基礎はどう学ぶと一番挫折しにくいか

![AI 数学学習の循環図](/img/course/math-study-loop-ja.png)

`03 AI 数学の最小必要基礎` に入って数式が多そうで不安になったら、まずは目標を少し下げましょう。1回目は数学体系を全部学ぶのではなく、モデルの直感をつかむことが目標です。

## この段階の全体原則

AI 数学の1回目で押さえるのは3つだけです。線形代数はデータとパラメータがどう表されるかを説明し、確率統計は不確実性と評価を説明し、微積分は損失と勾配によってモデルがどう更新されるかを説明します。

![AI 数学学習ガイドの最小ループ](/img/course/ch04-study-guide-math-minimum-loop-ja.png)

## おすすめの学習順序

1周目はまず線形代数を学びます。重点は、ベクトル、行列、行列積、線形変換、類似度を理解することです。後で Embedding、ニューラルネットワーク、Attention で、これらの概念が何度も出てきます。

2周目は確率統計を学びます。重点は、確率、分布、期待値、分散、条件付き確率、統計的推定、情報エントロピーを理解することです。これらは分類、評価、生成、検索で登場します。

3周目は微積分と最適化を学びます。重点は、導関数、偏導関数、勾配、連鎖律、勾配降下法を理解することです。最初から複雑な式を導ける必要はありませんが、モデルがなぜ少しずつ良くなるのかは分かるようにしましょう。

## 数式を直感に翻訳する表

数式が難しく見えたら、まずモデルの言葉に翻訳してみましょう。

| 用語 | 最初の直感 | あとで出てくる場所 |
|---|---|---|
| ベクトル | 1行の特徴量、または意味の座標 | embedding、類似度検索、モデル入力 |
| 行列 | 多くの行のサンプル、またはベクトルを変換する機械 | ニューラルネットワーク層、Attention の重み |
| 内積 | 2つの方向がどれくらい合っているか | コサイン類似度、Attention スコア |
| ノルム | 長さや強さ | 距離、正則化、勾配クリッピング |
| PCA (Principal Component Analysis) | 座標軸を回し直し、情報の多い軸だけ残す | 次元削減、可視化 |
| SVD (Singular Value Decomposition) | 複雑な行列を「回転-拡大縮小-回転」に分ける | 圧縮、推薦システム、PCA |
| MLE (Maximum Likelihood Estimation) | 観測データを最もよく説明するパラメータを選ぶ | 損失関数、ロジスティック回帰 |
| MAP (Maximum A Posteriori) | MLE に事前知識を足したもの | 正則化、weight decay |
| エントロピー | 不確実性や驚きの大きさ | 分類損失、言語モデル |
| KL ダイバージェンス (Kullback-Leibler divergence) | 2つの確率分布がどれくらい違うか | 蒸留、変分モデル |
| GD / SGD (Gradient Descent / Stochastic Gradient Descent) | 損失の地形を下り坂へ歩く | モデル学習、最適化器 |

右側を自分の言葉で言えれば、もう実用上かなり大事な部分をつかめています。厳密な導出はあとから戻ってきても大丈夫です。

## おすすめの学習ペース

| 内容の種類 | おすすめ時間 | 学習目標 |
|---|---|---|
| 線形代数のページ | 2～4 時間 | ベクトル、行列、データを結びつけて理解できる |
| 確率統計のページ | 2～4 時間 | 分類確率と評価指標を理解できる |
| 微積分と最適化のページ | 2～4 時間 | 勾配降下法の直感を説明できる |
| 小さな実験 | 4～8 時間 | コードで数学の概念を可視化できる |

## 段階プロジェクトの流れ

最初の小さな実験では、2次元ベクトルを使って類似度を描き、内積、距離、類似度を理解します。

2つ目の小さな実験では、乱数で確率分布を作り、平均、分散、サンプリングのばらつきを観察します。

3つ目の小さな実験では、簡単な関数を使って勾配降下法を示し、ある点が最小値に少しずつ近づく様子を確認します。

これらのプロジェクトはどれも難しくありませんが、数式とコードをつなぐ助けになります。

## つまずきやすいポイント

最もよくあるつまずきは、数学を「完全に」理解してから機械学習に進もうとすることです。AI 学習では、これはたいてい進度を遅くします。まずは最小限の直感を作り、あとで具体的なモデルに出会ったときに戻って補えば大丈夫です。

2つ目のつまずきは、数式だけを見てコードを書かないことです。各コア概念には、NumPy の小さな実験か図を必ずセットで用意するのがおすすめです。

3つ目のつまずきは、忘れることを怖がることです。数学は何度も繰り返し触れるものです。最初は、それがモデルの中でどんな役割を持つかだけ分かれば十分です。

## 合格基準

この段階を終えたら、次のことを自分の言葉で説明できるようになっているはずです。なぜデータは行列で表せるのか、なぜモデルは確率を出力するのか、なぜ損失関数がパラメータ更新を導けるのか。

コードで最小限の勾配降下の流れを示し、各ステップで何をしているか説明できれば、機械学習の段階に進んで大丈夫です。
