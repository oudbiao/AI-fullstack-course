# 4 AI 数学最小必要基礎

![AI 数学基礎メインビジュアル](/img/course/ch04-ai-math-ja.png)

この段階で解決したいのは、「モデルの中に出てくる数学を見ても、もう怖くない」状態になることです。数学者になることが目的ではありません。モデルでよく出てくる数学の対象、つまりベクトル、行列、確率、損失、勾配、最適化を理解できるようにすることが目的です。

## ストーリーで導入：モデルに“数学のメガネ”をかける

多くの初心者が AI 数学を苦手に感じるのは、数式が壁のように見えるからです。ここでは見方を変えて、数学を「メガネ」だと考えましょう。ベクトルはモデルに「方向と似ている度合い」を見せ、行列は大量のデータをまとめて処理し、確率は不確実さを表し、勾配はどこを直せばよいかを教えてくれます。最初から数学の専門家になる必要はありません。まずは、これらの道具がモデルの中でどんな役割を持つのかを読み取れれば十分です。

## 体験練習：コードで公式を画面にする

ベクトルを学ぶときは、2つの2次元の矢印を描いて、角度が小さいほど似ていることを観察してみましょう。確率を学ぶときは、乱数を生成して分布図を描きましょう。勾配降下法を学ぶときは、ランダムな点から出発して、そこから少しずつ最小値へ向かう様子を見てみましょう。数式を配列、曲線、動きのあるイメージに変えられれば、最も難しい最初の関門はもう越えています。

## プロジェクトの特典

この段階の特典は大きなプロジェクトではなく、いくつかの「数学ミニ実験」です。ベクトル類似度の可視化、確率分布の観察器、勾配降下法のデモです。あとで推薦システム、Embedding、ニューラルネットワーク、Transformer を学ぶとき、これらの小さな実験が実は本物の AI モデルの中に何度も出てきたものだと気づくでしょう。

`4.1`、`4.2`、`4.3` を学んだ後に、[4.4 実践：第 4 章フル数学ワークショップ](./hands-on-math-workshop.md) を実行してください。ベクトル類似度、確率シミュレーション、エントロピー/loss、勾配降下、SVG 図、見直せる証拠フォルダを 1 つにつなげます。

## 初心者向けの最小クリアコース

初心者は、完全な数学体系を目指さなくて大丈夫です。まずは、ベクトル、行列、確率、損失、勾配がモデルの中でそれぞれ何を解決しているのかを理解しましょう。NumPy でベクトル類似度、確率分布、勾配降下法の小さな実験が書ければ、最小クリアです。

## 上級者向けの深掘りコース

経験のある学習者は、行列積の幾何学的な意味、統計的推定、情報エントロピー、連鎖律、逆伝播をさらに理解できます。各数式にコード実験や図による説明を組み合わせ、後の機械学習や深層学習の数式を読む準備をしましょう。

## 初心者はまず何をする？ 上級者はその後何をする？

初めてこの段階を学ぶときは、数学を公式暗記にしないことが大切です。まずは、ベクトルは「対象の位置」を表す、確率は「不確実さ」を表す、勾配は「改善の方向」を表す、という3つの直感をつかみ、そのあとでモデルの中でどう使われているかを見直しましょう。

経験のある学習者は、モデルの説明に重点を置けます。なぜ行列積で特徴変換ができるのか、なぜ確率で予測の信頼度を表せるのか、なぜ勾配降下法でモデルを学習できるのかを理解しましょう。目標は、モデルの記事を読んだりパラメータ調整をしたりするときに、各数学概念が何を解決しているのか分かるようになることです。

## ここが「最小必要基礎」と呼ばれる理由

線形代数、確率論、微積分は、それぞれ長く学べる分野です。ただし AI の入門では、最初から完全な数学体系を追う必要はありません。まずは、最も役に立ち、よく出てきて、モデルにつなげやすい部分を押さえることが大切です。

## この段階の学習ルート

第1章では線形代数を学びます。ベクトル、行列、行列積、線形変換、固有値が、データ行列、Embedding、ニューラルネットワークのパラメータ、注意機構の計算にどう出てくるかを理解する必要があります。

第2章では確率と統計を学びます。確率、分布、期待値、分散、統計的推定、情報エントロピーを理解しましょう。これらは分類モデル、損失関数、評価指標、生成モデルに登場します。

第3章では微積分と最適化を学びます。導関数、偏微分、勾配、連鎖律、勾配降下法を理解しましょう。これらは、損失関数を通じてモデルのパラメータが少しずつ更新される仕組みを説明します。

## 学習後にできるようになること

- 表形式データを行列として理解し、1行のサンプルをベクトルとして理解できる
- 分類モデルが確率を出すことが多い理由を説明できる
- 損失関数、勾配降下法、パラメータ更新のおおまかな流れを理解できる
- NumPy や簡単なコードでベクトル演算、確率分布、勾配降下法を示せる
- 機械学習や深層学習の数式を見たときに、何を表しているのか大まかに判断できる

## よくある誤解

数学の細かい部分を完全に理解できないからといって、そこで止まらないでください。AI 数学は繰り返し学ぶものです。1回目は直感をつかめば十分で、あとで機械学習、深層学習、Transformer、RAG の中で何度も出会います。

また、数式を見るだけでコードを書かないのも避けましょう。エンジニア学習者にとっては、配列、画像、小さな実験で数学を理解するほうが、推論だけを見るより効果的なことが多いです。

## 数学つまずき劇場：公式は分かったのに使えないときは？

数式を見てすぐ混乱するなら、まずモデルの言葉に訳してみましょう。それが対象を表しているのか、不確実さを測っているのか、モデルにどこを直すか教えているのかを考えます。推論についていけないなら、まずは2次元の図、表、簡単な数値例で1回動かしてみましょう。数学とコードがつながらないなら、モデル入力、損失関数、パラメータ更新の中でその数式がどこにあるのかを探してみてください。

## 最小実行可能実験：コードでベクトル、確率、勾配を見る

この段階の最小実験では、完全な導出は求めません。代わりに、3つの小さな Notebook で抽象概念を観察できる形にします。つまり、余弦類似度で2つの学習テーマを比べ、乱数で分布を描き、1変数関数で勾配降下法を示します。

```python
import numpy as np

a = np.array([1, 1, 0])
b = np.array([1, 0.8, 0.2])
cosine = a @ b / (np.linalg.norm(a) * np.linalg.norm(b))
print(cosine)
```

この最小例を実行するには、コードを `math_similarity_demo.py` として保存し、次を実行します。

```bash
python math_similarity_demo.py
```

出力は次に近い値になります。

```text
0.9819805060619657
```

この数値がなぜ「似ていること」を表せるのか説明できれば、あとで Embedding、検索、推薦の理解がかなり楽になります。

## 数学失敗ケース集：まずモデルの言葉に訳す

| 現象 | よくある原因 | 切り分け方法 | 修正の方向 |
|---|---|---|---|
| 数式は分かったのに使えない | 入力、パラメータ、損失、出力とつながっていない | モデルの中でどんな役割かを問う | 小さな数値例と図で書き直す |
| 確率の概念が混乱する | 頻度、信頼度、モデルスコアを混同している | 確率変数と事象を見つける | サンプル、結果、確率を表で整理する |
| 勾配降下法が収束しない | 学習率が大きすぎる、または関数のスケールが不明 | loss の反復変化を描く | 学習率を調整し、経路を観察する |
| 数学とコードが分離している | 導出だけ見て、配列実験をしていない | NumPy で最小例を再現する | 各数式に実行可能な断片を付ける |
