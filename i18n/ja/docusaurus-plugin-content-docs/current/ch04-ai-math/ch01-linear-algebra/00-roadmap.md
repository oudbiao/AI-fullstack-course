# 4.1.1 学習ガイド：線形代数ってこの章で何を学ぶの？

![線形代数学習マップ](/img/course/ch04-linear-algebra-roadmap-vertical-ja.png)

:::tip 公式を急いで暗記しなくてOK
初心者にとって、この章でいちばん大事なのは定理の証明ではなく、まず地図を作ることです。

1. なぜ AI の中にはベクトルと行列があふれているのか

## 学習目標

- 線形代数のこの章が、カリキュラム全体のどこにあるかを知る
- 「ベクトル -> 行列 -> 固有値 -> 線形変換」の関係を理解する
- 初心者が重点的に押さえるべき内容と、まずは直感だけでよい内容を知る
- 本格的に公式へ入る前に、AI の場面での使い方の感覚をつかむ

## まず、とても大事な心構え

線形代数は、それだけで長く学べる科目です。  
なのでこの章の目的は、証明や定理、応用を全部やり切ることではなく、まずあなたに次の感覚を持ってもらうことです。

- ベクトルや行列を怖がらない
- いちばんよく出る計算が、何をしているのか分かる
- それらを AI のデータ、重み、類似度、PCA と結びつけられる

今この時点で、読み終えたあとに「問題をスラスラ解ける」必要はありません。  
それより大事なのは、次のことです。

- 「ベクトル / 行列」を、知らない記号の集まりではなく AI の言葉として見始められているか

---

## 一、なぜ AI には線形代数がたくさん出てくるのか？

AI の多くの対象は、本質的には「数字の並び」として表せます。  
そして線形代数は、その数字の並びをどう表すか、どう変換するか、どう比べるかを扱う言語です。

| AI の場面 | 目の前に見えるもの | 線形代数ではどう見るか |
|---|---|---|
| 1 人分のユーザーデータ | 年齢、収入、都市ランク、アクティブ度 | 1つのベクトル |
| たくさんのサンプル | 多くのユーザーデータをまとめたもの | 1つの行列 |
| テキストベクトル検索 | クエリと文書のどちらが似ているか | ベクトルの類似度 |
| ニューラルネットワークの1層 | 入力に重みを掛けてバイアスを足す | 行列乗算 |
| PCA の次元削減 | 特徴を圧縮しつつ情報の損失をできるだけ減らす | 固有値と固有ベクトル |

なので、この章は次のように理解できます。

- ベクトル：1つの対象を表せるようになる
- 行列：複数の対象をまとめて扱う、または対象に変換をかける
- 固有値：いちばん重要な方向を見つける
- ベクトル空間：次元、基底、変換を、より高い視点で理解する

---

## 二、この章の4つのレッスンはどうつながっているの？

![線形代数章の関係図](/img/course/ch04-linear-algebra-chapter-flow-ja.png)

この章全体は、次の一文にまとめられます。

> **まずデータをベクトルとして書けるようになり、次に行列でベクトルをまとめて処理できるようになり、最後にその変換の中からいちばん大事な構造を見つける。**

## 三、なぜこの章は AI にとって特に重要なの？

この先、あなたは何度も次のものに出会います。

- 1つのサンプルはベクトル
- たくさんのサンプルは行列
- ニューラルネットワークの1層は行列乗算
- 類似度検索には内積やコサイン類似度を使う
- PCA は重要な方向を見つける

つまり、AI における線形代数は「背景知識」ではありません。  
むしろ、

> **モデルとデータを見るときに、いちばんよく使う観察の言語です。**

## 四、始める前の小さな用語集

| 用語 | ここでの意味 | この章で出てくる理由 |
|---|---|---|
| `RAG` | Retrieval-Augmented Generation、検索拡張生成：資料を検索してから回答を生成する仕組み | 検索にはベクトル類似度が必要です。 |
| `PCA` | Principal Component Analysis、主成分分析 | 重要な方向を使ってデータを圧縮・可視化します。 |
| `X @ W + b` | 行列乗算にバイアスを足す形 | 多くのニューラルネットワーク層の基本形です。 |
| 内積 | 対応する位置を掛けて足し合わせる計算 | 類似度と行列乗算の中心になる操作です。 |
| コサイン類似度 | 2つのベクトルの角度を比べる指標 | ベクトルの長さに引っ張られすぎず、向きの近さを見たいときによく使います。 |

まだ慣れない用語があっても、ここで暗記のために止まらなくて大丈夫です。この表を小さな辞書として使い、同じ言葉に再会したときに戻って確認しましょう。

## 五、初心者はこの章をどう学べばいい？

### 第1段階：まずベクトルを理解する

最低限、次のことを理解しましょう。

- ベクトルは、順番のある数字の並び
- ベクトルの長さ、内積、コサイン類似度は、それぞれ何を測っているのか
- なぜ RAG、推薦システム、単語ベクトルで類似度が使われるのか

### 第2段階：行列を「一括処理マシン」として見る

最低限、次のことを理解しましょう。

- たくさんのサンプルをまとめると行列になる
- 行列乗算は、実は「行と列の内積」
- なぜニューラルネットワークの1層が `X @ W + b` と書けるのか

### 第3段階：固有値を「特別な方向」として見る

最低限、次のことを理解しましょう。

- ほとんどのベクトルは、行列をかけると向きが変わる
- ただし、ある特別な方向は伸びたり縮んだりするだけ
- PCA は、データの変化が最も大きい方向を探している

### 第4段階：ベクトル空間は理解を深めるための発展内容

この節は、前半で学んだ内容を、少し高い視点から見直すパートです。

もし今の目標が機械学習や深層学習を早く学ぶことなら、まずは次の内容を：

- 線形独立
- 基底
- 次元
- 線形変換

「説明できる」「コードで確かめられる」くらいまで理解してから、次へ進めば十分です。

### 初心者向けのおすすめ読書順

各節は、次の順番で読むのがおすすめです。

1. まずたとえ話と図を見る
2. 次に最小のコードを見る
3. 最後に公式と定義を見る

特に線形代数は、この順番のほうが、最初から記号を追うよりずっと分かりやすいです。

## 五、この章にはどれくらい時間を使うとよい？

この章を安定して学ぶなら、初心者向けの目安は次の通りです。

1. ベクトル：2〜4 時間  
   重点は計算問題よりも、「対象 -> ベクトル -> 類似度」の流れをきちんと理解することです。

2. 行列：3〜5 時間  
   「一括処理」と `X @ W` の主線を理解することを重視しましょう。

3. 固有値：3〜5 時間  
   まずは「特別な方向」と PCA の直感を作ることが大切で、難しい計算を完璧にする必要はありません。

4. ベクトル空間：2〜4 時間  
   これは理解を深めるパートなので、前の3節がある程度固まってから読むのがおすすめです。

1日に1節ずつ進めるほうが、4節をまとめて無理に進めるより、だいたい安定します。

## 六、この章でよくある勘違い

- 線形代数は公式のかたまりだと思ってしまうが、実際はまずデータと変換を表す言語である
- 証明が分からないと AI は学べないと思ってしまうが、まず図やコードが分かるだけでも十分価値がある
- 数学とコードは分けて学べると思ってしまうが、分けるとだんだん曖昧になりやすい
- 高次元ベクトルは図にできないから理解できないと思ってしまうが、2次元の図はあくまで直感を作るためのもの

## 七、まずはこの章を通して使う最小例を見てみよう

次の小さなコードは、この章の主線をかなりよくつないでいます。

- 1つのサンプルはベクトル
- 複数のサンプルをまとめると行列
- 重みとの内積でスコアが出る
- サンプル全体と重み行列を掛けると、複数の結果を一度に計算できる

```python
import numpy as np

student = np.array([90, 85, 92])
print("単一の生徒ベクトル:", student)

students = np.array([
    [90, 85, 92],
    [70, 88, 75],
    [95, 91, 89],
])
print("サンプル行列の形状:", students.shape)  # (3, 3)

weights = np.array([0.4, 0.2, 0.4])

single_score = student @ weights
print("単一の生徒の総合点:", round(single_score, 2))

all_scores = students @ weights
print("すべての生徒の総合点:", all_scores.round(2))
```

実行方法：

```bash
python linear_algebra_minimum_demo.py
```

期待される出力：

```text
単一の生徒ベクトル: [90 85 92]
サンプル行列の形状: (3, 3)
単一の生徒の総合点: 89.8
すべての生徒の総合点: [89.8 75.6 91.8]
```

出力が違う場合は、まず重みの合計が `1.0` のままか、サンプル行列の形状が `(3, 3)` のままか、そして `*` による要素ごとの掛け算ではなく `@` による行列/ベクトル乗算を使っているかを確認しましょう。

## 八、この章を終えた時点で、少なくとも何ができればよい？

- 1つのデータを見て、それをベクトルとして書けることが分かる
- たくさんのデータを見て、それを行列として書けることが分かる
- 内積を見て、それが何を測っているか分かる
- 行列乗算を見て、それが一括変換だと分かる
- 固有値を見て、それが PCA や主方向と関係あると分かる

## もしこの章が「まだ抽象的すぎる」と感じたら、どこを見直すべき？

いちばん見直す価値があるのは、最後の固有値ではなく、最初の次の3つです。

- ベクトルがどう対象を表すか
- 行列がどう一括で対象を処理するか
- 内積が何を比較しているか

この3つがしっかりすると、その後のもっと抽象的な内容はかなり楽になります。
