# 段階学習タスクシート：AI 数学の最小必要基礎

![数学最小クリアタスクシート図](/img/course/math-task-checklist-ja.png)

このステージの目標は、数学を試験科目として学ぶことではなく、AI モデルの背後にある核心的な直感を理解できるようになることです。ベクトルがどのようにサンプルを表すのか、行列がどのように変換を表すのか、確率がどのように不確実性を表すのか、勾配がどのようにモデル更新を導くのかを理解する必要があります。

## このタスクシートで出てくる用語

| 用語 | 英語の正式名称 | ここでの意味 |
|---|---|---|
| `ML` | Machine Learning | 機械学習。データでモデルを訓練し、予測や判断をできるようにすること |
| `RAG` | Retrieval-Augmented Generation | 検索拡張生成。関連資料を先に検索し、その文脈でモデルに回答させる方法 |
| `LLM` | Large Language Model | 大規模言語モデル。GPT 系のように token を予測・生成する大きなテキストモデル |
| `Notebook` | Jupyter Notebook | コード、グラフ、メモ、出力をまとめて扱える学習実験ファイル |

初学者は、これらの実務方向を今すぐ全部理解しようとしなくて大丈夫です。まずは「この数学があとでどこに使われるのか」として受け取ってください。

## このステージで必ず完了するタスク

| タスク | 産出物 | クリア基準 |
| --- | --- | --- |
| ベクトルと行列を理解する | ベクトル類似度の実験 1 つ | 内積、ノルム、コサイン類似度、行列積を説明できる |
| 確率と統計を理解する | 分布の可視化 Notebook 1 つ | 平均、分散、条件付き確率、よくある分布を説明できる |
| 情報量とエントロピーを理解する | 情報理論ノート 1 つ | エントロピー、クロスエントロピー、モデル損失の関係を説明できる |
| 勾配降下法を理解する | パラメータ更新の可視化実験 1 つ | 学習率、勾配の方向、収束過程を説明できる |
| ステージプロジェクトを完了する | 数学的直感の可視化ミニプロジェクト 1 つ | 数学概念を ML、RAG、または LLM の場面に結びつけられる |

最初の実行としては、[実践：第 4 章フル数学ワークショップ](./hands-on-math-workshop.md) がおすすめです。1 つの実行可能なスクリプトで、ベクトル、確率、エントロピー/loss、勾配のタスクをまとめて扱い、証拠フォルダも自動で保存します。

## おすすめの学習順序

まずベクトルと行列を学び、その後で確率統計を学び、最後に微積分と最適化を学びます。最初から厳密な証明を追い求めなくて大丈夫です。1 回目は、「これらの概念がモデルの中で何をしているのか」という直感を作ることに重点を置きましょう。

数学を学ぶときは、各概念に小さな実験を 1 つ対応させるのがおすすめです。たとえば、ベクトル類似度で検索を説明し、確率分布で分類の信頼度を説明し、勾配降下法でモデルがなぜ少しずつ良くなるのかを説明できます。

## 最小実験テンプレート

このステージの数学実験は、どれも同じ小さな構成で進められます。

1. 概念を 1 文で書く
2. とても小さな数値例を作る
3. グラフを 1 つ描く、または比較表を 1 つ出す
4. 何が変わったのか、なぜ変わったのかを説明する
5. 自分が避けられた誤解を 1 つ記録する

たとえば、ベクトル類似度の実験は大きくする必要はありません。3 つのテキストベクトル、1 つのコサイン類似度表、短い説明があれば、最初の通過としては十分です。

## AI 学習アシスタントプロジェクトとの関係

このステージは、AI 学習アシスタントの v0.4 における数学説明能力に対応します。学習アシスタントに「概念説明カード」機能を追加できます。数学概念を 1 つ入力すると、直感的な説明、AI の場面、簡単な例、よくある誤解を出力する機能です。

まずは静的テンプレートで実装するのがおすすめです。モデルを呼び出す必要はありません。大事なのは、抽象的な数学概念を、後続プロジェクトで使えるエンジニアリング用語に翻訳することです。

## よくあるつまずきポイント

よくある問題には、式を孤立した暗記対象としてしまうこと、導出は分かるのに応用場面が分からないこと、確率と頻度を混同してしまうこと、勾配の方向を理解できないこと、ライブラリを使うだけで指標の意味を理解していないことがあります。つまずいたときは、まず図、数値実験、具体的な AI の場面に戻るのを優先してください。

## ゆるく進める版 / 標準版 / チャレンジ版タスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| ゆるく進める版 | コードで類似度や確率指標を計算する | 初めて学ぶ人、時間が少ない人、または入門したばかりの学習者 |
| 標準版 | 2 つの指標を比較して違いを説明する | このステージを作品集に入れたい学習者 |
| チャレンジ版 | 1 つの指標を手計算例、コード、図でわかりやすく説明する | 基礎があり、より強いプロジェクト証拠を作りたい学習者 |

## このステージのバッジと Boss 戦

| 種類 | 内容 |
|---|---|
| Boss 戦 | 指標迷路 |
| 解放できるバッジ | ベクトル翻訳官、指標説明員 |
| 最小クリアの合言葉 | まず動かす、次に説明する、最後に失敗を記録する |
| 証拠保存のおすすめ | スクリーンショット、ログ、失敗サンプル、評価表を `reports/`、`evals/`、または `logs/` に保存する |

ゆるく進める版を完了すれば次へ進めます。標準版を完了したら、作品集に載せるのがおすすめです。チャレンジ版は、余力があるときだけ取り組みましょう。

## ステージ作品集の成果物

このステージの成果を作品集に残したいなら、少なくとも以下のファイル、または同等の素材を保管するのがおすすめです。

| 成果物 | 説明 |
| --- | --- |
| `vector_similarity.ipynb` | ベクトル、内積、コサイン類似度でサンプルの類似性を説明する |
| `probability_demo.ipynb` | 確率変数、分布、平均、分散、不確実性を可視化する |
| `gradient_descent.ipynb` | 学習率、勾配の方向、収束経路を示す |
| `math_cards.md` | ベクトル、確率、エントロピー、勾配を AI アプリの直感に翻訳する |
| `reflection.md` | これらの数学概念が機械学習、RAG、大規模モデルをどう支えているかを説明する |

これらの素材は、あなたが試験のために数学を学んでいるのではなく、数学をモデル理解、モデルデバッグ、システム説明の道具に変えられることを証明してくれます。

## ステージクリアの確認問題

学習後には、次の質問に答えられるようになっているはずです。なぜテキストや画像をベクトルで表せるのか、なぜ類似度を検索に使えるのか、なぜ確率でモデルの不確実性を表せるのか、なぜクロスエントロピーが分類でよく使われるのか、なぜ勾配降下法でモデルパラメータが少しずつ良くなるのか。

## 完了状態 Checklist

- [ ] AI におけるベクトル、行列、類似度の役割を説明できる。
- [ ] 少なくとも 1 種類の確率分布と不確実性を図表で説明できる。
- [ ] エントロピー、クロスエントロピー、分類損失の関係を説明できる。
- [ ] 小さな実験で勾配降下法の過程を示せる。
- [ ] 少なくとも 3 つの数学概念を、自分なりの AI アプリ説明カードとして書けている。
