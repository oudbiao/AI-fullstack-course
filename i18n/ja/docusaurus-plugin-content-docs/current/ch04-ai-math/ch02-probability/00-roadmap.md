# 学習ガイド：この章で学ぶ確率と統計は何か

![確率と統計の学習マップ](/img/course/ch04-probability-roadmap-vertical-ja.png)

初心者にとって、この章でいちばん起こりやすい問題は「公式が計算できないこと」ではなく、学びながら「これらの概念が AI とどう関係するのか」が分からなくなることです。

この章が実際に解決しようとしているのは、同じひとつのテーマです。

> **世界が不確実性に満ちているとき、私たちはその不確実性をどう表現し、データからどう法則を逆に推定し、モデルがどれだけ確からしいかをどう測るのか。**

## 学習目標

- 「確率 -> 分布 -> 推論 -> 情報理論」という章全体の流れをつかむ
- この章の各レッスンが AI の中でどこに位置するかを知る
- 初心者が先に押さえるべき直感と、その後に見るべき公式を知る

## 読む前に略語を先にほどいておこう

この章には短い英語略語が多く出てきます。単語だけを暗記するのではなく、まず「その用語がどんな問いに答えるのか」と結びつけましょう。

| 用語 | 英語の正式名称 | AI の中で答える問い |
|---|---|---|
| `PDF` | Probability Density Function | 連続値では、確率はどこに集中しているのか？ |
| `PMF` | Probability Mass Function | 離散的な選択肢では、それぞれにどれくらい確率が割り当てられるのか？ |
| `CDF` | Cumulative Distribution Function | ある値以下になる確率はいくらか？ |
| `MLE` | Maximum Likelihood Estimation | 観測データを最もよく説明するパラメータはどれか？ |
| `MAP` | Maximum A Posteriori | データと事前知識を合わせたとき、最も信頼できるパラメータはどれか？ |
| `EM` | Expectation-Maximization | 隠れた変数とパラメータを、2 つのステップを交互に使ってどう推定するか？ |
| `KL` | Kullback-Leibler divergence | ある確率分布は、別の確率分布からどれくらい離れているか？ |
| `A/B testing` | Controlled comparison experiment | A 版と B 版の差は本当に性能差なのか、それとも偶然なのか？ |
| `loss` | Training objective value | モデルはどれくらい間違っていて、最適化では何を小さくすべきか？ |

今すぐ正式な定義をすべて覚える必要はありません。まずは、確率の用語を見たら **「これはどんな不確実性を測る、または減らすためのものか？」** と考える習慣を作りましょう。

## まず、学習に対する大事な期待値を共有します

確率と統計は、そもそも数回の授業で「完全に学び終える」類の内容ではありません。  
なので、この章の現実的な目標は次の通りです。

- まずは確率の記号に苦手意識を持たない
- 「なぜモデルはいつも確率を出すのか」を理解する
- 分布、推論、情報理論が AI の中でそれぞれ何を担当しているのかを読み取れるようにする

最初からすべての公式をスラスラ導ける必要はありません。  
ただし、次のことは言えるようになりましょう。

- それぞれがどんな不確実性を表しているのか
- なぜこれらの概念がモデルの学習や判断に直接影響するのか

---

## 一、この章の4つの節はどうつながっているのか？

![確率統計の章の関係図](/img/course/ch04-probability-chapter-flow-ja.png)

この章は、次の4つの問いとして覚えると分かりやすいです。

1. 確率の基礎：ある出来事が起こる可能性はどれくらいか？
2. 確率分布：1回の事象ではなく、一連のランダムな現象全体はどんな形をしているか？
3. 統計的推論：データを見たとき、背後にあるパラメータや結論をどう逆算するか？
4. 情報理論：モデルの予測はどれくらい不確実で、真の分布とどれくらい離れているか？

この順番には意図があります。

- 確率は、不確実性を表すための語彙を与える
- 分布は、多くのランダムな結果を見える形にまとめる
- 推論は、観測データからパラメータ推定や判断を作る
- 情報理論は、不確実性を交差エントロピー loss のような学習信号に変える

---

## 二、この章と AI の関係

| 章 | AI でのもっとも直接的な役割 |
|---|---|
| 確率の基礎 | 分類確率、ベイズ更新、スパム判定 |
| 確率分布 | 正規分布、ノイズのモデリング、ランダム初期化、データ統計 |
| 統計的推論 | MLE、MAP、A/B テスト、パラメータ推定 |
| 情報理論 | エントロピー、交差エントロピー、KL ダイバージェンス、分類損失関数 |
| 歴史の主線 | Bayes、MLE、EM、Shannon がそれぞれ何を解決してきたか |

AI の用語は、表面だけを暗記するとかなり散らばって見えます。ですが、この流れに戻して見ると、ずっと分かりやすくなります。

たとえば：

- モデルが `0.93` を出すのは、実は確率を返している
- `CrossEntropyLoss` は、実は情報理論から来ている
- `MLE` は、実は「どのパラメータがこのデータをいちばんよく説明するか」を考えている
- `ベイズ` は、実は「新しい証拠が得られたあと、判断をどう更新するか」を考えている

ここで、小さいけれど重要な違いも先に押さえておきましょう。

- **確率** は、多くの場合、既知のモデルから出発して「どんなデータや出来事が起こりそうか」を考える
- **統計** は、多くの場合、観測されたデータから出発して「どんなモデル、パラメータ、結論が妥当か」を考える

機械学習は、この 2 つの間を行き来します。学習時にはデータからパラメータを推定し、予測時にはそのパラメータを使って確率を出します。

## 三、なぜ AI は特にこの章を離れられないのか？

なぜなら、AI が扱う世界は、ほとんどいつも完全には確定していないからです。

たとえば：

- モデルは「80% の確率で猫です」としか言えない
- 検出結果には必ず誤検出や見逃しがある
- テキスト分類も 100% 確実とは限らない
- 学習時の loss や交差エントロピーは、どちらも確率の見方と直結している

なので、この章はまず次のように理解するとよいです。

> **AI システムに、不確実性に向き合うための言葉を補う章。**

## 初心者はこの章をどう学ぶべきか？

### 4.1 まずは直感を先に、記号は後回しにする

たとえば：

- 条件付き確率は、「ある出来事が起きたあとで、別の出来事が起こる確率」と考える
- 分布は、「ランダムな現象全体がどんな形をしているか」と考える
- MLE は、「どのパラメータが観測データをいちばんうまく説明するか」と考える
- エントロピーは、「どれくらい不確実か」と考える

### 4.2 どの概念も、必ず AI の場面に結びつける

ある概念を学び終えたときに、「AI の中で何に使うの？」と答えられないなら、まだ本当の意味では理解できていない可能性が高いです。

### 4.3 いきなり全部できることを目指さない

この章の内容は、後で何度も出てきます。

- 5 機械学習入門から実践では、確率と統計的推論に引き続き触れます
- 6 深層学習と Transformer 基礎以降では、勾配、損失、最適化に引き続き触れます
- 7 大規模モデルの原理、Prompt と微調整、8 LLM アプリケーション開発と RAG では、交差エントロピー、KL ダイバージェンス、ベイズ的な見方に引き続き触れます

だからこそ、この章で大事なのは、まず最初の「はっきりした理解」を作ることです。

## 五、この章の時間配分はどうするのがよいか？

初心者に合った目安のペースは次の通りです。

1. 確率の基礎：2〜4 時間  
   重点は「条件付き確率」と「ベイズ更新」をスムーズに理解すること。

2. 確率分布：2〜4 時間  
   重点は「1回の事象」を「全体の規則性」に広げて考えること。

3. 統計的推論：2〜4 時間  
   MLE、MAP、A/B テストがなぜ成り立つのかを理解することに重点を置く。

4. 情報理論：2〜4 時間  
   エントロピー、交差エントロピー、KL ダイバージェンスがなぜそのまま loss に入るのかを理解することに重点を置く。

5. 確率統計の歴史の主線：20〜40 分  
   Bayes、MLE、EM、Shannon を AI 技術の進化の流れに戻して、どの後続章につながるのかを押さえる。

この学び方のほうが、「たくさんの記号を先に暗記する」よりずっと安定します。

## 六、この章をつなぐ最小の実行可能な流れ

章全体を 1 つの小さなプロジェクトでつなぐなら、二値分類の例を使うと分かりやすいです。

1. 確率で「スパム」と「非スパム」を表す
2. 分布で、単語数や出現頻度がどのようにばらつくかを見る
3. 推論で、小さなデータセットからモデルのパラメータを推定する
4. 交差エントロピーで、モデルの間違いが減っているかを測る

後のレッスンでは、これらの部品を分けて学びます。この 4 ステップの流れを説明できれば、確率と統計が AI にとって「おまけ」ではない理由が見えてきます。

---

## 七、この章を学び終えたら、少なくとも何ができるべきか？

- モデルが確率を出したとき、それを「絶対的な結論」だと思わない
- 条件付き確率、ベイズ更新、分布がそれぞれ何を言っているか分かる
- MLE、MAP、仮説検定が、だいたい何を解決するためのものか分かる
- エントロピー、交差エントロピー、KL ダイバージェンスが、なぜ AI の学習に出てくるのか分かる

## この章を読んでいて不安になったら、まず何を押さえるべきか？

最初に押さえる価値が高いのは、次の4つです。

1. 確率は「不確実性」を表す
2. 条件付き確率は「新しい情報を得たあとで、判断がどう変わるか」を表す
3. 分布は「ランダムな現象全体がどんな形をしているか」を表す
4. 交差エントロピーと情報理論は、最終的に loss の中にそのまま入る

この4つが安定すれば、この章はかなり大きな収穫になります。

## 初心者と上級者はどう読むべきか

初心者がこの章を初めて学ぶときは、まず主線と最小限の動く例をつかみましょう。すべての細部を一度で理解する必要はありません。この章が何を解決するのか、入力と出力は何か、最小プロジェクトはどう動くのかが説明できれば、先へ進んで大丈夫です。

経験のある学習者は、この章を「抜け漏れの確認」と「実践練習」の場として使えます。境界条件、失敗例、評価方法、コードの再現性、そして前後の内容とのつながりに注目しましょう。読み終えたら、本章の内容を自分の作品の README や実験記録にまとめておくのがおすすめです。

## 本章の小プロジェクトの出口

この章を学び終えたら、最小の練習として、いちばん核となる概念またはツールを1つ選び、実行できて、スクリーンショットを残せて、README に書ける小さな成果物を作るのがおすすめです。複雑である必要はありませんが、入力が何か、処理が何か、出力が何かを説明できることが大切です。

## 合格基準

この章の最後には、本章が何を解決するのか、前後の学習ステップとどうつながるのかを自分の言葉で説明でき、本章の小プロジェクトの出口の最小版を完成できるようになっているはずです。
