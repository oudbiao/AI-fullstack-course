# 事前ガイド：この微積分と最適化の章では何を学ぶのか

![微積分と最適化の学習地図](/img/course/ch04-calculus-roadmap.png)

線形代数が「データと変換をどう表すか」を教えてくれるなら、この章が答えるのは次の問いです。

> **モデルは、いったいどうやって学習していくのか？**

中心となる流れは、実はとてもシンプルです。

- 導関数：ある量がどれくらい速く変化するかを教えてくれる
- 勾配：多変数関数がどの方向に最も速く変化するかを教えてくれる
- 勾配降下法：損失を少しずつ下げる方法を教えてくれる
- 逆伝播：ニューラルネットワークのたくさんのパラメータの勾配を、どう効率よく計算するかを教えてくれる

## 学習目標

- 「導関数 -> 勾配 -> 勾配降下法 -> 逆伝播」という章全体の地図をつかむ
- 微積分が AI の学習で実際にどう役立つのかを知る
- 初学者がまず押さえるべき本質的な直感をつかみ、最初から導出の細かい部分にハマらないようにする

## まず、とても大事な学習イメージを共有しよう

微積分は、初学者がいちばん怖く感じやすい章です。というのも、「導関数、勾配、連鎖律」と聞くだけで、こう思ってしまいがちだからです。

- これは大量の導出を丸暗記する必要があるのでは？

でも、この章の現実的な目標は、導出を完璧に覚えることではありません。まずは次のことを理解するのが大事です。

- 導関数はなぜ変化率を表すのか
- 勾配はなぜモデルに「どちらへ修正すればよいか」を教えてくれるのか
- 逆伝播はなぜ、その作業を効率よく行うだけなのか

つまり、この章で最も大切なのは、まず **「なぜ学習ができるのか」** をきちんと理解することです。

---

## 一、この章の4つの節はどういう関係？

![微積分と最適化の章の関係図](/img/course/ch04-calculus-training-flow.png)

この章は、次の一文にまとめることができます。

> **まず変化を測れるようになり、その変化を使ってパラメータを更新し、最後にその変化を深いネットワークの中で効率よく伝える。**

---

## 二、この章と AI の関係

| 章 | AI での最も直接的な役割 |
|---|---|
| 導関数 | 「あるパラメータを少し変えると、損失はどう変わるか」を理解する |
| 勾配 | 複数パラメータのモデルをどの方向に更新すべきかを理解する |
| 勾配降下法 | モデル学習が、なぜ何度も繰り返す最適化なのかを理解する |
| 逆伝播 | ニューラルネットワークの多くの層で、なぜ勾配を計算できるのかを理解する |

あとで PyTorch で次のようなコードを見たとき：

```python
loss.backward()
optimizer.step()
```

その裏では、この章の内容がまるごと動いています。

## 三、なぜ AI はこの章に強く依存するのか？

モデルの学習は、本質的には同じことを何度も繰り返しています。

1. 今の結果がどれくらい間違っているかを見る
2. パラメータをどちらに直すべきかを判断する
3. 少しだけ修正する
4. もっと良くなったかをもう一度確認する

そして、この一連の動きを支える数学の言葉が、次のものです。

- 導関数
- 勾配
- 勾配降下法
- 逆伝播

だから、この章は一言で言うとこうです。

> **「モデルはなぜ学習できるのか」を説明している章。**

---

## 初学者はこの章をどう学べばいい？

### 4.1 まずは「変化率」という核心の直感をつかむ

最初から難しい数式に引っ張られないようにしましょう。まず覚えるのはこれです。

- 導関数は変化率
- 勾配は多変数版の変化率
- 負の勾配方向は、たいてい最も下がりやすい方向

### 4.2 それぞれの節を必ず「モデル学習」と結びつける

導関数を学ぶときに「損失はどう変わるのか」を考えていないと、勾配を学ぶときに「パラメータをどう調整するのか」を考えていないと、これらがただの数学問題に見えてしまいます。

### 4.3 まずは図とコードを読めるようにして、あとで導出を補う

AI を独学する初学者なら、優先順位は次の順番がおすすめです。

1. 図のイメージを理解する
2. 最小限のコードを理解する
3. 公式の意味を理解する
4. 最後に、より厳密な導出を見る

### 4.4 初学者に合った学び方の順番

各節は、次の順で読むと理解しやすいです。

1. まず生活のたとえを見る
2. 次に図を見る
3. そのあと最小コードを動かす
4. 最後にもう一度公式を見る

この順番のほうが、最初から連鎖律や導出に飛び込むより、ずっと安定します。

## 五、この章にどれくらい時間を使うべき？

初学者向けの目安は、だいたい次の通りです。

1. 導関数：2〜3時間  
   まずは「変化率」という言葉を本当の直感に変えましょう。

2. 偏導関数と勾配：2〜4時間  
   「1つの変数がどう変わるか」から「たくさんの変数がどう一緒に変わるか」へ広げます。

3. 勾配降下法：2〜4時間  
   「なぜモデルは何度も繰り返して学習するのか」を理解します。

4. 連鎖律と逆伝播：3〜5時間  
   ここは特に難しく感じやすいので、少しまとまった時間を確保するのがおすすめです。

もしここでの理解が遅く感じても、それはあなたの能力が低いからではありません。もともとこの章が抽象的だからです。

---

## 六、この章を終えたら、少なくとも何ができるべき？

- 導関数を見て、それが変化率を表すと分かる
- 勾配を見て、多変数関数が最も増える方向を表すと分かる
- 勾配降下法を見て、モデルが損失の小さい方向へ少しずつ進んでいると分かる
- 逆伝播を見て、それが本質的には連鎖律の応用だと分かる

## もしこの章を読んで「まだ抽象的すぎる」と感じたら、まず何を押さえるべき？

まず押さえる価値が高いのは、次の4つです。

1. 導関数 = ある量がどれくらい速く変わるか
2. 勾配 = 複数の量が一緒に変わるとき、どちらが最も速く変わるか
3. 勾配降下法 = 損失が小さくなる方向へ少しずつ進むこと
4. 逆伝播 = 多くの層の勾配を効率よく計算すること

この4つがしっかりしていれば、第6章で `loss.backward()` を見ても、もうブラックボックスだけには感じません。

## 初学者と上級学習者では、どう読めばいい？

初めてこの章を学ぶ人は、まず主線と最小の実行例をつかみましょう。すべての細部を一度に理解する必要はありません。この章が何を解決するのか、入出力は何か、最小プロジェクトをどう動かすのかが説明できれば、先に進んで大丈夫です。

経験のある学習者なら、この章を弱点補強と実践練習の場として使えます。境界条件、失敗例、評価方法、コードの再現性、そして前後の章とのつながりに注目しましょう。読み終わったら、本章の内容を自分の作品の README や実験記録にまとめておくとよいです。
