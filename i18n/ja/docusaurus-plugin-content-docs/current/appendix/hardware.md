# ハードウェアとクラウドリソースのガイド

![ハードウェアとクラウドリソースの意思決定ツリー](/img/course/appendix-hardware-cloud-decision-tree.png)

![ローカル・クラウド・API の3ルートにおけるコスト比較図](/img/course/appendix-hardware-local-cloud-api-cost-map.png)

:::tip 画像の見方
ハードウェア選びは、まずやることから考えます。基礎学習、Demo の実行、モデルの学習、アプリ開発では、必要なリソースがまったく違います。図を見るときは、ローカルPC、クラウドGPU、API の3つのルートを並べて比較してください。最初から「必ずグラボを買わないと」と思い込む必要はありません。
:::

このページの目的は、「高性能構成にすること」ではありません。学習の段階ごとに、機材を買うべきか、クラウドを借りたほうが得か、どの構成が本当に学習体験に影響するのかを判断できるようにすることです。

## まず結論

- 1～5 の学習ステーションでは、基本的に GPU は不要です。
- 6 深層学習と Transformer の基礎、10 コンピュータビジョン、11 自然言語処理は、まずクラウドGPUを使えば十分で、急いでグラボを買う必要はありません。
- 8 LLM アプリ開発と RAG、9 AI Agent とエージェントシステムは、多くのプロジェクトが実際にはエンジニアリング力に左右され、本格的なローカルGPUが必須とは限りません。
- 初心者が優先してお金をかける順番は、一般的に「メモリ > SSD > 安定した環境 > GPU」です。

## 1. ステージごとのハードウェア要件

| 学習ステーション | よくある作業 | ローカルの最低目安 | より快適な構成 | クラウド代替 |
|---|---|---|---|---|
| 1 開発者ツールの基礎、2 Python プログラミング基礎、3 データ分析と可視化 | Python、データクリーニング、グラフ作成、Notebook | 8GB メモリ / 256GB SSD | 16GB / 512GB SSD | 不要 |
| 4 AI 数学の最小必須基礎、5 機械学習入門から実践まで | 数学実験、従来型機械学習 | 8GB メモリ | 16GB メモリ | 不要 |
| 6 深層学習と Transformer の基礎 | PyTorch、CNN、RNN、Transformer の基礎 | 16GB メモリ | 16GB以上 / 512GB SSD | Colab / クラウドGPU |
| 7 大規模言語モデルの原理、Prompt とファインチューニング | LLM の原理、Prompt 実験、ファインチューニングの概念実験 | 16GB メモリ | 32GB メモリが望ましい | クラウドGPU のほうが現実的 |
| 8 LLM アプリ開発と RAG、9 AI Agent とエージェントシステム | RAG、Agent、アプリ開発 | 16GB メモリ | 16GB以上 / 安定したネットワーク | CPU + API で大半のプロジェクトは可能 |
| 10 コンピュータビジョン、11 自然言語処理 | CV、NLP の実験 | 16GB メモリ | 16GB以上 / 外部モニターがあると快適 | Colab / AutoDL / クラウドサーバー |
| 12 AIGC とマルチモーダル | マルチモーダルの体験、プロジェクトの組み合わせ | 16GB メモリ | 32GB メモリだとより快適 | 画像・動画生成はクラウド向き |

## 2. 3段階のデバイス構成

### 2.1 ちょうどよい入門構成

こんな人向け：

- これから体系的に学び始める
- 予算が限られている
- まずは教材のサンプルや小〜中規模プロジェクトを動かしたい

おすすめ：

- 16GB メモリ
- 512GB SSD
- 一般的な CPU のノートPCまたはデスクトップ
- GPU が必要なときは一時的にクラウドを使う

### 2.2 快適構成

こんな人向け：

- ローカル実験を頻繁に回す
- ブラウザ、IDE、Notebook、ターミナルを同時に開く
- 長時間の学習でも安定した体験がほしい

おすすめ：

- 32GB メモリ
- 1TB SSD
- 多コア CPU
- 必要なら中級クラスの独立GPUを付ける、または引き続きクラウドGPUを中心に使う

### 2.3 しっかり実験構成

こんな人向け：

- 明確に学習・訓練実験をやる予定がある
- やや大きめの画像系や深層学習タスクをよく回す
- ローカルで素早く試行錯誤したい

おすすめ：

- 32GB～64GB メモリ
- 1TB SSD以上
- 比較的新しいGPU
- ただし先に考えるべきです：それは本当に学習に必要か、それとも「機材不安」か

## 3. 初心者にとって本当に価値が高い構成は、GPUとは限らない

多くの人は最初、GPU の価値を高く見積もりすぎて、メモリや SSD の重要性を見落としがちです。

体験に大きく効くのは、たいてい次の要素です。

1. メモリが足りるか  
   ブラウザ、VS Code、Jupyter、ターミナルを同時に開くと、16GB と 8GB の差はかなり大きいです。

2. SSD が十分か  
   データセット、モデルのキャッシュ、仮想環境はかなり容量を使います。SSD が小さすぎると何度も整理する必要があり、学習の流れが止まりやすくなります。

3. 環境が安定しているか  
   同じ機械でも環境が乱れていると、たくさんの時間をトラブルシューティングに使うことになります。

4. 画面や入力機器が使いやすいか  
   コードや資料を長時間見るので、外部モニター、キーボード、マウスの効果はかなり実感しやすいです。

## 4. クラウドリソースはどう選ぶと合理的か

### 4.1 無料または低コストで試す

向いている人：

- 講座のサンプルを動かしたい
- 小さな実験をしたい
- GPU 訓練の流れに慣れたい

特徴：

- 始めやすい
- コストが低い
- 長期的で安定した訓練には向かない

### 4.2 時間単位で借りるクラウドGPU

向いている人：

- 訓練時間が長い
- より安定した VRAM と計算資源が必要
- コストを抑えたいが、一括でハードを買いたくない

おすすめ：

- まず小さなデータで流れを通してからクラウドに乗せる
- 起動前に、コード、データ、実験計画を準備しておく
- 訓練が終わったら、すぐ結果を保存してシャットダウンし、無駄な課金を止める

### 4.3 API 優先のアプリ開発ルート

向いている人：

- RAG、Agent、スマートアシスタントを作りたい
- 自分で基盤モデルを学習することより、プロダクトやエンジニアリングを重視したい

このルートでは、ローカルGPUは基本的に不要で、重要なのは次の点です。

- インターフェースの設計
- ログと監視
- 検索とツール呼び出し
- ユーザー体験とコスト管理

## 5. ローカルGPUを買う価値があるのはいつか？

少なくとも次の2つ以上に当てはまるなら、ローカルGPUを買う意味は大きくなります。

- 長期的に学習・訓練実験を続けることがほぼ決まっている
- クラウドの待ち時間や利用時間制限でよく作業が止まる
- ローカルでの試行速度に明確なニーズがある
- どんなタスクをどれくらいの VRAM で回すか、すでに把握している

まだ「いつか使うかも」くらいなら、急いで買う必要はありません。

## 6. よくある誤解

### 6.1 誤解1：GPU がないと AI は学べない

これは最もよくある誤解です。前半の多くの内容は GPU と関係ありませんし、後半のアプリ開発も、API とクラウドリソースで十分進められるものがたくさんあります。

### 6.2 誤解2：まず機材を全部そろえてから始めるべき

学習で一番もったいないのは、機材が普通なことではなく、なかなか始めないことです。まずは今ある環境で進め、実際のボトルネックに当たってからアップグレードするほうが、たいてい合理的です。

### 6.3 誤解3：GPU のために全体の安定性を犠牲にする

高性能GPUを買っても、メモリ、SSD、冷却、システム安定性が追いついていなければ、実際の体験はあまり良くないかもしれません。

## 7. 現実的な購入アドバイス

これから始めるなら、いちばん堅実なのはたいてい次の流れです。

- まず今のPCで 1～5 の学習ステーションを終える
- 6 深層学習と Transformer の基礎、10 コンピュータビジョン、11 自然言語処理は、必要に応じてクラウドGPUを借りる
- 8 LLM アプリ開発と RAG、9 AI Agent とエージェントシステムは、まずアプリ開発を中心に進める
- 自分が長期的に訓練を続けると明確になってから、本当にローカルハードを更新するか決める

これは、最初から予算を機材に全部つぎ込むよりも合理的で、長く学び続けるのにも向いています。
