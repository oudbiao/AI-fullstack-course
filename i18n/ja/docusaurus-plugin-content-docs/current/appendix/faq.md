![FAQ 不安のリセットと行動の分岐図](/img/course/appendix-faq-confidence-reset-map-ja.png)

:::tip 画像の読み方のヒント
FAQ の目的は、あなたに「唯一の正解」を与えることではなく、あいまいな不安を実行できる問題に分解することです。図を見るときは、まず自分が数学、環境、時間、プロジェクト、論文、就職のどこで止まっているかを確認し、対応する章に戻って必要最小限の内容を補いましょう。

このページでは概念の説明はせず、初心者がよくつまずく実践的な疑問にまとめて答えます。迷ったときにまず開く「付録」として使ってください。

## 数学が苦手でも AI は学べますか？

はい。むしろ多くの人は、数学を学び切ってから始めるのではなく、学びながら補っています。

初心者に向いている順番は次の通りです。

1. まず Python、データ処理、グラフ作成をスムーズにできるようにする。
2. そのあとで、線形代数や確率統計がコードのどこで使われているかを見る。
3. 最後に、公式や導出、より抽象的な理解を補う。

最初から数式に押しつぶされそうになるなら、たいていは向いていないのではなく、順番が違うだけです。

より安定したやり方は次の通りです。

- 行列を学ぶときは、同時に `NumPy` の配列演算を見る。
- 勾配降下法を学ぶときは、同時に最小の学習ループを見る。
- 確率を学ぶときは、分類タスクの損失関数や予測確率も一緒に見る。

目標は、先に「証明できる」ようになることではなく、まず「この数学の道具がモデルの中で何を担当しているか」を知ることです。

## GPU を買う必要はありますか？

多くの初心者は、最初は必要ありません。

| 学習ステージ | 主な内容 | GPU が必須か | 現実的なおすすめ |
|---|---|---|---|
| 1 開発者ツールの基礎、2 Python プログラミングの基礎、3 データ分析と可視化 | 開発環境、Python、データ分析 | 不要 | 個人のPCで十分 |
| 4 AI 数学の最小必要基礎、5 機械学習入門から実践まで | 数学実験、従来の機械学習 | 不要 | 個人のPCで十分 |
| 6 深層学習と Transformer の基礎 | PyTorch、CNN、RNN、Transformer の基礎 | 任意 | まずは Colab やクラウド GPU を使う |
| 7 大規模言語モデルの原理、Prompt と微調整 | LLM の原理、Prompt、微調整の理解 | 実験内容による | まず流れを理解してから、必要なら借りる |
| 8 LLM アプリ開発と RAG、9 AI Agent とエージェントシステム | RAG、Agent、アプリケーション開発 | 通常は不要 | CPU + API で多くのプロジェクトが作れる |
| 10 画像認識、11 自然言語処理、12 AIGC とマルチモーダル | 画像、テキスト、マルチモーダル、生成系プロジェクト | プロジェクトによる | 学習や生成タスクはクラウド GPU が向いている |

初めて体系的に学ぶなら、優先順位は通常次の通りです。

1. 安定したPC環境
2. 十分なメモリとストレージ
3. 長く続けられる学習リズム
4. 最後に GPU やグラフィックボード

## すべてのステージを学ぶ必要がありますか？

必ずしもそうではありませんが、主線はできるだけ途切れさせないのがおすすめです。

より実用的には次のように考えるとよいです。

- 1～6 学習ステージ：コアの内容。ほとんどの人におすすめ
- 7～9 学習ステージ：大規模言語モデルのアプリ開発の主線。原理、RAG から Agent システムまでつながる
- 10～12 学習ステージ：方向の拡張と卒業プロジェクト。興味に応じて CV、NLP、AIGC マルチモーダルを選べる

目標が「できるだけ早く AI アプリを作りたい」なら、次の流れがよいです。

`1 開発者ツールの基礎 -> 2 Python プログラミングの基礎 -> 3 データ分析と可視化 -> 6 深層学習と Transformer の基礎 -> 8 LLM アプリ開発と RAG -> 9 AI Agent とエージェントシステム`

目標が「モデルの原理をしっかり理解したい」なら、次の流れがよいです。

`1～6 コア基礎 -> 7 大規模言語モデルの原理、Prompt と微調整 -> 8 LLM アプリ開発と RAG -> 9 AI Agent とエージェントシステム`

## 1週間でどれくらい学ぶのがちょうどいいですか？

「1日何時間か」よりも、「12週間以上、安定して続けられるか」のほうが大事です。

| 週あたりの時間 | リズムのおすすめ | 向いている人 |
|---|---|---|
| 4～6 時間 | ゆっくりだが長く続けやすい | 会社員、授業が忙しい人 |
| 7～10 時間 | かなり理想的 | ほとんどの独学者 |
| 12～18 時間 | 進みは速いが疲れやすい | 明確なプロジェクト目標がある人 |

初心者によくある失敗は、最初の1週間で飛ばしすぎて、3週目で止まってしまうことです。より安定したやり方は次の通りです。

- 平日は毎日1時間
- 週末に1回の振り返りと1回のコード練習
- 2～3週間ごとに小さなプロジェクトか実験のまとめを1つ作る

## いつからプロジェクトを始めるのがよいですか？

早ければ早いほどよいですが、プロジェクトの難易度はその時点の学習段階に合わせる必要があります。

ちょうどよい流れは次の通りです。

- Python 基礎を学んだ後：小さなスクリプトやツールを作る
- データ分析を学んだ後：データクリーニングと可視化のプロジェクトを作る
- 機械学習を学んだ後：分類または回帰の完全なプロジェクトを作る
- 深層学習を学んだ後：学習の一連の流れを完結させるプロジェクトを作る
- RAG / Agent まで学んだ後：本当に見せられるアプリケーションプロジェクトを作る

「プロジェクト」を、必ず大きなシステムを作ることだと考えないでください。初心者にとって価値が高いのは次のようなものです。

- 目的がはっきりしている
- 入力と出力が明確
- 最小限の一連の流れを動かせる
- なぜそうしたのかを自分の言葉で説明できる

## コードがいつも動かないときはどうすればいいですか？

まず自分を疑いすぎないでください。多くの問題は、環境、パス、依存関係のバージョン、入力データにあります。

次の順番で確認するとよいです。

1. エラーメッセージの最後の10行を見る。最初の1行だけ見ない。
2. 正しい Python 環境を使っているか確認する。
3. 依存関係が現在の環境に入っているか確認する。
4. ファイルパス、データパス、モデルパスが正しいか確認する。
5. 問題を最小再現可能な例まで小さくする。

まず覚えておくとよい確認コマンドは次の通りです。

```bash
python --version
which python
pip --version
pip list
pwd
ls
```

まだトラブルシュートの習慣がないなら、付録の [A.8 学習のつまずき救援](./troubleshooting.md) もあわせて見てください。

## 論文が理解できないと AI は学べませんか？

いいえ。

初心者にとって論文は「理解を補強し、視野を広げるもの」であって、学習の入口ではありません。

おすすめの順番は次の通りです。

1. まずチュートリアルを見て、タスク、入力と出力、評価方法を理解する。
2. 次に実装や最小コードを見る。
3. 最後に、疑問を持ちながら論文を読む。

最初から無理に論文を読むと、次の2つが起こりやすいです。

- 用語だけ覚えて、タスクの感覚が身につかない
- 自分は何もできないと感じてしまい、だんだん手を動かさなくなる

## どのくらい学んだら就職活動や案件受注を始められますか？

現実的な基準は、「すべての知識を学び終えたか」ではなく、次の4つができるかどうかです。

1. 自力で環境を構築し、中程度の難しさのプロジェクトを動かせる
2. プロジェクトのデータフロー、モデル選択、評価方法、失敗例を説明できる
3. よくある Python / ML / LLM の開発コードを読める
4. 依頼を実行可能な小さなタスクに分解し、少しずつ完成させられる

すでに自分で説明できるプロジェクトが2～3個あるなら、「全部学び終わる」のを待たずに、履歴書や面接の準備を始めて大丈夫です。

## 自分の学習が遅い気がします。どうすればいいですか？

これは、ほとんどすべての独学者が経験する段階です。

本当に比べるべきなのは「他人が1日にどれだけ進んだか」ではなく、次の点です。

- 今月の自分は先月よりコードを理解できるようになっているか
- サンプルをコピーするだけでなく、自分で修正できるようになっているか
- 概念を他人に説明できるようになってきているか

この3つの答えがだんだん「はい」に近づいていれば、ちゃんと成長しています。

## つまずいたとき、最初に補うと効果が高い3つは何ですか？

今すでに少し混乱しているなら、優先して補うべきなのは次の3つです。

1. Python の基礎とデバッグ力
2. データ処理と可視化の力
3. 完全なプロジェクトの最小限の一連の流れを意識すること

多くの人は「もっと高度なモデル」が足りないと思いがちですが、本当に足りていないのは次のような力です。

- 入力と出力を理解する
- エラーを調べる
- 問題を分解する
- 結果を検証する

この4つを身につけると、その後の学習はずっとスムーズになります。
