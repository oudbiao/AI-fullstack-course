# コースのビジュアル強化計画

![コース画像アセット計画カンバン](/img/course/appendix-visual-enhancement-kanban.png)

![コース画像の不足から生成・公開までのフローチャート](/img/course/appendix-image-production-pipeline-map.png)

:::tip 画像の見方のヒント
画像は多ければ多いほどよいわけではなく、理解コストを下げることが大切です。読むときは「不足を見つける -> 図の意味を決める -> 画像を生成する -> ページに挿入する -> manifest/進捗を記録する -> ビルドで確認する」という流れで、一連のサイクルとして進めると、画像アセットが増えすぎるのを防げます。
:::

いまのコースには、各段階のトップページ用のビジュアルもあり、Mermaid のフローチャートもたくさんあります。次にやるべきことは、すべてのページに画像を入れることではなく、理解コストを最も下げられる場所に画像を置くことです。

:::info 章番号のルール
コースのソースディレクトリは、Web 表示の章番号とそろえています。`ch01-tools` は第 1 章、`ch05-machine-learning` は第 5 章に対応します。サイドバーの主線 1〜4 は学習グループであり、ディレクトリ階層ではありません。
:::

シンプルな原則は次のとおりです。

| コンテンツの種類 | より適したビジュアル形式 | 説明 |
|---|---|---|
| 抽象的な概念 | たとえのイラスト、構造図 | 初心者がまずイメージをつかみやすくなる |
| 複数ステップの流れ | Mermaid またはフローチャート | 手順の順番がわかる |
| 数学とデータ | コードで生成したグラフ | AI 生成画像より正確 |
| モデル構造 | モジュール構成図 | 入力、出力、中間層の関係がわかる |
| 実践プロジェクト | アーキテクチャ図、画面モック、結果サンプル画像 | 最終的に何を作るのかがわかる |
| 歴史的背景 | タイムライン図、人と論文のカード | アルゴリズムを時代の中に置いて理解できる |

## 優先順位のルール

| 優先度 | 先に作るとよい図 | 理由 |
|---|---|---|
| P0 | 各章トップページのメインビジュアル | すでに完成済み。章の雰囲気を作る役割がある |
| P1 | 各章の最初の本編の概念図 | 初心者の第一印象に最も影響する |
| P1 | プロジェクトページのシステム構成図と結果サンプル図 | もっとも取り組みやすくなる |
| P2 | 数学、学習、評価系のコード可視化 | 抽象的な過程を観察できる結果に変えられる |
| P2 | 歴史年表と論文ストーリー図 | 興味と記憶のきっかけになる |
| P3 | 装飾用イラスト | ページが味気なく、読むのが疲れるときだけ追加する |

## 生成戦略

まずは P1 の教学図から作り、装飾用の図は後回しにします。コースの図は、見た目をにぎやかにするためではなく、理解を助けるためにあります。

次に、プロジェクトページのスクリーンショットや画面モックを補います。プロジェクトページでは、初心者に「最終的な成果物がどんな見た目か」を伝えることがとても重要です。特に RAG、Agent、教材生成アシスタント、マルチモーダルのクリエイティブプラットフォームは優先度が高いです。

そのあとで、歴史上の人物、論文、アルゴリズムのストーリー図を補います。こうした図は歴史ページや章の冒頭に置くと、なぜその技術が生まれたのかを覚えやすくなります。

付録のバッチは、まず `ai-milestones`、`troubleshooting`、`hardware`、`job-prep` の 4 枚の P1 図を補い、そのあとでリソース、FAQ、継続学習、速覧系の図を追加するのがおすすめです。そうすれば参照効率が上がり、付録が画像の詰め込み場になりません。

## いまの画像生成スクリプトの使い方の提案

現在のスクリプト `scripts/generate_course_images.py` で、すでにコース画像アセットを管理できます。今後は、上の高優先度画像を `IMAGE_JOBS` に追加し、次の方法で生成できます。

```bash
pip install -r requirements-course-ai.txt
export OPENAI_API_KEY="your_api_key_here"
export OPENAI_BASE_URL="https://cliproxy.airoads.org/v1"
export OPENAI_IMAGE_MODEL="gpt-image-2"
npm run images:dry-run
npm run images:generate
```

数学曲線、学習曲線、検出ボックス、セグメンテーション mask については、正確さが必要なので、まずコードで生成するのがおすすめです。段階のメインビジュアル、プロジェクトの作業台、歴史ストーリー図については、画像生成モデルで作るのが向いています。
