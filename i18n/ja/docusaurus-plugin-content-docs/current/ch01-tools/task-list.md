# 段階学習タスク一覧：開発者ツール基礎

この段階の目標は、すべてのコマンドを覚えることではありません。後続の AI プロジェクト学習に必要な、基本的な開発ワークフローを身につけることです。学習後には、プロジェクトを作成し、ファイルを管理し、依存関係をインストールし、Git でバージョンを保存し、実行結果を記録できるようになっているはずです。

![開発者ツール段階タスクチェーン](/img/course/ch01-task-list-workflow.png)

## この段階で必ず完了するタスク

| タスク | 生成物 | 通過基準 |
| --- | --- | --- |
| ターミナルとよく使うコマンドを設定する | コマンド練習の記録 | コマンドでファイルを作成、移動、確認、削除できる |
| Python 開発環境を設定する | 実行可能な Python プロジェクトディレクトリ | `python --version` を実行し、依存関係をインストールしてスクリプトを実行できる |
| VS Code または同等のエディタを設定する | プロジェクトのスクリーンショットまたは設定説明 | プロジェクトを開き、ファイルを実行し、ターミナル出力を確認できる |
| Git の基本フローを身につける | 少なくとも 3 回の commit 記録 | add、commit、status、log の役割を説明できる |
| リモートリポジトリに接続する | リモートリポジトリのアドレスまたはスクリーンショット | push でき、リモート側でファイルの存在を確認できる |

## 推奨学習順序

まずターミナルとファイル操作を学び、その後に Python 環境を設定します。次にエディタを学び、最後に Git を学びましょう。最初から複雑なブランチモデルにこだわらず、まずは「ローカルでファイルを修正し、コミットし、プッシュする」という主な流れを通しましょう。

環境問題に出会ったら、エラーメッセージの原文、オペレーティングシステム、Python バージョン、実行ディレクトリ、完全なコマンドを優先して記録してください。今後の RAG、Agent、デプロイ学習でも、多くの問題の本質は環境、パス、依存関係にあります。

## AI 学習アシスタントプロジェクトとの関係

この段階は、AI 学習アシスタントの v0.1 プロジェクト骨組みに対応します。プロジェクトディレクトリを作成し、README を用意し、Git リポジトリを作り、最もシンプルなコマンドラインの入口を 1 つ書く必要があります。このバージョンには AI 機能は不要で、プロジェクトが継続的に改良できるエンジニアリング基盤を持っていれば十分です。

推奨ディレクトリ構成：

```text
ai-learning-assistant/
  README.md
  src/
    main.py
  notes/
    learning-log.md
  requirements.txt
```

`main.py` は最初は 1 行の歓迎メッセージを出力するだけで構いません。大切なのは機能ではなく、固定のコマンドで実行でき、その結果を README に書き込めるかどうかです。

## よくあるつまずき

最もよくある問題は、コマンドを実行するディレクトリが違う、Python バージョンが混乱している、依存関係が間違った環境にインストールされている、Git のユーザー名が設定されていない、リモートリポジトリの権限が失敗する、などです。このような問題を解決するときは、最後の 1 行のエラーだけをコピーするのではなく、完全なコマンドと完全な出力を残してください。


## やさしめ版 / 標準版 / チャレンジ版タスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| やさしめ版 | Python ファイルを 1 つ実行し、Git commit を 1 回行う | 初めて学ぶ人、時間が少ない人、または入門したばかりの学習者 |
| 標準版 | README、仮想環境の説明、コマンド記録を補う | この段階を作品集に入れたい学習者 |
| チャレンジ版 | あえてパスやコマンドのエラーを作り、トラブルシュート記録を書く | 基礎があり、より強いプロジェクト証拠を作りたい学習者 |

## この段階のバッジと Boss 戦

| 種類 | 内容 |
|---|---|
| Boss 戦 | ワークベンチの門番 |
| 解放できるバッジ | ターミナル生存者、Git アーカイブ職人 |
| 最小通過スローガン | まず動かす、次に説明する、次に失敗を記録する |
| 証拠保存のおすすめ | スクリーンショット、ログ、失敗サンプル、評価表を `reports/`、`evals/`、または `logs/` に保存する |

やさしめ版を完了すれば先に進めます。標準版を完了したら、作品集に書くのがおすすめです。チャレンジ版は、余力があるときだけ取り組んでください。

## 段階作品集の提出物

この段階の成果を作品集に残したい場合は、少なくとも以下のファイルまたは同等の材料を残すことをおすすめします。

| 提出物 | 説明 |
| --- | --- |
| `README.md` | プロジェクトの目的、環境要件、実行コマンド、サンプル出力 |
| `requirements.txt` | 現在のプロジェクト依存関係。最初は空でも残してよい |
| `notes/learning-log.md` | 毎日何をしたか、どんな環境問題があったかを記録する |
| `screenshots/` | ターミナル実行、エディタ、Git commit のスクリーンショットを保存する |
| Git commit 記録 | 少なくとも 3 回の小さな commit で、プロジェクトの反復過程が分かること |

これらの材料は、あなたが教材を見るだけでなく、AI プロジェクトを継続開発するための基本的なエンジニアリング習慣をすでに身につけていることを示してくれます。

## 段階通過の質問

学習後には、次の質問に答えられるはずです。今のターミナルはどのディレクトリにあるか、Python スクリプトはどの interpreter で実行されたか、プロジェクトの依存関係の記録はどこにあるか、Git に未コミットの変更は何があるか、なぜ小さな機能を 1 つ終えるたびに 1 回 commit すべきなのか。

## 完了状態 Checklist

- [ ] ターミナルでプロジェクトディレクトリに入り、今のコマンドがどこで実行されているか説明できる。
- [ ] Python ファイルを作成し、実行し、修正できる。
- [ ] Git で status を確認し、変更を commit し、commit 記録の意味を理解できる。
- [ ] AI 学習アシスタントのプロジェクト骨組み、README、実行スクリーンショットを作成済みである。
- [ ] 少なくとも 1 つの環境またはパスの問題と、その調査・解決の流れを記録した。
