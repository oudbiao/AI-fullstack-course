# 学習前ガイド：この記憶の章で何を学ぶのか

この章で解決するのは、Agent が「今この1回だけ答えられる」だけでなく、必要なときに過去の情報を覚え、検索し、使えるようになるにはどうすればよいか、ということです。

記憶の目的は、Agent を人間らしく見せることではありません。タスクをうまく進めるためです。記憶がシステムの目標達成、やり取りの重複削減、文脈の一貫性維持、経験の再利用に役立たないなら、それはむしろ複雑さとリスクを増やすだけかもしれません。

## この章のコース全体での位置づけ

これまでに、Agent の基礎、推論と計画、ツール呼び出しを学んできました。記憶の章では、課題が1回で終わるのではなく、複数ターン、複数ファイル、複数時間にまたがって進むとき、Agent が文脈と経験をどう管理するかを学びます。

ツールは Agent に行動させ、記憶は Agent に継続させます。記憶がなければ、Agent は毎回初めてそのタスクに出会ったかのように振る舞います。逆に、記憶設計が悪いと、誤って覚えたり、覚えすぎたり、情報が混ざったり、古い情報を事実として扱ってしまうことがあります。

## この章で本当に解決する問題

この章では、次の5つの問いに答えます。短期記憶と長期記憶の違いは何か。どの情報を覚えるべきで、どの情報を覚えるべきでないか。エピソード記憶と手続き記憶はそれぞれ何のために使うのか。記憶はどう書き込み、検索し、更新し、忘れるのか。記憶の汚染、プライバシーリスク、古い情報による誤導をどう防ぐのか。

初心者がいちばん誤解しやすいのは、「たくさん覚えているほど Agent は賢い」という考えです。実際には、記憶システムの質は、情報の選別、構造化、検索、更新の仕組みによって決まります。関係ない情報が多いほど、かえって意思決定の邪魔になりやすいのです。

## 初心者におすすめの学習順序

まずは記憶の全体像を学び、コンテキストウィンドウ、短期記憶、長期記憶、外部ストレージの違いを整理しましょう。次に短期記憶を学び、複数ターンの対話やタスク状態がどう保持されるかを理解します。その後、長期記憶を学び、好み、プロジェクトの背景、安定した事実、再利用できる経験がどう保存されるかを見ていきます。さらに、エピソード記憶と手続き記憶を見て、「何が起きたか」と「今後どうするか」は別の種類の情報だと理解します。最後に、記憶エンジニアリングの実践として、書き込みルール、検索戦略、更新の仕組み、安全の境界を学びましょう。

## この章で押さえるべき主線

この章の主線は、次のようにまとめられます。記憶システムは単なる保存庫ではなく、タスクに向けた文脈管理の仕組みである、ということです。

![Agent 記憶の書き込み・検索の閉ループ図](/img/course/ch09-memory-write-retrieve-loop-ja.png)

この流れがわかると、記憶で大事なのは「保存すること」ではなく、「いつ保存するか、何として保存するか、いつ取り出すか、取り出した内容は信頼できるか、期限が切れたらどう扱うか」だと理解できます。

## この章と後続章の関係

記憶は、MCP、多 Agent、評価、安全、デプロイの重要な土台です。MCP によって記憶が外部システムにつながることがありますし、多 Agent では役割ごとに記憶を共有したり分離したりします。評価と安全では、記憶がプライバシーや誤情報の拡散リスクを生まないかを確認します。デプロイ段階では、権限、監査、データ保持、ユーザーによる削除制御まで考えなければなりません。

この章が不安定なままだと、後でよく起きる問題は次のようになります。Agent が毎回同じことを聞き直す。長期記憶に古い内容や無関係な内容が残る。システムがユーザーの好みと事実を混同する。複数の Agent が共有してはいけない文脈を共有してしまう。検索結果を検証せず、そのまま真実として使ってしまう。

## 初心者と上級学習者の読み方

初心者がこの章を初めて学ぶときは、まず主線と最小実行例をつかみましょう。すべての細かい部分を一度に理解する必要はありません。この章が何を解決するのか、入出力は何か、最小プロジェクトをどう動かすかが説明できれば、先に進めます。

経験のある学習者は、この章を抜け漏れチェックと実装練習として読むとよいでしょう。境界条件、失敗例、評価方法、コードの再現性、そして前後の章とのつながりに注目してください。読み終えたら、本章の内容を自分の作品の README や実験記録にまとめておくとよいです。

## 学習時間と難易度の目安

| 学習方法 | 目安時間 | 目標 |
|---|---|---|
| ざっと読む | 20～30 分 | この章が何を解決するのかを理解し、後でどこで使うかを知る |
| 最小到達 | 1～2 時間 | 最小例を動かし、本章の小プロジェクトの出口まで到達する |
| 深く練習 | 半日～1 日 | エラー分析、比較実験、プロジェクト README の記録を補う |

## 本章の自己チェック問題

| 自己チェック問題 | 合格基準 |
|---|---|
| この章は何を解決するのか？ | 一言で、この章がコース全体のどこにあるか説明できる |
| 最小の入力と出力は何か？ | 例に何が必要で、どんな結果が出るか説明できる |
| よくある失敗点はどこか？ | エラー、精度低下、理解のずれの原因を少なくとも1つ挙げられる |
| 学んだ後に何を残せるか？ | 本章の成果を README、実験記録、ポートフォリオに書ける |

## 本章の小プロジェクトの出口

この章を学び終えたら、「記憶を持つ学習プラン支援アシスタント」を作るのがおすすめです。これは、ユーザーの学習目標、現在の段階、好みの学習ペース、完了したプロジェクトを記憶できます。次にユーザーが質問したとき、システムはこれらの情報を検索して、より適切な提案を返せます。

このプロジェクトで大事なのは、記憶ルールの設計です。どの内容を長期の好みとして保存するか、どれが今のタスク状態にすぎないか、どれをユーザーに確認すべきか、どれを期限切れにするか、または削除すべきかを決めます。

## 合格基準

この章の終わりには、短期記憶、長期記憶、エピソード記憶、手続き記憶の違いを説明でき、簡単な記憶の書き込みと検索の流れを設計でき、記憶の汚染、古い情報、プライバシーリスクがなぜ重要かを説明できるようになっているはずです。

複数ターンのタスクで、Agent が過去の情報を正しく使いながら、関係ない情報や古い情報を事実として扱わないようにできれば、あなたはもう記憶システムの入門レベルを身につけたと言えます。
