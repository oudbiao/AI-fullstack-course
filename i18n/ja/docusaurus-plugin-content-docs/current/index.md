# AI フルスタックエンジニアリングコース

![ここから始める4ステップ図](/img/course/intro-start-here-four-step-map-ja.webp)

この講座では、他のエンジニアが確認できる AI アプリケーションを作る力を育てます。動くコード、再現可能な環境、データメモ、モデルの証拠、Prompt/RAG/Agent のトレース、明確な限界を残します。

まず小さく始めます。AI例を1つ体験し、最小環境を準備し、能力マップを見て、第1章へ進みます。キャリア転換の目標は、AI の流行語を暗記することではありません。モデルの振る舞いを使えるプロダクトワークフローに変えられることを、プロジェクトの証拠で示すことです。

この講座には1本のエンジニアリング主線があります。第1-9章は、ツール、Python、データ、モデル、LLM、RAG、Agent まで進む中核です。第10-12章では、同じプロジェクト思考を computer vision、NLP、multimodal/AIGC に広げます。選択モジュールは、プロジェクトが必要としたときにデプロイ、Python 応用、classic ML を深めます。

## 最終的に作るもの

メインラインを終えるころには、最も強いプロジェクトで1つの AI アプリケーションを端から端まで説明できる状態を目指します。

| 証拠 | 何を示すか |
|---|---|
| セットアップと README コマンド | 他の人が作業を再実行できる |
| データ、Prompt、検索、ツールのトレース | アプリの挙動が観察可能で、隠れた魔法ではない |
| 指標、比較メモ、失敗例 | 1回のデモを信じるのではなく品質を判断できる |
| 安全性、プライバシー、コスト、遅延メモ | プロダクト上の制約を理解している |
| スクリーンショットまたは短いデモ | レビューする人がユーザー体験をすばやく理解できる |

## 1つのルール

短く読み、動かし、証拠を残す。各ステージの終わりには、他の人が確認できるものを残します。README コマンド、保存した出力、指標表、トレース、失敗メモ、小さなデモなどです。これが学習をポートフォリオの物語に変える方法でもあります。

## 残す証拠

第1章に入る前に、次の小さなカードを1枚書きます。

```text
目標職種: AIアプリケーションエンジニア、AIフルスタックビルダー、またはAIプロダクトエンジニア
プロジェクト軸: 章をまたいで改善する assistant、automation、analysis、または multimodal の案
最初のメインルート: クイック体験 -> 最小環境 -> 能力マップ -> 第1章
最初の証拠: スクリーンショット、保存した出力、README コマンド、または短い観察メモ
期待される成果: メインルート計画と、ポートフォリオの起点になる具体的な成果物
```
