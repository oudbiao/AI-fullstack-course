# ステージ学習タスクシート：コンピュータビジョン

このステージの目標は、AI が画像や動画をどのように扱うかを理解することです。単にビジョンモデルを呼び出して結果を得るのではなく、基本的な画像処理、ビジョンタスクの種類、データのアノテーション、モデル評価、エラーサンプル分析を身につける必要があります。

## このステージで必ず完了するタスク

| タスク | 成果物 | 合格基準 |
| --- | --- | --- |
| ビジョンタスクの種類を理解する | タスク比較表 1 枚 | 分類、検出、セグメンテーション、OCR、ビジュアル質問応答を区別できる |
| 画像処理を動かす | OpenCV の練習スクリプト 1 つ | 画像の読み込み、切り抜き、拡大縮小、グレースケール化、エッジ検出、または画像強調ができる |
| 画像分類の実験を完了する | 分類 Demo 1 つ | データ分割、学習/推論フロー、指標を説明できる |
| エラーサンプルを分析する | 誤判定サンプル記録 1 つ | 画質、ラベル、クラス混同、分布差から原因を分析できる |
| ステージプロジェクトを完了する | ビジョンアプリの小さなプロジェクト 1 つ | 入力・出力、実行方法、指標、制限の説明がある |

## おすすめの学習順序

まず、画像がコンピュータの中でどう表現されるかを理解し、その後で OpenCV の基本処理を学び、次に分類、検出、セグメンテーション、OCR などのタスクを学びます。最初から最新モデルばかりを追わず、まずは各ビジョンタスクの入力・出力・評価指標をしっかり確認しましょう。

ビジョンプロジェクトはデータ品質に強く依存します。学習前にはサンプルが鮮明か、クラスの偏りがないか、ラベルが一貫しているかを確認し、評価後には総合スコアだけでなくエラーサンプルを見るようにしましょう。

## AI 学習アシスタントプロジェクトとの関係

このステージでは、AI 学習アシスタントにビジョン機能を追加できます。たとえば、講義資料のスクリーンショットを認識したり、OCR で画像内の文字を抽出したり、学習資料の図表を分析したりできます。これは、後続のマルチモーダルステージに向けた入力能力としても使えます。

最小機能としては、講義資料のスクリーンショットをアップロードまたは読み込み、その中の文字や重要領域を抽出し、構造化された要約を出力し、失敗サンプルを記録するのがおすすめです。

## よくあるつまずきポイント

よくある問題には、画像パスの間違い、色チャンネルの BGR/RGB 混同、学習サンプルの少なさ、クラス不均衡、アノテーション枠の不正確さ、Demo 画像だけ見て実際の入力を見ないこと、ビジョンモデルの結果を絶対的な事実だと思い込むことなどがあります。確認するときは、まず元画像、前処理結果、ラベル、エラーサンプルを見ましょう。


## やさしめ版 / 標準版 / チャレンジ版タスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| やさしめ版 | 画像を読み込み、サイズ、モード、または予測結果を出力する | 1 回目の学習、時間が少ない人、または入門したばかりの学習者 |
| 標準版 | 成功例と失敗例の画像サンプルを保存する | このステージを作品集に入れたい学習者 |
| チャレンジ版 | ある認識失敗がデータ、ラベル、それともモデルに起因するかを分析する | 基礎があり、より強いプロジェクトの証拠を作りたい学習者 |

## このステージのバッジと Boss 戦

| 種類 | 内容 |
|---|---|
| Boss 戦 | ビジュアル手がかりハンター |
| 解放できるバッジ | 画像観察員、ビジョン失敗記録員 |
| 最小クリア合言葉 | まず動かす、次に説明する、そして失敗を記録する |
| 証拠保存のおすすめ | スクリーンショット、ログ、失敗サンプル、評価表を `reports/`、`evals/`、または `logs/` に保存する |

やさしめ版を完了すれば次に進めます。標準版まで完了したら、作品集に書くことをおすすめします。チャレンジ版は余力があるときに取り組みましょう。

## ステージ作品集の提出物

このステージの成果を作品集に残したい場合は、少なくとも以下のファイル、または同等の資料を保存することをおすすめします。

| 提出物 | 説明 |
| --- | --- |
| `opencv_demo.py` | 画像読み込み、前処理、基本可視化のスクリプト |
| `vision_dataset.md` | データソース、クラス、サンプル数、ラベル付け方法、制限 |
| `eval_results.md` | 分類精度、検出 mAP、OCR ヒット率、またはその他の指標 |
| `failure_cases.md` | 誤判定画像、考えられる原因、改善方向を保存する |
| `README.md` | プロジェクトの目的、実行コマンド、入出力例、適用範囲 |

これらの材料があると、ビジョンプロジェクトは「1 枚の画像を認識できる」段階から、「データ、指標、失敗、アプリケーションの境界を理解している」段階へ進めます。

## ステージ合格問題

学習後、あなたは次の問いに答えられるようになっているはずです。画像分類、物体検出、セグメンテーション、OCR の出力はどう違うのか、なぜラベル品質がモデル効果に影響するのか、mAP と IoU は何を解決するのか、なぜビジョンモデルは実際の画像で失敗しやすいのか。

## 完了状態 Checklist

- [ ] 代表的なビジョンタスクの入力、出力、指標を説明できる。
- [ ] OpenCV を使って基本的な画像処理を行い、結果を保存できる。
- [ ] 画像分類、OCR、または検出 Demo を 1 つ動かせる。
- [ ] いくつかのビジョンエラーサンプルを記録し、考えられる原因を分析できる。
- [ ] ビジョン機能をマルチモーダル、または AI 学習アシスタントプロジェクトにどう組み込むか説明できる。
