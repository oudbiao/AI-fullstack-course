# 学習ガイドとタスクシート：コンピュータビジョンをどう学ぶと一番混乱しにくいか

`第10章 コンピュータビジョン（方向選修）` に来て、モデルが多い、タスクも多いと感じたら、まずはビジョンタスクを出力の粒度で整理しましょう。分類、検出、セグメンテーションは並列に並ぶ単語の羅列ではなく、画像を粗いところから細かいところへ理解していく方法です。

## この段階の総原則

コンピュータビジョンの最初の学習では、1本の「タスクの粒度の線」をつかむことが大切です。まず画像そのものを理解し、次に画像全体のカテゴリを判断し、その後で対象の位置を特定し、最後にピクセル単位の領域を理解します。

![ビジョン出力粒度の学習ガイド図](/img/course/ch10-study-guide-output-granularity-map-ja.png)

## この段階で必ず完了するタスク

このタスクでビジョン学習を実作業に落とし込みます。「モデルが 1 枚の画像を認識した」で止めず、データ、指標、失敗サンプルを見える形で残します。

| タスク | 成果物 | 合格基準 |
|---|---|---|
| ビジョンタスクの種類を理解する | タスク比較表 1 枚 | 分類、検出、セグメンテーション、OCR、ビジュアル質問応答を区別できる |
| 画像処理を動かす | OpenCV の練習スクリプト 1 つ | 画像の読み込み、切り抜き、拡大縮小、グレースケール化、エッジ検出、または画像強調ができる |
| 画像分類の実験を完了する | 分類 Demo 1 つ | データ分割、学習/推論フロー、指標を説明できる |
| エラーサンプルを分析する | 誤判定サンプル記録 1 つ | 画質、ラベル、クラス混同、分布差から原因を分析できる |
| 再現可能なミニパイプラインを実行する | `cv_workshop_run/` 証拠フォルダ | labels、outputs、metrics、`reports/failure_cases.md` を説明できる |
| 段階プロジェクトを 1 つ完了する | ビジョンアプリの小さなプロジェクト | 入力・出力、実行コマンド、指標、制限の説明がある |

## おすすめの学習順序

1回目はまず画像の基礎と OpenCV を学びます。ピクセル、色空間、フィルタ、エッジ、基本的な画像処理を理解しましょう。

2回目は画像分類を学びます。分類はビジョン深層学習で最も直感的な入口であり、データ拡張、転移学習、学習テクニックの練習に向いています。

3回目は物体検出を学びます。バウンディングボックス、クラス、信頼度、IoU、mAP、YOLO 系列を重点的に理解しましょう。

4回目は画像セグメンテーションを学びます。意味的セグメンテーション、インスタンスセグメンテーション、ピクセル単位の出力を重点的に理解しましょう。

5回目に OCR、動画、顔、3D、医療画像などの方向のプロジェクトを選びます。

## 学習ペースの目安

| 内容タイプ | 目安時間 | 学習目標 |
|---|---|---|
| 画像の基礎 | 4～8時間 | 画像データと OpenCV の操作を理解する |
| 画像分類 | 8～16時間 | 分類学習の一連の流れを完成させる |
| 検出 / セグメンテーション | 12～24時間 | 入出力と評価指標を理解する |
| 総合プロジェクト | 16～32時間 | 1つのビジョン分野の作品を完成させる |

## 段階的なプロジェクトの進め方

最初のプロジェクトは画像分類がおすすめです。たとえば、ゴミ分類、花の分類、食品分類、手書き数字認識などです。

2つ目のプロジェクトは物体検出がおすすめです。たとえば、安全ヘルメット検出、車両検出、不良品検出、商品認識などです。

3つ目のプロジェクトでは、画像セグメンテーションや OCR を作れます。自分の方向性に合わせて、医療画像、文書認識、工業検査などを選びましょう。

大きなプロジェクトを選ぶ前に、まず [再現可能なビジョン・ミニパイプライン](./ch06-projects/03-hands-on-vision-workshop.md) を動かしてください。本ステージの baseline として、1 本のスクリプト、合成画像、分類、box、mask、指標、失敗例を確認できます。

## よくあるつまずきポイント

最もよくあるつまずきは、分類、検出、セグメンテーションが混ざってしまうことです。まず「出力は何か？」と考えてみましょう。1つのカテゴリなのか、複数の枠なのか、それとも各ピクセルのカテゴリなのかです。

2つ目のつまずきは、モデル構造ばかり追ってデータのラベルを見ないことです。ビジョンプロジェクトでは、データ品質、クラスのバランス、ラベルのルール、拡張戦略のほうが、モデルを変えることより大事な場合が多いです。

3つ目のつまずきは、指標がはっきりしないことです。分類では accuracy/F1 を見ます。検出ではよく mAP を見ます。セグメンテーションではよく IoU/Dice を見ます。

## 段階ポートフォリオ成果物

![ビジョン指標と失敗レビュー図](/img/course/ch10-workshop-metrics-iou-confusion-map-ja.png)

このステージの成果を作品集に残したい場合は、少なくとも以下のファイル、または同等の資料を保存することをおすすめします。

| 成果物 | 説明 |
|---|---|
| `opencv_demo.py` | 画像読み込み、前処理、基本可視化のスクリプト |
| `vision_dataset.md` | データソース、クラス、サンプル数、ラベル付け方法、制限 |
| `eval_results.md` | 分類精度、検出 mAP、OCR ヒット率、またはその他の指標 |
| `failure_cases.md` | 誤判定画像、考えられる原因、改善方向を保存する |
| `README.md` | プロジェクトの目的、実行コマンド、入出力例、適用範囲 |

これらの材料があると、ビジョンプロジェクトは「1 枚の画像を認識できる」段階から、「データ、指標、失敗、アプリケーションの境界を理解している」段階へ進めます。

## 段階通過質問

この段階を学び終えたら、分類、検出、セグメンテーションの3つのタスクの違いを説明できるはずです。そして、1つのビジョンプロジェクトについて、データ準備、学習、評価、結果の表示まで完了できるようになりましょう。

先へ進む前に、次の質問に答えられるか確認してください。

- 画像分類、物体検出、セグメンテーション、OCR の出力はどう違うのか。
- なぜラベル品質がモデル効果に影響するのか。
- mAP と IoU は何を解決するのか。
- なぜビジョンモデルは実際の画像で失敗しやすいのか。

1つのビジョンプロジェクトを、再現可能な Notebook かスクリプトとして整理し、モデルの失敗例まで説明できれば、その分野の入門基準に達しています。
