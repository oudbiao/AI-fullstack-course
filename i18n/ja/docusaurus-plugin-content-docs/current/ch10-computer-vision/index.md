# 10 コンピュータビジョン（方向選択）

![コンピュータビジョンのメインビジュアル](/img/course/ch10-computer-vision-ja.png)

この段階で解決するのは、「どうやってモデルに画像を理解させるか」です。これは方向選択です。主な目標が LLM アプリや Agent なら後から学んでも大丈夫です。画像、マルチモーダル、工業検査、OCR、医療画像をやりたいなら、体系的に学ぶことをおすすめします。

## ストーリーで導入：モデルに世界を見せる

人間は1枚の画像を見ると、物体、位置、境界、動作を自然に認識します。しかし、モデルにとって見えているのはただのピクセル行列です。コンピュータビジョンの役割は、モデルがピクセルから少しずつ「これは何か」「どこにあるか」「境界はどこまでか」を学べるようにすることです。分類から検出、そして分割へと進むにつれて、モデルはより細かく見られるようになります。

## 体験練習：同じ画像に3段階の質問をする

複数の物体が写っている画像を1枚用意して、まず「この画像の主なカテゴリは何か」と聞きます。次に「それぞれの物体はどこにあるか」、最後に「それぞれの物体の境界はどこか」と聞きます。この3つの質問は、それぞれ分類、検出、分割に対応します。視覚タスクの難しさは急に上がるのではなく、出力がだんだん細かくなるのだと分かるはずです。

## プロジェクトのちょっとしたお楽しみ

この段階のボーナス作品として、「ビジュアル検出ツール」を作れます。画像をアップロードすると、システムが前処理、対象物の認識、位置の表示を行い、信頼度と結果の説明を出力します。さらに、OCR 文書アシスタント、工業欠陥検出、マルチモーダル Q&A プロジェクトへ発展させることもできます。

| 情報 | 説明 |

## 初心者向けの最小クリア経路

初心者はまず、画像のピクセル、色空間、OpenCV 前処理、分類、検出、分割の違いを理解しましょう。最初から最新モデルを追いかける必要はありません。画像分類モデルを1つ学習させるか呼び出せて、さらに検出と分割が分類より何を追加で出力するのかを説明できれば、最小クリアです。

## 発展学習ルート

経験のある学習者は、データアノテーション、拡張戦略、YOLO、分割モデル、mAP、デプロイの場面、失敗ケース分析まで深掘りできます。さらに一歩進めて、ビジョンモデルを小さなアプリに組み込み、ラベル付きボックス、信頼度、誤りサンプルの説明を出すようにしてみましょう。

## ビジョンタスクはどう深くなっていくか

コンピュータビジョンは1つのタスクではありません。一般的には、出力の粒度がだんだん細かくなる形で複雑になります。まず画像全体が何かを判断し、次に対象物がどこにあるかを見つけ、最後に各ピクセルがどの領域に属するかを判断します。

![ビジョンタスク出力粒度の発展図](/img/course/ch10-visual-task-progression-map-ja.png)

## 初心者はまず何をするか、発展では何をするか

初めてこの段階を学ぶ人は、まず画像タスクの主線をつかみましょう。画像がどうやってテンソルになるのか、畳み込みがどう局所特徴を抽出するのか、分類・検出・分割がそれぞれ何を解決するのかを理解します。

経験のある学習者は、データと評価に重点を置けます。注釈の品質、クラス不均衡、IoU、mAP、失敗サンプル、デプロイ速度です。目標は、ビジョンの Demo を「説明できる」「評価できる」「改善できる」作品にすることです。

## この段階の学習ルート

10.1 節では CV の基礎と OpenCV を学び、画像のピクセル、色空間、フィルタ、エッジ、形態学、基本的な画像処理を理解します。

10.2 節では画像分類の発展として、データ拡張、現代的な分類アーキテクチャ、学習テクニックを学びます。

10.3 節では物体検出を学び、候補枠、クラス、信頼度、IoU、mAP、YOLO 系列を理解します。

10.4 節では画像分割を学び、意味分割、インスタンス分割、ピクセル単位の出力を理解します。

10.5 節では発展トピックとして、顔検出、動画解析、OCR、3D ビジョンを学びます。

10.6 節では総合プロジェクトを完成させ、データ、モデル、指標、アプリケーション場面をつなげます。

## 学び終えたあとにできること

- 分類、検出、分割の3種類のビジョンタスクの違いを説明できる
- OpenCV を使って基本的な画像処理ができる
- 画像分類モデルを学習または微調整できる
- 物体検出と分割の入力・出力・評価指標を理解できる
- 1つのビジョンプロジェクトのためにデータを準備し、モデルを学習し、結果を分析できる

## よくある誤解

最新のビジョンモデルだけを追いかけないようにしましょう。ビジョンプロジェクトで本当に難しいのは、データ収集、注釈の品質、クラス不均衡、指標の選び方、デプロイ場面です。

また、OpenCV と深層学習を切り離して考えないことも大切です。OpenCV は従来の画像処理や工程前処理に向いており、深層学習は複雑な認識タスクに向いています。実際には、この2つは一緒に使われることがよくあります。

## ビジョン失敗劇場：モデルの誤りはたいてい1つの原因ではない

分類結果が不安定なら、まず学習画像が鮮明か、クラスが偏っていないか、拡張がやりすぎていないかを確認します。物体検出で小さい対象を見落とすなら、注釈の品質、画像解像度、評価しきい値をチェックします。Demo の画像ではうまくいくのに実画像で悪いなら、まずデータ分布の不一致を疑いましょう。

## 最小実行実験：画像を読み込み、確認できる結果を出す

この段階の最小実験は OpenCV または PIL から始められます。1枚の画像を読み込み、サイズ、チャネル、切り抜き、またはグレースケール処理を行い、処理後の結果を保存します。そのあとで、分類、検出、OCR モデルに置き換えます。

```python
from PIL import Image

img = Image.open("sample.jpg")
print(img.size, img.mode)
small = img.resize((224, 224))
small.save("sample_224.jpg")
```

ビジョンプロジェクトでは、入力画像、処理結果、予測の可視化を必ず残しましょう。そうしないと、モデルが間違ったときに、問題がデータ、注釈、前処理、モデルのどこにあるのか判断しにくくなります。

## ビジョン失敗ケース集：まず入力品質と注釈境界を確認する

| 現象 | よくある原因 | 切り分け方法 | 修正方向 |
|---|---|---|---|
| 分類が不安定 | 画像がぼやけている、クラス不均衡、拡張のやりすぎ | 誤分類画像とクラス分布を見る | データをきれいにする、拡張戦略を調整する |
| 検出で小さい対象を見落とす | 解像度が低い、注釈が不一致、しきい値が高すぎる | ボックスと信頼度を可視化する | 解像度を上げる、注釈を確認する、しきい値を調整する |
| 分割の境界が粗い | 境界注釈が正確でない、またはモデル出力の尺度が低い | mask と元画像を比較する | 注釈ルールを改善し、より適切な指標を使う |
| Demo の画像は良いのに実画像は悪い | 学習データと実場面の分布が違う | 光、角度、背景を比較する | 実サンプルと場面説明を追加する |


## 段階プロジェクト

基礎版は、データ準備、学習、基本評価を含む画像分類プロジェクトを完成させます。標準版では、データ拡張、誤りサンプル分析、予測結果の可視化を追加します。チャレンジ版では、物体検出または分割プロジェクトを作り、注釈形式、mAP/IoU 指標、推論デモ、場面に応じたアプリケーション説明を加えます。

大きなプロジェクトを選ぶ前に、まず具体的な baseline を動かしたい場合は、[10.6.4 実践：再現可能なビジョン・ミニパイプラインを作る](./ch06-projects/03-hands-on-vision-workshop.md) から始めてください。ローカルスクリプトで画像、mask、box、指標、予測可視化、失敗レポートを生成できます。

より細かい学習ペースを知りたい場合は、[10.0 学習ガイド：コンピュータビジョンはどう学ぶと混乱しにくいか](./study-guide.md) を読んでください。

## AI 学習アシスタントと通しプロジェクトの関係

この段階では、AI 学習アシスタントにビジョン機能を追加できます。たとえば、教材スライドのスクリーンショットを認識したり、OCR で文字を抽出したり、学習用画像資料を分析したりできます。通しプロジェクトの流れで学んでいるなら、この段階の最後に少なくとも1回は版管理の記録を残すことをおすすめします。つまり、この段階で何が追加されたのか、どう実行するのか、入力と出力の例は何か、どんな問題に遭遇したか、次にどう直す予定か、をまとめておきましょう。
