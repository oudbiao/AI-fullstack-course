# 学習ガイド：Python プログラミング基礎をどう学ぶと一番混乱しにくいか

もしあなたがすでに `01 Python 编程基础` に来ているなら、最も大事な目標は、すべての文法を暗記することではなく、Python で「動く・直せる・説明できる」小さなプログラムを書けるようになることです。

## この段階の基本方針

Python を最初に学ぶときは、一本の流れをつかむことが大切です。つまり、入力がプログラムに入り、データ構造、条件分岐、ループ、関数で処理され、最後に結果を出力するか、ファイルに保存する、という流れです。この流れが安定していれば、あとでデータ分析、機械学習、RAG、Agent を学ぶときもずっとスムーズになります。

![Python 学習ガイド最小閉ループ](/img/course/ch02-study-guide-program-loop-ja.png)

## おすすめの学習順序

1回目は、まず最小限のプログラミングの流れを学びます。Python の概要、データ型、演算子、入出力、制御構文、リスト、辞書、関数、モジュールです。目標は、すべての細かい点を覚えることではなく、小さなスクリプトを書けるようになることです。

2回目は、コードを整理する力を補います。オブジェクト指向、例外処理、ファイルの読み書き、関数型プログラミング、イテレータとジェネレータ、型注釈、コード品質です。目標は、コードをより分かりやすく、より安定させ、より拡張しやすくすることです。

3回目は、プロジェクトに進みます。コマンドラインのタスク管理ツール、Web クローラー、Web API、AI API の簡単な体験です。プロジェクトを通して、文法、ファイル、例外、サードパーティライブラリ、デバッグの力をつなげていきます。

## おすすめの学習ペース

| 内容の種類 | おすすめ時間 | 学習目標 |
|---|---|---|
| 基本文法ページ | 1～2 時間 | サンプルを手で打ち込み、結果を変えられる |
| データ構造と関数 | 2～4 時間 | 重複する処理を関数に分けられる |
| 発展的な整理ページ | 2～4 時間 | ファイル、クラス、例外、モジュールの関係を読める |
| プロジェクトページ | 4～8 時間 | 独力で動かし、小さく改造できる |

## 段階ごとのプロジェクトルート

最初のプロジェクトは、コマンドラインのタスク管理ツールがおすすめです。入力と出力、リスト/辞書、ファイル保存、例外処理を練習できます。

2つ目のプロジェクトは、Web データ収集がおすすめです。HTTP リクエスト、HTML 解析、データ整形、ファイル保存を練習できます。

3つ目のプロジェクトは、Web API がおすすめです。リクエスト、レスポンス、ルーティング、パラメータ検証、サービス起動を理解できます。

4つ目のプロジェクトは、AI API の簡単な体験がおすすめです。大規模言語モデルの能力を自分の Python プログラムに組み込み、後の LLM アプリの本線に向けた土台を作ります。

## よくあるつまずきポイント

いちばんよくあるつまずきは、「サンプルは分かるのに、自分では書けない」というものです。対処法は、もっとたくさんのチュートリアルを見ることではありません。サンプルを閉じて、要件だけを残し、まずは一番単純な方法で第1版を書き切ることです。

2つ目のつまずきは、関数とモジュールです。最初は1つのファイルで最後まで書いてかまいません。コードが数十行を超えたら、重複している処理を関数に分け、関連する関数をモジュールに整理しましょう。

3つ目のつまずきは、エラーです。エラーを怖がらないでください。まず最後の1行のエラーメッセージを見て、次にファイル名と行番号を確認し、最後に最小限のコードで問題を再現します。

## 合格基準

この段階を学び終えたら、ひとりで小さな Python プロジェクトを完成でき、各ファイルの役割を説明でき、よくあるエラーに対処でき、ファイルの読み書きができ、サードパーティライブラリをインストールでき、簡単な API を呼び出せるようになっているはずです。

もし「コマンドラインのタスク管理ツール」をゼロから書けて、さらに検索、並べ替え、分類、エクスポートのような新機能を追加できるなら、次の段階に進んで大丈夫です。
