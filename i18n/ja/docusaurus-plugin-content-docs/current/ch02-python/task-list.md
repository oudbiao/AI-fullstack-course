# 段階学習タスクリスト：Python プログラミング基礎

この段階の目標は、Python を使って小さなプログラムの入力・処理・出力をはっきり表現できるようになることです。今後のデータ分析、機械学習、RAG、Agent はどれも Python に依存するので、ここで最も大切なのは、読みやすく、実行できて、デバッグしやすいコードを書くことです。

![Python 段階タスクの流れ](/img/course/ch02-task-list-workflow-ja.png)

## この段階で必ず完了するタスク

| タスク | 成果物 | 合格基準 |
| --- | --- | --- |
| 基本文法を身につける | 5 個の小さな練習スクリプト | 変数、条件分岐、ループ、関数を使って簡単な問題を処理できる |
| データ構造に慣れる | 1 つのデータ整理スクリプト | list、dict、set、tuple を選び、その理由を説明できる |
| 関数を書く | 複数の関数に分けた小さなプログラム | 各関数の役割が明確で、入力と戻り値がある |
| ファイルを扱う | テキストまたは JSON を読み書きするスクリプト | 結果をファイルに保存して、もう一度読み込める |
| 段階プロジェクトを完成させる | 1 つのコマンドライン小ツール | README で実行方法と出力例を説明できる |

## おすすめの学習順序

まずは変数、型、条件分岐、ループ、関数を一通り動かせるようにしてから、リスト、辞書、ファイル操作を学びます。オブジェクト指向、デコレータ、ジェネレータなどの発展内容は、まず用途を理解できれば十分で、最初から使いこなすことを目指さなくて大丈夫です。

Python を学ぶときは、文法の説明を見るだけにしないでください。文法を 1 つ学ぶたびに、すぐに小さなスクリプトを書きましょう。たとえば、タスクリスト、テキスト集計、ファイル整理、API レスポンス解析、学習ログ生成などです。

## AI 学習アシスタントプロジェクトとの関係

この段階は、AI 学習アシスタントの v0.2 コマンドライン対話版に対応します。プログラムに質問を読み込ませ、ログファイルに保存し、固定テンプレートの回答を返すようにできます。この時点ではまだ大規模言語モデルを呼び出す必要はありません。重点は、入力と出力、関数の分割、ファイル記録の練習です。

おすすめの実装機能：ユーザーが学習に関する質問を入力すると、プログラムがその質問、時間、現在の段階を `notes/questions.jsonl` に書き込み、そのあとで「記録しました。まずは該当する章を確認することをおすすめします」という案内を返します。

## よくあるつまずき

初心者によくある問題には、インデントエラー、変数名の上書き、list と dict の混同、ファイルパスの間違い、関数が print するだけで return しない、エラーメッセージが読めない、などがあります。エラーが出たら、まずどの行で起きたのかを特定し、その行で使っている変数の型と値が想定どおりかを確認しましょう。


## やさしめ版 / 標準版 / チャレンジ版のタスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| やさしめ版 | 追加、表示、完了のタスク管理と JSON 保存を実装する | 1 回目の学習、時間が少ない、または初心者の学習者 |
| 標準版 | 分類、検索、壊れたファイルへの案内をサポートする | この段階を作品集に入れたい学習者 |
| チャレンジ版 | 正常、空入力、壊れた JSON の 3 つのテストケースを用意する | 基礎があり、より強いプロジェクト証拠を作りたい学習者 |

## この段階のバッジと Boss 戦

| 種類 | 内容 |
|---|---|
| Boss 戦 | JSON ダンジョン管理者 |
| 解放できるバッジ | JSON 飼いならし者、例外キャッチャー |
| 最低合格スローガン | まず動かす、次に説明する、最後に失敗を記録する |
| 証拠の保存おすすめ先 | スクリーンショット、ログ、失敗サンプル、評価表を `reports/`、`evals/`、または `logs/` に保存する |

やさしめ版を完成できれば先に進んでよく、標準版まで完成したら作品集に載せるのがおすすめです。チャレンジ版は、余裕があるときに取り組みましょう。

## 段階作品集の提出物

この段階の成果を作品集として残したい場合は、少なくとも次のファイルまたは同等の材料を残すのがおすすめです。

| 提出物 | 説明 |
| --- | --- |
| `todo_cli.py` またはプロジェクトディレクトリ | 追加、表示、完了、保存に対応したコマンドラインタスク管理器のソースコード |
| `data/tasks.json` | ファイル読み書きとデータ構造設計を示すサンプルデータファイル |
| `api_demo.py` | エラー処理と戻り値の解析を含む Web API 呼び出し練習 |
| `README.md` | 実行コマンド、入力出力例、プロジェクト構成、既知の制限 |
| `debug_notes.md` | 少なくとも 2 個の Python エラー、その原因、修正過程の記録 |

これらの材料があると、Python 段階は「文法練習」から「小さなツールを自力で書ける、エラーを直せる、構造を説明できる」プロジェクト証拠になります。

## 段階クリア問題

学習を終えたら、次の質問に答えられるようになっているはずです。どのタイミングで関数を書くべきか、どのタイミングで辞書を使うべきか、ファイルパスは何を基準に相対的に決まるのか、print と return の違いは何か、なぜ実際のプロジェクトではロジックを複数の小さな関数に分ける必要があるのか。
