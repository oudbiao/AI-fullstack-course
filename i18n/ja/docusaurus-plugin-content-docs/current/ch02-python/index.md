# 2 Python プログラミング基礎

![Python プログラミング基礎メインビジュアル](/img/course/ch02-python-foundation-ja.png)

この段階で解決したいのは、「コードで問題を表現できるか、データを処理できるか、サービスを呼び出せるか」です。今後のデータ分析、機械学習、RAG、Agent は何度も Python に頼ることになるので、ここでの目標は文法を暗記することではなく、プログラミングの考え方とデバッグの習慣を身につけることです。

## ストーリーで入る：Python を最初の AI スイスアーミーナイフにしよう

小さな「AI スタジオ」を運営しているところを想像してみてください。毎日、タスクを整理し、資料を集め、データをファイルに保存し、他の人に使ってもらうためのインターフェースを提供し、最後には大規模言語モデルをつないで小さなアシスタントを作りたくなります。Python は、こうした作業をつなぐ道具箱です。最初は変数、ループ、関数に見えるだけかもしれませんが、文法を 1 つ学ぶたびに、あとで本物の力になります。たとえば、データの自動整理、ファイルの一括処理、API の呼び出し、サービスのカプセル化などです。

## 参加型練習：1 章学ぶごとに自分に 1 つ質問する

基礎文法を学ぶときは、「この知識は入力を受け取り、出力を作るのに役立つか」と考えましょう。データ構造を学ぶときは、「この情報のまとまりは、リスト・辞書・ファイルのどれで保存するのがよいか」と考えましょう。関数やモジュールを学ぶときは、「このロジックは今後くり返し使うことになるか」と考えましょう。プロジェクトを学ぶときは、「他の人に渡したとき、実行方法、終了方法、エラーの場所が分かるか」と考えましょう。

## プロジェクトのおまけ

この段階を終えると、あなたは単に「Python を学んだ人」ではなく、4 つの小さなプロジェクトを作品集としてまとめられるようになります。つまり、タスクを保存できるコマンドラインツール、Web データを収集できるクローラー、他の人が呼び出せる Web API、AI と会話できる API プログラムです。この 4 つをつなげると、後で RAG や Agent を作るときの最小の原型になります。

## 初心者向けの最小クリアコース

初心者はまず、変数、条件分岐、ループ、リスト、辞書、関数、ファイルの読み書き、例外処理といった使用頻度の高い力をつかみましょう。最初からすべての高度な文法にこだわる必要はありません。コマンドラインのタスク管理ツールを 1 人で完成でき、クローラー、Web API、AI API のサンプルの流れが読めれば、最小クリアです。

## 上級者向けの発展コース

経験のある学習者は、モジュール分割、型アノテーション、エラー処理、コード品質、プロジェクトのリファクタリングに重点を置けます。同じ機能を、スクリプト版、モジュール版、API 版の 3 通りで書いてみて、Python が小さなツールからエンジニアリングプロジェクトへ変わる感覚をつかみましょう。

## 初学者はまず何をするべきか、上級者は後で何をするべきか

初めてこの段階を学ぶ人は、まず「入力 → 処理 → 出力」を軸に小さなプログラムを書きましょう。変数、リスト、辞書、関数、ファイル、例外処理があれば、入門プロジェクトの大半は完成できます。最初から高度な文法を全部追いかける必要はありません。

経験のある学習者は、エンジニアリングの観点に重点を置けます。1 つの機能をどうモジュールに分けるか、例外をどこまで上に投げるか、型アノテーションでどう見やすくするか、スクリプトをどう API に発展させるか。目標は、後でデータ、モデル、LLM のプロジェクトでも再利用したくなる Python コードを書くことです。

## なぜ Python が AI の主流言語なのか

Python が AI の主流になったのは、文法が最強だからではありません。データ処理、機械学習、深層学習、Web API、自動化スクリプト、大規模言語モデルのエコシステムを、1 つの言語でつなげるからです。今後は Python でデータを読み、モデルを学習させ、LLM を呼び出し、RAG を構築し、ツールを封装し、Agent を作ることになります。

## この段階の学習ルート

第 1 章では、変数、型、演算子、入出力、制御フロー、データ構造、関数、モジュールなど、Python の言語基礎を学びます。特に「入力、処理、出力」の流れをしっかり理解しましょう。

第 2 章では、オブジェクト指向、例外処理、ファイルの読み書き、関数型プログラミング、イテレータとジェネレータ、型アノテーション、コード品質など、Python の発展内容を学びます。すべての高度な文法を一度に覚える必要はありませんが、それぞれが何の問題を解決するのかは知っておきましょう。

第 3 章では実践プロジェクトに入ります。前の知識を組み合わせて、コマンドラインのタスク管理ツール、Web クローラー、Web API、AI API のクイック体験を完成させます。

## よくある誤解

すべての文法を一気に暗記しようとしないでください。本当に大事なのは、プロジェクトの中で何度も使えることです。リスト、辞書、関数、ファイル操作、例外処理、サードパーティライブラリの呼び出しは、あまり使わない文法よりずっと頻繁に登場します。

また、早すぎる段階で「きれいに書くこと」にこだわりすぎないでください。最初の 1 回目は、まず動くこと、説明できること、変更できることのほうが大切です。いくつかプロジェクトを作ってから、型アノテーション、モジュール分割、コードスタイルを少しずつ意識していきましょう。

## プログラミング失敗劇場：コードが動かないとき、まずどこを見るか

プログラムが構文エラーを出したら、まずエラー行の近くにある括弧、インデント、コロンを確認しましょう。結果が合っていないときは、重要な変数を print して、入力・処理・出力の各ステップが期待どおりか確認しましょう。ファイルの読み書きに失敗したら、パス、エンコーディング、ファイルの存在を確認しましょう。API 呼び出しに失敗したら、リクエストパラメータ、ステータスコード、エラーメッセージを確認しましょう。

## 最小で動く実験：JSON タスクマネージャー

この段階の最小実験は、データを保存できるコマンドラインプログラムを書くことです。とてもシンプルでかまいません。リストでタスクを保存し、JSON ファイルで永続化し、新規追加・表示・完了をサポートします。この実験では、変数、リスト、辞書、関数、ファイルの読み書き、例外処理をまとめて使います。

```python
import json
from pathlib import Path

DATA = Path("tasks.json")

def load_tasks():
    if not DATA.exists():
        return []
    return json.loads(DATA.read_text(encoding="utf-8"))

def save_tasks(tasks):
    DATA.write_text(json.dumps(tasks, ensure_ascii=False, indent=2), encoding="utf-8")

tasks = load_tasks()
tasks.append({"title": "Python のファイル読み書きを学ぶ", "done": False})
save_tasks(tasks)
print(tasks)
```

最初から完全な CLI を作る必要はありません。まずは、プログラムが入力を受け取り、状態を保存し、もう一度読み込めることを確認しましょう。そのあとで、少しずつコマンドライン引数とエラー処理を追加します。

## Python 失敗ケース集：まず入力、パス、データ構造を見る

| 現象 | よくある原因 | 特定方法 | 修正の方向 |
|---|---|---|---|
| 構文エラー | インデント、括弧、コロン、引用符の不一致 | エラー行とその前の行を見る | コード範囲を狭め、少しずつ実行する |
| ファイルが見つからない | 実行中のディレクトリとファイルのあるディレクトリが違う | `Path.cwd()` を表示する | プロジェクトルートからの相対パス、または明示的なパスを使う |
| JSON の解析に失敗する | ファイルが空、形式が壊れている、またはエンコーディングが不一致 | 元のファイルを開いて内容を確認する | 例外処理とデフォルトの空データを追加する |
| 関数がどんどん分かりにくくなる | 入出力が不明確、グローバル変数が多い | コメントを書く：引数は何か、戻り値は何か | 関数を小さく分け、隠れた状態を減らす |
