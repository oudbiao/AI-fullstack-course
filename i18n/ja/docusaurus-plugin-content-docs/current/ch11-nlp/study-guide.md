# 学習ガイドとタスクシート：自然言語処理はどう学ぶと混乱しにくいか

`第 11 章 自然言語処理（方向選修）` に来て、token、embedding、分類、抽出、生成、BERT、GPT が全部ごちゃ混ぜに感じたら、まず NLP の主線に戻りましょう。テキストがどのようにモデルで扱える表現に変わるのか、という流れです。

## この段階の基本方針

NLP の最初の学習では、1 本の進化の流れをつかむことが大切です。  
テキストはまず前処理と分かち書き・分割を行い、それからベクトル表現に変わり、分類・抽出・生成などのタスクに入り、最後に事前学習言語モデルへ進みます。

![NLP テキストからモデルへの学習ガイド図](/img/course/ch11-study-guide-text-to-model-map-ja.png)

## この段階で必ず完了するタスク

このタスクで NLP を実際のテキスト作業に落とし込みます。強力な LLM があっても、明確なラベル、根拠、指標、失敗サンプルは必要です。

| タスク | 成果物 | 合格基準 |
|---|---|---|
| テキスト前処理を理解する | テキストクリーニングスクリプト | トークン化、大文字小文字、ストップワード、句読点、特殊文字を扱える |
| テキスト表現の実験を完了する | 表現方法の比較記録 | BoW、TF-IDF、Embedding、事前学習モデル表現を比較できる |
| テキスト分類タスクを完了する | 分類 Demo | ラベル、データ分割、指標、エラーサンプルを説明できる |
| 抽出または要約の演習を完了する | 情報抽出または要約の例 | フィールド境界、事実整合性、評価方法を説明できる |
| 再現可能な NLP ミニパイプラインを実行する | `nlp_workshop_run/` 証拠フォルダ | outputs、reports、metrics、`reports/failure_cases.md` を説明できる |
| 段階プロジェクトを 1 つ完了する | 小さなテキスト理解プロジェクト | 入力・出力、指標、失敗サンプル、README がある |

## おすすめの学習順序

1 周目はまずテキストの基礎を学びます。分かち書き、クリーニング、ストップワード、テキストの正規化、表現方法を理解しましょう。

2 周目は単語埋め込みと言語モデルを学びます。単語ベクトル、文脈表現、そして言語モデルがなぜテキストを予測できるのかを重点的に理解します。

3 周目はテキスト分類を学びます。これは入門に最も向いている NLP プロジェクトで、テキストからラベルへの流れを理解するのに役立ちます。

4 周目は系列ラベリングと Seq2Seq を学びます。これらはそれぞれ情報抽出と生成・翻訳タスクに対応します。

5 周目は事前学習言語モデルを学びます。BERT、GPT、T5、そして Transformers ライブラリが、それまでの概念を組み直してくれます。

## 学習のペースの目安

| 内容タイプ | 目安時間 | 学習目標 |
|---|---|---|
| テキスト基礎 | 4～8 時間 | テキストがどう特徴量になるかを理解する |
| 単語ベクトルと言語モデル | 6～12 時間 | 表現と文脈を理解する |
| 分類 / 抽出 / 生成 | 12～24 時間 | さまざまな NLP タスクを区別できるようにする |
| 事前学習モデル | 8～16 時間 | BERT/GPT/T5 の違いを理解する |
| 総合プロジェクト | 16～32 時間 | 1 つのテキストタスクのプロジェクトを完成させる |

## 段階的なプロジェクトの進め方

最初のプロジェクトは、感情分析、スパムメール判定、レビュー分類などのテキスト分類がおすすめです。

2 つ目のプロジェクトは、固有表現認識、履歴書の情報抽出、契約書の項目抽出などの情報抽出がおすすめです。

3 つ目のプロジェクトでは、質問応答システムやテキスト要約に取り組み、従来の NLP の考え方と事前学習モデルを組み合わせてみましょう。

大きなプロジェクトを選ぶ前に、まず [再現可能な NLP ミニパイプライン](./ch07-projects/05-hands-on-nlp-workshop.md) を動かしてください。本ステージの baseline として、1 本のスクリプト、ローカルテキストデータ、tokenization、TF-IDF、分類、検索 QA、要約、抽出、指標、失敗例を確認できます。

## よくあるつまずきポイント

最もよくあるつまずきは、大規模モデルにいきなり飛びついて、テキスト表現を軽視してしまうことです。LLM を使う場合でも、token、文脈、テキストの前処理、評価を理解しておく必要があります。

2 つ目のつまずきは、タスクの境界がはっきりしないことです。分類はカテゴリを出力し、系列ラベリングは各 token のラベルを出力し、生成タスクは新しいテキストを出力します。

3 つ目のつまずきは、モデルだけを見てデータを見ないことです。テキストタスクは、ラベル付けの品質、カテゴリ定義、評価セットの設計に大きく左右されます。

## 段階ポートフォリオ成果物

このステージの成果を作品集に残したい場合は、少なくとも以下のファイル、または同等の資料を保存することをおすすめします。

| 成果物 | 説明 |
|---|---|
| `text_cleaning.py` | テキストクリーニング、トークン化、正規化、サンプル出力 |
| `label_guide.md` | ラベル定義、境界ケース、正例と負例、アノテーション規則 |

| `README.md` | プロジェクト目標、実行方法、入力・出力、評価、制限 |

これらの材料があると、NLP プロジェクトは「モデルがテキストを出せる」から「タスク定義が明確で、評価が信頼でき、失敗を振り返れる」状態へ進めます。

## 段階通過質問

この段階を学び終えたら、テキストが元の文字列からモデル入力になるまでの大まかな流れを説明できて、テキスト分類または情報抽出のプロジェクトを 1 つ完成できるようになっているはずです。

先へ進む前に、次の質問に答えられるか確認してください。

- テキストクリーニングはなぜモデル性能に影響するのか。
- TF-IDF と Embedding の違いは何か。
- なぜテキスト分類にはラベルルールが必要なのか。
- 要約と抽出では、事実整合性をどう確認するのか。
- 従来型 NLP 手法が LLM より向いているのはどんなときか。

BERT と GPT の違いを、学習目標、適用タスク、使い方の観点からはっきり説明できれば、よりスムーズに大規模モデルの原理へ進めます。
