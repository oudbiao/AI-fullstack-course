# 学習ガイド：Seq2Seq と注意力のこの章では何を学ぶのか

## この章の位置づけ

この章で扱うのは、NLP の代表的な問題のひとつです。入力も出力も系列であるとき、モデルは文章をどうやって別の文章に変えるのか。翻訳、要約、対話生成、誤り訂正、タイトル生成は、根本的にはすべて「Sequence to Sequence」の問題として見ることができます。

またこの章は、RNN 時代から Transformer へ進むための重要な橋渡しでもあります。昔の Seq2Seq をそのまま将来の最終解にする必要はありませんが、なぜ Encoder-Decoder が必要なのか、なぜ普通の RNN は長い距離の情報を忘れてしまうのか、なぜ Attention が後の Transformer の中心的な考え方になったのかは、必ず理解しておく必要があります。

## この章が NLP の学習ルートの中でどこにあるか

![Seq2Seq と注意力の章の学習順序図](/img/course/ch11-seq2seq-chapter-flow.png)

前のテキスト分類では、たいてい「文章を入力して、ラベルを1つ出す」形でした。系列ラベリングでは「token の列を入力して、ラベルの列を出す」形でした。Seq2Seq ではさらに進んで、「token の列を入力して、別の token の列を出す」形になります。この変化によって、あなたは自然に生成モデルの世界へ入っていきます。

## この章の学習の流れ

まず、Encoder が入力系列を表現に圧縮し、Decoder がその表現を使って少しずつ出力を生成する流れを理解します。次に、なぜ1つの文脈ベクトルだけではボトルネックになるのかを学びます。文が長くなるほど、情報は失われやすくなるからです。3つ目に Attention を学びます。Decoder は各ステップで生成するとき、入力系列のさまざまな位置を見直すことができます。4つ目に、機械翻訳の実践を通して、入力、出力、teacher forcing、デコード、評価をつなげて理解します。最後に CTC と Deep Speech を読むと、音声認識のように「入力と出力がフレームごとに対応しない」場合に、系列モデルがどう学習するのかがわかります。

## Seq2Seq、Attention、Transformer の関係

| 概念 | 解決する問題 | 次につながる内容 |
|---|---|---|
| Encoder-Decoder | 入力と出力の長さが違うため、分類ではなく生成の問題になる | 要約、翻訳、質問応答生成 |
| Attention | Decoder が各 token を生成するときに、異なる入力位置に注目する必要がある | Transformer の中心的な仕組み |
| Teacher Forcing | 学習時に正しい過去出力を見せて、収束しやすくする | 生成モデル学習の直感 |
| Beam Search | 推論時に1つの token だけを貪欲に選ばない | LLM のデコード戦略の前提知識 |

この章で大事なのは、古いモデル名を覚えることではありません。生成系 NLP の基本問題を理解することです。つまり、学習時に入力と出力をどう対応づけるのか、推論時にどうやって1文字ずつ生成するのか、生成結果をどう評価するのか、という点です。

## 大規模言語モデルの学習とのつながり

この先 GPT、T5、Prompt、RAG、Agent を学ぶときにも、この章の考え方が何度も出てきます。GPT は自回帰生成、T5 はタスクを text-to-text に統一します。Prompt も本質的には、タスクを「入力系列」として書き換え、モデルに「出力系列」を生成させるものです。Seq2Seq を理解しておくと、大規模言語モデルが翻訳、要約、言い換え、質問応答、計画立案に使える理由が、よりわかりやすくなります。

## この章の小さな実践ゴール

「小さなテキスト言い換え、または翻訳実験」を1つ完成させるのがおすすめです。基礎版では、既存モデルや簡単なコードを使って、中英の短い文の翻訳を動かし、入力と出力を記録します。標準版では、greedy decoding と beam search の結果の違いを比べます。挑戦版では、講義の段落を使って「要約生成」の小実験を行い、どの文で重要情報が落ちやすいかを記録します。

README には少なくとも、入力系列は何か、出力系列は何か、モデルがどのように順番に生成するのか、どのデコード戦略を使ったのか、失敗例は何か、そして Attention がなぜ長文の情報損失をやわらげるのかを書いてください。

## よくある誤解

Seq2Seq を機械翻訳専用のモデルだと考えないようにしましょう。これは、入力と出力がどちらも系列であるようなモデリング手法の一種です。また、Attention を単なる「重み付き平均」だと思い込まないでください。本当に重要なのは、生成過程に動的な対応づけの能力を持たせることです。さらに、推論フェーズを軽く見てはいけません。多くの生成タスクでは、学習コードを書き終えたら終わりではなく、デコード戦略が出力品質に大きく影響します。

## 到達目標

この章を学び終えたら、Encoder-Decoder がなぜ必要だったのか、Attention がどんなボトルネックを解消したのか、機械翻訳タスクで入力と出力をどう組み立てるのか、そしてこれらの概念が Transformer、T5、さらに後の大規模言語モデルの応用へどうつながるのかを説明できるようになっているはずです。
