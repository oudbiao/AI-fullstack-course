# 学習ガイド：事前学習モデルのこの章では何を学ぶのか

![BERT GPT T5 比較図](/img/course/bert-gpt-t5-comparison.png)

## 本章の位置づけ

この章は、従来の NLP から大規模モデル応用へ進むための橋渡しです。これまでに、テキスト表現、テキスト分類、系列ラベリング、Seq2Seq を学んできましたが、これらは通常、単一タスクごとにモデルを学習する内容でした。ここからは、「まず汎用の土台を事前学習し、それをさまざまなタスクに転移する」という、現代 NLP の考え方に入っていきます。

この章で目指すのは、BERT、GPT、T5 の名前を覚えることではありません。モデルが大規模テキストからどのように言語能力を身につけるのか、なぜ事前学習の目的によって得意分野が変わるのか、そして transformers ライブラリがこれらのモデルをどう実プロジェクトに接続するのか、この3つを理解することです。

## この章がコース全体の中でどこにあるか

![事前学習言語モデル章の学習順序図](/img/course/ch11-pretrained-chapter-flow.png)

この章は、第11章の自然言語処理を締めくくると同時に、第7〜9章の大規模モデル、RAG、Agent への前置きにもなります。今後、大規模モデル API の呼び出し、テキスト検索、微調整、RAG の構築をするとき、tokenizer、embedding、コンテキスト長、生成方法、モデルの読み込みといった概念に何度も出会うことになります。

## BERT、GPT、T5 はそれぞれ何を表す考え方か

| モデル群 | コアの考え方 | 何を理解するのに向いているか |
|---|---|---|
| BERT | マスク予測を通じて双方向の意味表現を学ぶ | 分類、照合、抽出、検索 embedding の基本的な直感 |
| GPT | 自己回帰的に次の token を予測して生成能力を学ぶ | チャット、続きの生成、ツール呼び出し、Prompt と生成系アプリケーション |
| T5 | さまざまな NLP タスクを text-to-text に統一する | タスクの統一、命令化、マルチタスク転移 |

この3種類のモデルを、互いに置き換わるバージョン違いだと考えないでください。むしろ、学習と利用の3つのパラダイムだと捉えるとよいです。BERT は理解寄り、GPT は生成寄り、T5 はタスクをすべてテキストからテキストへ書き換えることを重視しています。現代の大規模モデルはこれらの考え方を取り入れていますが、まずここを学ぶことで、後の技術設計の理由が見えてきます。

## この章の学習順序

まず事前学習パラダイムを理解します。なぜ、各タスクをゼロから学習するよりも、大規模テキストで先に学び、その後で具体的なタスクへ転移するほうが効果的なのかを押さえます。次に BERT を見て、mask、双方向コンテキスト、CLS 表現、下流タスクへの微調整を理解します。続いて GPT を見て、自己回帰生成、コンテキストウィンドウ、Prompt を重点的に学びます。その次に T5 を見て、翻訳、要約、質問応答、分類をすべて text-to-text として統一する考え方を理解します。最後に transformers を使って最小例を動かし、tokenizer、model、pipeline、入力と出力をつなげます。

## 大規模モデルのコースとのつながり

この章では、大規模モデル応用でよく出てくる多くの「実装の常識」を先取りして理解できます。たとえば、tokenizer はテキストをモデルがどう読むかを決めます。コンテキスト長は、一度にどれだけの資料を入れられるかを決めます。embedding は検索やクラスタリングに使えます。生成モデルでは、温度、長さ、終了条件の制御が必要です。モデルの読み込みや推論コストは、デプロイ方法にも影響します。

RAG に進むと、BERT/embedding の理解がベクトル検索の理解に役立ちます。Prompt や Agent に進むと、GPT の自己回帰生成が、モデルがなぜ計画やツールのパラメータを一歩ずつ生成するのかを理解する助けになります。微調整に進むと、事前学習と下流タスク転移の関係がとても重要になります。

## この章のミニプロジェクトの出口

「事前学習モデルの比較ミニ実験」を1つ完成させることをおすすめします。基本版では、transformers の pipeline を使って、テキスト分類、要約、生成の例をそれぞれ動かし、入力と出力を記録します。標準版では、BERT 系モデルと GPT 系モデルを同じテキストタスクで比較し、それぞれの適用範囲を比べます。チャレンジ版では、ある講義段落を embedding 検索の小さな例にして、後の RAG の準備にします。

README には少なくとも次の内容を書いてください。どのモデルを使ったか、入力は何か、出力は何か、そのモデルは理解向きか生成向きか、実行時にどんな依存関係やダウンロード問題があったか、そしてこの実験が後の RAG や LLM 応用にどうつながるかです。

## よくある誤解

「事前学習モデル」は「より大きいモデル」と同じだと思わないでください。本当に重要なのは、学習目標、入力と出力の形式、転移の方法です。また、最初から最新のモデル名を追いかける必要はありません。まず BERT、GPT、T5 の基本的なパラダイムを理解しておけば、後で RoBERTa、DeBERTa、LLaMA、Qwen、DeepSeek などのモデルを見たとき、それらがどの種類の問題を解こうとしているのかが分かるようになります。

## クリア基準

この章を学び終えたら、次のことを説明できるはずです。BERT、GPT、T5 の学習の考え方はどう違うのか、なぜ transformers ライブラリで多くのモデルを統一して呼び出せるのか、どのタスクが理解型モデルに向いていて、どのタスクが生成型モデルに向いているのか、そしてこれらの概念が後の RAG、Prompt、微調整、Agent にどうつながるのか、です。
