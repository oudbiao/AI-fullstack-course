# 学習前ガイド：この章で扱うシーケンスラベリングとは何か

この章で解決するのは、次のことです。

> **文全体に1つのラベルを付けるのではなく、系列の各位置にラベルを付けること。**

## 歴史的背景：シーケンスラベリングはなぜ HMM から始まったのか？

この流れが最初にどう生まれたのかを知りたいなら、まずは定番の背景を押さえましょう。

- Transformer や BiLSTM が登場する前は、品詞付け、分かち書き、固有表現認識のようなタスクは、長い間 `HMM / CRF` という統計的な系列モデリングの手法に大きく依存していました。

初心者にとって、まず覚えておきたいのは次のことです。

> **シーケンスラベリングという主流は、もともと NLP の歴史の中でも非常に定番の「位置ごとの予測」手法だった。**

そのため、これから出てくる

- BIO ラベル付け
- BiLSTM + CRF

は、突然現れた新しいものではなく、より古い統計的系列モデリングの流れを引き継ぎながら発展してきたものです。

この歴史の流れをより正確に見たい場合は、先に [4.2 HMM、CRF とシーケンスラベリングの歴史的主流](./04-hmm-crf-history.md) を読んでから、NER と BiLSTM-CRF に進むのがおすすめです。

## 初心者と上級学習者はどう読むか

初心者がこの章を初めて学ぶときは、まず全体の流れと最小の実行例をつかみましょう。すべての細かい部分を一度に理解する必要はありません。この章が何を解決するのか、入力と出力は何か、最小プロジェクトをどう動かすのかが説明できれば、次へ進んで大丈夫です。

経験のある学習者は、この章を「理解の抜けを埋めること」と「実践的な練習」の場として使えます。境界条件、失敗例、評価方法、コードの再現性、そして前後の学習内容とのつながりに注目しましょう。読み終えたら、本章の内容を自分の作品の README や実験記録に残しておくとよいです。
