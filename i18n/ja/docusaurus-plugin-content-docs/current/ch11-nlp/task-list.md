# 段階学習タスクシート：自然言語処理

この段階の目標は、テキストタスクが、前処理・表現・モデリング・評価を経て、実際のアプリケーションにつながる流れを理解することです。大規模モデルがとても強くなった今でも、従来の NLP、テキスト注釈、タスク定義、エラー分析は、信頼できるテキストシステムを作るための基礎です。

## この段階で必ず完了するタスク

| タスク | 成果物 | 合格基準 |
| --- | --- | --- |
| テキスト前処理を理解する | テキストクリーニングスクリプト1本 | 分かち書き、大文字小文字、ストップワード、句読点、特殊文字を処理できる |
| テキスト表現の実験を完了する | 表現方法の比較記録1本 | BoW、TF-IDF、Embedding、事前学習モデルの表現を比較できる |
| テキスト分類タスクを完了する | 分類 Demo 1本 | ラベル、データ分割、指標、エラーサンプルを説明できる |
| 抽出または要約の練習を完了する | 情報抽出 / 要約のサンプル1本 | フィールド境界、事実一致性、評価方法を説明できる |
| 段階プロジェクトを完了する | テキスト理解の小規模プロジェクト1本 | 入出力、指標、失敗サンプル、README がある |

## おすすめの学習順序

まずテキストのクリーニングと表現を学び、そのあとで分類、系列ラベリング、事前学習モデル、プロジェクト実践を学びます。モデル名だけを見るのではなく、テキストタスクのラベル定義、サンプルの境界、評価指標、エラーの種類に注目しましょう。

NLP プロジェクトでは、「自然に見えるけれど事実と違う」という問題が特に起こりやすいです。要約、QA、情報抽出を行うときは、出典、証拠、失敗サンプルを残すことが大切です。

## AI 学習アシスタントプロジェクトとの関係

この段階では、AI 学習アシスタントにテキスト理解能力を追加できます。たとえば、学習に関する質問の分類、知識ポイントの抽出、要約の生成、復習テーマの認識、学習ログの構造化記録への変換などです。

おすすめの最小機能は、1つの学習質問を入力すると、所属する段階、キーワード、推奨章、信頼度を出力し、分類のエラーサンプルも記録することです。

## よくあるつまずき

よくある問題には、テキストのクリーニングをやりすぎて情報を失うこと、ラベルの境界が不明確でモデルが混乱すること、クラス不均衡で精度が高く見えてしまうこと、要約で重要な条件を落とすこと、抽出フィールドの型が安定しないことがあります。調査するときは、まず元のテキスト、ラベルルール、エラーサンプル、指標がタスク目標と合っているかを確認しましょう。

まず一度ガイド付きで動かしたい場合は、[実践：再現可能な NLP ミニパイプラインを作る](./ch07-projects/05-hands-on-nlp-workshop.md) を完了してください。そこで作る `nlp_workshop_run/outputs/`、`reports/*.json`、`reports/failure_cases.md` を、このステージの最小証拠として使えます。


## やさしい版 / 標準版 / チャレンジ版 タスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| やさしい版 | テキストラベルのサンプルを一式作成する | 初回学習の人、時間が少ない人、基礎から始めたい学習者 |
| 標準版 | 指標とエラーテキストを出力する | この段階の内容をポートフォリオに入れたい学習者 |
| チャレンジ版 | ラベル境界を作り直して前後の結果を比較する | 基礎があり、より強いプロジェクトの証拠を作りたい学習者 |

## この段階のバッジと Boss 戦

| 種類 | 内容 |
|---|---|
| Boss 戦 | テキストラベル審査官 |
| 解放できるバッジ | ラベル設計者、テキストエラー分析者 |
| 最小クリアの合言葉 | まず動かす、次に説明する、最後に失敗を記録する |
| 証拠の保存おすすめ場所 | スクリーンショット、ログ、失敗サンプル、評価表は `reports/`、`evals/`、`logs/` に保存する |

やさしい版を終えれば次に進んで大丈夫です。標準版まで完成したら、ポートフォリオに書くのがおすすめです。チャレンジ版は、余力があるときに取り組みましょう。

## 段階ポートフォリオの提出物

この段階の成果をポートフォリオに残したい場合は、少なくとも次のファイル、または同等の材料を残すことをおすすめします。

| 提出物 | 説明 |
| --- | --- |
| `text_cleaning.py` | テキストクリーニング、分かち書き、正規化、サンプル出力 |
| `label_guide.md` | ラベル定義、境界ケース、正例・負例、アノテーションルール |
| `classification_report.md` | 指標、混同行列、エラーサンプル、モデル比較 |
| `extraction_examples.jsonl` | 情報抽出または構造化出力のサンプル |
| `README.md` | プロジェクト目標、実行方法、入出力、評価、制約 |

これらの材料があると、NLP プロジェクトは「モデルがテキストを出力できる」状態から、「タスク定義が明確で、評価が信頼でき、失敗を振り返れる」状態へと進化します。

## 段階クリア問題

学習後には、次の問いに答えられるようになっているはずです。なぜテキストクリーニングがモデルの効果に影響するのか、TF-IDF と Embedding の違いは何か、なぜテキスト分類にはラベルルールが必要なのか、要約と抽出で事実一致性をどう確認するのか、どんなときに従来の NLP 手法が大規模モデルより適しているのか。

## 完了状態 Checklist

- [ ] 1 回のテキストクリーニングと表現方法の比較を完了できる。
- [ ] テキスト分類タスクのラベルと評価指標を定義できる。
- [ ] 分類、抽出、要約、または QA の小規模プロジェクトを完了できる。
- [ ] エラーサンプルを記録し、失敗がデータ、ラベル、モデル、生成のどれに由来するか説明できる。
- [ ] NLP の能力が RAG、Agent、または学習アシスタントプロジェクトをどのように支えるか説明できる。
