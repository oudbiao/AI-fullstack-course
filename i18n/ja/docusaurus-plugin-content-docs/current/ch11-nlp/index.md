# 11 自然言語処理（方向別選択）

![自然言語処理のメインビジュアル](/img/course/ch11-nlp-ja.png)

この段階で扱うのは、「モデルにテキストをどう扱わせるか」です。大規模モデルの時代では、多くの NLP の基礎はすでに LLM に組み込まれていますが、大規模モデル、テキストタスク、情報抽出、検索、QA システムをより深く理解したいなら、NLP は今でも非常に重要です。

## ストーリーで始める導入：モデルに人間の言葉の込み入った仕組みを読ませる

テキストは表のように整っておらず、画像のようにピクセルで固定されたものでもありません。ある単語は文脈によって意味が変わり、1つの文の中に感情、エンティティ、関係、意図が含まれることがあります。NLP の役割は、こうした一見ばらばらに見える文字情報を、モデルが計算・比較・分類・生成できる表現へ変換することです。

## インタラクティブ演習：1つの文から3層の情報を切り出す

ユーザーレビューを1つ選び、まず感情を判断し、次にその中にある人名・製品名・場所を見つけ、最後に本当に伝えたい意図を考えてみましょう。感情分類、固有表現認識、意図認識は、それぞれ NLP の異なるタスクに対応します。この練習は、「テキストを読む」ことを、モデル化できる問題に分解する助けになります。

## プロジェクトの小ネタ

この段階の小ネタ作品としては、「レビュー理解アシスタント」があります。レビューをまとめて入力すると、システムが自動で感情を判断し、キーワードを抽出し、テーマを要約し、代表サンプルも示します。これは従来の NLP と自然につながり、さらに LLM を使ったテキスト分析や RAG の文書理解へと発展させることもできます。

## この段階の位置づけ

| 情報 | 説明 |
|---|---|
| 対象者 | 深層学習の基礎を終え、テキストタスク、大規模モデルの原理、または NLP 分野をさらに学びたい人 |
| 予想学習時間 | 120～180 時間 |
| 前提条件 | 深層学習と Transformer の基礎を修了していること |
| 到達成果 | テキスト分類、QA システム、テキスト要約、または情報抽出のプロジェクト |

## 初心者向けの最小突破ルート

初心者はまず、テキストのクリーニング、分かち書き、Token、単語ベクトル、テキスト分類、事前学習モデルといった主線の概念を理解しましょう。最初からすべての従来型 NLP モデルを覚える必要はありません。テキスト分類かキーワード抽出のプロジェクトを1つ完成させ、テキストがどのようにモデル入力になるか説明できれば、最小突破といえます。

## 上級者向けの深掘りルート

経験のある学習者は、文脈表現、系列ラベリング、Seq2Seq、BERT、GPT、T5、そして Transformers のツールチェーンを深く学べます。さらに、従来手法、深層学習手法、LLM 手法を同じテキストタスク上で比較してみましょう。

## NLP と大規模モデルの関係

NLP は大規模モデルの重要な出発点の1つです。Token、Embedding、言語モデル、Seq2Seq、Attention、事前学習モデルといった概念は、大規模モデルの中でも引き続き使われています。NLP を学ぶのは古い方法にとどまるためではなく、大規模モデルの前に積み上げられた技術を理解するためです。

![NLP から大規模モデルへの技術主線図](/img/course/ch11-nlp-to-llm-backbone-ja.png)

## 初心者はまず何をし、上級者は何をするか

この段階を初めて学ぶ人は、まずテキストタスクの主線、つまり分かち書き、表現、分類、抽出、生成、評価をつかみましょう。最初からモデル名に入り込みすぎず、入力テキストがどうやって計算可能な特徴になるのかを理解することが大切です。

経験のある学習者は、タスクの境界と評価に重点を置けます。ラベル体系は明確か、抽出結果をどう正解と判定するか、生成タスクをどう評価するか、従来の NLP と大規模モデルのどちらを選ぶか、を考えてみましょう。

## この段階の学習パス

11.1 節では、NLP マップ、テキスト前処理、テキスト表現を含むテキスト処理の基礎を学びます。

11.2 節では、単語埋め込みと言語モデルを学び、単語ベクトル、文脈表現、言語モデルの基本思想を理解します。

11.3 節では、テキスト分類を学びます。これは多くの業務テキストタスクの入門プロジェクトです。

11.4 節では、固有表現認識や BiLSTM-CRF などを含む系列ラベリングを学びます。

11.5 節では、Seq2Seq と Attention を学び、機械翻訳と生成タスクの重要な歴史的な流れを理解します。

11.6 節では、BERT、GPT、T5、そして Transformers ライブラリを含む事前学習済み言語モデルを学びます。

11.7 節では、NLP の総合プロジェクトを完成させます。

## 学習後にできるようになること

- テキストがモデルで扱える表現にどう変わるか説明できる
- 単語ベクトル、文脈表現、言語モデルの違いを理解できる
- テキスト分類、系列ラベリング、要約系プロジェクトを実行できる
- BERT、GPT、T5 などの事前学習モデルのおおまかな違いを説明できる
- 大規模モデルの tokenizer、embedding、文脈モデリングをよりスムーズに理解できる

## よくある誤解

LLM があれば NLP はまったく不要、と思わないでください。LLM によって多くのタスクはやりやすくなりましたが、テキストのクリーニング、タスク定義、ラベル形式、評価指標、情報抽出、意味検索は、今でも NLP 的な考え方が欠かせません。

また、最初から従来手法の細部にはまりすぎないことも大切です。最初の学習では、「テキストをどう表すか」「タスクをどうモデル化するか」「事前学習がどうパラダイムを変えたか」を重点的に押さえましょう。

## NLP エラー劇場：テキストタスクはまずラベルと境界を確認

分類モデルが近いカテゴリをよく混同するなら、まずラベル定義が重なっていないか確認しましょう。抽出結果で情報が抜けるなら、アノテーション規則とテキストの分割を確認します。生成結果が自然でも信頼できないなら、事実確認、引用、または人手レビューを必ず追加する必要があります。

## 最小実行実験：テキストを評価可能なフィールドに変える

この段階の最小実験は、テキスト分類か情報抽出から始められます。20件の短いテキストを用意し、2～3個のラベルまたはフィールドを定義し、ルール・従来モデル・LLM のいずれかで構造化結果を出力し、予測の正誤を記録してみましょう。

```python
samples = [
    {"text": "RAG が正しい文書を検索できない", "label": "retrieval"},
    {"text": "JSON 出力から required フィールドが抜けている", "label": "format"},
]

for item in samples:
    print(item["text"], "=>", item["label"])
```

NLP プロジェクトの鍵は、流ちょうな文章を出すことではなく、タスクの境界、ラベル定義、フィールドの schema、評価方法が明確かどうかです。

## NLP 失敗事例集：まずラベル、フィールド、事実根拠を確認

| 現象 | よくある原因 | 特定方法 | 修正方向 |
|---|---|---|---|
| 分類カテゴリの混同 | ラベル定義が重なっている、またはサンプル不足 | 混同行列と誤り例を見る | ラベル説明を書き直し、境界ケースを増やす |
| 抽出フィールドが不安定 | schema が不明確、またはテキスト境界が曖昧 | 原文と抽出 JSON を比較する | フィールド定義、正例・負例、検証ルールを補う |
| 要約は流ちょうだが不正確 | 生成内容が原文から離れている | 各文の出典を確認する | 引用チェックと事実一致チェックを追加する |
| QA で分からないのに乱答する | 知識の境界と拒否ルールがない | 答えのない質問を用意する | 答えなし処理と人手レビューを追加する |


## 段階プロジェクト

ベーシック版は、テキストクリーニング、特徴表現、学習、評価を含むテキスト分類プロジェクトを完成させることです。スタンダード版では、情報抽出、要約、または QA タスクを加え、異なるモデルや Prompt の効果を比較します。チャレンジ版では、レビュー理解アシスタントや分野別文書抽出システムを作り、感情、エンティティ、テーマ、代表サンプルの分析結果を出力できます。

大きなプロジェクトを選ぶ前に、まず具体的な baseline を動かしたい場合は、[11.7.6 実践：再現可能な NLP ミニパイプラインを作る](./ch07-projects/05-hands-on-nlp-workshop.md) から始めてください。ローカルスクリプトでデータファイル、TF-IDF 分類、検索 QA、抽出型要約、情報抽出結果、指標、失敗レポートを生成できます。

より細かい学習ペースを見たい場合は、[11.0 学習ガイド：自然言語処理をどう学ぶと最も混乱しにくいか](./study-guide.md) を読んでください。

## AI 学習アシスタントとの一貫プロジェクトとの関係

この段階では、AI 学習アシスタントに従来の NLP 能力を追加できます。テキスト分類、情報抽出、要約、事前学習モデルの比較です。一貫プロジェクトの流れで学んでいるなら、この段階が終わる時点で少なくとも1回は版を記録することをおすすめします。つまり、この段階で何が追加されたか、どう実行するか、入出力の例は何か、どんな問題に出会ったか、次にどう改善するか、を残しましょう。
