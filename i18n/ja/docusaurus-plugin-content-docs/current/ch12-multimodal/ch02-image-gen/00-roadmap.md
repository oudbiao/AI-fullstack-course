# 学習ガイド：画像生成のこの章では何を学ぶのか

この章で扱うのは、画像は分類・検出・理解できるだけでなく、モデルが少しずつ生成・編集・制御することもできる、ということです。

前のコンピュータビジョンがより「画像を見て理解する」ことを重視していたのに対して、画像生成は「画像をつくる」ことをより重視します。これはタスクの目的が違うだけでなく、システムの流れも違います。分類はラベルを出力し、検出は枠を出力し、セグメンテーションは画素領域を出力します。一方、生成タスクはテキスト、ラフスケッチ、参照画像、姿勢、スタイル条件などを手がかりに、新しい画像結果を少しずつ構築していきます。

## この章がカリキュラム全体の中でどこにあるか

あなたはすでに、ビジョン基礎、多モーダル基礎、大規模モデルの応用を学んできました。画像生成の章では、これらの能力を AIGC の制作ワークフローにつなげます。テキストが意図を伝え、画像モデルが生成を担い、制御モジュールが構図を制約し、微調整の方法がスタイルやキャラクターの一貫性を支え、後処理とレビューが納品品質を担保します。

画像生成は「プロンプトを書いて、あとは画像が出るのを待つ」だけではありません。実際のワークフローには、要件分解、プロンプト設計、参考素材、制御条件、生成の反復、部分修復、スタイル統一、著作権と安全性のチェックが含まれます。

## この章で本当に解決したい問題

この章では、次の5つの問いに答えます。拡散モデルはなぜ「ノイズ付加とノイズ除去」で画像を生成できるのか。Stable Diffusion のテキストエンコーダ、U-Net、VAE、latent 空間はそれぞれ何をしているのか。プロンプト、ネガティブプロンプト、サンプリングステップ数、seed、CFG などのパラメータは結果にどう影響するのか。ControlNet、image-to-image、局所再描画、参照画像はどうやって制御性を高めるのか。LoRA や DreamBooth のような微調整手法は、なぜ特定のスタイルやキャラクターを学習できるのか。

初心者が最も誤解しやすいのは、画像生成能力はモデル名だけで決まる、という考え方です。実際には、生成品質はワークフロー設計に大きく左右されます。つまり、プロンプト、制御条件、素材の出どころ、パラメータの選び方、後処理、人手での選別が重要です。

## 初心者におすすめの学習順序

まずは拡散モデルの直感を理解することをおすすめします。学習時にはノイズの中から画像を復元する方法を学び、生成時にはノイズを少しずつ除去して結果を得る、という流れです。次に Stable Diffusion の構造を見て、テキスト条件、latent 表現、U-Net のノイズ除去、VAE のデコードを1枚の図の中でつなげて理解します。そのあと、文生成画像、画像生成画像、局所再描画、スタイル変換などのよくある応用ワークフローを学びます。最後に、制御条件と微調整を見ましょう。最初から複雑なプラグインを追いかける必要はありません。

## この章で押さえるべき主線

この章の主線は、次のようにまとめられます。画像生成は「意図の表現 + 条件制御 + 段階的なノイズ除去 + 編集とレビュー」の一連の流れです。

この線が理解できると、同じプロンプトでも結果が変わる理由、参照画像や制御画像で構図を変えられる理由、LoRA でスタイルに影響を与えられる理由、生成結果に後処理とレビューが必要な理由がわかります。

## この章と次の章との関係

画像生成は、動画生成、デジタルヒューマン、AIGC 総合プロジェクトの基礎です。動画生成は、時間軸に沿って連続的に生成し、一貫性を保つものと考えられます。デジタルヒューマンには、画像、音声、動作、アイデンティティの一貫性が必要です。総合プロジェクトでは、画像生成を文章作成、レビュー、出力、製品画面に接続する必要があります。

この章が安定していないと、後でよくある問題として、プロンプトは書けるのに結果が不安定な理由がわからない、ControlNet と LoRA が解決する問題が別だとわからない、素材の利用許諾や肖像権リスクを見落とす、モデル出力を最終成果物だと思って編集やレビューの流れを作らない、などがあります。

## 初心者と上級学習者はどう読むか

初心者がこの章を初めて学ぶときは、まず主線と最小実行例をつかみましょう。すべての細部を一度で理解する必要はありません。この章が何を解決するのか、入力と出力は何か、最小プロジェクトをどう動かすかが説明できれば、先へ進んで大丈夫です。

経験のある学習者は、この章を穴埋めとエンジニアリング練習として読むとよいでしょう。境界条件、失敗例、評価方法、コードの再現性、そして前後の段階との接続に注目します。読み終えたら、本章の内容を自分の作品の README や実験記録に整理しておくのがおすすめです。

## 学習時間と難易度の目安

| 学習方法 | 目安時間 | 目標 |

| じっくり練習 | 半日～1 日 | エラー分析、比較実験、またはプロジェクト README の記録を補足する |

## この章の自己チェック問題

| 自己チェック問題 | 合格基準 |

## この章のミニプロジェクトの出口

この章を学び終えたら、「講義の表紙生成ワークフロー」を作るのがおすすめです。講義テーマ、対象学習者、スタイル要件、サイズ仕様を入力すると、システムがプロンプトを生成し、3～5枚の候補表紙を出力します。そして、それぞれの画像のプロンプト、パラメータ、良い点・悪い点、修正案を記録します。

さらに発展させるなら、参照画像、ControlNet による構図制約、または LoRA によるスタイル制御を加え、最終出力の前に著作権とコンテンツ安全性のチェックリストを追加できます。

## 合格基準

この章の終わりには、拡散モデルのノイズ付加とノイズ除去の直感をやさしい言葉で説明でき、Stable Diffusion におけるテキスト条件、U-Net、VAE、latent 空間のおおまかな役割を説明でき、文生成画像、画像生成画像、局所再描画、ControlNet、LoRA の使い分けができるようになっているはずです。

プロンプト、パラメータ記録、生成の反復、人手による選別、後処理、レビューの手順を含む画像生成ワークフローを設計できれば、AIGC の画像分野の入門基準に到達したと言えます。
