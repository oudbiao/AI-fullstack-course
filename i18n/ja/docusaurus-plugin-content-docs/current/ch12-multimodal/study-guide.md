# 学習ガイドとタスクシート：マルチモーダルと AIGC をどう学べば最も混乱しにくいか

`第 12 章 AIGC とマルチモーダル` に入って、画像、音声、動画、デジタルヒューマン、そしてさまざまな新しいモデルがバラバラに見えるときは、まず Demo を片っ端から追わないでください。マルチモーダルを最初に学ぶときは、異なるモダリティがどのように同じシステムに入るのかを理解することが大切です。

## この段階の総合原則

マルチモーダルと AIGC は、1 本のシステムの流れとして捉えましょう。つまり、異なるモダリティを表現にエンコードし、モデルが理解または生成を行い、最後に制作、編集、審査、納品のワークフローにつなげます。

![多モーダル学習ガイドのワークフロー図](/img/course/ch12-study-guide-modal-workflow-map-ja.png)

## この段階で必ず完了するタスク

このタスクで、マルチモーダル作品を提出できる形にします。大事なのは 1 回の生成結果の見た目だけではなく、入力、バージョン、レビュー、権利の境界、エクスポートが明確かどうかです。

| タスク | 成果物 | 通過基準 |
|---|---|---|
| マルチモーダルの入出力を理解する | マルチモーダルの処理フロー図 1 枚 | テキスト、画像、音声、動画、ドキュメントがどのようにシステムへ入るか説明できる |
| 画像理解または画像生成を動かす | 最小 Demo 1 つ | 入力、Prompt、出力、手動選別の結果を記録できる |
| ドキュメントまたはスクリーンショット理解を完成する | マルチモーダル解析サンプル 1 つ | 元のページ番号、領域、またはスクリーンショットの証拠を残せる |
| レビューと提出フローを追加する | レビュー用チェックリスト 1 つ | 著作権、肖像、センシティブ内容、事実リスクを確認できる |
| 再現可能なクリエイティブパッケージを実行する | `multimodal_workshop_run/` 証拠フォルダ | prompts、assets、書き出しプレビュー、安全レビュー、失敗例を説明できる |
| 段階プロジェクトを 1 つ完成する | マルチモーダル作品またはクリエイティブ作業台 | 入出力、バージョン管理、レビュー、エクスポート、README がある |

## おすすめの学習順序

最初の 1 周目は、まずマルチモーダル基礎を学びます。画像とテキストのアライメント、ビジョン・ランゲージ・モデル、マルチモーダル入力と出力、典型的な応用を理解してください。

2 周目は画像生成です。拡散モデル、Stable Diffusion、Prompt、ControlNet、LoRA、そしてよく使うワークフローを重点的に理解します。

3 周目は動画生成と音声生成です。時系列コンテンツがなぜより複雑なのか、また TTS、デジタルヒューマン、動画生成がどう組み合わさるのかを理解します。

4 周目は最先端トレンドと倫理です。AIGC は著作権、肖像権、偏見、虚偽情報、規制の境界に直接関わるため、技術的な出来栄えだけを見てはいけません。

5 周目は総合プロジェクトを行い、生成能力を使えるクリエイティブ・ワークフローとしてまとめます。

## 学習ペースのおすすめ

| 内容の種類 | 目安時間 | 学習目標 |
|---|---|---|
| マルチモーダル基礎 | 4～8 時間 | 異なるモダリティがどうアライメントされるかを理解する |
| 画像生成 | 8～16 時間 | 1 つの画像生成ワークフローを動かせるようにする |
| 動画 / 音声 / デジタルヒューマン | 8～20 時間 | 時系列生成と素材の流れを理解する |
| 倫理とコンプライアンス | 3～6 時間 | コンテンツ安全性と著作権意識を身につける |
| 総合プロジェクト | 16～32 時間 | 生成系プロダクトの試作品を完成させる |

## この段階のプロジェクト方針

最初のプロジェクトは、画像生成ワークフローにするのがおすすめです。たとえば、テーマに基づいてポスター、表紙、講座用の挿絵を生成します。

2 つ目のプロジェクトは、画像とテキストのマルチモーダル Q&A にするとよいです。たとえば、画像をアップロードして、モデルに説明、分類、または注釈の生成をさせます。

3 つ目のプロジェクトは、クリエイティブ・コンテンツ・プラットフォームの試作にできます。テーマを入力すると、コピー文、挿絵、音声、または動画スクリプトを生成し、さらに審査とエクスポートの流れを追加します。

大きな卒業プロジェクトを選ぶ前に、まず [再現可能なマルチモーダル・クリエイティブパッケージ](./ch05-projects/02-hands-on-multimodal-workshop.md) を動かしてください。本ステージの baseline として、1 本のスクリプト、creative brief、Prompt version、生成 SVG 資産、storyboard、安全レビュー、書き出しプレビュー、失敗例を確認できます。

## よくあるつまずきポイント

最も多いつまずきは、新しいモデルばかり追ってワークフローを理解しないことです。生成系プロダクトは 1 つのモデルだけでは足りません。Prompt、素材、制御条件、後処理、審査、納品も必要です。

2 つ目のつまずきは、著作権と肖像権のリスクを軽視することです。AIGC プロジェクトでは、最初から素材の出所、利用許諾、人物の肖像、コンテンツ安全性を考える必要があります。

3 つ目のつまずきは、マルチモーダルを「画像に文字を足したもの」と考えてしまうことです。本当のマルチモーダルシステムでは、異なるモダリティ同士をどうアライメントし、参照し、編集し、協力してタスクを完成させるかを考えなければなりません。

## 段階ポートフォリオ成果物

![マルチモーダルレビューと書き出しフロー図](/img/course/ch12-workshop-review-export-map-ja.png)

このステージの成果を作品集に残したい場合は、少なくとも次のファイル、または同等の資料を残すことをおすすめします。

| 成果物 | 説明 |
|---|---|
| `multimodal_pipeline.md` | 入力、解析、生成、レビュー、エクスポートのシステムフロー |
| `prompts/` | マルチモーダル Prompt、参考画像、ネガティブプロンプト、バージョン記録 |
| `outputs/` | 生成結果、候補バージョン、手動選別の理由、最終提出ファイル |
| `safety_review.md` | 著作権、肖像、センシティブ内容、事実性、使用範囲の確認 |
| `README.md` | プロジェクト目標、実行方法、素材の出典、サンプル出力、制限の説明 |

これらの資料があると、マルチモーダルプロジェクトは「見た目の良い結果を 1 つ作る」から、「要件、出典、バージョン、レビュー、提出の境界を説明できる完成度の高い作品」へとレベルアップします。

## 段階通過質問

この段階を学び終えたら、テキスト、画像、音声、動画を受け取り、理解または生成を行うマルチモーダルシステムがどう動くかを説明できるはずです。

先へ進む前に、次の質問に答えられるか確認してください。

- マルチモーダル入力でなぜ出典を残す必要があるのか。
- 画像生成でなぜ Prompt とバージョンを記録するのか。
- PDF/スクリーンショット理解でなぜ証拠を確認するのか。
- コンテンツ生成でなぜ手動レビューが必要なのか。
- どの場面でマルチモーダル機能を RAG や Agent に接続するのが適しているのか。

入力、生成、編集、審査、エクスポートの手順を備えた AIGC の小さなプロダクト試作を作れれば、方向性としての入門基準を満たしたことになります。
