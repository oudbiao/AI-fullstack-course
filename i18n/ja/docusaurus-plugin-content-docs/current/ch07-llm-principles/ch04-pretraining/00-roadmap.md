# 学習ガイド：事前学習技術のこの章では何を学ぶのか

## 本章の位置づけ

この章が扱う核心的な問題は、モデルの汎用能力はいったいどこから生まれるのか、ということです。これまでに Transformer の構造、Prompt、そして大規模モデルの基礎を学んできましたが、「モデルが大きい」「データが多い」だけを知っていても、なぜ同じモデルがコードを書けて、文書を要約し、質問に答え、推論し、さまざまなタスクに適応できるのかは、まだ理解しにくいはずです。

事前学習技術は、この問いに答えるための鍵です。大量のデータ、学習目標、モデル構造、計算資源のエンジニアリング、データガバナンスをつなぎ合わせ、モデルにまず汎用的な表現を学習させ、その後で Prompt、ファインチューニング、RAG、または Agent によって具体的なアプリケーションへ接続します。

## この章がコース全体の中で持つ位置

![事前学習技術の章の関係図](/img/course/ch07-pretraining-chapter-flow-ja.png)

この章は、前半で学んだ Transformer の深掘りを受け継ぎ、後半の Prompt、ファインチューニング、アラインメント、そして LLM アプリケーションの土台にもなります。実際にゼロから大規模モデルを学習する必要はありませんが、データが能力の上限にどう影響するのか、目的関数がモデルの振る舞いをどう形作るのか、学習エンジニアリングがなぜコスト・安定性・再現性を左右するのかは説明できるようになる必要があります。

## 本章の学習の主線

![事前学習データ・目的・エンジニアリングの三角図](/img/course/ch07-pretraining-data-objective-engineering-map-ja.png)

学ぶときは、これらを論文用語の一覧として覚えるのではなく、常に「この設計はどんな失敗を解決しているのか？」と問い続けてください。たとえば、重複排除は記憶のしすぎやデータリークを減らすため、品質フィルタリングは有効な学習シグナルを高めるため、混合精度は学習コストを下げるため、チェックポイントは長時間の学習が失敗したときに復旧できなくなるのを防ぐためです。

## 本章の各節の読み方

| 小節 | 重点 प्रश्न | 学び終わったあとに説明できるようになること |
|---|---|---|
| 事前学習データ | モデルはそもそもどんなデータから学ぶのか | データソース、クリーニング、重複排除、著作権、安全性、バイアスがモデル能力にどう影響するか |
| 事前学習方法 | なぜモデルは予測タスクから汎用能力を学べるのか | 自己回帰、マスク付き言語モデル、マルチタスク目標がそれぞれどんなモデル形態に向いているか |
| 学習エンジニアリング | なぜ大規模モデルの学習はシステム工学なのか | 分散学習、メモリ、スループット、チェックポイント、ログ、障害復旧の重要性 |

初心者なら、最初の一周では「データが上限を決め、目標が学び方を決め、エンジニアリングが最後まで学習できるかを決める」という点だけを押さえれば十分です。深層学習の経験があるなら、学習の安定性、データガバナンス、そしてその後のファインチューニング/RAG/アラインメントとの境界に注目するとよいでしょう。

## よくある誤解

1つ目の誤解は、事前学習は「大量の Web ページをモデルに食べさせるだけ」だと思ってしまうことです。実際の事前学習は、データとエンジニアリングのパイプラインに近いものです。収集、フィルタリング、重複排除、トークン化、配分、学習、監視、評価のすべてが最終的なモデルに影響します。

2つ目の誤解は、モデルが大きければ必ず良いと思ってしまうことです。モデルサイズは要素のひとつにすぎず、データ品質、学習目標、推論コスト、デプロイ環境、タスクの種類も同じくらい重要です。講義の QA アシスタントには、必ずしも最大のモデルは必要ありません。むしろ、良い RAG、引用、評価セットのほうが重要かもしれません。

3つ目の誤解は、事前学習、ファインチューニング、RAG をごちゃ混ぜにしてしまうことです。事前学習は汎用能力の獲得、ファインチューニングは振る舞いパターンの変更や強化、RAG は外部知識への接続を担当します。3つは互いの代替ではなく、それぞれ異なる階層の問題を解決します。

## 本章の小プロジェクトの出口

この章では、大規模モデルを実際に学習する必要はありません。おすすめは「事前学習の意思決定メモカード」を作ることです。たとえば、講義の QA アシスタント、コードアシスタント、学習計画アシスタントのようなモデルの利用シーンを1つ選び、それにどんなデータが必要か、どのデータは使ってはいけないか、ファインチューニングが必要か、RAG のほうが適しているか、そして公開後にどんな品質指標を観察すべきかを整理します。

基礎版は Markdown 1ページで十分です。標準版では、重複テキストの削除、短すぎるサンプルの除外、token 長の集計といったデータクリーニングの例を追加できます。チャレンジ版では、小さな 2つのモデル、または 2つの Prompt/RAG 設定を比較し、「事前学習の土台が違うとどんな差が出るのか」を説明してみましょう。

## 合格基準

この章を学び終えたら、あなた自身の言葉で次のことを説明できるはずです。なぜ事前学習によって汎用能力が生まれるのか、なぜ単純なデータ量よりもデータ品質のほうが重要なのか、なぜ学習エンジニアリングがモデルの安定的な出力を左右するのか、そしてなぜ多くのアプリケーション課題は再事前学習ではなく別の方法で解くべきなのか、ということです。
