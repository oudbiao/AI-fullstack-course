# 7 大モデル原理、Prompt と微調整

![大モデル原理メインビジュアル](/img/course/ch07-llm-principles-ja.png)

このステージで解決するのは、「大モデルの能力はどこから来るのか、どうやって制御し、適応させるのか」です。単にいくつかのモデル名を学ぶのではなく、Token、Embedding、Transformer、事前学習、Prompt、微調整、整合の関係を理解します。

## ストーリーで導入：チャットボットの魔法箱を分解する

大モデルは魔法のように見えます。1文入力するだけで、コードを書いたり、文書を要約したり、役を演じたり、タスクを計画したりできます。ですが、このステージでやることは、その魔法箱を分解することです。まずテキストは Token に分割され、Token はベクトルになり、Transformer が文脈の中で関係を計算し、事前学習によってモデルは言語能力を獲得し、さらに Prompt と微調整によって、その能力を具体的なタスクへ導いていきます。

## 体験練習：実験するように Prompt を書く

「この Prompt は良いか？」とだけ考えるのではなく、毎回1つの変数だけを変えてみましょう。たとえば、役割を付けるか、例を与えるか、手順を分けて書かせるか、出力形式を制限するか、評価基準を与えるか、です。異なる版の入力と出力を保存していくと、自分だけの Prompt 実験ノートが少しずつ形になります。

## プロジェクトの特典

このステージの特典作品は、「Prompt 実験マップ」です。同じタスクを選び、少なくとも5種類の指示方法を設計して、出力品質、安定性、形式の制御しやすさ、コストを比較します。後で RAG、Agent、構造化出力、ツール呼び出しを学ぶとき、このマップがあなたのプロンプトエンジニアリングの出発点になります。

## 初心者向けの最短クリアコース

初心者はまず、Token、Embedding、Attention、Transformer、事前学習、Prompt、微調整、整合の関係を理解しましょう。最初から自分で大モデルを学習させる必要はありません。安定した Prompt を設計でき、モデルに構造化された結果を出させることができ、タスクが Prompt 向きか、RAG 向きか、微調整向きかを判断できれば、最短クリア達成です。

## 応用学習コース

経験のある学習者は、Transformer の変種、事前学習データ、LoRA/QLoRA、指示微調整、RLHF、モデル評価まで深く学べます。さらに、ある分野タスクに対する微調整方案を設計し、データ形式、学習コスト、評価指標、リスクの境界を明確にしてみましょう。

## 大モデルの AI 史における位置づけ

大モデルは突然現れたわけではありません。深層学習、NLP、Transformer、事前学習という流れを受け継いでいます。本当の変化は、モデル規模、データ規模、指示への整合によって、言語モデルが「単一の NLP タスクをこなすもの」から「言語インターフェースを通じて大量のタスクをこなすもの」へ変わったことです。

![大モデル能力の源流図](/img/course/ch07-llm-capability-backbone-ja.png)

## 初学者は何を先にやるか、上級者は何をやるか

初めてこのステージを学ぶ人は、大モデルを「文脈に応じてタスクをこなすインターフェース」として捉えるところから始めましょう。まずは安定した Prompt、構造化出力、簡単な評価を練習し、その後で事前学習、微調整、整合の背景を理解します。

経験のある学習者は、どの解決策を選ぶかに重点を置くとよいです。いつ Prompt で十分なのか、いつ RAG が必要なのか、いつ微調整を考えるのか、どうやって評価セットを設計して効果を確かめるのか。目標はモデル名を追いかけることではなく、実際の課題に対して適切な大モデルの方法を選べるようになることです。

## このステージの学習ルート

7.1 節では、tokenizer、embedding、事前学習済みモデル、HuggingFace の簡単な体験を含む NLP の核心を速習します。

7.2 節では、大規模言語モデルの全体像を学び、LLM の発展史、核心概念、業界構造を理解します。

7.3 節では Transformer を深く学び、特にアーキテクチャ、変種、高効率 Attention、スケール計算を理解します。

7.4 節では、データ、学習方法、エンジニアリング上の課題を含む事前学習技術を学びます。

7.5 節では Prompt Engineering を学び、入力の組み立てによってモデルの挙動をどう導くかを理解します。

7.6 節では微調整を学び、特に LoRA、QLoRA、PEFT、データラベリングを理解します。

7.7 節では RLHF と整合を学び、なぜモデル能力が高くても、それだけでは信頼性・制御性・安全性があるとは限らないのかを理解します。

## 学び終えたらできるようになること

- Token、Embedding、Attention、コンテキストウィンドウの基本的な意味を説明できる
- 事前学習、指示微調整、Prompt、微調整の違いをはっきり説明できる
- 基本的な Prompt を設計し、モデルに構造化結果を出力させられる
- あるタスクが Prompt、RAG、微調整のどれに向いているか判断できる
- LoRA/QLoRA の用途と適用範囲を理解できる
- 今後の LLM アプリ、RAG、Agent システムのために、モデル挙動の直感を築ける

## よくある誤解

大モデルを「より大きい検索エンジン」や「知識を持つデータベース」と考えないでください。大モデルの本質は、あくまで文脈に基づいて token を生成するモデルです。間違った内容を生成することもありますし、Prompt、文脈、タスク定義が不明確だと、動作が不安定になることもあります。

また、最初から微調整を追いかける必要もありません。多くのアプリケーション課題は、まず Prompt、構造化出力、RAG、システム設計で解決すべきです。微調整は通常、最初の一手ではありません。

## 大モデル失敗劇場：回答が不安定なとき、まず何を確認するか

モデルの出力がずれているなら、まずタスク説明が明確か、出力形式に例があるか、文脈に矛盾情報が入っていないかを確認します。構造化出力がよく失敗するなら、まず項目数を減らし、例を増やし、パース検証を入れます。それでも不安定なら、単発の体験で判断せず、固定の問題集で Prompt、RAG、微調整の案を比較しましょう。

## 1回目の読み方：必読、プロジェクト参照、選修の深掘り

| 読み方ラベル | 推奨章 | 学習目標 |
|---|---|---|
| 必読 | NLP 速習、LLM 核心概念、Prompt 基礎、構造化出力 | まずモデルの入出力、文脈、タスク表現を理解する |
| プロジェクト参照 | Prompt 実践、微調整概観、データラベリング、ステージプロジェクト | Prompt 比較や分野適応プロジェクトを行うときに重点的に見る |
| 選修の深掘り | Transformer 深掘り、事前学習エンジニアリング、LoRA/QLoRA、RLHF と整合 | モデル理解、微調整、評価の方向に進みたいときに深掘りする |

1回目は、Transformer や事前学習の細部をすべて暗記対象にしなくて大丈夫です。より大切なのは、あるタスクに Prompt、RAG、微調整、Agent のどれを使うべきかを判断できることです。

## ステージ復習カード：モデル原理からタスク適応まで

このステージを学び終えたら、次の表で「大モデルはどう生まれ、どう使い、どう適応させるのか」を本当に理解できているか確認しましょう。

| 復習問題 | あなたが答えられるべきこと |
|---|---|
| Token と Embedding | なぜテキストを先に token に分け、それをベクトルに変えるのか？ |
| Transformer | Attention メカニズムはなぜ文脈関係を扱えるのか？ |
| 事前学習 | モデル能力は主にデータと学習目標から何を学んだのか？ |
| Prompt | どの問題なら、より明確なタスク説明で先に解決できるのか？ |
| 微調整 | どの問題が、知識更新ではなく長期的な挙動適応に当たるのか？ |
| 整合 | なぜ「モデルが答えられる」ことは「モデルが信頼でき、安全で、意図に合う」ことと同じではないのか？ |

このステージの本当の出口は、たくさんのモデル名を暗記することではありません。あるタスクが本当に Prompt、構造化出力、RAG、微調整、それとも後の Agent システムで解決すべきかを判断できることです。

## ステージプロジェクト

ベーシック版は、Prompt 比較実験を完成させ、通常の指示、役割指示、手順付き指示、構造化出力の違いを記録します。スタンダード版では、同じタスクを中心に Prompt 実験マップを作り、安定性、形式制御、コスト、エラーの種類を比較します。チャレンジ版では、分野微調整の方案、または最小限の微調整実験を設計し、データ源、ラベリング規範、学習方法、評価方法、安全リスクを説明します。

プロジェクトを選ぶ前に一度通しで動かしたい場合は、[7.8.4 実践：第 7 章フルワークショップ](./ch08-projects/03-stage-hands-on-workshop.md) から始めてください。オフラインで実行できる流れの中で、token、Prompt 版、構造化出力の検証、手法選択、失敗証拠をまとめて確認できます。

もっと細かい学習リズムを見たい場合は、[7.0 学習ガイド：大モデル原理をどう学べば混乱しにくいか](./study-guide.md) を読んでください。
