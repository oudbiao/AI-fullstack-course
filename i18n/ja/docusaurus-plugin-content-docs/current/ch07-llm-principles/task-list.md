# 段階学習タスクシート：大規模モデルの原理、Prompt、微調整

この段階の目標は、大規模モデルがなぜ文章を生成できるのか、Prompt が出力にどう影響するのか、そしていつ構造化出力、RAG、微調整が必要になるのかを理解することです。Prompt を「おまじない」だと思わず、バージョン管理でき、テストでき、振り返りできる工程入力として扱いましょう。

## この段階で必ず完了するタスク

| タスク | 成果物 | 合格基準 |
| --- | --- | --- |
| 大規模モデルの基本原理を理解する | LLM ワークフロー図 1 枚 | token、文脈、確率生成、Transformer の関係を説明できる |
| Prompt 基礎演習を終える | Prompt 比較サンプル一式 | 役割、文脈、例、制約が出力に与える影響を比較できる |
| 構造化出力を実装する | JSON 出力デモ 1 つ | フィールドを定義し、型を検証し、解析失敗を処理できる |
| RAG と微調整の境界を理解する | 技術選定表 1 枚 | いつ Prompt、RAG、微調整、ルールを使うべきか説明できる |
| 段階プロジェクトを完成する | Prompt アシスタント、または振り返りカード生成器 1 つ | Prompt のバージョン、固定入力、出力比較、失敗サンプルがある |

## おすすめの学習順序

まず token、文脈、生成メカニズムを理解してから、Prompt 基礎、応用 Prompt、構造化出力、微調整の境界を学びましょう。最初からモデルを訓練しようとせず、まずは「モデル出力をどう安定して呼び出し、どう制約するか」を固めることが大切です。

Prompt を変更するたびに、必ずバージョンとテストサンプルを記録しましょう。Prompt エンジニアリングは、感覚で長い文章を書くことではありません。変数を制御し、出力を比較し、失敗を記録し、うまくいった変更を定着させる作業です。

## AI 学習アシスタントプロジェクトとの関係

この段階は、AI 学習アシスタントの v0.7 Prompt アシスタントに対応します。学習記録に基づいて、学習計画、知識カード、振り返り要約、または間違い問題の解説を生成し、Prompt のバージョンと失敗サンプルを記録できます。

最小機能としては、学習テーマと現在のレベルを入力すると、構造化された学習提案を出力すること、Prompt のバージョンを保存すること、固定サンプルで出力の安定性をテストすることをおすすめします。

## よくあるつまずき

よくある問題には、Prompt を長くしても効果が安定しない、JSON 出力に余計な説明が混ざる、フィールド型がずれる、モデルが存在しない資料をでっち上げる、Prompt を変えたあとに昔のサンプルの結果が悪くなる、などがあります。調べるときは、入力、Prompt のバージョン、モデルの生出力、解析ロジックを先に確認し、最終表示テキストだけを見ないようにしましょう。


## やさしい版 / 標準版 / チャレンジ版のタスク

| 難易度 | やること | 向いている人 |
|---|---|---|
| やさしい版 | モデルに固定フォーマットで 5 回出力させる | 1 回目の学習、時間が少ない人、入門者 |
| 標準版 | schema で 10 個の固定入力を検証する | この段階を作品集に入れたい人 |
| チャレンジ版 | 2 つの Prompt バージョンを比較し、改善記録を書く | 基礎があり、より強い実績を作りたい人 |

## この段階の作品集提出物

この段階の成果を作品集に残したいなら、少なくとも次のファイル、または同等の資料を残すことをおすすめします。

| 提出物 | 説明 |
| --- | --- |
| `prompts/` | Prompt テンプレート、バージョン番号、適用タスク、変更説明を保存する |
| `prompt_eval_cases.csv` | 固定入力、期待出力の要点、実際の出力、評価を記録する |
| `structured_output_schema.json` | 構造化出力のフィールド、型、必須項目、列挙値を定義する |
| `failure_cases.md` | JSON 解析失敗、フィールド欠落、幻覚、スタイルのずれなどのサンプルをまとめる |
| `README.md` | プロジェクトの目的、実行方法、Prompt バージョン、評価結果、制限を書く |

これらの材料があると、Prompt プロジェクトは「プロンプトを書ける」から「モデル出力を安定して、テスト可能で、保守しやすいアプリのインターフェースにできる」へとレベルアップします。

## 段階の合格問題

学習後には、次の質問に答えられるようになっているはずです。token と文脈はなぜ重要か、Prompt の例はなぜ出力に影響するのか、構造化出力になぜ検証が必要なのか、いつ Prompt を足すより RAG を使うべきなのか、いつ微調整を検討すべきなのか。
