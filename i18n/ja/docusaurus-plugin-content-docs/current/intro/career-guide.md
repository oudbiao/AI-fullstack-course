# AI キャリア方向ガイド

![役割別 AI 学習ルート](/img/course/intro-role-based-paths-map-ja.png)

何かを作る前に職種名を決め急がないでください。まず共通基礎を終え、どの作業が楽しいかを見てから、ポートフォリオ方向を 1 つ選びます。

## 方向カード

| 方向 | こんな作業が好きなら向いている | コース重点 | ポートフォリオ証拠 |
| --- | --- | --- | --- |
| LLM アプリエンジニア | プロダクト、API、実用的な流れを作る | 第 7-8 章 | 評価付き RAG または LLM アプリ |
| AI Agent エンジニア | 多段階自動化、ツール、trace、権限 | 第 9 章 | Agent trace、ツール schema、安全境界 |
| データ / ML エンジニア | データ品質、指標、実験、baseline | 第 3-6 章 | 指標と失敗例を含むモデルレポート |
| CV / マルチモーダル | 画像、スクリーンショット、視覚結果 | 第 10 章と第 12 章 | 前後比較の視覚例とレビュー |
| NLP / RAG | テキスト抽出、検索、分類、Q&A | 第 7-8 章と第 11 章 | ラベル付き例、引用、評価セット |
| MLOps / デプロイ | 信頼性、コスト、ログ、デプロイ | デプロイと工程化の参照 | Docker、運用手順、ログ、監視メモ |

## 簡単セルフチェック

| 質問 | はいなら向きやすい方向 |
| --- | --- |
| 早くプロダクトの反応を見たいか？ | LLM アプリまたは Agent |
| エラーを測って説明するのが好きか？ | データ / ML |
| 画像、動画、文書が好きか？ | CV / マルチモーダル |
| テキスト、検索、構造化抽出が好きか？ | NLP / RAG |
| 信頼性、コスト、ログ、デプロイが好きか？ | MLOps / AI エンジニアリング |

## いつ決めるか

| ステージ | 判断 |
| --- | --- |
| 第 3 章前 | まだ決めない。ツール、Python、データの基礎を作る |
| 第 6 章後 | モデル深掘りかアプリ構築かを考える |
| 第 9 章後 | ポートフォリオ方向を 1 つ選んで強化する |
| 卒業制作 | 手元の材料と評価方法で選ぶ |

一番安全な標準方向は、LLM アプリ + RAG + Agent です。早く見えるプロジェクトを作りやすく、それでいて本物の工程証拠も必要になるからです。
