# 学習ブロッカー診断マップ

![学習ブロッカー診断フローチャート](/img/course/intro-blocker-diagnosis-flow-ja.png)

詰まったとき、まず資料を増やさないでください。詰まっている層を見つけ、最小テストを実行し、修復を記録します。

## 速診断表

| 症状 | 可能性が高い層 | まず戻る場所 | 最小テスト | 証拠 |
| --- | --- | --- | --- | --- |
| コマンドが動かない | ツール/環境 | 第 1 章、環境準備 | 新しいターミナルで README 通りに再実行 | コマンドログ |
| Python がファイルを読めない | コード/プロジェクト構造 | 第 2 章 | `Path.cwd()` を出し、小さな読み書きを行う | 入出力ファイル |
| データの結論が怪しい | データ品質 | 第 3 章 | 列名、欠損、重複を出力 | データ辞書 |
| 指標や loss が分からない | 数学/モデル評価 | 第 4-6 章 | 5 サンプルで指標を再現 | 指標メモ |
| LLM JSON が不安定 | Prompt/schema | 第 7 章 | 10 個の固定入力をパーサで検査 | Prompt バージョン表 |
| RAG の根拠が弱い | 検索/引用 | 第 8 章 | 生成前に検索結果だけ表示 | 検索ログ |
| Agent がループまたは越権する | ツール/権限 | 第 9 章 | 3 ステップに制限して trace を保存 | `agent_traces.jsonl` |
| 自分の環境でしか動かない | 交付/デプロイ | 交付基準、デプロイ章 | クリーン clone で README 通りに実行 | クリーン実行ログ |
| プロジェクトを説明できない | ポートフォリオ表現 | ポートフォリオチェックリスト | 問題、入出力、評価、失敗を追加 | README 更新 |

## 検索する前の 4 つの質問

1. どのコマンドまたは入力で問題が起きたか？
2. 何を期待していたか？
3. 実際には何が起きたか？
4. もっと小さい例で再現できるか？

## ブロッカーメモテンプレート

```md
## ブロッカー

層：
コマンドまたは入力：
期待：
実際：
最小再現：
修正：
回帰チェック：
```

順調な学習とは、失敗しないことではありません。似た失敗を次にもっと早く見つけられることです。
