# 現代 AI アプリケーション技術スタック

![現代 AI アプリケーション技術スタック全景図](/img/course/intro-modern-ai-stack-map-ja.png)

コース後半で学ぶのは、LLM を1回呼び出すことだけではありません。知識を使い、行動し、現実の入力を扱い、評価でき、長く動くシステムを作ることです。

## 1. まず5つのブロックを覚える

| ブロック | 解決する問題 | 主なステージ |
|---|---|---|
| RAG | モデルに私有知識や最新情報が必要 | 8 |
| Agent | 計画、ツール、複数手順が必要 | 9 |
| マルチモーダルAI | 入出力に画像、PDF、音声、動画、スクリーンショットがある | 10-12 |
| モデルエンジニアリング | 品質、遅延、コスト、プライバシーを調整する | 6-8 |
| Ops | Prompt、評価、ログ、trace、デプロイ、ロールバックを管理する | 7-9 と卒業制作 |

## 2. 最小システム視点

| 層 | 確認するもの |
|---|---|
| 入力 | ユーザータスク、ファイル、画像、権限 |
| 文脈 | 文書、chunk、metadata、会話状態 |
| モデルまたはツール | モデル呼び出し、ツール呼び出し、パラメータ、コスト |
| 出力 | 回答、引用、アクション、生成素材 |
| 評価 | 固定テスト、人の確認、指標 |
| 可観測性 | ログ、trace、エラー、遅延 |
| 改善 | Prompt、検索、ツール schema、モデル、ワークフローの変更 |

入力と出力だけなら、まだデモです。使ったデータ、呼んだツール、失敗理由、改善方法を説明できると、エンジニアリングに近づきます。

## 3. 何に何を使うか

| 必要なこと | まず使うもの |
|---|---|
| 文書から答える | 検索ログが見える RAG |
| 安全に行動する | まずワークフロー。手順が不確定なら Agent |
| スクリーンショットや PDF を理解する | マルチモーダルモデルと出典追跡 |
| コストを下げる | 小さいモデル、ルーティング、キャッシュ、短い文脈 |
| 品質を安定させる | 評価セット、ログ、trace、Prompt 版管理 |

初回はこの地図を覚え、各ブロックを最小の動くプロジェクトで理解しましょう。
