# 役割別の学習ルート：目標に合わせてAIフルスタックを学ぶ

![役割別の学習ルート地図](/img/course/intro-role-based-paths-map-ja.png)

## このセクションの位置づけ

同じコースでも、学習目標が違えば読み方も変わります。初学者は1～9章の順番で進めばOKです。はっきりした職業目標がある人は、目標の役割に合わせて、じっくり読む範囲と流し読む範囲を調整できます。

このページの目的は、基礎を飛ばすことではありません。どの内容を必ず身につけるべきか、どの内容はまず「どこにあるか」だけ分かればよいか、どのプロジェクトがポートフォリオに最も向いているかを判断するためのものです。

## 30秒で役割を選ぶ

| 何をしたいか | 優先ルート |
|---|---|
| AIを製品や業務システムに組み込む | AIアプリケーションエンジニアルート |
| モデルに会社資料、PDF、ナレッジベースを安定して読ませる | RAGエンジニアルート |
| AIにタスク分解、ツール操作、自動実行をさせる | Agent開発者ルート |
| モデルの学習、微調整、評価、デプロイを行う | モデルエンジニアルート |
| 就職・転職準備や、実力のアピールをしたい | ポートフォリオ就職ルート |

## AIアプリケーションエンジニアルート

AIアプリケーションエンジニアの中心スキルは、モデルを実際の製品に組み込むことです。バックエンドAPIを書けること、モデルを呼び出せること、文書やユーザー入力を処理できること、ログ・評価・デプロイの説明を追加できることが求められます。

おすすめは第1、2、3、7、8、9章を丁寧に読み、第4～6章はざっと目を通しつつも完全には飛ばさないことです。モデルを学習しない場合でも、データ、評価、ベクトル、Embedding、モデルコスト、失敗分析は理解しておく必要があります。

代表的な作品は、ログイン機能や簡単なフロントエンドを備えたコースQ&Aアシスタントです。ファイルアップロード、RAG検索、引用表示、ログ記録、基本的な評価をサポートできるとよいでしょう。

## RAGエンジニアルート

RAGエンジニアの中心スキルは、外部知識を大規模モデルに安定して接続することです。文書解析、分割、Embedding、ベクトルデータベース、Hybrid Search、Reranking、Query Rewrite、GraphRAG、Multimodal RAG、RAGOpsを理解する必要があります。

おすすめは第2、3、5、7、8章を丁寧に読み、第9章でAgentic RAGを補うことです。第12章のマルチモーダル部分も学ぶ価値があります。実際のナレッジベースには、PDF、表、画像、スクリーンショットが含まれることが多いからです。

代表的な作品は、評価用データセット付きの企業ナレッジベースDemoです。検索された断片、出典の引用、再現率、回答の忠実性、失敗サンプル、改善記録を示せるとよいでしょう。

## Agent開発者ルート

Agent開発者の中心スキルは、「モデルに自由に考えさせる」ことではなく、制御可能なタスク実行システムを設計することです。Prompt、ツール呼び出し、MCP、タスク計画、メモリ、ワークフロー、マルチAgent、評価、安全性、デプロイ、AgentOpsを理解する必要があります。

おすすめは第7、8、9章を丁寧に読み、あわせて第2章のPythonエンジニアリング能力と第3章のデータ処理能力も見直すことです。Agentはコード、データベース、ファイル、検索、RAG、外部APIを呼び出すことが多いため、基礎的なエンジニアリング力がとても重要です。

代表的な作品は、リサーチアシスタントAgentです。問題を分解し、資料を検索し、ツールを呼び出し、レポートを生成し、実行の流れを記録し、リスクの高いステップでは人の確認を求められるようにします。

## モデルエンジニアルート

モデルエンジニアリングの方向では、モデル内部の仕組み、学習、微調整、推論最適化、デプロイにより重点を置きます。数学、機械学習、深層学習、Transformer、微調整、LoRA / QLoRA、量子化、蒸留、小型モデル、ハイブリッドデプロイをより深く理解する必要があります。

おすすめは第4、5、6、7章を丁寧に読み、その後で第8章のモデルデプロイの章に進むことです。第10～12章は、興味に応じて視覚、マルチモーダル、生成のいずれかを選べます。

代表的な作品は、モデル比較実験です。異なるモデル、異なる量子化方式、異なるPrompt、または微調整案について、性能・遅延・コストの差を比較し、実験レポートを書きます。

## ポートフォリオ就職ルート

ポートフォリオルートで最も重要なのは、すべての章を最深部まで読むことではなく、継続的に成果を出すことです。各段階で、実行できて、スクリーンショットを見せられて、説明できて、READMEに書けるプロジェクトを残しましょう。

おすすめは、ルート1に従って1から9まで進みつつ、「貫通プロジェクト：AI学習アシスタント成長ルート」で成果をつなげることです。第10～12章は、1つの方向を選んで卒業プロジェクトとして仕上げれば十分で、すべての方向を深くやる必要はありません。

ポートフォリオの代表例としては、Python小ツール、データ分析レポート、機械学習baseline、深層学習実験、Promptアシスタント、RAGナレッジベース、Agent自動化アシスタント、そして完全なAIアプリのデプロイ説明が含まれます。

## 役割別の能力レーダー

役割ルートを選ぶときは、6つの観点から今いちばん不足しているものを見てみましょう。プログラミング実装、データ処理、モデル理解、AIアプリケーションエンジニアリング、評価と振り返り、デプロイと納品です。どの役割にも重点の違いはありますが、どのルートでも評価とエンジニアリング化を完全に飛ばすことはできません。

| 役割 | 必ず強化したい能力 | 先にざっと読んでもよい内容 | もっとも補うべき作品の証拠 |
|---|---|---|---|
| AIアプリケーションエンジニア | API、Prompt、RAG、バックエンド、ログ、デプロイ | 深いモデル学習の詳細 | 実行可能なアプリ、デプロイ説明、エラー処理 |
| RAGエンジニア | 文書処理、検索、引用、評価、RAGOps | マルチAgent協働の詳細 | chunks、retrieval logs、eval questions |
| Agent開発者 | ツールschema、trace、権限、失敗復旧 | 複雑なモデル学習 | tool logs、agent traces、安全境界 |
| モデルエンジニアリング志向 | 数学、ML、DL、Transformer、微調整 | フロントエンド表示の詳細 | 実験記録、指標比較、学習曲線 |
| ポートフォリオ就職志向 | README、デモ、振り返り、プロジェクト表現 | ニッチな拡張テーマ | スクリーンショット/GIF、評価表、失敗サンプル、デモ台本 |

## 各ルートの最低プロジェクトセット

時間が限られているなら、すべてのプロジェクトを大きくする必要はありません。以下は、それぞれの目標に対する最低限のプロジェクトセットです。終わったら、余力に応じて標準版へアップグレードしましょう。

| 目標 | 最低プロジェクトセット | アップグレード方向 |
|---|---|---|
| AIアプリ開発に転向する | Python APIの小プロジェクト + Promptアシスタント + RAG Q&A + Agentツール呼び出し | フロントエンド、デプロイ、ログ、評価データセットを追加 |
| RAG/ナレッジベース分野に転向する | データクレンジングプロジェクト + 文書分割Demo + RAG Q&A + RAG評価レポート | hybrid search、rerank、citation checkを追加 |
| Agent/自動化分野に転向する | ツール呼び出しDemo + 複数ステップタスクAgent + 安全確認のサンプル | trace replay、MCP、権限監査を追加 |
| モデルエンジニアリングを目指す | ML baseline + DL学習実験 + Transformer/微調整比較 | 学習診断、コスト分析、モデルデプロイを追加 |
| 総合ポートフォリオを作る | データ分析 + ML baseline + RAGアシスタント + Agentアシスタント + 卒業プロジェクト | READMEのスタイルとプロジェクト紹介のストーリーを統一 |

これらのセットは、コースを削るためではなく、優先順位をつけるためのものです。基礎章は自力で原因を特定できるかを決め、プロジェクト章は能力を見せられるかを決め、評価と振り返りはプロジェクトの信頼性を決めます。

## どう選ぶか

まだ目標がはっきりしていないなら、まずはAIアプリケーションエンジニアルートに進んでください。最も守備範囲が広く、プロジェクトも作りやすいルートです。学習を進める中で、ナレッジベース、Agent、モデル学習、マルチモーダル生成のどれかにもっと興味が出てきたら、そのルートに切り替えて深掘りしましょう。

どのルートを選んでも、ツール名を集めるだけではいけません。1つ技術を学ぶたびに、次のことに答えられるようにしましょう。その技術は何を解決するのか、いつ使わないのか、最小プロジェクトはどう作るのか、どう評価するのか、どうポートフォリオに書くのか。
