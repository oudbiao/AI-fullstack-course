# 能力チェックポイント

![ステージチェックポイントカード](/img/course/intro-stage-checkpoint-cards-ja.png)

このページは、「次に進んでよいか」だけに答えます。読んだ記事数ではなく、見える能力で確認しましょう。

## 1. 4つのレベル

| レベル | できること | 証拠 |
|---|---|---|
| 理解 | 何を解決する考え方か説明できる | メモ、概念図、短い説明 |
| 実行 | 最小例を自分で動かせる | コマンド、スクリーンショット、動くスクリプト |
| 構築 | 入力と出力を持つ小さなプロジェクトを作れる | README、ソースコード、例 |
| 改善 | 評価し、失敗を見つけ、反復できる | 指標、ログ、失敗サンプル、変更メモ |

序盤の章は、実行または構築までで十分です。RAG、Agent、卒業制作では改善レベルを目指します。

## 2. 次に進む前に

5つの質問を確認します。

| 質問 | 合格の目安 |
|---|---|
| 簡単に説明できるか | 記事を見ずに用途を説明できる |
| 自分でコードを動かしたか | コマンドと出力が残っている |
| プロジェクト証拠を残したか | README、スクリプト、Notebook、図、Demo がある |
| 失敗を記録したか | 何が壊れ、なぜ壊れた可能性があるか分かる |
| 来週もう一度実行できるか | コマンドと環境メモが明確 |

2つ以上弱い答えがあるなら、次へ進む前にプロジェクト証拠を補強しましょう。

## 3. AIアプリ段階の追加基準

| 種類 | 追加チェックポイント |
|---|---|
| Prompt | Prompt の版と固定テスト入力がある |
| RAG | 検索ログと引用チェックが見える |
| Agent | ツール schema、trace、ステップ上限、権限境界が記録されている |
| デプロイ | 環境変数、起動コマンド、ログ、戻し方のメモがある |

本当の進歩は、道具の名前をたくさん知ることではありません。問題、データ、モデル、エンジニアリング、評価、振り返りを、自分で説明できるループにつなげることです。
