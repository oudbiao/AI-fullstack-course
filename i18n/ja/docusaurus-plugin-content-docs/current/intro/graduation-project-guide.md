# 卒業プロジェクト設計ガイド

![卒業プロジェクト閉ループ設計図](/img/course/intro-graduation-project-loop-ja.png)

卒業プロジェクトは、現実の問題を実行・評価できる AI システムに変えられることを示します。機能の多さではなく、閉ループがあることが大切です。

## 卒業プロジェクトの閉ループ

| 層 | 最小要件 |
| --- | --- |
| 問題 | ユーザーは誰で、どの痛みを解くか？ |
| 材料/データ | 入力はどこから来て、どう整えるか？ |
| AI 能力 | Prompt、RAG、Agent、モデル、マルチモーダルワークフロー |
| 入口 | CLI、Notebook、API、簡単な Web 画面 |
| 評価/ログ | 固定テスト、失敗例、引用、trace、指標 |
| Demo/レビュー | README、スクリーンショット、デモ台本、次の一歩 |

## テーマを選ぶ

| 手元にあるもの | 向いているテーマ | 評価の焦点 |
| --- | --- | --- |
| コース文書やナレッジベース | RAG / AI 学習アシスタント | 検索ヒット、引用の支え、回答不能時の処理 |
| CSV、Excel、業務データ | データ分析 Agent | 正しさ、グラフ説明、安全なコード |
| 画像、スクリーンショット、PDF | マルチモーダル助手 | 解析品質、不確実性、人手レビュー |
| ドメイン文章 | NLP / 専門助手 | 抽出精度、ラベル境界、事実一貫性 |
| 明確な案がない | AI 学習アシスタント | 再現性、ログ、評価、Demo |

## 最小バージョン

AI 学習アシスタントなら、最小卒業プロジェクトは次で十分です。

```text
Markdown 文書 -> 分割/索引 -> ユーザー質問 -> 検索された出典 -> 引用付き回答 -> Q&A ログ -> 10 個の評価質問
```

最初の版に、複雑な UI、マルチ Agent、長期記憶は不要です。

## README に必ず入れるもの

| セクション | 答えること |
| --- | --- |
| 目標 | 何の問題を、誰のために解くか？ |
| 実行 | コマンドと環境変数は何か？ |
| 例 | 入力と出力は何か？ |
| アーキテクチャ | LLM、RAG、Agent、データ、ログはどこか？ |
| 評価 | 固定テストまたは指標は何か？ |
| 失敗例 | どこで壊れるか？ |
| 次の一歩 | なぜその改善をするか？ |

## デモ台本

| 時間 | 見せるもの |
| --- | --- |
| 1 分 | 問題とユーザー |
| 2 分 | アーキテクチャとデータの流れ |
| 3 分 | 成功例 1 つ |
| 2 分 | 失敗例 1 つと診断 |
| 2 分 | 評価結果と次の一歩 |

失敗例をはっきり見せられることは、失敗を隠すより強い場合があります。エンジニアリング判断を示せるからです。
