# AI エンジニアリング公開チェックリスト

![AI プロジェクト速習参照マップ](/img/course/appendix-project-quick-reference-map-ja.png)

Demo だけでは不十分です。プロジェクトが「準備できた」と言う前に、再現、評価、調査、境界設定ができるかを確認します。

## Demo から使えるシステムへ

| 層 | 確認 |
| --- | --- |
| 問題境界 | ユーザー、入力、出力、対象外、拒否条件 |
| 評価 | 固定質問/タスク、期待結果、失敗例 |
| ログ/trace | リクエスト、検索、Prompt バージョン、ツール呼び出し、エラー |
| コスト/遅延 | token、呼び出し数、再試行、遅いステップ |
| 安全 | API key、機密データ、高リスクツール、人間確認 |
| デプロイ | `.env.example`、起動コマンド、クリーン実行メモ |
| ポートフォリオ | README、スクリーンショット、評価、制限、次の一歩 |

## プロジェクト別チェック

| プロジェクト種別 | 必ず見せるもの |
| --- | --- |
| RAG | 文書出典、chunk 方法、検索ログ、引用チェック、回答不能時の処理 |
| Agent | ツール schema、最大ステップ数、停止条件、trace、権限境界 |
| マルチモーダル | 入力素材、生成バージョン、人手レビュー、エクスポート制約 |
| デプロイ | 環境変数、ヘルス/エラーログ、ロールバックまたはリセットコマンド |

## 公開前の最後の質問

明日、システムが誤答し、間違った出典を検索し、違うツールを呼び、急に高コストになったら、どの層が失敗したか分かりますか？

分からないなら、機能追加より先にログ、評価、境界を追加してください。
