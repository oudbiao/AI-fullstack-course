# AIフルスタック用語集

![概念分類ボード](/img/course/intro-concept-sorting-board-ja.png)

これは暗記用ではなく、すばやく確認するためのページです。分からない言葉があっても、まず「何を解決する言葉か」だけ押さえ、使う章で詳しく戻りましょう。

## 調べ方

| その言葉が... | まず見る分類 |
|---|---|
| コマンド、リポジトリ、パッケージ、API、JSON | 開発基礎 |
| データセット、特徴量、ラベル、baseline、指標 | データと機械学習 |
| tensor、loss、embedding、attention、token | 深層学習とLLM基礎 |
| Prompt、RAG、chunk、citation、hallucination | LLMアプリケーション |
| Agent、tool、trace、memory、guardrails | Agent とエンジニアリング |

## 主要用語

| 用語 | ひとことで言うと |
|---|---|
| Terminal | コマンドを実行する場所 |
| 現在のディレクトリ | コマンドが実行されているフォルダ |
| パッケージマネージャ | Python や Node の依存関係を入れる道具 |
| 仮想環境 | Python プロジェクトの依存関係を分ける仕組み |
| Git commit | コード変更を保存した記録 |
| README | プロジェクトの目的と実行方法を書くファイル |
| API | サービスを呼び出すための約束 |
| JSON | API、設定、ログでよく使う構造化テキスト |
| Dataset | 分析、学習、評価に使うデータ |
| Feature | モデルが使う入力情報 |
| Label | 教師あり学習で予測したい正解 |
| Baseline | 比較用の簡単な出発点 |
| データリーク | 学習時に見てはいけない情報を見てしまうこと |
| 過学習 | 学習データでは良く、新しいデータでは弱い状態 |
| 指標 | 結果を判断する数字やルール |
| Tensor | 深層学習で使う多次元配列 |
| Loss | 予測と正解の差 |
| 逆伝播 | 誤差をネットワークの各層へ戻す方法 |
| Embedding | テキスト、画像、カテゴリのベクトル表現 |
| Attention | 入力のどこが重要かを選ぶ仕組み |
| Transformer | 多くの現代的な言語モデルの基礎構造 |
| Token | モデルが扱うテキストの単位 |
| コンテキストウィンドウ | モデルが一度に見られる入力範囲 |
| Prompt | モデルへの指示と入力 |
| 構造化出力 | JSON など固定フィールドで返させること |
| Function calling | モデルにツール呼び出し引数を作らせること |
| RAG | 回答前に外部資料を検索する流れ |
| Chunk | 検索に使う文書片 |
| ベクトルDB | ベクトルを保存し、類似検索する仕組み |
| Rerank | 生成前に検索結果を並べ替えること |
| Citation | 回答が参照したと示す出典 |
| Hallucination | もっともらしいが根拠のない出力 |
| Agent | 手順を計画し、ツールを使い、状態を持つシステム |
| Tool | システムが呼び出せる外部機能 |
| Trace | 手順、入力、出力、ツール呼び出しの記録 |
| Guardrails | 入力、出力、ツール、権限を守るための規則と検査 |
| 可観測性 | ログ、指標、trace、エラーでシステムを見えるようにすること |
| デプロイ | ローカル以外からプロジェクトを使えるようにすること |

## 使い方

知らない言葉が出ても、学習全体を止めないでください。分類し、一行の意味を読み、関連例を動かし、使った後で自分の言葉に直しましょう。
