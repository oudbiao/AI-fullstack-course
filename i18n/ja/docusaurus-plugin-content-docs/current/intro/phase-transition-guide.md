# ステージ移行ガイド

![AIメインリレーマップ](/img/course/appendix-ai-main-relay-map-ja.png)

各ステージは、足りない能力を1つずつ足していきます。次の話題が急に見えるときは、このページで何を補うのか確認しましょう。

## 移行マップ

| 前 | 次 | 新しく得る能力 |
|---|---|---|
| ツール | Python | コードで流れを表す |
| Python | データ分析 | ファイル、表、ログ、乱れた入力を読む |
| データ分析 | 機械学習 | データを特徴量、目標、指標、baseline にする |
| 機械学習 | 深層学習 | テンソルとニューラルネットで表現を学ぶ |
| 深層学習 | LLM | token、embedding、文脈、Transformer、モデル挙動を理解する |
| LLM | RAG | モデル記憶だけでなく検索した出典で答える |
| RAG | Agent | ツール、状態、trace、権限でタスクを進める |
| Agent | 卒業制作 | 必要な部品を組み合わせて再現できるシステムにする |

## 次へ進む前に

| 次のステージ | 1つ確認すること |
|---|---|
| Python | スクリプトを動かし、Git commit できるか |
| データ | ファイルを読み、フィールドを説明できるか |
| 機械学習 | 目標と指標を定義できるか |
| 深層学習 | train/test 分割、loss、過学習、baseline を説明できるか |
| LLM | API 呼び出しまたは小さなテキストモデルを動かせるか |
| RAG | 生成前に検索 chunk を確認できるか |
| Agent | ツール呼び出しを再生し、危険な操作を止められるか |
| 卒業制作 | 他の人が README 通りに実行できるか |

この順序は飾りではなく、実務的な順序です。後半のAIアプリは、前半の証拠、デバッグ、プロジェクト習慣の上に立っています。
