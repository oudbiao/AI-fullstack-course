# プロジェクトロードマップとポートフォリオ

![プロジェクトポートフォリオ成長はしご](/img/course/intro-project-portfolio-roadmap-ja.png)

プロジェクトで学びます。ただし、完成できる大きさにします。動く、説明できる、評価できるプロジェクトだけがポートフォリオ材料になります。

## 3 つのプロジェクト段階

| 段階 | 目標 | 最初に追わないもの |
| --- | --- | --- |
| ステージプロジェクト | 入力、出力、実行コマンド | 多機能や美しい UI |
| 統合プロジェクト | ログ、指標、失敗例 | 成功スクリーンショット 1 枚だけ |
| 卒業プロジェクト | README、評価、デモ台本 | すべての技術を 1 つに詰め込むこと |

## おすすめプロジェクトはしご

| コース領域 | プロジェクト出力 |
| --- | --- |
| ツール + Python | CLI 学習アシスタント |
| データ分析 | グラフと限界を含む EDA レポート |
| ML / DL | baseline モデル、指標、エラーサンプル |
| LLM / Prompt | 構造化計画または振り返りカード助手 |
| RAG | 引用と評価質問付きのコース Q&A |
| Agent | trace と権限付きのツール利用助手 |
| マルチモーダル / 卒業制作 | デモできる最終プロジェクト |

## 最小ポートフォリオ README

````md
# プロジェクト名

## 目標
何の問題を解くか？

## 入力と出力
入力：
出力：

## 実行方法
```bash
python main.py
```

## 評価
結果をどう判断するか？

## 失敗サンプル
何が壊れ、何を学んだか？

## 次の改善
続けるなら最初に何を改善するか？
````

各プロジェクトにこれらがあれば、ポートフォリオはコースと一緒に自然に育ちます。
