# 通しプロジェクト：AI 学習アシスタント

![AI 学習アシスタント バージョンロードマップ](/img/course/ai-learning-assistant-roadmap-ja.png)

AI 学習アシスタントは、このコースで最もおすすめする通しプロジェクトです。各ステージでゼロから始めるのではなく、同じプロダクトに小さな能力を 1 つずつ足します。

このページは概要です。実装の詳細は [バージョンロードマップ](/intro/ai-learning-assistant-version-roadmap) と [プロジェクトリポジトリテンプレート](/intro/ai-learning-assistant-template) を見てください。

## 1 つのプロダクト、各ステージで 1 つの能力

| ステージ | 追加する能力 | 証拠 |
| --- | --- | --- |
| 1 ツール | プロジェクト作業台がある | README、Git commit、実行スクリーンショット |
| 2 Python | アシスタントが学習タスクを記録する | CLI 出力、`tasks.json`、エラー処理 |
| 3 データ | アシスタントが学習記録を集計する | 整理済みデータ、グラフ、結論 |
| 4-6 モデル | 学習リスクを分類または予測する | baseline、指標、エラーサンプル |
| 7 LLM/Prompt | 計画や振り返りカードを生成する | Prompt バージョン、schema チェック |
| 8 RAG | コース資料に基づいて答える | 検索ログ、引用、評価質問 |
| 9 Agent | 手順を計画し、ツールを呼ぶ | trace、権限表、停止ルール |
| 10-12 発展 | 1 つの専門メディアを扱う | 前後比較例とレビュー記録 |
| 卒業制作 | デモできるプロダクトになる | Demo、README、評価レポート |

## プロダクトのルール

最初から巨大なプロダクトを作らないでください。各ステージで足すのはこれだけです。

1. ユーザー操作 1 つ。
2. 実行コマンド 1 つ。
3. サンプル入力と出力 1 つ。
4. 失敗例 1 つ。
5. 次の一歩メモ 1 つ。

これだけで、章ごとに別プロジェクトを作らなくても、自然にポートフォリオの物語ができます。

## 最終基準

最終アシスタントに多くの機能は不要ですが、明確なループが必要です。

```text
学習目標または材料 -> コンテキスト読解 -> 検索またはツール利用 -> 回答または計画 -> 証拠、ログ、評価
```

他の人が実行でき、例を確認でき、失敗を見られ、次の改善を理解できるなら、ポートフォリオに入れられます。
