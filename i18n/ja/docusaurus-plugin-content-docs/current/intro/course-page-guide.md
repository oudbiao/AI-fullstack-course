# コースページの使い方

![おすすめ学習ルート選択図](/img/course/intro-learning-path-selection-ja.png)

このページは、**次にどのページを開くか**に答えます。

## 1. 初回の順番

| 手順 | 開くページ | やること |
|---|---|---|
| 1 | [30分 AI クイック体験](/intro/quick-experience) | 小さなAI例を1つ動かす |
| 2 | [環境準備](/intro/environment-setup) | Python、Git、プロジェクトフォルダを準備する |
| 3 | [AIフルスタック能力マップ](/intro/ai-fullstack-map) | 細部より先に図を見る |
| 4 | [おすすめ学習ルート](/intro/learning-path) | ルートを1つ選ぶ |
| 5 | [第1章：開発者ツール](/ch01-tools) | 学習用ワークスペースを作り始める |

## 2. ページの使い分け

| ページ種類 | 使い方 |
|---|---|
| Intro ページ | クイックスタート、環境、マップ、ルート、用語 |
| ステージトップ | そのステージの意味と作るものを確認する |
| `0.0 学習ガイドとタスク表` | 学習順、必須タスク、合格基準 |
| レッスン | 概念、コード、出力、よくあるミス |
| ワークショップまたはプロジェクト | 主な理論の後に動く成果物を作る |
| 補足ヘルプ | FAQ、トラブル対応、ポートフォリオ、キャリア、深掘りルート |

## 3. 標準学習ループ

1. まず図を見る。
2. 最小コードを動かす。
3. 期待出力と比べる。
4. コマンド、結果、エラー、スクリーンショットのどれかを残す。
5. ステージタスクに戻る。

迷ったら、今の詰まりがツール、コード、データ、モデル挙動、LLMアプリ、Agentの動作、デプロイのどれかを先に分け、対応する章かトラブルシューティングページを開きます。
