# Debug 事件捜査ミッション集

![Debug 事件捜査ミッション集](/img/course/debug-detective-missions-ja.png)

Debug は失敗の証拠ではありません。エンジニアリング力が育つ場所です。

何かが壊れたら、ランダムに直さず、毎回同じ調査ループを使います。

```mermaid
flowchart LR
  A["症状を保存"] --> B["原因を 2 つ推測"]
  B --> C["最小テストを実行"]
  C --> D["層を特定"]
  D --> E["修復メモを書く"]
```

## 事件カード

| 事件 | 症状 | まず確認すること | 残す証拠 |
| --- | --- | --- | --- |
| コマンド消失 | `python`、`pip`、`npm`、`docusaurus` が見つからない | 現在のフォルダ、PATH、インストール済みバージョン、有効な環境 | ターミナル出力とバージョンメモ |
| JSON 破損 | `JSONDecodeError` | 空ファイル、括弧不足、余分なカンマ、JSON/JSONL の混同 | 壊れたサンプルと復旧ロジック |
| DataFrame 列不足 | `KeyError: 'minutes'` | `df.columns.tolist()`、区切り文字、ヘッダー行、空白 | 整理済み列名とデータ辞書 |
| 良すぎるモデルスコア | accuracy が不自然に高い | データリーク、重複行、正解列が特徴量に入っていないか、baseline | baseline 指標とリーク確認 |
| LLM JSON ずれ | 出力が schema から外れる | Prompt 例、パーサ検証、再試行ロジック | Prompt バージョン表と失敗出力 |
| RAG が根拠を拾えない | 答えに有効な出典がない | 生成を止め、検索結果だけ表示する | 検索ログと失敗質問 |
| 引用が答えを支えない | 引用はあるが回答を支えていない | 文ごとに根拠を確認する | 引用チェック表 |
| Agent がループする | 同じ計画やツールを繰り返す | 最大ステップ数、停止条件、trace フィールド | `agent_traces.jsonl` |
| ツール権限が強すぎる | Agent が書き込み、送信、削除を自由に呼ぶ | ツール危険度、allowlist、人間確認 | 権限表とブロックテスト |
| 自分の環境でしか動かない | 他の人が実行すると失敗する | README、依存関係、`.env.example`、サンプルデータ | クリーン環境での実行ログ |

## 最小修復メモ

`failure_cases.md`、`debug_notes.md`、README のどれかに書きます。

```md
## 事件

症状：
期待したこと：
実際に起きたこと：
疑わしい層：
最小テスト：
根本原因：
修正：
回帰チェック：
```

目的はエラーを集めることではありません。次に似たエラーが出たとき、もっと早く場所を見つけることです。
