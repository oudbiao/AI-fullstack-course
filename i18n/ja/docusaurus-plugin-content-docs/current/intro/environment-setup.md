# 環境準備

> **目標：** あなたのパソコンが要件を満たしているか確認し、必要なツールを理解する  
> **説明：** ここでは全体像だけを紹介します。各ツールの詳しいインストール手順は、1 開発者ツール基礎の該当章で、手順つきで解説しています。

## 初心者はまず何を入れる？

```mermaid
flowchart LR
  A["Python 3.11"] --> B["VS Code"]
  B --> C["Git"]
  C --> D["Miniconda"]
  D --> E["Jupyter"]
  E --> F["あとで章ごとに必要なライブラリを追加"]
```

| 段階 | 環境の方針 |
|---|---|
| 1～5 章 | 普通のパソコンでOK。まずは Python、VS Code、Git、Miniconda を整える |
| 6 章以降 | 深層学習が必要になってから、Colab、クラウドGPU、またはローカルGPUを検討する |
| 8～9 章 | API Key、ベクトルデータベース、ログ、環境変数を重点的に追加する |
| 12 章 | マルチモーダルプロジェクトに応じて、画像・音声・動画ツールを追加する |

## ハードウェア要件

### 1～5 学習章（ツール + Python + データ分析 + 数学 + ML）

普通に使えるパソコンなら大丈夫です。

| 項目 | 最低要件 | 推奨構成 |
|-------|---------|---------|
| CPU | 任意のデュアルコア | 4コア以上 |
| メモリ | 4GB | 8GB以上 |
| ストレージ | 20GB の空き容量 | SSD、50GB の空き容量 |
| GPU | **不要** | 不要 |
| OS | Windows 10/11、macOS 10.15+、Ubuntu 20.04+ | いずれも可 |

:::tip
パソコンがかなり古くても心配いりません。1～5 学習章のコードはすべて [Google Colab](https://colab.research.google.com) で実行できます。必要なのはブラウザだけです。
:::

### 6 深層学習と Transformer 基礎から

6 深層学習と Transformer 基礎からニューラルネットワークの学習を始める場合、GPU が必要です。

| 方案 | 説明 | 料金 | おすすめ度 |
|------|------|------|:---:|
| **Google Colab** | 無料の T4 GPU、設定不要 | 無料（Pro 版 $10/月） | ⭐⭐⭐⭐⭐ |
| **AutoDL** | 国内クラウドGPU、時間課金 | 約 2-3 元/時間 | ⭐⭐⭐⭐ |
| **ローカル NVIDIA GPU** | VRAM ≥ 8GB | 初期費用が必要 | ⭐⭐⭐ |

:::info GPU を先に買う必要はありません
1～5 学習章を終えるには、だいたい 4～6 か月かかります。GPU のことは、6 深層学習と Transformer 基礎に進んでから考えれば大丈夫です。コースでは、深層学習に入る前に詳しい[ハードウェアとクラウドリソースのガイド](/appendix/hardware)を用意しています。
:::

## おすすめの環境構成

前半のコースは、章ごとに少しずつライブラリを入れていけば進められます。  
ただし、6 深層学習と Transformer 基礎から 9 AI Agent とエージェントシステム、12 AIGC とマルチモーダルまで一気に学ぶなら、最初から「コース用のサンプル環境」を用意するほうが安心です。

リポジトリのルートには、すでに2つの依存関係リストがあります。

| ファイル | 対象範囲 |
|------|----------|
| `requirements-course-core.txt` | 1～5 学習章 + ほとんどの従来型機械学習 / エンジニアリング例 |
| `requirements-course-ai.txt` | 6 深層学習と Transformer 基礎以降でよく使う深層学習 / Transformers / LLM ツール依存関係 |

### おすすめのインストール順

まず、新しい conda 環境を作成します。

```bash
conda create -n ai-course python=3.11 -y
conda activate ai-course
```

次に、基本依存関係をインストールします。

```bash
pip install -r requirements-course-core.txt
```

6 深層学習と Transformer 基礎以降も学ぶなら、AI 関連の依存関係を追加で入れます。

```bash
pip install -r requirements-course-ai.txt
```

:::warning PyTorch について
`requirements-course-ai.txt` にはすでに `torch` が含まれています。  
次のいずれかを使う場合は、

- NVIDIA GPU
- Apple Silicon
- 特定の CUDA バージョン

先に PyTorch 公式のインストール手順に従って、あなたの環境に合う版を入れてから、このファイルにある他のライブラリをインストールするほうがおすすめです。
:::

### さらに安定したやり方

今後のコースページに出てくる `pip install ...` は、個別に使っても問題ありません。  
ただ、コース全体を通して体系的に進めるなら、この統一環境を優先して管理するのがおすすめです。そうすると、次のようなトラブルを避けやすくなります。

- 章ごとに個別インストールして、ライブラリのバージョンがぶつかる
- サンプルコードの内容は理解できるのに、手元で実行できない

## Python バージョンの選び方

**Python 3.11 を推奨します。** 理由は次のとおりです。

- 3.11 は 3.10 より 10～60% 高速
- 主要な AI ライブラリは、今のところ 3.11 に対応している
- 3.12/3.13 は新しすぎて、まだ対応していないライブラリがある

:::warning Python 3.8 以前は使わないでください
多くの新しい AI ライブラリは、すでに 3.8/3.9 をサポートしていません。もしパソコンに古い Python が入っていても、アンインストールする必要はありません。Miniconda で新しい 3.11 環境を作れば大丈夫です（1 開発者ツール基礎でやり方を説明します）。
:::

## OS ごとの補足

### Windows ユーザー

- **Windows Terminal** のインストールをおすすめします（Windows 11 には標準搭載、Windows 10 は Microsoft Store から入手）
- コマンドラインは **PowerShell** または **Git Bash** の使用がおすすめです
- Python パッケージのインストールで問題が出たら、まず Miniconda の使用を検討してください

### macOS ユーザー

- **Homebrew** パッケージマネージャーのインストールをおすすめします
- macOS に最初から入っている Python 2 は使わないでください。Miniconda で Python 3.11 を入れましょう
- Apple Silicon（M1/M2/M3/M4）の PyTorch 対応はすでにかなり良く、MPS 加速を使えます

### Linux（Ubuntu）ユーザー

- ほとんどの AI ツールは Linux との相性が最も良いです
- Ubuntu 22.04 LTS をおすすめします
- NVIDIA GPU ドライバのインストールには、追加の手順が必要になることがあります（1 開発者ツール基礎で扱います）

## ネットワーク環境

一部のリソースには、海外アクセス用の設定が必要です。

| リソース | 海外アクセスが必要か | 代替案 |
|------|:---:|---------|
| Google Colab | 必要 | AutoDL、ローカル Jupyter |
| GitHub | 地域によって必要 | Gitee をミラーとして使う |
| HuggingFace | 地域によって必要 | HuggingFace ミラーサイト |
| PyPI（pip ソース） | 不要だが、海外ソースは遅い | 清華大学 / アリババのミラーを使う |
| OpenAI API | 必要 | 国内の大規模モデル API（通義千問、DeepSeek） |

### pip の国内ミラーを設定する（おすすめ）

国内から使う場合、pip のインストールは遅くなりがちです。次のコマンドを実行すると、清華大学のミラーを恒久的に設定できます。

```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

設定すると、以後の `pip install` はすべて清華大学のミラーからダウンロードされ、かなり速くなります。

:::tip 環境でつまずいても大丈夫
環境構築は、すべての開発者が経験する「つらい」部分です。もし止まってしまったら、次のようにしてください。
1. エラーメッセージをそのまま全部コピーする
2. Google 検索に貼り付ける（英語検索のほうがよく見つかります）
3. 環境問題の 99% は、すでに誰かが経験しています。Stack Overflow にはきっと答えがあります
4. どうしても解決しないなら、まずは Google Colab で学習を続ける。環境問題はあとで解決してもOKです
5. 5 章の機械学習以降のコードが動かない場合は、まず上の統一環境構成どおりに依存関係を入れたか確認してください
:::
