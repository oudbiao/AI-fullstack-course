# AI 学習アシスタント バージョンロードマップ

![AI 学習アシスタント バージョンロードマップ](/img/course/ai-learning-assistant-roadmap-ja.png)

各バージョンでは能力を 1 つ追加し、4 種類の証拠を残します。

| 必要な証拠 | なぜ重要か |
| --- | --- |
| 実行コマンド | 機能を再現できることを示す |
| サンプル入力と出力 | 機能が何をするか見せる |
| 失敗例 | システムの境界を示す |
| 評価または確認 | 偶然動いた Demo を防ぐ |

## バージョンカード

| バージョン | 対応ステージ | 能力 | 最小証拠 |
| --- | --- | --- | --- |
| v0.1 | 第 1 章 | プロジェクト骨格 | README、依存関係メモ、最初の commit |
| v0.2 | 第 2 章 | CLI 学習タスクアシスタント | `tasks.json`、追加/表示/完了コマンド、壊れたファイル対応 |
| v0.3 | 第 3 章 | 学習データ分析 | グラフ、クリーニングメモ、結論 1 つ |
| v0.4 | 第 4-5 章 | 質問分類またはリスク baseline | ルールまたは ML baseline、指標、エラーサンプル |
| v0.5 | 第 6 章 | 学習診断実験 | 設定、loss カーブ、失敗記録 |
| v0.7 | 第 7 章 | Prompt 学習アシスタント | Prompt バージョン、schema チェック、失敗出力 |
| v0.8 | 第 8 章 | RAG コース Q&A | 文書一覧、検索ログ、引用チェック |
| v0.9 | 第 9 章 | 学習計画 Agent | ツール trace、権限ルール、停止条件 |
| v1.0 | 第 10-12 章 / 卒業制作 | デモ可能な製品または発展機能 | Demo、README、評価レポート |

バージョン番号は調整できます。証拠は省かないでください。

## バージョン記録テンプレート

```md
## v0.x バージョン名

目標：
実行コマンド：
サンプル入力：
サンプル出力：
評価/確認：
失敗例：
次バージョン：
```

プロジェクトを見せるときは、最終スクリーンショットだけで終わらせないでください。「CLI から始め、データ、評価、RAG、trace、デプロイ証拠を順に追加した」と説明できるほうが強いです。
