# 学習パス FAQ

このページでは、学習を始める前にもっとも迷いやすい質問に答えます。多くの質問に唯一の正解はありません。大事なのは、まず前に進めるルートを1つ選び、「学ぶべきか、何から学ぶか、飛ばせるか」で長く止まりすぎないことです。

## まずは最頻出の 5 つの答えを確認

| 問題 | 短い答え |
|---|---|
| 完全初心者でも学べる？ | はい。ただし、まずはツール、Python、データの3ステップから |
| Python ができるなら飛ばせる？ | 早めに読めますが、段階ごとのプロジェクト確認は必ず完了しましょう |
| RAG/Agent だけやりたい。数学は必要？ | 使えるレベルの直感は学ぶ必要があります。特にベクトル、類似度、評価です |
| GPU は必要？ | 最初の一通りでは、ほとんど不要です |
| つまずいたら？ | ルートを変えず、まずは1つのコマンド、ファイル、入力まで絞り込みます |

## まったくの初心者ですが、この講座は学べますか？

はい、学べます。ただし、いきなり大規模モデル、RAG、Agent から始めないでください。まずは開発者ツール、Python の基礎、データ分析の3つを終えましょう。最初からアルゴリズムの専門家になる必要はありませんが、コードを実行できること、ファイルを読み書きできること、表形式データを扱えること、基本的なプロジェクト構成を理解できることは必要です。

もしプログラミング経験がまったくないなら、各ステージのタスクリストで最小成果物を作ることを優先してください。すべての章を理解しきることより、小さなプロジェクトを動かせることのほうが大切です。

## すでに Python はできます。前半は飛ばせますか？

素早く流し読みすることはできますが、完全には飛ばさないことをおすすめします。少なくとも、段階タスクのプロジェクトを自力で完成できるか確認しましょう。たとえば、コマンドラインの小ツール、ファイルの読み書き、API 呼び出し、データクリーニング、EDA レポートです。これらができるなら、主要な時間は機械学習、大規模モデルアプリ、RAG、Agent に回せます。

## RAG と Agent だけやりたいです。数学と深層学習は必要ですか？

「十分に使える」レベルまでは学ぶ必要がありますが、最初から厳密に導出する必要はありません。ベクトル、類似度、embedding、学習/テスト、評価指標、過学習、Transformer の基本的な直感を理解しましょう。そうしないと、RAG でなぜ検索が失敗したのか判断しにくくなりますし、Agent で評価や安全境界を設計しにくくなります。

より現実的なルートは、まず開発ツール、Python、データ分析、LLM API、RAG、Agent を学び、数学と深層学習はプロジェクトで出てきた課題に合わせて補うことです。

## フロントエンドができなくても、AI フルスタックはできますか？

できます。AI フルスタックの重点は、最初から完全なフロントエンドを作れることではなく、データ、モデル、インターフェース、ツール、プロジェクト納品をつなげられることです。最初の段階では、コマンドライン、Notebook、または簡単な API でプロジェクトを作れば十分です。RAG や Agent のプロトタイプが安定してから、Web 画面を追加するか考えましょう。

就職や発表が目的なら、最終的には少なくとも簡単にデモできる入口を1つ作るのがおすすめです。たとえば、コマンドライン Demo、Streamlit ページ、FastAPI インターフェース、または軽量なフロントエンドです。

## GPU は必要ですか？

多くの段階では不要です。Python、データ分析、機械学習、LLM API、RAG、Agent は、普通のパソコンで学べます。GPU が特に役立つのは、ローカルで深層学習モデルを学習する、モデルを微調整する、または大きめのマルチモーダル実験をする場合です。

最初の学習では、ハードウェアに重点を置くことはおすすめしません。まずは小さなデータ、小さなモデル、API、またはクラウドサービスで流れを通し、その後プロジェクトに応じて GPU を考えましょう。

## 英語が苦手だと学習に影響しますか？

少し影響はあります。エラー、ドキュメント、モデルのパラメータ、ライブラリ名の多くが英語だからです。ただし、英語が得意でないと始められないわけではありません。大切なのは、path、module、import、permission、timeout、shape、token、quota、schema のようなエラーのキーワードを読めることです。

よくある英語のエラーは、自分のトラブルシュート記録にまとめておくことをおすすめします。見慣れてくると、多くのエラーは決まったパターンに見えてきます。

## コース全体を終えるのに、どれくらいかかりますか？

AI アプリを素早く体験したいだけなら、2〜4週間で LLM API、RAG、そして小さなプロジェクトを動かせます。体系的に主線を学びたいなら、通常は3〜6か月です。モデルの基礎、エンジニアリング、Agent、安全性、ポートフォリオまできちんと整えたいなら、6〜12か月かかることもあります。

かかる時間より、何を作れたかのほうが大切です。より良い指標は、段階タスクリストを終えたか、実行できるプロジェクトがあるか、README、スクリーンショット、評価、失敗サンプルがあるかです。

## 理論を先に見るべきですか、それとも先にプロジェクトですか？

まずは最小プロジェクトで「どんな問題を解くのか」をつかみ、そのあと理論を補うのがおすすめです。たとえば RAG を学ぶときは、まず「文書を読み込む、断片を検索する、答えを生成する、引用を表示する」という最小の流れを通し、その後に chunk、embedding、rerank、評価を深く学びます。こうすると理論がずっと理解しやすくなります。

理論をまったく見ないと壊れやすいシステムになりやすく、理論だけだと手応えがなくなりがちです。おすすめのリズムは、最小例を動かす → 核となる概念を読む → プロジェクトを作る → 失敗を振り返る、です。

## 各段階で必ずプロジェクトを作る必要がありますか？

少なくとも1つは、最小プロジェクトを作ることをおすすめします。プロジェクトは小さくて構いませんが、実行できること、説明できること、振り返れることが必要です。たとえば、Python の段階ではコマンドラインツール、データの段階では EDA レポート、機械学習の段階では baseline、RAG の段階ではコース Q&A、Agent の段階では学習プランニング Agent を作ります。

時間が限られているなら、AI 学習アシスタントの継続プロジェクトに関係する版だけを作るのでも大丈夫です。そうすれば、細かい課題が増えすぎません。

## どのルートを選べばいいですか？

とにかく早くアプリを作りたいなら、アプリ型 AI フルスタックの主ルートを選びましょう。モデルや学習を理解したいなら、モデル理解強化ルートです。就職や発表が目的なら、プロジェクトポートフォリオルートが向いています。まだ決められないなら、まずはアプリ型の主ルートから始めるのがよいです。プロジェクトを通して一番フィードバックを得やすいからです。

ルート選びは一度きりの決定ではありません。まずはアプリ主線でプロジェクトを動かし、そのあと数学、深層学習、微調整、マルチモーダルを深めれば大丈夫です。

## 卒業プロジェクトはいつ始められますか？

RAG か Agent の最小閉ループを1人で完成できるようになったら、卒業プロジェクトを始めてよいです。すべての章が終わるのを待つ必要はありません。卒業プロジェクトは v0.1 から段階的に改善していけばよく、まずは最小で動く版を作り、その後で評価、ログ、安全性、デプロイ、多モーダル対応を追加していきます。

準備ができているかの判断基準は、ユーザーの課題を定義できること、入力から出力まで動かせること、失敗サンプルを記録できること、次にどう改善するか説明できることです。

## 学習の途中でつまずいたら、どうすればいいですか？

まずはルートを変えたり、チュートリアルをやり直したりしないでください。今の段階のタスクリストに戻って、つまずきが環境、コード、データ、モデル、評価、プロジェクトの境界のどこにあるか確認します。そのうえでトラブルシュート索引を調べ、最小再現を作って、問題を1つのファイル、1つのコマンド、1つの入力まで小さくしましょう。

1つの章が理解できなくても、まず最小プロジェクトを終えてから戻って読めば大丈夫です。AI の学習では、理解の多くはプロジェクトの中で補われます。最初の読解だけで完全に理解できるとは限りません。
