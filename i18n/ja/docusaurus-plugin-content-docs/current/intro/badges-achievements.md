# スキル証拠カード

![ステージチェックポイントカード](/img/course/intro-stage-checkpoint-cards-ja.png)

バッジは、証拠につながっているときだけ役に立ちます。たくさんの名前を集めるより、各ステージで 1 枚の証拠カードを残します。

## 証拠カードのルール

| 項目 | 書くこと |
| --- | --- |
| 成果物 | 何を作ったか |
| 実行コマンド | 他の人がどう再現するか |
| 証拠 | スクリーンショット、ログ、グラフ、指標、trace、引用チェック |
| 失敗 1 件 | 実際に出会った問題 |
| 改善 | 何を直したか、次に何を直すか |

## 初回のステージカード

| ステージ | カードの焦点 | 証拠例 |
| --- | --- | --- |
| 1 ツール | 動く作業環境 | README、Git commit、ターミナル出力 |
| 2 Python | プログラムがデータを保存する | JSON ファイル、CLI 出力、エラー処理 |
| 3 データ | データから結論を出す | 整理済み CSV、グラフ、結論 |
| 4 数学 | 数式をコードにする | 小実験と説明 |
| 5 ML | baseline と指標 | 指標表、エラーサンプル |
| 6 DL | 学習の信号 | loss カーブ、設定、失敗記録 |
| 7 LLM/Prompt | 安定した構造化出力 | Prompt バージョン、schema チェック |
| 8 RAG | 根拠付きの回答 | 検索ログ、引用表 |
| 9 Agent | 制御できるツール利用 | trace、権限表、停止ルール |
| 10-12 | 1 つの発展方向 | 前後比較例と評価 |

## カードテンプレート

```md
## ステージ証拠カード

成果物：
実行コマンド：
証拠ファイル：
失敗 1 件：
修正または次の一歩：
```

時間が限られているなら、空のバッジをたくさん集めるより、各ステージに明確な証拠カードを 1 枚残すほうが役立ちます。
