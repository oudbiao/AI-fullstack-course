# 4週間・8週間・12週間の進め方

![学習ペースカード](/img/course/intro-learning-pace-cards-ja.png)

このコースは、読んだ記事数で計画しないほうが進めやすいです。毎週の成果物を決めましょう。見える出力、短い振り返り、次の改善点の3つを残します。

## 1. ペースを選ぶ

| 今の状況 | おすすめ | 主な目標 |
|---|---|---|
| すでにコードを書けて、AIアプリを早く作りたい | 4週間 | 説明できる RAG または Agent のプロトタイプを作る |
| 転職・就職用のポートフォリオを作りたい | 8週間 | 段階ごとのプロジェクトを作品ストーリーにする |
| 初学者、または基礎から整えたい | 12週間 | ツールから卒業制作まで順番に積み上げる |

## 2. ペースカード

| ペース | 毎週の重点 | 最終的な証拠 |
|---|---|---|
| 4週間 | 基礎を速く確認し、LLM、RAG、Agent、デプロイメモに時間を使う | 動くAIアプリ、README、評価セット、失敗サンプル |
| 8週間 | ツール、Python、データ、機械学習、深層学習、LLM、RAG、Agent を段階的に追加する | ポートフォリオREADME、スクリーンショット、デモ台本、振り返り |
| 12週間 | コース順に進め、毎週小さな成果を残し、デバッグ時間を確保する | 学習リポジトリ、段階成果物、卒業制作の下書き |

4週間ルートでも、評価、ログ、失敗サンプルは省かないでください。そこがデモとプロジェクトを分けます。

## 3. 毎週の作業ループ

| 手順 | やること |
|---|---|
| 月曜 | 章の目標と成果物を1つ決める |
| 学習中 | 最小のコード経路を先に動かし、その後で機能を足す |
| 終了前 | スクリーンショット、実行コマンド、出力例、失敗例を1つずつ残す |
| 週末 | READMEを更新し、次のボトルネックを決める |

## 4. 調整ルール

| もし... | こう調整する |
|---|---|
| Python が得意 | 1-2章を圧縮し、RAG と Agent 評価に時間を使う |
| 機械学習や深層学習の経験がある | 理論は速く読み、指標、曲線、誤差分析は残す |
| 環境構築でよく止まる | 12週間ルートを選び、トラブルシューティング記録を残す |
| 時間が少ない | 機能を減らし、証拠は減らさない。README、実行コマンド、例、失敗を残す |

## 5. 週次レビュー用テンプレート

```md
## Week N Review

- 成果物：
- 実行コマンド：
- 入出力例：
- 失敗例：
- 確認したこと：
- 次のボトルネック：
```

コースを終えた証拠は、全ページを一度読んだことではありません。他の人が理解し、再現し、あなたと議論できる動く成果を残すことです。
