# プロジェクト提出基準

![プロジェクト早見図](/img/course/appendix-project-quick-reference-map-ja.png)

このページは、「完成したら何を提出するか」だけに答えます。ステージプロジェクトの後に確認してください。

## 1. 提出ループ

| 手順 | 証拠 |
|---|---|
| 目的 | 誰が使い、何を解決し、何を出力するか |
| 実行 | インストールメモ、環境変数、最小実行コマンド |
| 例 | 実際の入力と実際の出力 |
| 評価 | 指標、チェックリスト、固定質問、または手動確認ルール |
| 失敗 | 再現できる失敗例と考えられる原因 |
| 次 | 次の版で直す具体的な点 |

時間が少ないときは、実行コマンド、出力例、失敗サンプルを先に残します。

## 2. 最小 README

```md
# プロジェクト名

## 目的

## 実行方法

## 入出力例

## 評価

## 既知の失敗

## 次の一手
```

## 3. AIプロジェクトの追加証拠

| プロジェクト | 追加するもの |
|---|---|
| Prompt | Prompt 版、固定テスト、構造化検証 |
| RAG | 分割方針、検索ログ、引用チェック |
| Agent | ツール schema、trace、最大ステップ、権限境界 |
| マルチモーダル | 元素材、生成設定、人による確認リスト |

提出物は小さくても構いません。ただし具体的で、他の人が README 通りに動かせることが大切です。
