# おすすめ学習ルート

![おすすめ学習ルート選択図](/img/course/intro-learning-path-selection-ja.png)

このコースは大きいです。初回からすべての分岐を平らに読もうとしないでください。まず 1 つのルートを選び、小さな成果を作り、プロジェクトが必要としたときに深掘りします。

## まず 1 つ選ぶ

| 今の状況 | 最初のルート | 初回のルール |
| --- | --- | --- |
| どこから始めるか迷う | 初心者完走ルート | 1-9 章を順に進み、各ステージで小さな成果を残す |
| すでにコードを書ける | 高速アプリルート | 1-6 を軽く確認し、Prompt、RAG、Agent、deploy に集中する |
| 作品集が必要 | ポートフォリオルート | 各プロジェクトで README、画像、ログ、失敗メモを残す |
| モデルを深く知りたい | モデル理解ルート | 数学、ML、深層学習、Transformer に時間をかける |

迷うなら **初心者完走ルート** を選びます。完璧な計画より、1 つの完整な流れを動かす方が大切です。

## 時間で絞る

| 使える時間 | やること |
| --- | --- |
| 週 3-5 時間 | コアページだけ読み、最小プロジェクトを終える |
| 週 6-10 時間 | 主ルートを進み、短いプロジェクトメモを書く |
| 就職・作品集スプリント | 先に作り、プロジェクトで見えた理論の穴を埋める |
| 研究・モデル志向 | 数学、ML、深層学習、評価でゆっくり進む |

## 何度も乗り換えない

現在のプロジェクトが本当の不足を示したときだけ、戻って補います。

| 症状 | 戻る場所 |
| --- | --- |
| コードを実行・再現できない | ツールと Python |
| データや引用が信頼できない | データ、RAG、評価 |
| モデル結果を説明できない | 数学、ML、深層学習 |
| Agent を制御できない | tool calling、trace、permission、recovery |

良いリズムは、短く読む、動かす、証拠を保存する、そして進む、です。
