# ステージ確認カード

![ステージ確認カード](/img/course/intro-stage-checkpoint-cards-ja.png)

このページは、以前の “Boss battle” 風の考え方を、もっと単純なルールに置き換えます。

> 各ステージで、実行できる成果物、証拠、短い振り返りを 1 つずつ残す。

## 確認カードの型

| 部分 | 意味 | 例 |
| --- | --- | --- |
| Artifact | 何を作ったか | script、notebook、API、RAG demo、Agent trace |
| Evidence | どう動いたと分かるか | screenshot、CSV、log、metric table、citation check |
| Reflection | 何を学び、何を直したか | limitation、failure case、next step |

## ステージカード

| ステージ | 最小成果物 | 証拠 |
| --- | --- | --- |
| 1 Tools | 実行できる project folder | README と Git commit |
| 2 Python | 小さな CLI または API | sample input/output と error handling |
| 3 Data | cleaned dataset report | charts、quality notes、limitations |
| 4 Math | 実行できる mini experiment | formula-to-code explanation |
| 5 Machine learning | baseline model | metrics と error samples |
| 6 Deep learning | training run | loss curve と checkpoint |
| 7 LLM/Prompt | 安定した prompt test | fixed inputs と output checks |
| 8 RAG | 引用付き Q&A デモ | 検索ログと引用表 |
| 9 Agent | 制御できるタスク実行 | ツール trace と停止条件 |
| 10-12 拡張方向 | 1 つの方向プロジェクト | 前後比較と評価 |
| 卒業制作 | 完成した AI プロダクト | デモ、評価レポート、デプロイメモ |

## 振り返りは短く

各ステージの最後に、このテンプレートだけを書きます。

```md
## ステージ確認カード

成果物：
証拠：
失敗例：
修正または次の一手：
```

カードが埋まったら次へ進みます。空欄が多いなら、読む量を減らし、まず最小成果物を作ってください。
