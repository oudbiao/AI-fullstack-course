# 各パートをどこまで読むか？

![学習ルート選択マップ](/img/course/intro-learning-path-selection-ja.png)

すべてのページを同じ重さで読む必要はありません。3 つの深さで読み分けます。

| 深さ | 使う場面 | 合格ライン |
| --- | --- | --- |
| 必読 | 今のプロジェクトがこれなしでは進まない | 最小例を実行し、入力と出力を説明できる |
| 速読 | 今は位置づけだけ分かればよい | 何を解決するか、後でどこに戻るか言える |
| 選択 | 専門方向や卒業制作に必要 | プロジェクトが必要としたときだけ実験を追加する |

## 標準の読書深度

| コース領域 | 初回の深さ | 理由 |
| --- | --- | --- |
| ツール、Python、データ | 必読 | 後のすべてがコード実行とデータ処理に依存する |
| 数学、機械学習、深層学習 | まず速読。モデル重視なら必読 | 指標、学習、類似度、失敗分析の直感が必要 |
| LLM、Prompt、RAG、Agent | アプリ開発では必読 | LLM アプリの本線だから |
| CV、NLP、マルチモーダル | 初回は選択 | 卒業制作で必要な方向だけ選ぶ |
| デプロイ、評価、安全 | 作品集では必読 | 他人が実行・評価できない Demo は弱い |

## 先へ進んでよいルール

次の 3 つがあれば、先へ進んで大丈夫です。

1. 動く最小例がある。
2. 入力と出力を 1 文で説明できる。
3. 証拠を 1 つ保存した：スクリーンショット、ログ、指標、README メモ、失敗例など。

この 3 つがなければ、読む量を減らし、まず最小成果物を作ってください。
