# よくあるエラーとトラブルシューティング索引

![トラブルシューティング救急マップ](/img/course/appendix-troubleshooting-rescue-map-ja.png)

学習が止まる原因は、「AIを理解していないこと」よりも、ディレクトリ、環境、依存関係、データ形式、API、検索、ツール、デプロイのずれであることが多いです。順番に切り分けましょう。

## 1. 症状別の最初の一手

| 症状 | まず確認するもの | 最小の確認 |
|---|---|---|
| コマンドが失敗する | 現在のディレクトリ、コマンドの有無 | `pwd`、`which <command>`、project scripts |
| Python import が失敗する | インタープリタとパッケージのインストール先 | `python -m pip show <package>` |
| ファイルを読めない | 相対パス、ファイル名の大小文字、文字コード | `pwd` を表示し、小さなファイルを開く |
| JSON 出力が不安定 | Prompt schema と検証 | 最小の JSON だけを返させる |
| RAG の答えが違う | モデル呼び出し前の検索結果 | top-k 検索結果を表示する |
| Agent がループする | ステップ上限、停止条件、trace | 最大3ステップで実行する |
| デプロイで失敗する | 環境変数、起動コマンド、ファイルパス | クリーンな環境で README 通りに実行する |

## 2. デバッグループ

1. 完全なエラーと正確なコマンドを保存する。
2. ディレクトリ、実行環境、依存関係の版、設定を確認する。
3. 最小入力で再現する。
4. 失敗した層を特定する。環境、コード、データ、モデル、検索、ツール、デプロイのどれか。
5. 修正し、予防策を README やスクリプトに書き戻す。

スタックトレースの最後の行だけを見ないでください。実際のパス、インタープリタ、モデル名、フィールド名、ツール引数など、重要な手がかりは途中に出ることがよくあります。

## 3. 層別チェック表

| 層 | すぐ確認する質問 |
|---|---|
| 環境 | 正しいフォルダで、正しいランタイムを使っているか |
| 依存関係 | 実行している Python や Node 環境にインストールされているか |
| データ | パス、列、文字コード、サンプル行は正しいか |
| モデル | 入力形状、ラベル、モデル名、上限、タイムアウトは正しいか |
| RAG | Chunk、metadata、top-k、rerank、引用がログに出ているか |
| Agent | ツール schema、ステップ上限、権限、trace が記録されているか |
| デプロイ | 環境変数、ポート、起動コマンド、本番ログが見えるか |

## 4. トラブル記録テンプレート

```md
## 問題タイトル

### 症状
実行したコマンドと見えたエラー。

### 重要なエラー
最後の行だけでなく、重要な行を貼る。

### 再現方法
1. ディレクトリ：
2. コマンド：
3. 入力または設定：

### 確認したこと
環境、パス、依存関係、データ、パラメータ、ログのどれを確認したか。

### 修正
何を変え、なぜ効いたか。

### 予防
README、検証スクリプト、ログ項目、テストケースに追加すべきこと。
```

解決したエラーは、次回のための分かりやすい文書や安全なコマンドとして残しましょう。
