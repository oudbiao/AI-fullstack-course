# 初心者向けやさしい学習ガイド

![30 分 AI クイック体験ループ](/img/course/intro-quick-experience-loop-ja.png)

コースマップが大きく感じるときは、このページを負担を下げる入口として使ってください。すべてを理解してから始める必要はありません。

最初の 1 周目は、このルールだけで十分です。

> 小さな結果を 1 つ動かす。入力と出力を 1 つ説明する。証拠を 1 つ残す。そして次へ進む。

## 最初の 1 週間のリズム

| 日 | やること | ここまでできたら止める |
| --- | --- | --- |
| 1 日目 | 30 分クイック体験を動かす | AI の結果またはスクリーンショット 1 つ |
| 2 日目 | Python、VS Code、Git、プロジェクトフォルダを準備する | `python --version` と `git --version` が動く |
| 3 日目 | 能力マップを開く | 今の層を言える：ツール、データ、モデル、LLM、アプリ、Agent、工程 |
| 4 日目 | 第 1 章を始める | プロジェクトフォルダと README |
| 5 日目 | 小さな Python スクリプトを書く | コマンド 1 つと出力 1 つ |
| 6 日目 | 本物のエラーを 1 つ記録する | エラーメッセージ、原因、修正メモ |
| 7 日目 | 振り返り、次のレッスンを選ぶ | 次にやることを 1 文で書く |

「何ページ読んだか」だけで測らないでください。まずは **結果 + 証拠 + 次の一歩** です。

## 初回は気にしなくてよいもの

| 今は気にしない | 理由 |
| --- | --- |
| 完璧な理論ノート | 例を動かした後のほうが概念は分かりやすい |
| すべての選択ルート | まず標準ルートを進む |
| 大規模モデルの学習 | 基礎ができる前は高コストで遅い |
| 複雑なフレームワーク | 基本の入力出力ループが見えにくくなる |
| 作品集の細かな仕上げ | まず動くプロジェクトを作る |

任意ページはブックマークで十分です。後で役立ちますが、スタート地点ではありません。

## 毎日の最小ループ

![ステージチェックポイントカード](/img/course/intro-stage-checkpoint-cards-ja.png)

1 回の学習は、とても小さくても構いません。

| 部分 | 例 |
| --- | --- |
| 入力 1 つ | コマンド、ファイル、Prompt、質問 |
| 出力 1 つ | 1 行の結果、グラフ、JSON ファイル、モデル回答 |
| 記録 1 つ | README メモ、スクリーンショット、エラー記録、指標表 |

この 3 つがあれば、その学習は有効です。

## 章が分からないとき

同じページを 5 回読み直す前に、次の質問に答えてください。

| 質問 | 役立つ理由 |
| --- | --- |
| このページは何の問題を解くのか？ | 目的が見える |
| 入力は何か？ | 作業の始まりが見える |
| 出力は何か？ | 成功の形が見える |
| 最小例を動かせるか？ | 読むだけから行動へ移れる |
| どのプロジェクトタスクで使うか？ | レッスンを本線へ戻せる |

これに答えられれば、先へ進んで大丈夫です。細かい部分は、プロジェクトで必要になったときに戻って補います。

## 短い振り返り Prompt

実際のコマンド、出力、エラーがあるときだけ使ってください。

```text
私は AI フルスタックコースの第 X 段階を学習しています。
今日実行した内容はこれです：
[コマンド、出力、エラー]

何を学んだのか、詰まりがどの層に属するのか、30 分以内でできる次の一歩を 1 つ教えてください。
```

役に立つのは「AI が全部説明してくれる」ことではありません。自分の学習記録を、小さく振り返れる形に整えることです。
